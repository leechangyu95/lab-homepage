Week 11. 피드백 제어 — PID로 시작하는 제어공학

결과를 보고 명령을 다시 조정하는 단순한 발상이 어떻게 자율 시스템의 토대가 되는지.

목차

  1. 개루프 vs 폐루프
  2. 피드백 시스템의 블록 다이어그램
  3. PID 제어기 — P·I·D 세 항
  4. 각 항의 직관과 효과
  5. 튜닝 — Ziegler-Nichols와 실무 가이드
  6. 실전 함정과 해결책

1. 개루프 vs 폐루프

제어 시스템은 크게 두 가지로 나뉩니다.

1.1 개루프(Open-loop)

입력을 정해 명령만 보내고 결과는 확인하지 않음. 외란이 없을 때만 잘 동작.

  • 예: 토스터(시간 타이머), 신호등, 일반 세탁기 시간 모드.
  • 장점: 단순하고 안정적이다(불안정해질 일이 없음).
  • 단점: 외란·시스템 변동에 그대로 노출. "모터에 부하가 더 걸려도" 명령은 그대로.

1.2 폐루프(Closed-loop) = 피드백

출력을 측정해 목표와의 오차를 계산하고, 그 오차를 줄이는 방향으로 입력을 조정.

  • 예: 자율주행 차량의 속도 제어, 드론 자세 제어, 에어컨 온도 제어.
  • 장점: 외란을 자동으로 보상. 시스템 불확실성에 강함.
  • 단점: 잘못 설계하면 발진·발산 가능 (안정성 분석 필요).

샤워기 온도 조절이 일상의 가장 친근한 폐루프 — 너무 뜨거우면 찬물 쪽으로, 너무 차가우면 따뜻한 쪽으로. 결과(피부 감각)를 보고 입력(밸브)을 조정.

2. 피드백 시스템의 블록 다이어그램

각 신호의 의미:

  • $r(t)$: 참조(reference) — 원하는 값 (예: 목표 속도)
  • $y(t)$: 출력 — 실제 측정된 값
  • $e(t) = r(t) - y(t)$: 오차
  • $u(t)$: 제어 입력 — 제어기가 만들어 대상에 보내는 명령

전체 폐루프 전달 함수 (단위 피드백, $H = 1$):

$$ \frac{Y(s)}{R(s)} = \frac{C(s) P(s)}{1 + C(s) P(s)} $$

분모 $1 + CP$가 0이 되는 $s$ 값이 폐루프 시스템의 극점 — 이것이 좌반평면에 있어야 안정.

3. PID 제어기 — 가장 보편적인 컨트롤러

제어기 $C(s)$의 가장 많이 쓰이는 형태가 PID(Proportional-Integral-Derivative)입니다.

$$ u(t) = K_p\,e(t) + K_i\!\int_0^t e(\tau)\,d\tau + K_d\,\dot{e}(t) $$

라플라스 영역에서:

$$ C(s) = K_p + \frac{K_i}{s} + K_d s $$

세 항의 직관:

  • P (비례): 현재 오차에 비례. 오차가 크면 큰 명령. 빠른 응답을 만들지만 정상상태 오차가 남음.
  • I (적분): 누적된 오차에 비례. 작은 오차도 시간이 지나면 누적되어 컨트롤러가 강하게 반응 → 정상상태 오차 0.
  • D (미분): 오차의 변화율에 비례. "오차가 줄어드는 중이면 명령을 미리 줄여" 오버슈트를 억제.

4. 각 항의 효과

이득 ↑상승시간오버슈트정착시간정상상태 오차
$K_p$감소증가변화 적음감소
$K_i$감소증가증가0으로
$K_d$변화 적음감소감소변화 없음

이 표가 PID 튜닝의 직관적 가이드가 됩니다. 너무 큰 $K_p$는 오버슈트·발진을, 너무 큰 $K_i$는 늦은 정착을, 너무 큰 $K_d$는 잡음 증폭을 일으킵니다.

5. 튜닝 방법

5.1 Ziegler-Nichols 공식 (가장 고전)

$K_i = K_d = 0$로 두고 $K_p$를 천천히 키우다가 시스템이 등진동(일정 진폭 진동)을 시작하는 임계 이득 $K_u$와 그때의 진동 주기 $T_u$를 측정. 다음 표로 이득을 결정:

제어기$K_p$$T_i$$T_d$
P$0.5 K_u$——
PI$0.45 K_u$$T_u/1.2$—
PID$0.6 K_u$$T_u/2$$T_u/8$

여기서 $K_i = K_p/T_i$, $K_d = K_p T_d$. 시스템을 한계까지 밀어붙이는 방식이라 가전·산업의 안전 시스템에는 직접 못 씁니다.

5.2 실무 권장 — 점진적 튜닝

  1. P만 살리고 시작: $K_p$를 작게 시작해 응답이 빠르되 발진 직전에서 멈춤.
  2. I 추가: 정상상태 오차가 보이면 $K_i$를 천천히 증가 (너무 크면 적분 와인드업).
  3. D 추가: 오버슈트가 거슬리면 $K_d$를 추가. 단, 잡음을 증폭하니 D 앞에 LPF를 두는 게 표준.
  4. 스텝 응답 측정과 반복.

6. 실전 함정과 해결책

6.1 적분 와인드업(Integral Windup)

액추에이터가 포화 상태(예: 모터 PWM이 100%로 다 차 있음)인데 오차가 계속 누적되면 적분 항이 무한정 커집니다. 그 후 오차가 0이 되어도 적분이 너무 커져 시스템이 한참 동안 잘못된 명령을 내보내요.

해결: 액추에이터 포화를 감지하면 적분을 멈추는 anti-windup 기법. 또는 적분값에 상한을 둠.

6.2 미분 잡음 증폭

D 항은 입력을 미분하므로 고주파 잡음을 키웁니다. 그래서 실제 PID에서 D 앞에는 거의 항상 LPF를 둡니다 — "D + LPF"가 한 모듈로 패키지된 형태.

6.3 디지털 구현 — 샘플링 주기

임베디드에서 PID는 다음과 같이 이산화:

e = ref - y;                        // 현재 오차
integral += e * dt;                 // 적분
derivative = (e - e_prev) / dt;     // 미분
u = Kp*e + Ki*integral + Kd*derivative;
e_prev = e;

샘플링 주기 $dt$는 시스템 시정수의 1/10 이하로 잡는 게 보통. 모터 제어는 보통 1kHz(1ms), 자세제어는 100~500Hz.

7. 응용 예 — 자율주행 속도 제어

차량의 목표 속도 60km/h에 맞추는 PID:

  • $y$ = 휠 엔코더로 측정한 실제 속도
  • $u$ = 스로틀(또는 브레이크) 명령
  • $P$: 빠르게 속도를 따라가게
  • $I$: 오르막에서 속도가 처지면 보상해 정확히 60에 맞춤
  • $D$: 가속이 너무 급격하면 미리 누름
