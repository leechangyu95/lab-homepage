Week 6. 마이크로프로세서 프로그래밍 — Arduino로 시작하기

센서 신호를 읽고, 알고리즘으로 처리하고, 액추에이터를 움직이는 두뇌. Sense-Think-Act가 한 칩 안에서 도는 자리.

목차

  1. 마이크로프로세서·MCU·SoC의 차이
  2. Arduino UNO — 가장 친근한 메카트로닉스 보드
  3. GPIO — 디지털 입출력
  4. 아날로그 입력(ADC)
  5. PWM — 아날로그 출력 흉내내기

1. 용어 정리

  • 마이크로프로세서(μP): CPU 코어만 들어 있는 칩. 인텔 i7 같은 것.
  • 마이크로컨트롤러(MCU): CPU + 메모리 + GPIO + ADC + 타이머가 한 칩에. 임베디드용. ATmega328 (Arduino UNO), STM32, ESP32 등.
  • SoC(System on Chip): CPU + GPU + 무선·메모리·센서까지 한 칩에. 스마트폰 AP, 라즈베리파이.

메카트로닉스에서 가장 자주 쓰는 건 MCU이고, 입문 학습에 표준은 Arduino입니다.

3. GPIO — 디지털 입출력

3.1 디지털 출력 — LED 깜빡이기

void setup() {
  pinMode(13, OUTPUT);   // 13번 핀을 출력으로
}
void loop() {
  digitalWrite(13, HIGH);  // 5V 출력 (LED ON)
  delay(500);              // 0.5초 대기
  digitalWrite(13, LOW);   // 0V 출력 (LED OFF)
  delay(500);
}

"Hello, World"에 해당하는 임베디드 입문 코드. 이 한 페이지로 GPIO·시간·반복문이 모두 실습됩니다.

3.2 디지털 입력 — 버튼 읽기

void setup() {
  pinMode(2, INPUT_PULLUP);    // 내부 풀업 저항 사용
  pinMode(13, OUTPUT);
}
void loop() {
  if (digitalRead(2) == LOW) { // 버튼 눌리면 LOW
    digitalWrite(13, HIGH);
  } else {
    digitalWrite(13, LOW);
  }
}

핵심 — 입력 핀은 항상 풀업 또는 풀다운이 필요합니다. 그렇지 않으면 핀이 떠 있어 잡음으로 마구 흔들립니다(floating). Arduino의 INPUT_PULLUP는 칩 내부의 풀업 저항을 켭니다.

4. 아날로그 입력 (ADC)

0~5V 아날로그 신호를 0~1023 정수로 변환 (10-bit). 분해능은 약 4.88mV.

void setup() {
  Serial.begin(9600);
}
void loop() {
  int raw = analogRead(A0);             // 0..1023
  float voltage = raw * (5.0 / 1023.0); // V로 환산
  Serial.println(voltage);
  delay(100);
}

주의 — Arduino UNO의 ADC는 약 100μs/샘플로 그리 빠르지 않습니다(<10kHz 표본화). 고주파 신호는 STM32(12-bit, MHz 단위)나 외장 ADC가 필요합니다.

5. PWM — 디지털로 아날로그 만들기

MCU는 진정한 아날로그 출력이 어려워 대신 고주파 사각파의 듀티 사이클로 평균 전압을 조절합니다.

$$ V_{\text{avg}} = V_{\text{HIGH}} \cdot D \quad (D = \text{듀티}, 0 \le D \le 1) $$

예: 5V·듀티 50% PWM의 평균은 2.5V. 부하가 RC 저역통과처럼 동작하면 정말로 2.5V DC가 나오는 것처럼 보입니다.

void setup() {
  pinMode(9, OUTPUT);
}
void loop() {
  for (int v = 0; v <= 255; v++) {
    analogWrite(9, v);   // 0..255 → 듀티 0..100%
    delay(10);
  }
}

5.1 PWM의 실제 응용

  • LED 밝기 조절 — 듀티만 바꾸면 됨.
  • DC 모터 속도 제어 — H-브릿지(MOSFET 4개)와 PWM으로 속도와 방향 조절.
  • 서보 모터 — 50Hz 주기에 1~2ms 펄스 폭으로 각도 지정.
  • 전압 변환(SMPS) — Buck/Boost 컨버터의 동작 원리.

5.2 PWM 주파수 선택

너무 낮으면 부하가 PWM의 사각파를 따라 깜빡거리고, 너무 높으면 MCU와 회로의 부담이 커집니다. 보통:

  • LED·서보: 50–500 Hz
  • DC 모터: 1–20 kHz (가청 진동음 회피)
  • 전력 변환: 20 kHz–MHz

6. 인터럽트와 타이머

delay()로 시간을 잰다는 건 그동안 MCU가 아무것도 못 한다는 뜻. 실시간 시스템에서는 절대 그렇게 짜지 않습니다. 대신 두 가지 도구를 씁니다.

6.1 외부 인터럽트

핀에 신호 변화(상승/하강 에지)가 들어오면 즉시 등록된 함수(ISR)를 호출. 메인 루프와 별개로 동작.

volatile int count = 0;     // ISR에서 쓸 변수는 volatile

void onPulse() {            // ISR
  count++;
}

void setup() {
  pinMode(2, INPUT_PULLUP);
  attachInterrupt(digitalPinToInterrupt(2), onPulse, FALLING);
  Serial.begin(9600);
}
void loop() {
  Serial.println(count);
  delay(500);
}

응용: 엔코더 펄스 카운팅, 버튼 디바운싱, 외부 RTC 동기화.

6.2 타이머 인터럽트

일정 주기로 ISR을 호출. 제어 루프를 정확한 주기로 돌리는 데 핵심입니다 — PID 제어, 데이터 로깅, 표본화 모두 이걸로.

// 1ms마다 ISR 호출 (정밀 제어 루프)
// 라이브러리 예: TimerOne, MsTimer2 등 사용

6.3 ISR 작성 규칙

  • 짧게 — 수 마이크로초 안에 끝낼 일만.
  • 공유 변수는 volatile — 컴파일러 최적화로 사라지지 않게.
  • delay·Serial.print는 금지 — 다른 인터럽트가 막힘.
  • 플래그만 세우고 실제 처리는 메인 루프에서 — 가장 안전한 패턴.

7. 시리얼 통신과 I2C/SPI

7.1 UART (시리얼)

가장 기본. 두 핀(TX, RX)로 1:1 통신. PC와 디버그용으로 가장 자주 씀.

7.2 I2C

두 핀(SDA, SCL)에 여러 장치를 매달 수 있음 (각자 주소를 가짐). MPU6050 IMU, OLED 디스플레이, EEPROM 등에 표준.

7.3 SPI

4핀(MOSI, MISO, SCK, SS) 고속 통신. SD카드, 무선 모듈, 정밀 ADC에 자주 씀.

통신핀 수속도장치 수
UART2~1Mbps1:1
I2C2100k–400kHz다대다(주소)
SPI4+~수십 MHz마스터-슬레이브

8. 실전 패턴 — 비차단(non-blocking) 루프

제어 시스템은 여러 일을 동시에 해야 합니다 — 센서 읽기, 제어 계산, 통신, LED 깜빡임. delay()로 줄 세우면 안 되고, millis()로 시간을 직접 관리합니다.

unsigned long lastSense = 0, lastCtrl = 0;
const unsigned long PERIOD_SENSE = 10;    // 10ms = 100Hz
const unsigned long PERIOD_CTRL  = 50;    // 50ms = 20Hz

void loop() {
  unsigned long now = millis();
  if (now - lastSense >= PERIOD_SENSE) {
    lastSense = now;
    readSensors();
  }
  if (now - lastCtrl >= PERIOD_CTRL) {
    lastCtrl = now;
    runControl();
  }
}

이 패턴 하나만 익히면 임베디드 코드의 90%를 짤 수 있습니다.
