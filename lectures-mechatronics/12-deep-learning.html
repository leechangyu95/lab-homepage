Week 12. 딥러닝 in 메카트로닉스

모델식을 모르는 시스템·인지가 어려운 환경에서 데이터로부터 직접 학습. 그리고 그 출력을 제어 루프에 연결하기.

1. 왜 딥러닝인가

지난 W7~W11에서 우리는 시스템을 미분방정식으로 모델링하고 PID로 제어했어요. 하지만 실제 메카트로닉스에는 두 가지 어려운 상황이 있습니다.

  • 모델을 적기 어려운 경우: 마찰·온도·재료 변형 같은 비선형성, 인간 운전자의 행동, 비정형 환경.
  • 인지가 필요한 경우: 카메라 영상에서 보행자·차선·표지판을 알아내야 함. 미분방정식으로는 답이 안 나오는 문제.

이런 자리에 데이터에서 직접 함수를 학습하는 딥러닝이 들어옵니다. 충분한 데이터가 있다면, 모델을 손으로 적지 않아도 입력→출력 관계를 잡아줍니다.

2. 퍼셉트론과 다층 신경망

2.1 퍼셉트론(Perceptron)

가장 작은 인공 뉴런. 입력에 가중치를 곱해 합한 뒤 활성화 함수를 통과:

$$ y = f\!\left(\sum_{i=1}^{n} w_i x_i + b\right) $$

$w_i$는 가중치(weights), $b$는 바이어스. 학습이란 결국 데이터로 이 $w$, $b$ 값을 찾는 일입니다.

2.2 활성화 함수 — 비선형성의 도입

가중합만으로는 선형 함수만 표현 가능. 활성화 함수가 비선형성을 더해줍니다.

  • Sigmoid $\sigma(x) = 1/(1+e^{-x})$: 0~1 출력, 확률 해석에 좋음
  • ReLU $\max(0, x)$: 가장 흔히 씀. 학습이 빠르고 vanishing gradient 적음
  • Tanh $\tanh(x)$: −1~1 출력, RNN의 게이트에 자주

왜 비선형성이 필요한가? 활성화 함수가 없거나 모두 선형이면, 아무리 층을 깊게 쌓아도 결국 한 번의 행렬 곱 $y = W_{\text{eff}}\,x + b_{\text{eff}}$로 합쳐집니다 — XOR 같은 단순한 문제도 못 풀어요. 비선형 활성화가 끼어들면 층마다 새로운 "꺾임"이 생겨 임의의 곡선·결정경계를 만들 수 있게 됩니다.

🎮 인터랙티브: 활성화 함수와 그 도함수(=그래디언트)

같은 입력에 대해 세 활성화의 모양과 도함수를 비교하세요. 도함수가 학습에 직결됩니다 — 도함수가 0이 되는 영역이 많으면 그래디언트가 사라져(vanishing gradient) 학습이 멈춰요.

  • Sigmoid: 양 끝(|x|>4)에서 도함수 ≈ 0 → 깊은 망에서 학습 멈춤. 1986년 이후 한참 사용했지만 ReLU에 밀림.
  • Tanh: 0 중심, sigmoid보다 좋음. 하지만 여전히 양 끝에서 saturate.
  • ReLU: 양수 영역에서 도함수가 1 → 그래디언트가 살아 있음. 음수 영역에선 도함수 0 ("Dying ReLU" 문제).

2.3 다층 신경망(MLP)과 "Deep"

퍼셉트론을 여러 층으로 쌓으면 다층 신경망(MLP, Multi-Layer Perceptron). 그중 은닉층(hidden layer)이 여러 개 있는 신경망을 심층 신경망(DNN, Deep Neural Network)이라 부르고, 이를 학습하는 분야가 딥러닝입니다.

"Deep"의 핵심은 깊이에서 오는 표현력입니다. 1개 은닉층으로도 임의 함수를 근사할 수 있다는 정리(보편 근사 정리)가 있지만, 실제로는 깊게 쌓을수록 같은 정확도를 더 적은 파라미터로 달성합니다.

🎮 인터랙티브: 퍼셉트론 1개로 풀 수 없는 문제 — XOR

퍼셉트론(은닉층 없음) = 직선 결정 경계만 가능 → XOR(대각선 패턴)을 절대 못 풀어요. 은닉층을 추가하면 휘어진 경계가 가능해지고 XOR이 풀립니다. 데이터 패턴을 바꿔보고, 은닉 뉴런 수를 0과 4로 바꿔 결과를 비교하세요.

3. 학습의 3요소

3.1 손실 함수(Loss Function)

예측 $\hat{y}$가 정답 $y$에 얼마나 가까운가를 수치로. 가장 흔한 두 가지:

  • 회귀 (연속값): 평균제곱오차(MSE) $L = \frac{1}{N}\sum (y_i - \hat{y}_i)^2$
  • 분류 (클래스 라벨): 교차 엔트로피 $L = -\sum y_i \log \hat{y}_i$

3.2 역전파(Backpropagation)

각 가중치 $w$가 손실 $L$에 미치는 영향(편미분 $\partial L/\partial w$)을 효율적으로 계산하는 알고리즘. 핵심은 연쇄 법칙(chain rule)으로 출력층에서 입력층으로 거슬러 올라가며 그래디언트를 계산.

$$ \frac{\partial L}{\partial w} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z} \cdot \frac{\partial z}{\partial w} $$

이 알고리즘이 1986년 다시 주목받으면서 신경망의 두 번째 봄이 시작되었어요(W7-W8 딥러닝 트랙에서 자세히).

3.3 옵티마이저(Optimizer)

그래디언트 정보로 가중치를 업데이트하는 규칙. 가장 단순한 형태는 경사 하강법:

$$ w \leftarrow w - \eta \frac{\partial L}{\partial w} $$

$\eta$는 학습률(learning rate). 너무 크면 발산, 너무 작으면 학습이 더딤. 실무에서는 Adam이 표준에 가까움.

직관 — "안개 낀 산에서 가장 낮은 곳 찾기": 손실 함수는 가중치 공간에서의 지형이고, 우리는 가장 낮은 골짜기(=손실 최소점)를 찾고 싶어요. 그래디언트는 "지금 위치에서 가장 가파르게 올라가는 방향"이므로 그 반대로 한 발씩 내려가면 됩니다. 한 발의 크기가 학습률 $\eta$.

🎮 인터랙티브: 경사 하강법으로 손실 최소점 찾기

1차원 손실 $L(w) = (w-2)^2$의 최소점은 $w=2$입니다. 시작점과 학습률을 바꾸며 경로를 관찰하세요. 학습률이 너무 크면(>1) 발산하고, 너무 작으면(<0.05) 한참 걸려요. "적당히 큰 학습률 + 모멘텀"이 실무의 표준입니다.

4. 메카트로닉스에서의 응용 4가지

4.1 인지 (Perception)

카메라·LiDAR로 본 환경을 이해. 자율주행에서 가장 큰 사용처.

  • 객체 검출: YOLO, Faster R-CNN — 보행자·차량·표지판 위치와 종류.
  • 의미 분할: 픽셀마다 클래스 라벨 — 차선·도로·하늘 구분.
  • 3D 인식: PointPillars, PointNet — LiDAR 점군에서 객체 검출.

이 모든 출력이 다시 제어 루프의 입력으로 들어갑니다. 예: "전방 5m에 보행자" → 정지 명령.

4.2 시스템 식별 (System Identification)

물리 모델을 적기 어려운 시스템에서 입출력 데이터로 직접 동역학을 학습. 비선형 마찰·간극·온도 효과까지 잡아낼 수 있음.

  • 응용 예: 4족 로봇의 다리 동역학, 부드러운 로봇(soft robot), 산업 펌프의 마찰 모델.
  • 방법: 신경망 ODE, RNN/LSTM, Transformer 기반 dynamics 모델.

4.3 강화학습 제어 (Reinforcement Learning)

"보상이 큰 행동을 점점 더 자주" 하도록 학습. PID로는 풀기 어려운 다관절·접촉 풍부한 작업에 강력.

  • 응용 예: 로봇 팔의 물체 잡기(grasping), 4족 로봇 보행, 드론 곡예비행.
  • 주의: 시뮬레이션→실제(sim-to-real) 격차, 안전 보장이 약점.

4.4 예지 보전 (Predictive Maintenance)

모터 진동·전류·온도 시계열을 분석해 고장이 일어나기 전에 예측. 산업 IoT의 표준 응용.

  • 방법: LSTM, Transformer, Autoencoder 기반 이상 탐지.
  • 가치: 갑작스런 정지를 사전 경보로 바꿔 가동률·안전 향상.

5. 실시간성과 임베디드 배포

제어 루프는 보통 1~1000Hz로 돕니다. 신경망이 들어가도 같은 속도로 동작해야 한다는 게 임베디드의 큰 도전.

5.1 모델 경량화 기법

  • 양자화(Quantization): 32-bit float → 8-bit int. 모델 크기 4배, 속도 2~4배.
  • 가지치기(Pruning): 영향이 작은 가중치를 0으로 만들고 희소 행렬로 처리.
  • 지식 증류(Knowledge Distillation): 큰 모델의 지식을 작은 모델로 옮김.
  • 전용 NPU/GPU: Jetson Nano/Orin, Coral TPU, Raspberry Pi AI Kit 등.

5.2 안전 보장의 어려움

전통적 제어공학은 안정성 증명이 가능하지만, 신경망 정책은 보장하기 어렵습니다. 그래서 실제 차량·항공기에서는:

  • 인식·계획에는 딥러닝을 쓰되
  • 저수준 제어는 여전히 PID/MPC를 쓰고
  • 안전 조건 위반 시 폴백(fallback)으로 강제 전환하는 구조

신경망과 고전 제어가 함께 가는 하이브리드가 현 단계 산업의 표준입니다.
