Week 3. 신호 조절(Signal Conditioning)

센서가 내놓은 거친 신호를 쓸 만한 형태로 다듬기 — 증폭·필터·ADC.

목차

  1. 왜 신호 조절이 필요한가
  2. 증폭(Amplification)
  3. 필터(Filter) — 노이즈 제거의 핵심
  4. ADC — 아날로그를 디지털로
  5. 표본화 정리와 에일리어싱
  6. 이동 평균 필터(Moving Average)

1. 왜 신호 조절인가

센서 출력은 그대로 MCU의 ADC에 넣을 수 있는 깨끗한 신호가 거의 없습니다. 보통 다음 문제 중 하나 이상을 가집니다.

  • 크기가 너무 작음 — 스트레인 게이지 출력은 mV 단위, ADC 입력은 0~5V. 1000배 증폭 필요.
  • 잡음이 섞임 — 50/60Hz 전원 잡음, 고주파 EMI, 기계 진동에 의한 흔들림.
  • 전압 범위가 안 맞음 — 음수 출력을 양수로, 또는 0~10V를 0~5V로 스케일링.
  • 임피던스가 안 맞음 — 신호원 출력저항이 너무 커서 ADC가 신호를 빨아먹어버림.

이 모든 문제를 해결하는 단계가 신호 조절이고, 보통 다음 순서로 구성됩니다.

2. 증폭(Amplification)

작은 신호를 키우는 단계. 기전공학기초 W11의 op-amp가 핵심 도구입니다. 메카트로닉스에서 자주 쓰는 두 형태:

2.1 비반전 증폭기 (Non-inverting)

$$ v_O = \left(1 + \frac{R_2}{R_1}\right) v_S $$

입력 임피던스가 매우 커서 약한 신호원에 부담을 안 줍니다. 일반적인 센서 신호 증폭에 1순위.

2.2 계측 증폭기 (Instrumentation Amp)

3-op-amp 구조로 차분 신호만 증폭하고 동상 잡음을 제거합니다 (CMRR이 매우 높음). 스트레인 게이지·서모커플·심전도 등 미약한 차분 신호 측정의 표준.

$$ v_O = \frac{R_f}{R_1}(v_2 - v_1) \times \text{(추가 이득)} $$

휘트스톤 브릿지(기전공학기초 W5)의 출력을 받아 ADC로 보내는 자리에 거의 항상 들어갑니다. INA125·INA126 같은 IC가 표준 부품.

3. 필터(Filter)

특정 주파수 대역만 통과시키고 나머지를 걸러내는 회로. 종류는 4가지:

이름약자통과 대역대표 응용
저역통과LPF$f < f_c$고주파 잡음 제거 (가장 흔함)
고역통과HPF$f > f_c$DC 오프셋 제거, AC 결합
대역통과BPF$f_1 < f < f_2$특정 주파수만 (라디오 튜너)
대역저지BSF/Notch특정 주파수만 막음

3.1 RC 저역통과 필터(LPF)

저항 + 커패시터 조합. 가장 단순하고 가장 자주 쓰는 잡음 제거기.

$$ \frac{V_{\text{out}}}{V_{\text{in}}} = \frac{1}{1 + j\omega RC}, \qquad f_c = \frac{1}{2\pi RC} $$

차단 주파수 $f_c$ 위는 점차 약해지고 아래는 그대로 통과. 기전공학기초 W8 임피던스에서 보드 플롯과 함께 자세히 다룹니다.

3.2 적용 예 — IMU 가속도 센서

차량의 IMU 가속도 신호에는 엔진 진동(수백 Hz)이 큰 잡음으로 섞여 들어옵니다. 차량 자세 제어에 필요한 주파수는 보통 ~10Hz 이하이므로 $f_c \approx 20\,\text{Hz}$인 LPF를 통과시키면 깨끗한 신호가 됩니다.

4. ADC — 아날로그를 디지털로

아날로그 신호를 MCU가 처리할 수 있는 비트열로 변환. 두 단계로 나뉩니다.

4.1 표본화(Sampling)

연속 신호 $x(t)$를 일정 간격 $T_s$로 측정 → 이산 신호 $x[n] = x(nT_s)$. 측정 빈도를 표본화 주파수 $f_s = 1/T_s$라 합니다.

4.2 양자화(Quantization)

각 표본의 값을 유한한 비트로 표현. $N$비트 ADC라면 $2^N$단계로 분해. Arduino UNO의 ADC는 10비트 = 1024단계 = 0~5V를 약 4.88mV 단위로.

4.3 ADC 분해능과 양자화 오차

$$ \text{LSB} = \frac{V_{\text{ref}}}{2^N} $$

분해능은 LSB와 같고, 양자화 오차는 ±LSB/2. 5V·10비트면 LSB ≈ 4.88mV, 오차 ±2.44mV. 이보다 작은 신호 변화는 ADC가 인식 못 합니다 — 그래서 증폭이 필요한 또 다른 이유.

5. 표본화 정리와 에일리어싱

5.1 나이퀴스트-섀넌 정리

"신호의 최고 주파수 성분의 2배 이상으로 표본화하면 원 신호를 완전히 복원할 수 있다."

$$ \boxed{f_s \ge 2 f_{\max}} $$

이 한계 $f_s/2$를 나이퀴스트 주파수라 합니다. 음성(20kHz) 녹음의 표본화율 44.1kHz가 이 정리에서 나온 값이에요.

5.2 에일리어싱(Aliasing)

표본화율이 $2f_{\max}$ 미만이면 고주파 성분이 저주파로 잘못 보입니다. 회전하는 바퀴가 영화에서 거꾸로 도는 것처럼 보이는 현상이 그 예. 한 번 에일리어싱이 일어나면 어떤 디지털 처리로도 복원 불가.

5.3 안티앨리어싱 필터

예방책 — ADC 직전에 LPF를 둬 $f_s/2$ 이상의 성분을 미리 제거. 이를 안티앨리어싱 필터(anti-aliasing filter)라 합니다. 모든 데이터 수집 시스템의 표준 구성:

센서 → 증폭 → 안티앨리어싱 LPF → ADC → MCU

5.4 예제 — 10Hz 신호의 표본화

$x(t) = \cos(2\pi \cdot 10 \cdot t)$를 다양한 $f_s$로 표본화하면:

  • $f_s = 100\,\text{Hz}$ → 매 주기당 10점 → 잘 표현됨.
  • $f_s = 25\,\text{Hz}$ → 주기당 2.5점 → 한계 근처지만 OK.
  • $f_s = 18\,\text{Hz}$ → 나이퀴스트(20Hz) 미만 → 에일리어싱! 2Hz 신호처럼 보임.

6. 이동 평균 필터(Moving Average)

가장 단순한 디지털 LPF. 최근 $N$개 샘플의 평균을 출력으로 내보냅니다.

$$ y[n] = \frac{1}{N}\sum_{k=0}^{N-1} x[n-k] $$

6.1 효과와 한계

  • 효과: 평균이 잡음을 누르고 신호의 추세를 부드럽게 함. 구현이 매우 쉬움 (3~4줄 C 코드).
  • 한계: 응답이 늦어짐(평균창 크기만큼 지연). 빠른 변화에 둔감해짐.

6.2 Python 예제

import numpy as np
def moving_average(x, N):
    return np.convolve(x, np.ones(N)/N, mode='valid')

# 잡음 섞인 신호
t = np.linspace(0, 1, 200)
clean = np.sin(2*np.pi*5*t)
noisy = clean + 0.3*np.random.randn(len(t))

filtered = moving_average(noisy, N=10)

이동 평균은 연산량이 적어 임베디드에 자주 쓰지만 정밀 측정에는 IIR/FIR 필터를 별도로 설계합니다.
