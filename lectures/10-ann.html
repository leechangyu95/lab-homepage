1. 왜 신경망인가?

한 줄 비유부터

여러분이 강아지와 고양이 사진을 친구에게 보여 주면, 친구는 1초도 안 돼 둘을 구분합니다. 그런데 "어떻게 구분했어?"라고 물으면 "그냥 보면 알아"라는 답이 돌아오죠. 사람의 뇌가 하는 일은 수많은 픽셀(입력)을 받아서 "강아지냐 고양이냐"라는 답(출력)을 만드는 함수를 굴리는 것과 같습니다. 다만 그 함수가 너무 복잡해서 손으로 식을 적을 수가 없을 뿐이에요.

인공 신경망(Artificial Neural Network, ANN)은 이 "너무 복잡한 함수"를 데이터로부터 자동으로 만들어 내는 기계입니다. 이름 그대로 뇌의 뉴런을 흉내 낸 작은 계산 단위(퍼셉트론)를 수천·수만 개 엮어, "이렇게 하면 정답이 나오더라"를 학습합니다.

왜 선형 회귀로는 부족한가?

L9에서 배운 선형 회귀는 $y = w x + b$로 데이터를 직선에 맞춥니다. 직선은 단순해서 좋지만, 강아지/고양이 분류 같은 문제에는 어림도 없죠. 이미지 픽셀 수천 개를 직선 하나로 쪼갤 방법이 있을 리가 없으니까요.

  • 선형 모델의 한계 — 입력과 출력 사이의 관계가 직선/평면이라고 가정. 데이터가 곡선을 그리면 끝.
  • 특징 공학(feature engineering)의 한계 — 사람이 "이 데이터엔 $x^2$이 중요해", "$\sin x$가 중요해" 하면서 수동으로 비선형 특징을 만들어 줘야 함. 이미지·음성처럼 입력 차원이 큰 문제에서는 불가능에 가깝습니다.
  • 신경망의 답 — "비선형 변환을 사람이 짜지 말고, 모델이 데이터로부터 직접 배우게 하자."

로봇공학에서 신경망이 등장하는 자리

로봇은 카메라·라이다·IMU 같은 센서에서 엄청난 양의 입력을 받아 "어떤 행동을 할지"를 결정해야 합니다. 이 입력→행동 매핑은 보통 비선형이며, 사람이 모든 경우를 손으로 코딩하기 어렵습니다.

  • 지각(perception) — 카메라 이미지에서 보행자·차선·장애물을 검출 (CNN, L11–L12에서 다룸).
  • 행동 정책(policy) — 강화학습에서 상태 → 행동 매핑을 신경망으로 표현.
  • 모델 학습(model learning) — 데이터로부터 로봇의 동역학($\dot{\mathbf x} = f(\mathbf x, \mathbf u)$)을 학습.
  • 상태 추정 — 칼만 필터를 보강하거나 대체하는 학습 기반 추정기.

2. 퍼셉트론 (뉴런 한 개)

한 줄 비유부터 — 뇌의 뉴런

생물 뉴런은 여러 개의 시냅스로 다른 뉴런들의 신호를 받습니다. 들어온 신호의 합이 어떤 임계값을 넘으면 자신도 "발화(fire)"해서 다음 뉴런으로 신호를 보내고, 안 넘으면 조용히 있어요. 입력 신호마다 시냅스의 강도가 달라서, 같은 신호라도 어떤 시냅스로 들어오느냐에 따라 영향력이 다릅니다.

인공 뉴런(퍼셉트론)은 이걸 그대로 수학으로 옮긴 것입니다.

  • 입력 $x_1, x_2, \ldots$ — 다른 뉴런(또는 센서)에서 받은 신호.
  • 가중치 $w_1, w_2, \ldots$ — 시냅스 강도. 큰 값일수록 그 입력을 더 중요하게 여깁니다.
  • 편향 $b$ — "기본적으로 발화하기 쉬운 뉴런/까다로운 뉴런"의 차이. 임계값을 옮기는 역할.
  • 활성화 함수 $\sigma$ — "임계값 넘으면 발화"를 부드럽게 표현한 비선형 함수.

가장 기본 단위는 입력에 가중치를 곱하고 활성화 함수를 통과시키는 작은 함수입니다.

$$ z = w_1 x_1 + w_2 x_2 + b, \quad a = \sigma(z) $$

식 한 줄씩 풀어보기

  • $w_1 x_1 + w_2 x_2$ — 입력에 가중치를 곱해서 더한 가중합. 벡터 표기로는 $\mathbf w^\top \mathbf x$. "여러 시냅스에서 들어온 신호를 합산"하는 단계.
  • $+ b$ — 편향을 더해 활성화 임계를 조정. $b$가 크면 약한 입력에도 발화하기 쉬워지고, 음수면 강한 입력이 와야 발화합니다.
  • $z$ — 가중합 + 편향의 결과. 아직 선형입니다.
  • $\sigma(z)$ — 비선형 활성화. 여기서 처음으로 "선형이 아닌 무언가"가 등장합니다. 이 한 단계가 신경망 전체의 표현력을 결정합니다.
  • $a$ — 뉴런의 출력. 다음 층 뉴런들의 입력으로 들어갑니다.

선형 회귀와 무엇이 다른가?

이게 선형 회귀와 무엇이 다를까요? 활성화 함수 $\sigma$가 비선형이라는 점입니다. 선형 회귀는 $\hat y = \mathbf w^\top \mathbf x + b$에서 끝나지만, 퍼셉트론은 그 결과를 한 번 더 비선형으로 비틉니다. 이 작은 차이가 누적되면 "직선밖에 못 그리는 모델" → "어떤 곡선이든 그릴 수 있는 모델"로 도약합니다.

숫자로 직접 풀어보기 — 입력 $\mathbf x = (2, -1)$, 가중치 $\mathbf w = (0.5, 1.0)$, 편향 $b = 0.3$, ReLU 활성화일 때:

$z = 0.5 \cdot 2 + 1.0 \cdot (-1) + 0.3 = 1.0 - 1.0 + 0.3 = 0.3$
$a = \mathrm{ReLU}(0.3) = \max(0, 0.3) = 0.3$

같은 입력이지만 $b = -0.5$였다면 $z = 1.0 - 1.0 - 0.5 = -0.5$, $a = \max(0, -0.5) = 0$. 편향 하나만 바뀌어도 뉴런이 발화하느냐 침묵하느냐가 갈립니다.
흔한 오해 — "퍼셉트론은 0/1만 출력한다"고 알고 계신 분이 많습니다. 1958년 Rosenblatt의 원조 퍼셉트론은 계단 함수(step function)였지만, 현대에는 거의 항상 ReLU/Sigmoid 같은 연속 함수를 씁니다. 이유는 미분이 가능해야 역전파가 작동하기 때문 — 계단 함수는 거의 모든 곳에서 기울기가 0이라 학습이 안 됩니다.

🎮 인터랙티브: 퍼셉트론 — 가중치·편향이 결정경계가 된다

2D 데이터(파랑 클래스 0, 빨강 클래스 1)를 한 개 뉴런이 분류합니다. 가중치 $w_1, w_2$와 편향 $b$를 움직이면 직선 $w_1 x_1 + w_2 x_2 + b = 0$이 회전·평행이동합니다. 배경 색은 sigmoid 출력이고, "학습 시작" 버튼은 한 스텝씩 가중치를 갱신해 자동 분류해 봅니다.

3. 활성화 함수

왜 비선형이 꼭 필요한가?

"활성화 함수가 그렇게 중요한가?" 매우 중요합니다. 비선형이 없으면 신경망은 아무리 깊게 쌓아도 결국 한 층 선형 모델과 같아집니다. 한 줄로 증명해 봅시다.

선형 변환만 두 층 쌓으면 $\mathbf a^{(2)} = W^{(2)}(W^{(1)} \mathbf x + \mathbf b^{(1)}) + \mathbf b^{(2)} = (W^{(2)} W^{(1)}) \mathbf x + (W^{(2)} \mathbf b^{(1)} + \mathbf b^{(2)})$. 괄호 친 부분이 그냥 새로운 행렬과 새로운 벡터일 뿐이므로, 결국 $\mathbf a^{(2)} = W' \mathbf x + \mathbf b'$ 한 층 선형 모델로 환원됩니다. 100층을 쌓아도 똑같아요. 비선형 $\sigma$가 중간에 끼어들어야 비로소 깊이가 의미를 갖습니다.

$$ \text{ReLU: } \sigma(z) = \max(0, z), \quad \text{Sigmoid: } \sigma(z) = \frac{1}{1+e^{-z}}, \quad \text{Tanh: } \sigma(z) = \tanh z $$

비선형이 끼어들지 않으면 아무리 많은 층을 쌓아도 결국 하나의 직선과 같습니다. 활성화 함수가 신경망을 강력하게 만듭니다.

대표 활성화 함수의 직관

  • ReLU가 현재 기본값 — 계산이 빠르고 기울기 소실이 적음. 단점: "죽은 뉴런".
  • Sigmoid는 출력이 (0, 1)이어서 확률 해석 좋지만 깊은 망에서 기울기 소실.
  • Tanh는 평균이 0이어서 Sigmoid보다 학습이 안정적.
  • 현대 변형: Leaky ReLU, GELU, Swish 등이 Transformer/ResNet에서 널리 쓰입니다.

각 함수를 일상 비유로

  • ReLU — "부정적인 신호는 무시, 긍정적인 신호만 그대로 통과." 시험 채점에서 마이너스 점수는 0 처리하고 플러스만 합산하는 느낌. 미분이 0 또는 1로 단순해서 계산이 가볍습니다.
  • Sigmoid — "확률을 출력". 신호가 매우 작으면 0에 가까운 확률, 매우 크면 1에 가까운 확률. 단점은 입력이 ±5만 넘어가도 기울기가 거의 0이 되어 학습이 멈춘다는 점(기울기 소실).
  • Tanh — Sigmoid를 -1~1로 잡아 늘인 형태. 평균이 0이라 다음 층의 입력이 한쪽으로 치우치지 않아 학습이 더 안정적입니다.
  • Leaky ReLU — ReLU에서 "음수도 아주 살짝(0.01배 정도) 통과"시킴. "죽은 뉴런(항상 0만 출력해서 학습이 멈춘 뉴런)" 문제를 완화합니다.
"죽은 뉴런(dying ReLU)"이란? ReLU는 입력이 음수면 출력도 0, 미분도 0입니다. 어떤 뉴런이 큰 음수 영역에 갇히면 그라디언트가 영원히 0이라 학습이 멈춰 버려요. Leaky ReLU·ELU·GELU 같은 변형이 이걸 해결합니다.

4. 다층 신경망 (MLP)

한 줄 비유부터 — 커피숍 추천 시스템

여러분이 친구에게 카페를 추천한다고 해봅시다. 입력으로 (분위기 점수, 가격 점수, 거리)를 받아, 첫 단계에서 "조용한 공부형 카페인가?", "가성비 카페인가?", "분위기 데이트형 카페인가?" 같은 중간 개념을 만들어 봅니다. 그다음 단계에서 친구의 성향에 맞춰 이 중간 개념들을 조합해 최종 점수를 매기죠.

다층 신경망이 하는 일이 정확히 이거예요. 은닉층(hidden layer)은 입력을 "더 추상적인 중간 특징"으로 변환하고, 출력층은 그 특징들을 조합해 최종 답을 냅니다. 사람이 "조용함은 분위기 점수에서 가격 점수를 뺀 값"이라고 안 알려줘도 모델이 데이터로부터 이런 조합을 알아서 찾아냅니다.

뉴런을 층(layer)으로 쌓으면 임의로 복잡한 함수를 근사할 수 있습니다 (보편 근사 정리).

$$ a^{[l]} = \sigma\!\left(W^{[l]} a^{[l-1]} + b^{[l]}\right) $$

식 한 줄씩 풀어보기

  • $a^{[l-1]}$ — 이전 층의 활성화(=출력) 벡터. $l=1$이면 입력 $\mathbf x$ 그 자체.
  • $W^{[l]}$ — $l$번째 층의 가중치 행렬. 행 수 = 이 층의 뉴런 수, 열 수 = 이전 층의 뉴런 수.
  • $W^{[l]} a^{[l-1]}$ — 이전 층 모든 뉴런의 출력을 가중합. 행렬·벡터 곱 한 번으로 한 층의 모든 뉴런을 동시에 계산합니다.
  • $+ b^{[l]}$ — 각 뉴런의 편향을 더함.
  • $\sigma(\cdot)$ — 원소별(element-wise) 비선형 활성화. 입력 벡터의 각 원소에 같은 $\sigma$를 적용.
  • $a^{[l]}$ — $l$번째 층의 출력. 다음 층 입력이 됩니다.

보편 근사 정리 (Universal Approximation Theorem)

한 문장으로: "은닉 뉴런이 충분히 많으면, 한 층짜리 MLP만으로도 임의의 연속 함수를 원하는 정밀도까지 근사할 수 있다." Cybenko(1989), Hornik(1991)이 증명했습니다.

이 정리가 의미하는 바: "표현력은 이미 충분하다". 그렇다면 왜 굳이 층을 깊게 쌓을까요?

  • 효율 — 같은 함수를 표현하더라도 깊게 쌓으면 뉴런 수가 기하급수적으로 줄어듭니다.
  • 학습 용이성 — 깊은 망이 데이터에서 계층적 특징(저수준 → 고수준)을 자연스럽게 학습합니다.
  • 일반화 — 적절히 깊은 망이 새 데이터에 더 잘 일반화되는 경향(경험적 사실).

그림의 구조는 입력 2개 → 은닉 4개 → 출력 1개입니다. 가중치는 입력→은닉이 $2 \times 4 = 8$개, 은닉→출력이 $4 \times 1 = 4$개, 편향까지 합치면 총 $8 + 4 + 4 + 1 = 17$개의 학습 파라미터가 있습니다.

층 크기 감각 잡기 — 실전 모델의 파라미터 수는 어마어마합니다. 작은 MLP는 수만 개, 이미지 분류 CNN은 수백만 개, 최근 대형 언어 모델은 수십~수천억 개입니다. "파라미터가 많을수록 더 복잡한 함수를 표현"하지만, 동시에 데이터도 더 많이 필요합니다(과적합 위험).

5. 순전파와 손실 함수

순전파(Forward Pass) — 입력에서 출력까지

입력 $\mathbf x$를 신경망에 넣어 출력 $\hat{\mathbf y}$를 얻는 과정이 순전파입니다. 한 줄 표어: "입력 → 층마다 행렬곱 + 활성화 → 출력".

$$ \mathbf a^{[0]} = \mathbf x, \quad \mathbf a^{[l]} = \sigma(W^{[l]} \mathbf a^{[l-1]} + \mathbf b^{[l]}), \quad \hat{\mathbf y} = \mathbf a^{[L]} $$

코드로 적으면 단순한 for 루프입니다.

a = x
for l in range(1, L+1):
    z = W[l] @ a + b[l]
    a = sigma(z)
y_hat = a

손실 함수 — "얼마나 틀렸나"

예측 $\hat{\mathbf y}$와 정답 $\mathbf y$가 얼마나 멀리 있는지를 한 숫자로 요약하는 함수가 손실 함수(loss)입니다. 학습이란 "이 손실을 작게 만드는 가중치를 찾는 일"이에요.

회귀에서: 평균 제곱 오차 (MSE)

$$ \mathcal{L}_{\text{MSE}} = \frac{1}{2N}\sum_{i=1}^N (y_i - \hat{y}_i)^2 $$
  • 각 데이터의 (정답 − 예측)을 제곱해서 평균. 부호 상관없이 "거리"만 측정.
  • 제곱이라 큰 오차에 더 민감. 한 점이 크게 빗나가면 손실이 폭발적으로 늘어남.
  • $\frac{1}{2}$가 붙은 이유: 미분할 때 $2$가 떨어져서 식이 깔끔해짐(편의용 상수).

분류에서: 교차 엔트로피 (Cross-Entropy)

$$ \mathcal{L}_{\text{CE}} = -\frac{1}{N}\sum_{i=1}^N \sum_{k=1}^K y_{ik} \log \hat{y}_{ik} $$
  • $y_{ik}$ — 정답 클래스의 원-핫 라벨 (1 또는 0).
  • $\hat y_{ik}$ — 모델이 출력한 클래스 $k$의 확률 (소프트맥스 통과 후).
  • "정답 클래스의 확률을 1에 가깝게" 만들수록 손실이 줄어듭니다. 정답에 0을 출력하면 $-\log 0 = \infty$로 무한히 큰 벌점.
숫자로 직접 풀어보기 — 강아지/고양이 2-클래스 분류에서 정답이 "강아지"(라벨 $\mathbf y = (1, 0)$), 모델 출력이 $\hat{\mathbf y} = (0.7, 0.3)$이라면:

$\mathcal{L}_{\text{CE}} = -(1 \cdot \log 0.7 + 0 \cdot \log 0.3) = -\log 0.7 \approx 0.357$

같은 정답에 모델이 $\hat{\mathbf y} = (0.99, 0.01)$로 더 자신 있게 맞혔다면 $-\log 0.99 \approx 0.010$로 손실이 훨씬 작습니다. "확신 있게 정답을 맞히면 손실이 빠르게 0에 가까워진다"가 교차 엔트로피의 직감.
왜 회귀는 MSE, 분류는 CE인가? 둘 다 통계적으로는 최대우도추정(MLE)에서 자연스럽게 유도됩니다. 회귀에서 출력 노이즈가 가우시안이라 가정하면 MLE가 MSE가 되고, 분류에서 출력이 카테고리 분포라 가정하면 MLE가 교차 엔트로피가 됩니다. 즉 손실 함수 선택은 "데이터 분포에 대한 가정"입니다.

6. 역전파와 경사 하강법

경사 하강법 — 산을 내려가는 등산객

손실 $\mathcal L(\mathbf w)$을 작게 만드는 $\mathbf w$를 어떻게 찾을까요? 손실을 "산"이라 생각하면 우리는 골짜기를 찾고 싶은 등산객입니다. 산의 모든 지점을 다 가볼 수는 없으니, "지금 서 있는 곳에서 가장 가파른 내리막을 따라 한 발 내려가자" — 이게 경사 하강법(gradient descent)이에요.

$$ \mathbf w_{\text{new}} = \mathbf w - \eta \, \nabla_{\mathbf w} \mathcal{L} $$
  • $\nabla_{\mathbf w} \mathcal L$ — 손실의 그라디언트(기울기 벡터). "가장 가파른 오르막 방향"을 가리킵니다.
  • $-\nabla \mathcal L$ — 그 반대, 즉 "가장 가파른 내리막 방향".
  • $\eta$ (또는 $\alpha$) — 학습률(learning rate). 한 발의 크기.

역전파(Backpropagation) — 오차를 거꾸로 흘려보낸다

그라디언트 $\partial \mathcal L / \partial W^{[l]}$을 어떻게 계산할까요? 신경망은 합성 함수가 깊게 쌓인 구조이므로, 미분의 연쇄 법칙(chain rule)을 그래프로 펼쳐서 적용합니다. 핵심 직감 한 줄: "오차를 출력에서 입력 쪽으로 거꾸로 흘려보낸다."

이 한 문장이 역전파의 모든 것입니다. 출력 쪽에서 "예측이 정답에서 얼마나 빗나갔는지"를 측정한 다음, 그 오차 신호를 층마다 거꾸로 전달하면서 각 층의 가중치가 그 오차에 얼마만큼 책임이 있는지를 계산합니다.

한 층 그라디언트의 모양

$l$번째 층의 가중치에 대한 그라디언트는 다음 형태로 깔끔하게 떨어집니다.

$$ \frac{\partial \mathcal L}{\partial W^{[l]}} = \boldsymbol\delta^{[l]} \, (\mathbf a^{[l-1]})^\top $$
  • $\mathbf a^{[l-1]}$ — 이 층 앞쪽(입력 쪽)에서 들어온 활성화. 순전파 때 저장해 둠.
  • $\boldsymbol\delta^{[l]}$ — 이 층 뒤쪽(출력 쪽)에서 거꾸로 흘러온 오차 신호.

한 줄로 외우면: "가중치의 그라디언트 = (앞 층의 활성화) × (뒤에서 거꾸로 흘러온 오차)". 이 패턴이 모든 층에서 똑같이 반복됩니다.

오차 신호의 재귀

$\boldsymbol\delta$가 거꾸로 어떻게 전파되는지를 보는 식:

$$ \boldsymbol\delta^{[l]} = \left( (W^{[l+1]})^\top \boldsymbol\delta^{[l+1]} \right) \odot \sigma'(\mathbf z^{[l]}) $$
  • $(W^{[l+1]})^\top \boldsymbol\delta^{[l+1]}$ — 다음 층의 오차를 가중치 행렬의 전치를 통해 거꾸로 보내는 단계. 순전파에서는 $W$를 곱했으니 역방향에서는 $W^\top$.
  • $\sigma'(\mathbf z^{[l]})$ — 활성화 함수의 도함수. ReLU라면 $z > 0$인 자리만 1, 나머지는 0.
  • $\odot$ — 원소별 곱(Hadamard product).

여러 층의 미분은 연쇄 법칙(chain rule)으로 뒤에서 앞으로 전파됩니다 — 이것이 역전파(backpropagation)입니다.

왜 굳이 역방향인가? 만약 그라디언트를 정방향(입력→출력)으로 계산하면 각 가중치마다 전체 그래프를 한 번씩 훑어야 해서 비용이 $O(WN)$($W$=가중치 수, $N$=뉴런 수)로 폭발합니다. 역방향으로 한 번 훑으면 모든 가중치의 그라디언트를 $O(W)$에 얻을 수 있어요. 1986년 Rumelhart·Hinton·Williams의 결정적 통찰입니다.

가장 작은 신경망에서 손으로 따라가는 역전파

식만 보면 어려우니, 입력 1개 · 은닉 1개 · 출력 1개짜리 미니 신경망에서 한 번 풀어 봅시다. 뉴런 한 개씩이라 행렬 없이 곱셈·덧셈만으로 끝납니다.

  • 모델: $z_1 = w_1 x + b_1$, $a_1 = \sigma(z_1)$, $\hat y = w_2 a_1 + b_2$ (출력은 그냥 선형, 활성화 없음).
  • 샘플: $x = 1.0$, 정답 $y^* = 1.0$. 초기 가중치 $w_1 = 0.5,\ b_1 = 0,\ w_2 = 0.5,\ b_2 = 0$, 활성화는 ReLU.
  • 손실: $\mathcal L = \tfrac12 (\hat y - y^*)^2$.

① 순전파. $z_1 = 0.5 \cdot 1 + 0 = 0.5$, $a_1 = \mathrm{ReLU}(0.5) = 0.5$, $\hat y = 0.5 \cdot 0.5 + 0 = 0.25$. 오차는 $\hat y - y^* = -0.75$, $\mathcal L = 0.281$.

② 출력층 그라디언트. 손실의 $\hat y$에 대한 미분이 곧 출력 오차 신호 $\delta_2 = \hat y - y^* = -0.75$. 여기서 $\partial \mathcal L / \partial w_2 = \delta_2 \cdot a_1 = -0.75 \cdot 0.5 = -0.375$, $\partial \mathcal L / \partial b_2 = \delta_2 = -0.75$.

③ 은닉층으로 거꾸로. 다음 층 가중치를 통해 오차가 흘러옵니다: $\delta_1 = w_2 \cdot \delta_2 \cdot \sigma'(z_1) = 0.5 \cdot (-0.75) \cdot 1 = -0.375$ (ReLU의 도함수는 $z_1>0$이므로 1). 그래서 $\partial \mathcal L / \partial w_1 = \delta_1 \cdot x = -0.375 \cdot 1 = -0.375$, $\partial \mathcal L / \partial b_1 = \delta_1 = -0.375$.

④ 한 발 내려가기. 학습률 $\eta = 1.0$로 업데이트: $w_2 \gets 0.5 - 1.0 \cdot (-0.375) = 0.875$, 마찬가지로 $w_1 \gets 0.875$. 다시 순전파해 보면 $\hat y = 0.766$, $\mathcal L = 0.0274$로 손실이 약 1/10로 줄었습니다 — 이게 한 스텝의 역전파입니다.

핵심 한 줄 — 역전파는 거창해 보이지만, "각 층마다 (앞 층의 활성화 × 뒤에서 온 오차)를 곱해 그라디언트를 만든다"는 같은 패턴을 반복하는 일이에요. 위 예제를 손으로 두 번만 풀어 보면 식의 모든 모양이 자연스러워집니다.

미니배치 SGD

전체 데이터 $N$개의 손실을 매번 계산하면 너무 느립니다. 보통은 작은 묶음(미니배치, 보통 32–256개)을 무작위로 뽑아 그 묶음에서 추정한 그라디언트로 업데이트합니다 — 이를 미니배치 확률적 경사 하강법(mini-batch SGD)이라 부릅니다.

  • 한 번의 업데이트가 빠름 (전체 대신 일부만 봄).
  • 그라디언트가 노이즈를 머금어 오히려 얕은 골짜기를 빠져나오기 좋음.
  • $N=$ 32~256이 표준. GPU 메모리에 따라 결정.

현대 옵티마이저: SGD + 모멘텀, Adam

  • Momentum — 최근 그라디언트의 지수 가중 평균을 따라 진행. "공이 골짜기에서 굴러갈 때 속도가 누적되듯" 진동을 줄이고 수렴을 빠르게 합니다.
  • Adam — 모멘텀 + 적응적 학습률(파라미터마다 학습률을 자동 조절). 거의 모든 기본 설정에서 잘 작동해 현대 딥러닝의 기본 옵티마이저로 자리 잡았습니다.

🎮 인터랙티브: 손실 곡면 위의 경사 하강 — η와 모멘텀

늘어진 타원 손실 $f(x,y) = x^2 + 10y^2$ 위에서 GD가 골짜기로 내려갑니다. $\eta$가 너무 크면 진동·발산하고, 모멘텀 $\beta$를 키우면 진동이 누적 평균으로 잡혀 수렴이 빨라집니다. 캔버스를 클릭하면 시작점을 옮길 수 있어요.

7. 학습 하이퍼파라미터와 과적합

학습률 $\eta$ — 한 발의 크기

  • $\eta$가 너무 크면 골짜기를 건너뛰어 발산 — 손실이 NaN이 되거나 진동.
  • $\eta$가 너무 작으면 학습이 한없이 느려짐 — 수만 에폭 후에도 수렴 못 함.
  • 실전에선 보통 $10^{-4} \sim 10^{-2}$에서 시작, 학습 진행에 따라 감소(learning rate schedule)시킵니다.

배치 크기 — 한 번에 몇 개를 볼까

  • 작은 배치(예: 8) — 그라디언트 노이즈 큼. 일반화에 유리하지만 GPU 활용도 낮음.
  • 큰 배치(예: 1024) — 그라디언트 안정. GPU 효율은 좋지만 과적합·일반화 저하 우려.
  • 일반적으로 32, 64, 128 같은 2의 거듭제곱이 GPU 친화적입니다.

에폭(epoch)

전체 학습 데이터를 한 바퀴 다 본 횟수입니다. 에폭이 너무 적으면 과소적합(underfitting), 너무 많으면 과적합(overfitting)이 일어납니다.

과적합 — "외워 버린" 모델

과적합이란 모델이 훈련 데이터를 통째로 외워서, 처음 보는 데이터에는 형편없이 동작하는 현상입니다. 시험 전날 기출문제만 통째로 외운 학생이 살짝 변형된 문제에서 무너지는 것과 같습니다.

  • L2 정규화(weight decay) — 손실에 $\lambda \|\mathbf w\|^2$ 항을 추가해 가중치 크기를 억제. (L9 정규화 참고)
  • 드롭아웃(dropout) — 학습 중 일부 뉴런을 무작위로 끔(예: 50% 확률). 매 미니배치마다 다른 부분망이 학습되는 셈이라 일종의 앙상블 효과.
  • 조기 종료(early stopping) — 검증 손실이 더 이상 줄지 않으면 학습을 중단. "외우기 직전에 멈추기."
  • 데이터 증강(data augmentation) — 이미지 회전·잘라내기·색상 변화로 훈련 데이터를 늘림.
  • 배치 정규화(batch norm) — 각 층의 입력 분포를 매 미니배치에서 정규화. 학습 안정화.
훈련 손실 vs 검증 손실 — 둘 다 모니터링하는 게 핵심. 훈련 손실은 계속 줄어들지만 검증 손실이 다시 올라가기 시작하면 그 시점이 과적합 시작점입니다. 이때 멈추는 게 조기 종료의 본질.

9. 로봇공학에서의 활용

End-to-End 학습

전통적 자율주행은 "카메라 → 차선 검출 → 경로 계획 → 핸들 각도"처럼 단계마다 사람이 손으로 설계한 모듈을 거칩니다. End-to-end 학습은 이 모든 단계를 한 신경망으로 대체합니다 — 입력은 카메라 이미지, 출력은 핸들 각도. NVIDIA의 PilotNet(2016)이 대표 사례입니다. 장점은 단순함과 강한 성능, 단점은 디버깅 어려움과 데이터 의존성입니다.

강화학습 정책망

강화학습(L13–L15에서 다룸)에서 정책(policy)은 "상태 → 행동" 매핑인데, 이걸 신경망으로 표현하는 것이 표준입니다. 로봇 팔이 물체를 잡거나 사족 보행 로봇이 거친 지형을 걷는 정책이 모두 이 방식으로 학습됩니다. Boston Dynamics와 ETH Zurich의 ANYmal이 신경망 정책으로 산악 지형을 누비는 영상을 본 적 있을 거예요.

모델 학습 (System Identification)

로봇의 동역학 $\dot{\mathbf x} = f(\mathbf x, \mathbf u)$를 신경망으로 학습. L7에서 다룬 시스템 식별의 데이터 기반 버전입니다. 마찰·풍하중처럼 모델링하기 어려운 효과가 있을 때 유용합니다.

지각(Perception)

카메라 이미지를 입력받아 보행자·차선·차량을 검출하는 데는 보통 합성곱 신경망(CNN, L11에서 다룸)이 쓰입니다. MLP의 가중치 행렬이 모든 입력 픽셀을 직접 연결하는 데 비해, CNN은 작은 윈도우(필터)를 슬라이딩하며 공유된 가중치를 사용해 파라미터 수가 훨씬 적습니다.

학습 기반 상태 추정

L4–L5에서 배운 칼만 필터를 신경망으로 보강하거나 대체할 수 있습니다. 예를 들어 IMU 노이즈가 비가우시안일 때 학습 기반 추정기가 더 잘 동작하는 경우가 있습니다.

주의 — 안전이 중요한 로봇에서 신경망은 강력하지만 훈련 분포 밖(out-of-distribution)에서 어떻게 동작할지 보장하기 어렵습니다. 자율주행·의료 로봇처럼 안전이 중요한 응용에서는 보통 신경망 단독보다 전통 제어(L8 PID, L11 MPC)와 결합한 하이브리드 접근을 선호합니다.

10. PyTorch 코드

위의 sin 함수 학습 데모를 PyTorch로 작성하면 다음과 같습니다. 핵심은 model 정의 + loss.backward() 한 줄로 역전파가 자동 수행된다는 것입니다.

import torch
import torch.nn as nn

model = nn.Sequential(
    nn.Linear(1, 20),
    nn.ReLU(),
    nn.Linear(20, 1),
)
loss_fn = nn.MSELoss()
opt = torch.optim.Adam(model.parameters(), lr=1e-2)

x = torch.linspace(-3.14, 3.14, 200).unsqueeze(1)
y = torch.sin(x)

for ep in range(2000):
    pred = model(x)
    loss = loss_fn(pred, y)
    opt.zero_grad(); loss.backward(); opt.step()
print("loss:", loss.item())

줄별 해설

  • nn.Sequential(...) — 층을 차례로 연결한 모델. 1차원 입력 → 20개 ReLU 은닉 → 1차원 출력.
  • nn.MSELoss() — 평균 제곱 오차 손실 함수.
  • torch.optim.Adam(..., lr=1e-2) — Adam 옵티마이저. 학습률 0.01.
  • opt.zero_grad() — 이전 미니배치의 그라디언트 누적을 초기화. 이 줄을 빼먹는 게 입문자의 단골 실수.
  • loss.backward() — 자동 미분으로 모든 파라미터의 그라디언트 계산. 역전파가 한 줄로 끝납니다.
  • opt.step() — 그라디언트를 사용해 파라미터 업데이트.

10.5. 신경망 학습 흐름 정리

  1. 입력·출력 데이터 정규화.
  2. 모델 정의 (층 수, 뉴런 수, 활성).
  3. 손실 함수 정의 (회귀 MSE, 분류 CE).
  4. 옵티마이저 선택 (Adam이 표준).
  5. 매 에폭: forward → loss → backward → step.
  6. 검증 손실로 조기 종료.
  7. 테스트 평가, 시각화.

10.6. 보편 근사 정리의 의미

"은닉층 1개에 뉴런만 충분하면 어떤 연속 함수든 임의 정밀도로 근사 가능." 이론적으로 1층이면 충분하지만, 실제로는 깊이 쌓는 게 표현력·학습 효율이 모두 우수합니다(표현력의 깊이 vs 너비).

10.7. 자주 만나는 문제

  • 학습이 안 됨 — 학습률 ÷10, 데이터 정규화 확인.
  • 과적합 — 드롭아웃, weight decay, 데이터 증강.
  • NaN 손실 — 학습률 큼 또는 log(0). gradient clipping.
  • 죽은 ReLU — 음수 영역 항상 0. Leaky ReLU 또는 He 초기화.

10.8. 로봇 응용에서의 신경망

  • End-to-End 자율주행 — 카메라 → 조향각 (NVIDIA PilotNet).
  • 로봇 매니퓰레이션 — 픽셀 → 액션 (Open-X Embodiment).
  • 강화학습 — 가치 함수·정책 함수 근사.
  • System ID — 데이터로부터 동역학 학습 → MPC 모델로.

10.10. 학습률 튜닝 4단계

  1. $10^{-1}$로 시작. 발산하면 ÷10.
  2. 안정 영역에서 정밀 탐색.
  3. cosine annealing으로 점진적 감소.
  4. warmup이 필요한 모델(트랜스포머)은 처음 몇 에폭 점진적 증가.

11. 다음 강의 미리보기

L9·L10에서 데이터로부터 함수를 학습하는 도구를 갖췄으니, 이제 다시 제어 이야기로 돌아갑니다. L7의 PID는 "지금 오차"만 보고 명령을 만들었습니다. 하지만 운전을 잘하려면 코너 100 m 앞에서 미리 감속해야 하죠. 미래를 예측해 한 번에 최적의 행동을 푸는 게 다음 주제예요.

L11. 모델 예측 제어 (MPC)에서는 ① L2·L3에서 만든 동역학 모델로 앞으로 $N$ 스텝을 시뮬레이션하고, ② 그 미래에서 비용(목표 추종 오차 + 입력 크기)이 가장 작아지는 입력 시퀀스를 최적화로 풀고, ③ 그중 첫 입력만 실행한 뒤 다시 측정값을 받아 처음부터 반복합니다. L9·L10의 학습된 모델을 동역학 자리에 끼워 넣으면 데이터 기반 MPC가 되고, 이는 L12에서 다룹니다.

11.5. 신경망 디버깅 체크리스트

  • 입력 모양 출력해 (B, 차원) 확인.
  • 입력 통계 (mean, std) 확인. 표준화 안 됐으면 먼저.
  • 작은 데이터(10개)로 손실 0 수렴 확인 (sanity check).
  • 학습률 한 자릿수씩 변화시키며 실험.
  • 학습/검증 손실 곡선 그리기.
  • 그래디언트 노름 모니터링.
  • 활성·초기화 페어링(ReLU+He, tanh+Xavier) 확인.

13. 손계산 워크쓰루 — 작은 MLP 한 번 굴리기

입력 2, 은닉 2, 출력 1, ReLU. $W_1 = [[1, 0], [0, 1]]$, $b_1 = (0, 0)$. $W_2 = [1, 1]$, $b_2 = 0$.

입력 $\mathbf x = (2, -1)$.

  • 은닉 입력 $z_1 = (2, -1)$.
  • ReLU 후 $h = (2, 0)$ — 음수 잘림.
  • 출력 $y = 1\cdot 2 + 1 \cdot 0 = 2$.

이 한 흐름이 모든 신경망의 본질.

18. 한 페이지 핵심 식 모음

  • 가중합: $z = Wx + b$
  • 활성: $h = \text{ReLU}(z) = \max(0, z)$
  • 출력: $\hat y = \sigma(W_2 h + b_2)$ (분류)
  • 손실: $L = -\sum y \log \hat y$ (CE)
  • 업데이트: $\theta \leftarrow \theta - \alpha \nabla L$

19. 신경망과 제어의 만남 — 미리보기

L10 신경망과 L7 PID·L11 MPC가 만나는 지점:

  • 학습된 동역학 — 신경망이 $f(x, u)$를 학습 → MPC의 모델로.
  • 강화학습 정책 — 신경망이 직접 $\pi(u|x)$를 출력.
  • 제어기 모방 — MPC의 입력을 신경망에 모방학습 → 빠른 추론.
  • Imitation Learning — 사람의 시연을 신경망이 학습 (자율주행).

이 모든 것이 L11 MPC와 L12 Advanced MPC에서 다시 등장합니다.

20. 한 학기 전체 진도판 — L10 시점

  • L1~L8 — 모델링·추정·제어 ✓
  • L9~L10 — 데이터 학습 ◉ 현재
  • L11~L12 — MPC + Learning-based

한 학기의 큰 분기점에 도착했습니다.

24. 부록 — Hello World 신경망

import torch
import torch.nn as nn

# XOR
X = torch.tensor([[0,0],[0,1],[1,0],[1,1]], dtype=torch.float)
y = torch.tensor([[0],[1],[1],[0]], dtype=torch.float)

model = nn.Sequential(nn.Linear(2,4), nn.ReLU(), nn.Linear(4,1), nn.Sigmoid())
opt = torch.optim.Adam(model.parameters(), lr=0.05)
loss_fn = nn.BCELoss()

for ep in range(2000):
    yh = model(X)
    L = loss_fn(yh, y)
    opt.zero_grad(); L.backward(); opt.step()
print((model(X) > 0.5).int().squeeze())  # tensor([0, 1, 1, 0])
