Week 12. 디지털 논리 — 0과 1로 짓는 컴퓨터

잡음에 강한 0/1의 세계, 부울 대수와 논리 게이트, 그리고 "기억하는 회로"의 첫 발걸음.

  3. 부울 대수와 기본 7개 공리
  4. 논리 게이트 — AND·OR·NOT·NAND·NOR·XOR

1. 왜 디지털인가

아날로그 카세트 테이프는 복사할수록 음질이 나빠지지만 MP3는 100번 복사해도 원본과 똑같습니다. 그 차이는 단 하나 — 아날로그는 연속값, 디지털은 0/1 두 값만이라는 점입니다.

1.1 잡음 강건성

전송선을 지나며 잡음이 섞이는 건 피할 수 없습니다.

  • 아날로그라면 3.2V를 보냈는데 3.5V로 도착하면 영원히 복원 불가.
  • 디지털이라면 "3V 이상은 1, 2V 이하는 0"이라는 약속만 있으면 3.5V도 그냥 1로 읽고 끝 — 원본 완벽 복원.

1.2 왜 하필 2진수인가

0~5V를 10단계로 쪼개면 한 칸이 0.5V — 잡음 0.3V만 섞여도 헷갈립니다. 2단계로 쪼개면 한 칸이 2.5V — 잡음 1V가 와도 안전. 가장 덜 쪼갠 것이 2진수이고, 그래서 잡음·회로 단순성 모두에서 압도적으로 유리해 디지털 시스템의 표준이 되었습니다.

2. 2진수와 2의 보수

10진수 235 = $2 \cdot 100 + 3 \cdot 10 + 5 \cdot 1$ (자리값 100, 10, 1).
2진수 $(1011)_2$ = $1 \cdot 8 + 0 \cdot 4 + 1 \cdot 2 + 1 \cdot 1 = 11$ (자리값 8, 4, 2, 1).

  • 비트(bit) = 2진수 한 자리 (Binary Digit)
  • MSB = 가장 큰 자리 비트 (Most Significant Bit)
  • LSB = 가장 작은 자리 비트 (Least Significant Bit)

2.0 진법 변환 빠른 가이드

2진수 ↔ 10진수

  • 2 → 10: 각 자리 비트 × 2의 거듭제곱을 합산. 예) $1011_2 = 8+0+2+1 = 11_{10}$.
  • 10 → 2: 2로 나눈 나머지를 거꾸로 적기. 예) $13 \div 2 = 6\,r1, 6\div 2 = 3\,r0, 3\div 2 = 1\,r1, 1\div 2 = 0\,r1 \to 1101$.

16진수 (Hex) 한 줄 정리

2진수 4비트를 한 자리 16진수로 압축 표기. 컴퓨터 메모리 주소·색상 코드에 흔함.

  • 0~9는 그대로, 10=A, 11=B, 12=C, 13=D, 14=E, 15=F.
  • 예: $1010_2 = A_{16}$, $11111111_2 = FF_{16} = 255_{10}$.
  • 웹 색상 #FF0000 = 빨강 (R=255, G=0, B=0).

2.1 음수 표현 — 2의 보수

컴퓨터는 덧셈 회로만 잘 만들어두고 뺄셈은 "−값을 더하는 것"으로 처리합니다. 그러려면 −3을 어떻게 2진수로 쓸지 정해야 하는데, 표준이 2의 보수(two's complement)입니다.

만드는 법 (2단계)

  1. 모든 비트를 뒤집는다 (1의 보수)
  2. 1을 더한다

예제: $(1101100)_2$의 2의 보수.

  1. 뒤집기: $1101100 \to 0010011$
  2. +1: $0010011 + 1 = 0010100$

검산: $1101100 + 0010100 = 10000000$. 맨 왼쪽 1은 오버플로로 버려져 결과는 0 — 두 수의 합이 0이므로 서로 반대 부호 관계 ✓.

2의 보수가 1의 보수보다 표준이 된 이유는 1의 보수는 +0과 −0 두 가지가 따로 존재해 혼란스럽기 때문입니다(2의 보수는 0이 하나뿐).

$N$비트로 표현 가능한 범위 — 2의 보수 8비트는 $-128 \sim +127$, 16비트는 $-32{,}768 \sim +32{,}767$. MSB가 부호 비트 역할을 하지만 단순히 부호만 떼는 게 아니라 자리값이 음수($-2^{N-1}$)가 됩니다. 예를 들어 8비트 $11111111$은 $-128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = -1$.

🎮 인터랙티브: 2의 보수 변환기 — 비트 뒤집기 + 1

10진수를 입력하면 8비트 2의 보수로 변환하는 과정을 단계별로 보여줍니다. 부호 있는 8비트로 표현 가능한 범위는 −128 ~ +127.

3. 부울 대수

변수가 0 또는 1만 가지는 특별한 수학. 영국의 조지 부울(George Boole)이 1850년대에 만들었고, 디지털 회로 설계의 공용 언어가 되었습니다. 단 세 가지 기본 연산만 있으면 모든 디지털 회로를 표현할 수 있어요.

연산기호의미스위치 비유
AND (곱)$A \cdot B$ 또는 $AB$둘 다 1일 때만 1직렬 스위치 (둘 다 켜야 전등 ON)
OR (합)$A + B$하나라도 1이면 1병렬 스위치 (하나만 켜도 전등 ON)
NOT (부정)$\bar{A}$ 또는 $A'$0↔1 반전반대로 작동

3.1 기본 공리 7개

$$ A \cdot 0 = 0, \;\; A \cdot 1 = A, \;\; A + 0 = A, \;\; A + 1 = 1 $$ $$ A \cdot A = A, \;\; A + A = A, \;\; A \cdot \bar{A} = 0, \;\; A + \bar{A} = 1 $$

일반 대수와 가장 다른 두 가지: $A + A = A$, $1 + 1 = 1$. "켜짐 + 켜짐 = 켜짐", 그냥 켜짐일 뿐입니다. 외울 때 한 줄: 0과의 연산 — AND는 먹음, OR는 무시 / 1과의 연산 — AND는 무시, OR는 먹음.

3.2 세 가지 핵심 법칙

분배법칙: $A(B+C) = AB + AC$ (일반 대수와 동일)

드모르간 법칙 — 회로 변환의 핵심:

$$ \overline{A \cdot B} = \bar{A} + \bar{B}, \qquad \overline{A + B} = \bar{A} \cdot \bar{B} $$

"바깥의 NOT = 안에서 AND/OR를 바꾸고 각자에 NOT". 즉 NAND를 NOT-OR(입력 둘 다 반전)로, NOR를 NOT-AND로 변환할 수 있어요.

3.5 부울 대수 — 7개 기본 공리

  • $A + 0 = A$, $A \cdot 1 = A$ (항등원)
  • $A + 1 = 1$, $A \cdot 0 = 0$ (지배 원소)
  • $A + A = A$, $A \cdot A = A$ (멱등성)
  • $A + \bar A = 1$, $A \cdot \bar A = 0$ (보원)
  • $\bar{\bar A} = A$ (이중 부정)
  • 교환·결합·분배 법칙 (일반 대수와 동일)

3.6 드모르간의 법칙 — 회로 변환의 만능 도구

$$ \overline{A \cdot B} = \bar A + \bar B, \quad \overline{A + B} = \bar A \cdot \bar B $$

"전체 부정 = 각자 부정 + 연산자 교환"으로 외워두세요. 이 법칙이 NAND·NOR로 모든 게이트를 구현 가능하게 만드는 핵심.

3.7 부울 식 간략화 — 예제

$F = A\bar B + AB$를 간략화:

$$ F = A(\bar B + B) = A \cdot 1 = A $$

$F = AB + \bar A B + AB$ (중복 항):

$$ F = AB + \bar A B = (A + \bar A)B = B $$

4. 논리 게이트

입력 신호를 받아 규칙대로 출력을 내보내는 작은 부품(gate = 문). 모든 디지털 IC가 이 게이트들의 조합입니다.

게이트출력진리표
NOT$\bar{A}$$0\to 1, 1\to 0$
AND$AB$11일 때만 1
OR$A+B$둘 다 0일 때만 0
NAND$\overline{AB}$11일 때만 0
NOR$\overline{A+B}$둘 다 0일 때만 1
XOR (배타적 OR)$A \oplus B$다를 때만 1
XNOR (등가)$A \odot B$같을 때만 1
NAND의 놀라운 성질 — NAND 게이트 하나만 가지고 다른 모든 게이트(AND·OR·NOT)를 만들 수 있습니다. NOT: 두 입력에 같은 신호 → NAND. AND: NAND 결과를 다시 NAND(=NOT). OR: 두 입력을 각각 NOT한 뒤 NAND. CMOS 공정에서 NAND가 가장 만들기 쉬워서 모든 디지털 IC의 기본 블록이 됩니다.

🎮 인터랙티브: 논리 게이트 7가지

입력 A, B를 토글하며 7가지 게이트의 출력을 동시에 봅니다. 출력이 1이면 파란색.

4.1 XOR의 의미와 응용

XOR 진리표: $00 \to 0$, $01 \to 1$, $10 \to 1$, $11 \to 0$. "두 입력이 다르면 1".

  • 비밀번호 일치 검사 — 두 비트열이 정확히 같은지 빠르게 확인
  • 덧셈 회로 — 캐리 없이 두 비트의 합 (반가산기)
  • 패리티 검사 — 데이터 무결성 확인

4.5 NAND·NOR — 만능 게이트의 비밀

NAND 게이트 하나로 모든 다른 게이트를 만들 수 있습니다.

  • NOT: NAND의 두 입력에 같은 신호 → $\overline{A \cdot A} = \bar A$.
  • AND: NAND 출력에 NAND-NOT → $\overline{\overline{A \cdot B}} = AB$.
  • OR: 입력 둘 다 부정 후 NAND → $\overline{\bar A \cdot \bar B} = A + B$ (드모르간).

그래서 CMOS 칩 제조에서 NAND를 가장 많이 만듭니다 — "한 게이트로 다 만들 수 있어 비용 절약."

4.6 진리표 작성 — 단계별

  1. 입력 변수 수 N 결정 → 행 수 = $2^N$.
  2. 모든 입력 조합을 0~$2^N-1$로 나열 (2진수).
  3. 각 조합에 대해 출력 값 계산.
  4. 출력이 1인 행만 골라 SOP (최소항의 합) 식으로 정리.

5. 조합 논리 회로 — 진리표에서 게이트로

"지금 입력만 보고 지금 출력을 결정"하는 회로. 설계 절차는 거의 항상 동일합니다.

  1. 원하는 동작을 진리표로 작성. 입력이 $n$개면 행 수는 $2^n$.
  2. 논리식 유도 — 출력이 1인 행마다 입력 조합을 AND로 묶고, 그것들을 OR로 합침(SOP, Sum of Products).
  3. 간략화 — 부울 대수 법칙이나 카르노 맵(K-map)으로 식을 단순화.
  4. 게이트 조립 — 간략화된 식을 그대로 게이트로 그림.

5.1 카르노 맵(K-Map)으로 식 간소화

SOP로 뽑은 식은 보통 너무 복잡합니다. 카르노 맵은 진리표를 격자로 그려놓고 인접한 1을 묶어 식을 단순화하는 시각적 도구예요. 2변수, 3변수 K-map의 핵심 규칙은 "인접한 1들의 직사각형(2의 거듭제곱 크기)을 가능한 크게 묶는다"입니다.

그루핑 규칙 4가지

  1. 크기는 1, 2, 4, 8, ... — 2의 거듭제곱만 가능. 3개나 5개로는 묶지 못함.
  2. 가능한 크게 묶기. 한 그룹이 클수록 식의 항이 단순해짐 (한 그룹 = 한 곱항).
  3. 모든 1은 적어도 한 그룹에 속해야 함 (중복 허용).
  4. K-map의 행/열은 그레이 코드 순서(00, 01, 11, 10) — 인접 칸은 비트 한 개만 다름. 그래서 양쪽 끝(맨 위/아래, 좌/우)도 인접 취급해 "랩어라운드 묶음"이 가능.

한 그룹이 같은 변수의 두 값(0과 1) 모두를 포함하면 그 변수는 식에서 사라집니다. 그래서 그룹이 클수록 살아남는 변수가 적어 식이 짧아져요.

5.2 반가산기(Half Adder)와 전가산기(Full Adder)

두 비트 $A$, $B$의 덧셈을 생각해 봅시다. 진리표:

$A$$B$합 $S$자리올림 $C$
0000
0110
1010
1101
  • 합 $S = A \oplus B$ (XOR — 다르면 1)
  • 자리올림 $C = A \cdot B$ (AND — 둘 다 1일 때만 올림)

이것이 반가산기. 자리올림 입력 $C_{\text{in}}$까지 받는 전가산기는:

$$ S = A \oplus B \oplus C_{\text{in}}, \qquad C_{\text{out}} = AB + (A \oplus B)C_{\text{in}} $$

전가산기를 $n$개 직렬로 연결하면(이전 단의 $C_{\text{out}}$이 다음 단의 $C_{\text{in}}$로) $n$비트 덧셈기가 만들어집니다 — CPU의 ALU(Arithmetic Logic Unit) 핵심 블록의 시작점입니다. 빼기는 W12.2의 2의 보수로 처리하니 같은 덧셈기 회로가 빼기에도 그대로 쓰여요.

🎮 인터랙티브: 반가산기 — 두 비트의 합과 자리올림

$S = A \oplus B$ (XOR), $C = A \cdot B$ (AND). 두 입력 비트로 단순한 1비트 덧셈을 시연합니다 — CPU 가산기의 첫 블록.

5.3 디코더와 멀티플렉서

  • 디코더(Decoder) — $n$비트 입력을 받아 $2^n$개 출력 중 하나만 1로 내보냄. 메모리 주소 디코딩, 7-세그먼트 표시 등에.
  • 멀티플렉서(MUX) — $2^n$개 입력 중 하나를 선택해 출력. 데이터 버스 공유, ALU 연산 선택에.

5.5 카르노 맵 (Karnaugh Map) — 시각적 간략화

진리표를 2차원 격자로 재배치해 인접한 1을 묶어 변수를 소거. 식을 손쉽게 줄이는 도구.

2변수 카르노 맵

$2 \times 2 = 4$칸. 가로 = $B$, 세로 = $A$. 인접 칸끼리 묶어서 공통 변수만 남김.

3변수 카르노 맵

$2 \times 4 = 8$칸. 가로 = $BC$ (00, 01, 11, 10 순서!), 세로 = $A$. 그레이 코드 순서라 옆 칸끼리 한 비트만 차이.

묶음 크기 = 사라지는 변수

  • 2칸 묶음 → 1변수 사라짐.
  • 4칸 묶음 → 2변수 사라짐.
  • 8칸 묶음 → 3변수 사라짐.

5.6 인코더 vs 디코더 — 짝꿍 회로

  • 인코더: 8개 입력 중 1개만 1 → 3비트 코드. 키보드, 우선순위 결정기.
  • 디코더: 3비트 코드 → 8개 출력 중 1개만 1. 메모리 주소 선택, 7-segment 디스플레이.

5.7 가산기 — 컴퓨터의 기본 산술

  • 반가산기 (Half Adder): 두 비트 더하기 → Sum (XOR), Carry (AND).
  • 전가산기 (Full Adder): 입력 캐리까지 받음. n비트 가산기는 전가산기 n개 직렬.
  • CPU의 ALU는 전가산기 32~64개 + 시프터·논리연산기 모음.

5.8 멀티플렉서 (MUX) — 신호 선택기

$2^n$개 입력 중 $n$비트 선택 신호로 하나를 출력. CPU의 레지스터 선택, 데이터 버스에 흔함.

6. 순차 논리 회로 — 기억하는 회로

지금까지 배운 회로는 지금 입력만 보고 출력을 결정합니다. 그런데 카운터·메모리는 과거의 상태도 기억해야 해요. 이 "기억"을 가능하게 하는 것이 피드백 루프 — 출력이 입력으로 다시 돌아오는 구조입니다.

6.1 SR 래치 — 가장 간단한 1비트 메모리

NOR 두 개를 교차 연결한 회로:

  • S(Set) = 1 → Q = 1로 기록
  • R(Reset) = 1 → Q = 0으로 기록
  • S = R = 0 → 이전 상태 유지 (이것이 "기억"!)
  • S = R = 1 → 금지(둘 다 동시에 1로 만들 수 없음)

6.2 D 플립플롭 — 클럭에 동기화된 1비트 저장소

래치는 입력이 변하면 즉시 출력이 변하지만, 플립플롭은 클럭(clock) 신호의 모서리(edge)에서만 입력을 받아들입니다. 모든 디지털 시스템이 같은 클럭에 맞춰 일제히 갱신되도록 동기화하는 핵심 부품이에요.

D 플립플롭의 동작: 클럭 상승 에지에서 D 입력 값이 Q로 복사되고, 그 다음 클럭까지 유지.

$$ Q_{\text{next}} = D \quad (\text{at rising edge of clock}) $$

🎮 인터랙티브: SR 래치 — 1비트 메모리

S, R을 펄스로 쳐보세요. S=1로 설정, R=1로 리셋, 둘 다 0이면 이전 값 유지 — 이것이 "기억"의 본질. 스텝으로 시간 진행.

6.3 레지스터와 카운터

  • 레지스터(Register) — D 플립플롭 $n$개를 같은 클럭에 묶어 $n$비트 데이터 저장. CPU의 작업 메모리.
  • 카운터(Counter) — 클럭마다 1씩 증가하는 회로. 시계, 타이머, PWM의 기반.

6.5 메모리 소자 — 비트를 어떻게 저장하나

래치 (Latch) — 단순 1비트 저장소

NOT 게이트 두 개를 입력-출력으로 서로 연결 → 두 가지 안정 상태 (00 vs 01에 11 vs 10) → 1비트 저장. 간단하지만 입력이 들어오면 즉시 반응해 잡음에 약함.

플립플롭 — 클록 동기화 메모리

  • D 플립플롭: 클록 에지에 입력 D를 출력 Q에 복사. CPU 레지스터의 기본 단위.
  • JK 플립플롭: SR 래치 개량형. J=K=1이면 토글.
  • T 플립플롭: 토글 전용. 카운터 만들 때 사용.

왜 플립플롭이 래치보다 안전한가

래치는 입력이 변할 때마다 출력이 따라가지만, 플립플롭은 클록 에지에서만 반응 → 잡음에 강하고 동기화가 깔끔함. 모든 현대 CPU·메모리는 플립플롭 기반.

6.6 카운터·시프트 레지스터

  • 카운터: T 플립플롭 n개 직렬 → n비트 이진 카운터 (0~$2^n-1$). 디지털 시계의 초·분·시.
  • 시프트 레지스터: D 플립플롭 n개 직렬, 클록마다 데이터가 한 칸씩 이동. 직병렬 변환에 사용 (USB·UART 통신).

7. 유한상태기계(FSM)

플립플롭으로 현재 상태를 저장하고, 조합 논리로 입력에 따라 다음 상태와 출력을 결정하는 구조. 신호등 제어, 자판기, 게임 컨트롤러, 통신 프로토콜 — 모든 디지털 시스템의 행동 패턴이 결국 FSM입니다.

7.1 FSM의 두 종류 — Moore vs Mealy

  • Moore 머신 — 출력이 현재 상태에만 의존. 신호등처럼 "지금 무슨 상태냐"만 보면 출력이 정해짐.
  • Mealy 머신 — 출력이 현재 상태와 입력에 모두 의존. 보통 더 적은 상태로 같은 동작을 표현 가능.

7.2 신호등 예제 — Moore 머신

3상태 신호등 (Green → Yellow → Red → Green ...). 타이머 입력 $T$가 만료되면 다음 상태로 전이. 상태를 2비트로 인코딩: G=00, Y=01, R=10.

상태 천이표(state transition table):

현재 ($S_1 S_0$)$T$다음 ($S_1' S_0'$)출력 (R Y G)
00 (G)000 (G)001
00 (G)101 (Y)001
01 (Y)001 (Y)010
01 (Y)110 (R)010
10 (R)010 (R)100
10 (R)100 (G)100

이 표를 → 다음 상태 비트 $S_1'$, $S_0'$ 각각에 대한 부울식 → 게이트로 만들어 D 플립플롭의 입력으로 연결하면 회로가 완성됩니다. 출력 $R$, $Y$, $G$는 현재 상태 비트만의 함수라서 (Moore!) 별도 조합 논리로 디코드.

실무에서는 이 과정 전체를 Verilog/VHDL 같은 HDL로 기술하고 합성기가 게이트로 변환합니다 — 회로를 손으로 그리지는 않지만 안에서 일어나는 일은 정확히 위와 같아요.

7.3 클럭 주파수와 타이밍

FSM은 클럭마다 한 번씩 상태가 갱신됩니다. 두 가지 핵심 타이밍 제약:

  • Setup time — 클럭 에지 전에 입력이 안정화되어 있어야 하는 시간
  • Hold time — 클럭 에지 후에 입력이 변하지 않아야 하는 시간

이 두 시간을 위반하면 메타스테이블(metastable) 상태가 발생해 회로가 잘못 동작합니다. 클럭 주파수의 한계는 결국 "한 클럭 안에 조합 논리가 다 끝낼 수 있는가"로 결정돼요. 이래서 GPU·CPU의 클럭이 무한정 못 올라가는 거고요.

7.5 FPGA·ASIC — 커스텀 디지털 칩

  • FPGA (Field-Programmable Gate Array): 게이트와 플립플롭이 미리 박혀 있어 사용자가 회로 구성을 프로그래밍. 시제품·소량 생산에 적합.
  • ASIC (Application-Specific IC): 특정 용도로 마스크부터 설계해 대량 생산. 단가 낮음, 성능 최고. 스마트폰 SoC가 대표.

7.6 HDL — 디지털 회로의 프로그래밍 언어

Verilog, VHDL — 회로를 코드로 기술. 합성 도구가 코드를 게이트 회로로 자동 변환. 한 학기 동안 손으로 그린 회로를 코드 한 줄로 표현.

8. 디지털과 아날로그의 접점

실제 시스템은 디지털만으로 안 됩니다 — 센서는 아날로그 신호를 내고, 액추에이터는 아날로그 전력을 받아야 합니다. 그 사이에 두 가지 변환기가 들어갑니다.

  • ADC (Analog-to-Digital Converter) — 아날로그 전압을 0/1 비트열로. 마이크 → 디지털 오디오 등.
  • DAC (Digital-to-Analog Converter) — 비트열을 아날로그 전압으로. 디지털 오디오 → 스피커 등.

이 두 변환기가 들어가는 자리에는 거의 항상 W11의 op-amp(저역통과 필터·계측 증폭기)가 같이 등장합니다.

9. 우리가 다룬 모든 것의 종착지

아두이노 같은 마이크로컨트롤러 안에는 다음이 한 칩에 들어 있습니다.

  • CPU 코어 — 트랜지스터(W10) 수만 개로 만든 게이트(W12)들의 조합으로 만들어진 ALU와 제어 유닛.
  • 메모리 — 플립플롭과 더 정교한 셀 구조의 거대한 배열.
  • ADC/DAC — 아날로그 회로(W6–W11)의 결정체.
  • GPIO/PWM — 디지털 출력으로 외부 트랜지스터 스위치(W10)를 제어해 모터·LED 구동.

한 학기 동안 배운 옴의 법칙(W2)부터 디지털 논리(W12)까지의 모든 조각이 이 한 칩 안에 다 들어 있습니다. 기전공학 전공자에게 마이크로컨트롤러는 "전기 위에서 도는 작은 컴퓨터"가 아니라 — 이번 학기에 배운 모든 회로 개념의 통합 결과물입니다.

9.5 디지털 통신 프로토콜 — 회로 너머의 세계

  • UART: 가장 간단한 직렬 통신. 시작 비트 + 8 데이터 비트 + 종료 비트. 아두이노의 시리얼 통신.
  • SPI (Serial Peripheral Interface): 4선 (CS, CLK, MOSI, MISO) 마스터-슬레이브. 빠르고 단순.
  • I²C: 2선 (SCL, SDA) 멀티-슬레이브. 여러 센서 한 버스에 연결.
  • USB: 4선 (Vbus, GND, D+, D−), 차동 신호로 잡음에 강함.

모든 프로토콜의 기본은 D 플립플롭과 시프트 레지스터의 조합. 디지털 회로 한 학기 분량이 USB·I²C·SPI 등 모든 통신 표준의 기반입니다.

10. Falstad 실습

  • NAND로 NOT/AND/OR 만들기 — Falstad의 NAND 게이트만으로 다른 게이트의 동작을 직접 구현해 보세요. 진리표를 입력 토글로 검증.
  • 반가산기 — XOR 1개 + AND 1개로 반가산기 회로. 두 입력 비트를 토글하며 합 $S$와 자리올림 $C$를 확인.
  • SR 래치 — NOR 두 개를 교차 연결. S, R 입력을 토글하며 출력 Q가 1비트 메모리처럼 동작하는지 확인.
  • D 플립플롭 + 클럭 — 클럭 상승 에지에서만 입력이 출력으로 복사되는 모습을 스코프로 관찰.
다음 단계 — 이 강의로 회로의 기초가 끝납니다. 다음 학기/심화 과목에서는 다음을 다룹니다:
  • 마이크로컨트롤러 프로그래밍 — Arduino C++로 GPIO·ADC·PWM·인터럽트·타이머 다루기.
  • 센서와 액추에이터 — 온도·압력·IMU·인코더의 신호 조절과 모터의 토크-속도 곡선.
  • 모터 제어 — H-bridge, PWM, 인코더 피드백, P 제어 → 로봇공학특론 L7의 PID로 자연스럽게 이어집니다.

11. 한 학기 마무리

축하합니다. 옴의 법칙 한 줄에서 시작해 디지털 IC가 어떻게 만들어지는지까지 한 바퀴 돌았습니다. 회로 분야는 깊이가 끝없지만, 이 12주의 기초가 단단히 자리 잡혀 있으면 어떤 심화 과목·연구 분야에서도 든든한 발판이 될 거예요.

12. 2진수 ↔ 10진수 ↔ 16진수 변환 — 친절 가이드

디지털 회로는 모든 정보를 0과 1로 표현합니다. 그 0/1의 묶음이 바로 2진수. 16진수는 4비트 한 묶음을 한 글자로 줄여 적은 표기법입니다.

12.1 2진수 → 10진수

각 자리수를 $2^n$ 가중치로 곱해서 더합니다.

비트 자리$2^7$$2^6$$2^5$$2^4$$2^3$$2^2$$2^1$$2^0$
가중치1286432168421
예: $\text{10110101}_2$10110101

$1\cdot128 + 0\cdot64 + 1\cdot32 + 1\cdot16 + 0\cdot8 + 1\cdot4 + 0\cdot2 + 1\cdot1 = 181$.

12.2 10진수 → 2진수

2로 계속 나누면서 나머지를 거꾸로 적습니다. 예 $13 \div 2 = 6 \text{ R } 1$, $6 \div 2 = 3 \text{ R } 0$, $3 \div 2 = 1 \text{ R } 1$, $1 \div 2 = 0 \text{ R } 1$ → 거꾸로 1101. 즉 $13_{10} = 1101_2$.

12.3 2진수 ↔ 16진수

4비트 묶음 단위로 16진수 한 글자로 변환합니다.

4비트10진수16진수
000000
000111
010155
101010A
110012C
111115F

예: $\text{10110101}_2 = 1011\,0101 = \text{B}5_{16}$.

12.4 변환 연습 5문항

  1. $\text{1010}_2$ = ?₁₀
  2. $\text{11111111}_2$ = ?₁₀
  3. $25_{10}$ = ?₂
  4. $\text{F}_{16}$ = ?₂
  5. $\text{2A}_{16}$ = ?₁₀

정답 — ① 10 ② 255 ③ 11001 ④ 1111 ⑤ 42.

13. 부울 대수 — 한 페이지 정리

2진수 회로의 식 변환·간소화에 사용하는 대수입니다. 이걸 아는 만큼 회로를 작게 만들 수 있습니다.

법칙식
항등$A + 0 = A,\quad A \cdot 1 = A$
지배$A + 1 = 1,\quad A \cdot 0 = 0$
중복$A + A = A,\quad A \cdot A = A$
상보$A + \overline{A} = 1,\quad A \cdot \overline{A} = 0$
이중부정$\overline{\overline{A}} = A$
교환$A + B = B + A,\quad A \cdot B = B \cdot A$
결합$(A+B)+C = A+(B+C)$
분배$A(B+C) = AB + AC$
흡수$A + AB = A,\quad A(A+B) = A$
드 모르간$\overline{A+B} = \overline{A} \cdot \overline{B},\quad \overline{A \cdot B} = \overline{A} + \overline{B}$

13.1 식 단순화 예제

$F = A\overline{B} + AB + \overline{A}B$를 단순화하라.

풀이
1. 분배: $F = A(\overline{B}+B) + \overline{A}B$.
2. 상보: $\overline{B}+B = 1$. → $F = A + \overline{A}B$.
3. 흡수: $A + \overline{A}B = A + B$.
답: $F = A + B$. 게이트 수가 5개 → 2개로 줄어듭니다.

14. 카르노 맵 입문 — 시각적 단순화

3~4 변수 부울 식의 단순화를 표로 그리는 도구입니다. 시험에서 자주 등장하는 단원입니다.

14.1 2변수 카르노 맵

$B=0$$B=1$
$A=0$$\overline{A}\,\overline{B}$$\overline{A}\,B$
$A=1$$A\,\overline{B}$$A\,B$

출력이 1인 칸을 묶고(2개·4개·8개), 묶음이 가로·세로 인접한 변수만 남깁니다.

14.2 4변수 카르노 맵 예제

$F(A,B,C,D) = \sum m(0,2,5,7,8,10,13,15)$ — 8개 1을 묶으면 $F = D + B$로 단순화. 게이트 8개의 회로가 OR 1개로 줄어듭니다.

15. 디지털 회로 응용 — 8가지 사용 예

  • CPU — 명령어 디코더는 거대한 조합 회로.
  • 메모리 — 주소 디코더 + 플립플롭 어레이.
  • 디지털 시계 — 카운터 + 디스플레이 디코더.
  • 신호등 제어 — 상태 기계(state machine).
  • 키패드 스캔 — 행·열 디코더.
  • UART 통신 — 시프트 레지스터 + 보드 레이트 카운터.
  • SPI/I2C — 클록 + 데이터 동기 라인.
  • FPGA — 프로그래머블 게이트 + 룩업 테이블.

16. 흔한 오답 패턴

  1. NAND vs NOR 혼동 — NAND는 모두 1이면 0, NOR는 모두 0이면 1. 헷갈리면 진리표 그리기.
  2. 드 모르간 부호 반전 누락 — 곱이 합이 되거나 합이 곱이 될 때 모두 부호도 같이 반전.
  3. 음수 표현 혼동 — 2의 보수는 부호 비트(MSB)와 크기를 같이 본다.
  4. 플립플롭 클록 에지 무시 — 상승 에지 vs 하강 에지 항상 명시.
  5. 글리치(spike) 무시 — 조합 회로의 중간 시점 출력은 일시적으로 잘못된 값을 가질 수 있음.

17. Falstad 데모 — 4비트 카운터

  1. Draw → Logic Family → Add D Flip-Flop로 D 플립플롭 4개 연결.
  2. 각 플립플롭의 $\overline{Q}$를 다음 단의 D로 연결 (T 플립플롭처럼 동작).
  3. 클록 입력에 1Hz 사각파.
  4. 출력 4비트가 0000 → 0001 → 0010 → ... → 1111 → 0000 순환 확인.
  5. BCD 디코더 + 7-세그먼트 LED를 연결하면 0~F가 디스플레이에 표시됨.

19. FAQ — 디지털 회로 자주 묻는 질문

Q1. CPU 안에는 정말 트랜지스터가 200억 개?

네. 2024년 기준 고급 CPU/SoC 한 개당 트랜지스터 수가 100억~200억 개. CMOS 공정이 5nm/3nm까지 미세화되어 가능해졌습니다.

Q2. 디지털 회로는 정말 0/1만 다루나요?

겉보기는 그렇지만, 안은 매우 정밀한 아날로그 회로입니다. 트랜지스터 한 개의 작은 전압 노이즈가 전체 시스템을 망가뜨릴 수 있어, 사실 가장 정밀한 회로 중 하나입니다.

Q3. 클록 주파수가 빠를수록 빠른 컴퓨터?

대체로 그렇지만 한계가 있습니다. 클록이 빨라지면 발열·전력 소모가 급증하고, 신호가 게이트를 지나가는 데 걸리는 시간이 한 사이클 내에 끝나야 합니다. 그래서 멀티코어·파이프라인 같은 다른 방법이 사용됩니다.

Q4. 양수 논리(positive logic)와 음수 논리(negative logic) 차이?

양수 논리는 1=H, 0=L. 음수 논리는 1=L, 0=H. 일반적으로는 양수 논리이지만 일부 통신 표준이나 액티브-로우 신호에서는 음수 논리가 사용됩니다.

Q5. FPGA는 무엇인가요?

Field Programmable Gate Array. 디지털 회로를 소프트웨어처럼 "프로그래밍"할 수 있는 칩. 고정된 ASIC보다 유연하고, CPU보다 병렬 처리에 강합니다. 신호처리·딥러닝 가속에 사용됩니다.

22. 한 학기 자기 평가 — 5단계 체크

  1. 옴의 법칙 — 단위까지 정확히 적기. (W2)
  2. KCL/KVL + 분배기 — 직병렬·노드·메쉬 빠르게 풀이. (W3~W4)

  4. RC·RL·페이저 — 시간 변화·교류 대응. (W6~W8)
  5. 다이오드·트랜지스터·op-amp·디지털 — 부품별 동작 직관. (W9~W12)

23. 학기 후 첫 미니 프로젝트 — 추천 5가지
