Week 4. 블록선도

시스템을 그림으로 그리고 식으로 줄이기. 세 가지 결합 규칙(직렬·병렬·피드백)만 알면 어떤 복잡한 회로도 단일 전달함수로 압축할 수 있습니다.

목차

  1. 블록선도의 기호
  2. 세 가지 기본 결합
  3. 폐루프 전달함수 — 자동제어의 가장 중요한 식
  4. 등가 변환의 트릭
  5. 인터랙티브: 블록선도 → 단일 전달함수
  6. 외란이 들어올 때

1. 블록선도의 기호

블록선도(Block Diagram)는 신호의 흐름과 변환을 그림으로 표현한 것입니다. 단 네 가지 기호만 씁니다.

2. 세 가지 기본 결합

② 직렬(Cascade) 결합

두 블록이 일렬로 놓이면 전달함수는 곱해집니다.

$$ Y(s) = G_1(s) G_2(s) U(s) \quad \Rightarrow \quad G_{eq}(s) = G_1(s)\, G_2(s) $$

② 병렬(Parallel) 결합

같은 입력이 두 블록으로 나뉘어 들어가고 출력이 합쳐집니다.

$$ G_{eq}(s) = G_1(s) \pm G_2(s) $$

③ 피드백(Feedback) 결합 — 자동제어의 주역

출력이 센서 $H(s)$를 거쳐 입력으로 되돌아옵니다.

$Y = G(R - HY)$를 풀면:

$$ \boxed{\,\frac{Y(s)}{R(s)} = \frac{G(s)}{1 + G(s) H(s)}\,} $$

이게 폐루프 전달함수입니다. 자동제어 책 한 권이 사실상 이 한 줄에서 시작합니다. $H = 1$인 경우(센서가 이상적, 단위 피드백)에는 더 단순해지죠.

$$ \frac{Y(s)}{R(s)} = \frac{G(s)}{1 + G(s)} $$

3. 폐루프 전달함수가 왜 그렇게 중요한가?

분모 $1 + G(s)H(s) = 0$이 폐루프 시스템의 특성방정식입니다. 이 식의 근(극점)이 시간응답·안정성을 결정합니다. 즉:

  • W5–W6 시간응답 분석은 폐루프 극점의 위치 → 응답 모양 매핑
  • W8 Routh-Hurwitz는 특성방정식의 근이 좌반평면에 있는지 판정
  • W9 근궤적은 게인 $K$가 변할 때 극점이 어떻게 움직이는지 추적

모든 길이 한 분모에서 갈라집니다.

4. 등가 변환의 트릭

실제 시스템의 블록선도는 가산점·분기점이 얽혀 있어 한 번에 위 세 결합으로 안 읽힐 수 있습니다. 그때 쓰는 변환 규칙 두 개만 기억하세요.

  • 가산점 이동: 가산점을 블록 앞/뒤로 옮기려면 그 블록의 전달함수로 곱하거나 나누어 보정.
  • 분기점 이동: 분기점을 블록 앞/뒤로 옮기려면 그 블록을 한 번 더 끼워 넣거나 빼서 보정.

왜 곱하거나 나누어 보정해야 하는가? — 한 줄 직관

핵심은 단 한 가지: 이동 후의 신호 흐름이 이동 전과 정확히 같은 식을 표현하도록 새 블록을 넣어 균형을 맞추는 일입니다.

왜 그런가? "변환 전의 출력 = 변환 후의 출력"이 되어야 하니, 새 신호 경로가 잃어버린 곱셈을 빨간 블록이 채워주는 것입니다. 처음에 헷갈리면 "변환 전 출력 = ?, 변환 후 출력 = ? 두 식이 같아지려면 어디에 G를 끼워 넣어야 할까?"를 종이에 적어 보세요. 다섯 번 정도 해보면 자동으로 손에 익습니다.

실무에서는 이 변환을 손으로 하는 일이 거의 없고, 아래 Python 한 줄로 풀어버립니다.

5. 인터랙티브: 블록선도 → 단일 전달함수

🎮 인터랙티브: PI 제어기 + 1차 플랜트의 폐루프

제어기 $C(s) = K_p + K_i/s$, 플랜트 $G(s) = 1/(\tau s + 1)$, 단위 피드백. 폐루프 전달함수와 그 단위 스텝 응답을 동시에 봅니다.

관찰:

  • $K_i = 0$ (P 단독): 정상상태 오차가 남는다 — 분자에 $K_p$만 있고 분모에 $K_i = 0$.
  • $K_i > 0$: 적분 항 덕에 정상상태에서 1.0에 정확히 수렴.
  • $K_p$를 너무 키우면 오버슈트와 진동이 생긴다 — W6의 ζ 감소.

6. 외란이 들어올 때

플랜트 입력에 외란 $D(s)$가 더해진다고 합시다. 같은 폐루프에 두 입력 ($R$과 $D$)이 있는 셈인데, 선형이므로 중첩의 원리로 따로 계산해 더하면 됩니다.

$$ Y(s) = \underbrace{\frac{G C}{1 + G C}}_{T(s)} R(s) + \underbrace{\frac{G}{1 + G C}}_{S(s)} D(s) $$

$T(s)$를 기준 추적 함수(complementary sensitivity), $S(s)$를 외란 감도 함수(sensitivity)라 부릅니다. $|GC|$가 클수록 $T \to 1$, $S \to 1/(GC) \to 0$. 즉 게인을 키우면 외란이 자동으로 억제됩니다. 하지만 너무 키우면 W11에서 보듯 안정여유가 사라집니다 — 자동제어의 영원한 트레이드오프.

6-B. 외란 응답의 물리적 의미

실제 시나리오 — 항온조에 외풍이 들어올 때

항온조의 입력은 히터 전력 $u$, 출력은 내부 온도 $y$. 외란 $d$ 는 외풍·문 개폐 등으로 들어오는 열손실.

  • P 제어 ($C = K_p$): $S(0) = \dfrac{1}{1 + K_p G(0)}$. $K_p \to \infty$ 면 0 에 가까워지지만 무한 게인은 불가능.
  • PI 제어 ($C = K_p + K_i/s$): $s=0$ 에서 $C \to \infty$, 따라서 $S(0) = 0$. 외란을 완전히 제거.

이게 W7 정상오차에서 다시 나오는 핵심 결과. 적분기 한 개의 위력.

6-C. 노이즈 감도 — 또 하나의 트레이드오프

센서에 측정 노이즈 $n(t)$ 가 있을 때:

$$ Y(s) = T(s) R(s) - T(s) N(s) + S(s) D(s) $$

(노이즈는 부호 반대로 출력에 영향) 게인 $C$ 를 키우면:

  • $T \to 1$ → 추종 성능 ↑ but 노이즈도 그대로 출력에 통과.
  • $S \to 0$ → 외란 억제 ↑.

즉 고게인 = "외란은 막지만 노이즈는 그대로". 노이즈가 큰 시스템에서는 게인을 무한정 키울 수 없다 — $S + T = 1$ 항상 성립.

$S + T = 1$ 의 의미: 외란 감도와 추종 성능은 동시에 개선이 불가능. 한쪽을 0 에 가깝게 만들면 다른 쪽이 1 에 가까워짐. 이걸 깨려면 별도 필터·관측기 추가.

6-D. 블록선도 변환의 표준 규칙들

변환전후
합점 이동 (블록 앞 → 뒤)$X \to G \to Y$, $Y + W$$X + W/G \to G \to Y + W$
분기점 이동 (블록 앞 → 뒤)$X \to G \to Y$, $X$ 분기$X \to G \to Y$, $Y/G$ 분기
폐루프 → 단일 블록$\dfrac{G}{1 \pm GH}$등가 단일 전달함수
두 블록 직렬$G_1 \to G_2$$G_1 G_2$
두 블록 병렬$G_1 \pm G_2$$G_1 \pm G_2$ (덧셈/뺄셈)

이 5 가지 변환을 반복하면 어떤 복잡한 블록선도도 단일 전달함수로 환원 가능. 시험에서 자주 쓰임.

6-E. 손계산 예제 — 다중 피드백 환원

문제

다음 블록선도를 단일 전달함수로 환원하시오: $X \to [+] \to G_1 \to [+] \to G_2 \to Y$. 첫 번째 합점에 $H_1 \cdot Y$ 가 음피드백, 두 번째 합점에 $H_2 \cdot$ ($G_2$ 후) 가 음피드백.

풀이 — 안쪽 루프부터

안쪽 루프 ($G_2$ 와 $H_2$): 등가 = $\dfrac{G_2}{1 + G_2 H_2}$.

이제 시스템은 $X \to [+] \to G_1 \to \tilde G \to Y$ ($\tilde G = G_2/(1+G_2 H_2)$). $H_1$ 음피드백.

최종 폐루프: $T = \dfrac{G_1 \tilde G}{1 + G_1 \tilde G H_1} = \dfrac{G_1 G_2}{1 + G_2 H_2 + G_1 G_2 H_1}$.

이렇게 안쪽부터 차근차근 환원하면 4-5 단의 다중 피드백도 정리 가능. 자동차 ESC 같은 다중 루프 제어가 이런 형태.

6-F. 응용 — 캐스케이드 제어 (Cascade Control)

2 단 캐스케이드 구조

외부 루프 (느린 변수) + 내부 루프 (빠른 변수) 의 중첩 구조. 산업에서 매우 자주 쓰임.

  • 외부 루프: 위치 제어 (느림, 명령은 속도).
  • 내부 루프: 속도 제어 (빠름, 명령은 토크).
  • 최내부: 토크 제어 (가장 빠름, 명령은 전류).

왜 캐스케이드가 좋나?

  • 각 루프를 독립적으로 튜닝 가능 — 빠른 루프부터 안정화.
  • 내부 루프가 외란 (모터 마찰 등) 을 빠르게 흡수 → 외부 루프는 매끄러운 명령만 처리.
  • 드론·로봇·자동차 ESC 모두 이 구조.

예 — 산업로봇 관절 제어

위치 명령 → [위치 P/PI] → 속도 명령 → [속도 PI] → 토크 명령 → [토크 PI] → 전류 → 모터.

각 루프의 게인은 안쪽이 가장 큼 (가장 빠른 응답), 바깥은 작음. 이 위계가 안정성을 보장합니다.

6-G. 손계산 예제 — 외란 응답 정량화

문제

$G(s) = \dfrac{1}{s+1}$, $C(s) = K_p$ (P 제어). 단위 외란 $d(t) = 1$ 에 의한 정상상태 출력 $y_{ss}$ 를 $K_p$ 의 함수로 구하시오.

풀이

$Y(s) = S(s) D(s) = \dfrac{G}{1 + GC}\cdot D = \dfrac{1/(s+1)}{1 + K_p/(s+1)}\cdot \dfrac{1}{s} = \dfrac{1}{s(s+1+K_p)}\cdot 1$.

$y_{ss} = \lim_{s\to 0}sY(s) = \dfrac{1}{1+K_p}$.

  • $K_p = 1$: $y_{ss} = 0.5$ (외란의 50% 가 출력에 남음).
  • $K_p = 10$: $y_{ss} = 0.091$ (9.1%).
  • $K_p = 100$: $y_{ss} = 0.0099$ (1%).

게인을 10 배 키우면 외란이 거의 1/10 로 줄지만, 위상 안정여유 (W11) 도 같이 줄어 진동·불안정 위험.

6.5 블록선도 변환 8가지 표준 규칙

규칙변환
직렬 결합$G_1 G_2$
병렬 결합 (덧셈)$G_1 + G_2$
피드백 (음)$G/(1+GH)$
피드백 (양)$G/(1-GH)$
합점 이동 (앞으로)$G$ 통과 후 합산

분점 이동 (앞으로)$1/G$로 보정
분점 이동 (뒤로)$G$로 보정

6.6 블록선도 단순화 절차

  1. 가장 안쪽 루프 식별.
  2. 피드백 공식 적용.
  3. 직렬 결합 단순화.
  4. 병렬 결합 단순화.
  5. 합점·분점 이동.
  6. 최종 단일 블록.

6.7 자주 만나는 함정

  • 합점·분점 이동 시 $G$, $1/G$ 보정 잊음.
  • 음피드백·양피드백 부호 혼동.
  • 중복된 신호 경로의 처리 실수.
  • 비선형 요소는 블록선도 한계.

6.8 추가 워크북 — 5문제

6.8.1 직렬 결합

$G_1 = 1/(s+1), G_2 = 2/(s+2)$. 직렬?

$G = 2/((s+1)(s+2))$.

6.8.2 병렬 결합

$G_1 = 1/s, G_2 = 1$. 병렬?

$G = (1+s)/s$. 영점 추가.

6.8.3 단위 피드백

$G = K/(s+1)$. 폐루프?

$T = K/(s+1+K)$.

6.8.4 일반 피드백

$G = 1/(s+1), H = K_d s$. 폐루프?

$T = 1/((1+K_d)s+1)$.

6.8.5 외란 억제

외란 D에 대한 출력 이득?

$Y/D = -T$. T가 작으면 잘 억제.

6.9 응용 — 자동차 ACC 블록선도

입력: 설정 속도. 외란: 경사. 플랜트: 차량+엔진. 제어기: PID. 폐루프 $T = G_pC/(1+G_pC)$.

6.10 응용 — 모터 위치 제어 캐스케이드

3중 캐스케이드 — 외부(위치) → 중간(속도) → 내부(전류). 블록선도로 한 페이지에 표현. 안쪽 루프가 5~10배 빠르게 동작하도록 게인 설정.

7. 핵심 정리

  • 블록선도는 시스템 그림 + 식이다. 직렬·병렬·피드백 세 결합으로 거의 모든 구조를 줄일 수 있다.
  • 폐루프 전달함수: $\dfrac{G}{1 + GH}$. 자동제어 강의의 가장 중요한 한 줄.
  • 분모 $1 + GH$ = 특성방정식. 모든 안정성·응답 분석이 여기서 출발.
  • 고게인은 외란 억제에 좋지만 안정여유를 잡아먹는다.

8. Mason의 게인 공식 (선형 신호 흐름 그래프)

복잡한 다중 루프 시스템에 사용. 블록선도 단순화의 일반화.

$T = \frac{1}{\Delta} \sum_k P_k \Delta_k$, 여기서 $P_k$는 k번째 전방 경로, $\Delta$는 시스템 결정자.

9. 블록선도 vs 신호 흐름 그래프

  • 블록선도 — 박스와 화살표.
  • SFG — 노드와 가지.
  • 두 표현은 동등. 복잡한 시스템에서 SFG가 자주 더 명료.

10. 외란·잡음 모델링

  • 입력 외란 d — 플랜트 입력에 더해짐. $u + d \to G$.
  • 출력 외란 d_o — 플랜트 출력에 더해짐. $G u + d_o = y$.
  • 측정 잡음 n — 센서 출력에 더해짐. $y_m = y + n$.
  • 각 외란/잡음에 대한 sensitivity 함수가 다름.

11. 4가지 sensitivity 함수

  • $S = 1/(1+L)$ — 명령 → 오차.
  • $T = L/(1+L)$ — 명령 → 출력.
  • $SG = G/(1+L)$ — 외란 → 출력.
  • $CS = C/(1+L)$ — 잡음 → 입력.

12. Python 코드

import control as ct

# 직렬 결합
G1 = ct.tf([1], [1, 1])
G2 = ct.tf([2], [1, 2])
G_series = G1 * G2

# 병렬 결합
G_parallel = G1 + G2

# 피드백 (음, 단위 피드백)
T = ct.feedback(G1, 1)

# 일반 피드백
H = ct.tf([1], [1, 5])
T_gen = ct.feedback(G1, H)

print(T)

  • 1자유도 — 단일 피드백 루프.
  • 2자유도 — 피드포워드 + 피드백.
  • 캐스케이드 — 다중 루프 직렬.
  • 병렬 — 여러 제어기 결합.
  • 분산 — 여러 노드의 분산 제어.

  • 외란·잡음 입력 위치 명확.

25. 부록 — Python에서 블록선도 표현

import control as ct
G_plant = ct.tf([1], [1, 1, 0])      # 1/(s²+s)

L = ct.series(C_pid, G_plant)        # 직렬
T = ct.feedback(L, 1)                # 폐루프
ct.step_response(T)

$U = F R + C(R - Y)$. F는 피드포워드, C는 피드백.

69. 부록 — 두 자유도 제어 깊이

2-DOF 구조: 명령 R → 피드포워드 F → + → 플랜트 G → 출력 Y. 동시에 R - Y → 피드백 C → +.

전달함수: $Y/R = (FG + CG)/(1+CG)$. 추적 성능과 외란 거부를 분리해 설계.

72. 부록 — Inner-Outer Loop 설계

73. 부록 — Feedforward 설계

74. 부록 — Cascade vs 2-DOF

  • Cascade — 다중 상태(위치, 속도) 직렬 루프.
  • 2-DOF — 한 상태에 FF + FB 결합.
  • 둘 다 자주 결합 사용.

76. 부록 — Sensitivity 함수의 해석

$|S(j\omega)| < 1$: 외란이 출력에서 줄어듦.
$|S(j\omega)| = 1$: 그대로.
$|S(j\omega)| > 1$: 증폭.
Bode 적분 정리: $\int |S| d\omega \geq 0$. "외란을 어디서 줄이면 다른 곳에 손실."
