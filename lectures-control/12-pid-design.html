Week 12. PID 제어기 설계

전 세계 산업용 제어기의 95%가 여전히 PID. 단순한 만큼 잘 쓰는 게 어렵습니다. Ziegler-Nichols 튜닝, 안티 와인드업, 미분 필터까지 실무에서 바로 쓰는 모든 것.

1. PID 식과 각 항의 역할 복습

오차 $e(t) = r(t) - y(t)$에 대한 표준 PID 식:

$$ u(t) = K_p\, e(t) + K_i \int_0^t e(\tau)\,d\tau + K_d\, \dot{e}(t) $$

라플라스로:

$$ C(s) = K_p + \frac{K_i}{s} + K_d s = \frac{K_d s^2 + K_p s + K_i}{s} $$
항역할키우면주의
P (비례)현재 오차에 비례응답 빨라짐, 진동↑정상오차 잔류
I (적분)누적 오차 보정정상오차 0 (Type +1)위상여유↓, 와인드업
D (미분)변화 속도로 미리 제동댐핑↑, 오버슈트↓잡음에 매우 민감

2. Ziegler-Nichols 튜닝

1942년에 Ziegler와 Nichols가 제안한, 모델이 없을 때도 게인을 정해주는 경험적 공식. 여전히 산업 현장의 출발점입니다.

방법 ① 임계 진동법 (Closed-loop)

  1. I, D는 끄고 P만으로 시작.
  2. $K_p$를 천천히 키워 출력이 막 지속 진동을 시작하는 임계 게인 $K_u$를 찾는다.
  3. 그때의 진동 주기 $T_u$를 측정.
제어기$K_p$$T_i$ (적분 시간)$T_d$ (미분 시간)
P$0.5\,K_u$——
PI$0.45\,K_u$$T_u/1.2$—
PID$0.6\,K_u$$T_u/2$$T_u/8$

($K_i = K_p/T_i$, $K_d = K_p T_d$)

W8에서 본 예제 시스템 $G(s) = 1/[s(s+1)(s+5)]$의 임계 게인은 $K_u = 30$, 진동 주파수 $\omega_u = \sqrt{5}$. 주기 $T_u = 2\pi/\omega_u \approx 2.81$. PID 추천값 $K_p = 18$, $T_i = 1.4$, $T_d = 0.35$. 이게 안정 경계의 60% 위치에 있어 실무 안전망.

방법 ② 계단응답법 (Open-loop)

플랜트에 단위 스텝을 가하고 계단응답에서 두 값을 측정합니다.

  • $L$ (지연시간, dead time): 스텝을 가한 시점부터 변곡점 접선이 시간축을 가로지를 때까지의 시간. "시스템이 반응을 시작하기까지의 늑장".
  • $T$ (시정수, time constant): 그 접선이 시간축을 가로친 점에서 정상값 $K$에 닿기까지의 추가 시간. "반응이 시작된 뒤 얼마나 빨리 자라는가".
  • $K$ (정상값): 충분히 긴 시간 뒤 출력이 도달하는 값.

이를 1차 + 시간지연 모델 $G(s) \approx K e^{-Ls}/(Ts+1)$로 근사한 뒤:

제어기$K_p$$T_i$$T_d$
P$T/L$——
PI$0.9\,T/L$$L/0.3$—
PID$1.2\,T/L$$2L$$0.5L$
실측 팁 — 노이즈가 있는 실제 응답에서 변곡점 위치를 정확히 잡기는 어렵습니다. 실무에서는 (a) 응답을 사보쳬프 필터로 부드럽게 만들고, (b) 출력의 28%, 63% 도달 시간 두 값으로 "두 점법(Two-point method)"을 쓰는 것이 더 안정적입니다 — $T = 1.5(t_{63\%} - t_{28\%})$, $L = t_{63\%} - T$.
🤔 왜 ZN 추천값은 오버슈트가 20~50%로 클까? — "1/4 감쇠비"의 정체

Ziegler와 Nichols(1942)는 화학공정 엔지니어들에게 "한 진동 다음 진동의 진폭이 1/4로 줄어드는 응답"을 표준이라 보고 표를 맞췄습니다. 이를 Quarter Amplitude Decay (QAD)라고 부릅니다.

  1. "한 진동 → 다음 진동의 진폭이 1/4"는 한 주기 동안 감쇠 $e^{-\zeta\omega_n T_d} = 1/4$를 뜻합니다.
  2. 한 주기 $T_d = 2\pi/\omega_d$이므로 $\zeta\omega_n \cdot 2\pi/\omega_d = \ln 4$.
  3. $\omega_d = \omega_n\sqrt{1-\zeta^2}$를 대입하면 $\zeta/\sqrt{1-\zeta^2} = \ln 4 / (2\pi) \approx 0.221$.
  4. 풀면 $\zeta \approx 0.215$.

$\zeta \approx 0.22$ → 오버슈트 약 50%! 그래서 ZN 추천값을 그대로 쓰면 화면에서 응답이 한참 출렁입니다. 1942년 화학공정에서는 외란 회복(load disturbance rejection)이 우선이라 이 정도 진동이 허용됐지만, 모터·로봇처럼 정밀이 중요한 곳에서는 매우 거칠죠.

실무 보정:

  • 오버슈트 줄이기: $K_p$를 ZN 추천값의 50% 정도로 (또는 "Tyreus-Luyben" 표 사용 — $K_p = K_u/3.2$, $T_i = 2.2 T_u$).
  • 로봇/모터: ITAE-튜닝(시간 가중 절대 오차 적분 최소) 또는 IMC-PID(내부모델 기반)가 더 부드러운 응답을 줌.
  • 오토튜닝 옵션: Åström-Hägglund의 릴레이 방법은 ZN 임계 진동법을 자동화한 것으로 산업 컨트롤러에 내장돼 있음.

3. 안티 와인드업 — 적분기를 살아있게 하기

액추에이터에는 한계(예: 모터 PWM 0~100%, 스로틀 0~100%)가 있습니다. 큰 오차가 한참 지속되면 적분 항이 한없이 누적되는데, 출력은 이미 포화 상태라 실제 시스템엔 반영되지 않습니다. 나중에 오차가 줄어들기 시작해도 거대한 적분값을 모두 풀어내는 동안 시스템이 큰 오버슈트를 냅니다.

두 가지 표준 해법

  • 조건부 적분 (Conditional Integration): 출력이 포화되어 있을 때 적분 갱신을 멈춘다 — 한 줄짜리 if 문으로 끝.
  • 역계산 (Back-calculation): 포화로 잘려나간 양 $u_{cmd} - u_{sat}$을 적분기에서 빼주는 방식. $\dot{x_i} = K_i e + K_t (u_{sat} - u_{cmd})$. $K_t$는 시정수의 역수쯤으로 잡는다.
class PID:
    def __init__(self, kp, ki, kd, dt, u_min=-1, u_max=1):
        self.kp, self.ki, self.kd = kp, ki, kd
        self.dt = dt; self.u_min, self.u_max = u_min, u_max
        self.iE = 0.0; self.prev_e = 0.0

    def step(self, ref, y):
        e = ref - y
        # 미분 (잡음 약화 위해 측정값 미분: 부호 유의)
        dE = (e - self.prev_e) / self.dt
        # 적분 (조건부)
        u_unsat = self.kp*e + self.ki*self.iE + self.kd*dE
        u = max(self.u_min, min(self.u_max, u_unsat))
        if u_unsat == u:                          # 포화 안 됐을 때만 적분 갱신
            self.iE += e * self.dt
        self.prev_e = e
        return u

4. 미분 필터 & D-on-measurement

이상적인 미분 $K_d s$는 고주파에서 무한히 증폭됩니다. 측정 잡음이 살짝만 있어도 출력이 진동. 실무에서는 두 가지 방어책이 표준.

  1. 미분 필터: $K_d s \to \dfrac{K_d s}{1 + s\tau_d}$. $\tau_d$는 $K_d/(N \cdot K_p)$, 보통 $N = 8\sim 20$.
  2. D-on-measurement: 오차 $e$ 대신 측정값 $-y$를 미분. 이러면 setpoint $r$이 갑자기 바뀔 때 미분이 거대해지는 "도함수 킥(derivative kick)"을 막는다. $u = K_p e + K_i \int e - K_d \dot{y}$.

5. 인터랙티브: PID 슬라이더로 직접 튜닝

🎮 인터랙티브: 2차 플랜트 + PID 폐루프

플랜트 $G(s) = 1/(s^2 + 0.4s + 1)$ ($\omega_n = 1$, $\zeta = 0.2$). Kp, Ki, Kd를 직접 만져 오버슈트와 정착시간을 줄여 보세요. Ziegler-Nichols 추천값 버튼으로 출발점 확인.

실험 가이드:

  • Ki, Kd = 0 두고 Kp만 키우기 → 오버슈트 증가, 정상오차 잔류.
  • Ki를 추가하면 정상오차 → 0, 그러나 오버슈트가 더 커지고 진동.
  • Kd를 추가하면 진동·오버슈트가 줄어들면서 부드러워짐.
  • Ziegler-Nichols 버튼 누르고 → 너무 격렬하면 Kp를 절반으로.

6. 구현: 이산화와 코드 한 줄

실제 마이크로컨트롤러는 일정 주기 $\Delta t$마다 PID를 한 번 계산합니다. 적분은 좌측합 또는 사다리꼴(Tustin), 미분은 후방 차분이 표준.

$$ u_k = K_p e_k + K_i \sum_{j=0}^{k} e_j \Delta t + K_d \frac{e_k - e_{k-1}}{\Delta t} $$

위 Python 클래스를 그대로 ARM Cortex-M C 코드로 옮길 수 있고, 각종 RTOS의 PID 라이브러리도 같은 구조입니다. $\Delta t = $ 1~10 ms가 모터 제어, 100 ms~1 s가 공정 제어의 보통 값.

7. 실무 체크리스트 (Cheat Sheet)

  1. 출력 한계 알기 → 안티 와인드업 필수.
  2. 측정 신호 잡음 확인 → 필요시 LPF 또는 미분 필터.
  3. Setpoint 변화가 큰 응용이면 D-on-measurement 사용.
  4. 샘플링 시간 $\Delta t$는 시스템 시정수의 1/10 이하.
  5. 먼저 P만으로 안정/응답 확인 → I 추가로 정상오차 제거 → D는 마지막에 미세 조정.
  6. Bode로 PM ≥ 45° 확인 후 현장 투입.

8. 핵심 정리

  • P = 현재, I = 누적, D = 변화율. 직관 기반 만능 도구.
  • Ziegler-Nichols로 출발점, 응답 보면서 줄여간다.
  • 안티 와인드업·미분 필터·D-on-measurement는 실무에서 빠질 수 없는 3종 세트.
  • PID의 한계: 다입출력·제약이 본격적으로 들어오면 MPC로 — 로봇공학특론 L11.
