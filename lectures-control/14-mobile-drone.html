1. 크루즈 컨트롤 — 1D PID의 정석

설정 속도 $v_{ref}$ (예: 100 km/h)를 유지하는 시스템. 외란은 도로 경사·풍속·차량 무게 변화.

1자유도 종방향 모델:

$$ m \dot{v} = u_{throttle} - b\, v - mg\sin\alpha $$

$u$는 스로틀 토크, $b$는 공기·구름 마찰, $\alpha$는 도로 경사각.

왜 PI(D)면 충분한가?

속도제어는 본질적으로 1차 시스템(W5)이라서 PI 한 줄이면 정상오차 0, 빠른 응답이 모두 됩니다. 차량 제조사들도 거의 모두 PI 기반 + 안티 와인드업 + 스로틀 한계 처리 + 단순한 게인 스케줄링(저속/고속)으로 끝.

class CruiseControl:
    def __init__(self, m, b, dt):
        self.m, self.b, self.dt = m, b, dt
        self.kp, self.ki = 800, 200
        self.iE = 0.0
        self.u_max = 4000  # N (스로틀 토크 한계)

    def step(self, v_ref, v):
        e = v_ref - v
        u_unsat = self.kp*e + self.ki*self.iE
        u = max(-self.u_max, min(self.u_max, u_unsat))
        if u_unsat == u:
            self.iE += e * self.dt   # 안티 와인드업
        return u

2. 모바일 로봇 경로 추종 — 거리·헤딩 PID 분리

차륜 로봇은 두 입력이 있습니다: 직진 속도 $v$와 회전 각속도 $\omega$. 그러나 한 번에 PID 하나로 두 입력을 결정하지 않습니다.

  • 거리 PID → $v$ 결정. 목표 점까지의 직선 거리에 비례.
  • 헤딩(요) PID → $\omega$ 결정. 목표 점이 보이는 방향과 현재 진행방향의 각도 차에 비례.
$$ v = K_{p,d}\, d, \qquad \omega = K_{p,\psi} (\psi_{target} - \psi_{robot}) + K_{d,\psi}\,(\dot\psi_{target} - \dot\psi_{robot}) $$

이 단순한 분리만으로도 라인트레이서·창고 AGV·자동 청소기가 충분히 동작합니다. 더 정밀한 차량용 경로 추종은 로봇공학특론 L7의 Pure Pursuit·Stanley로 발전합니다.

3. 인터랙티브: 모바일 로봇 경로 추종

🎮 인터랙티브: 두 PID로 곡선 경로 따라가기

로봇이 초록 곡선을 따라가도록 거리 PID + 헤딩 PID를 적용. 게인을 만져 진동·코너 컷팅 양상을 확인하세요.

관찰:

  • 전방 주시 거리 짧음 + 헤딩 Kp 큼 → 진동(좌우 핸들 떨림).
  • 전방 주시 거리 김 → 부드럽지만 코너 안쪽으로 컷팅.
  • 적정 균형이 자율주행 분야의 영원한 튜닝 문제.

4. 쿼드콥터 자세 제어 — 캐스케이드 PID

쿼드콥터는 4개 모터로 3축 회전과 1축 추력을 동시에 만듭니다. 자세 제어는 보통 2단 캐스케이드로 합니다.

  • 외부 루프 (자세각): 목표 roll·pitch·yaw → 각속도 명령. ~50–100 Hz.
  • 내부 루프 (각속도): 각속도 명령 → 모터 RPM 차이. ~500–1000 Hz, IMU의 자이로 신호 사용.

각 축에 PID 한 세트씩, 총 6개 PID. 그 출력을 모터 분배 행렬로 곱해서 각 모터의 RPM 명령을 만듭니다. Pixhawk·Betaflight·DJI 모두 이 구조의 변형입니다.

5. 인터랙티브: 1축 자세 제어 (외란 가하기)

🎮 인터랙티브: 쿼드콥터 1축 캐스케이드 PID

회전 관성 J인 1자유도 시스템. 자세 PD + 각속도 PID 캐스케이드. 버튼으로 갑작스러운 외부 토크(돌풍) 외란을 가해보세요.

꼭 만져볼 것:

  • 각속도 Kp가 너무 작으면 외부 자세 PD가 명령해도 따라가지 못해 응답이 둔하다.
  • 각속도 Kp가 충분하면 외란 후 빠르게 회복.
  • 자세 Kp가 너무 크면 진동, 자세 Kd가 너무 크면 응답이 둔해진다.
  • 실제 드론은 같은 PID를 roll·pitch·yaw 세 축에 한 벌씩 — 각 축이 서로 약하게 결합.

6. PID의 한계와 다음 단계 — MPC로 가는 길

지금까지 PID 한 가지로 1D·2D·3D 시스템을 모두 다뤄봤습니다. 그런데 다음 같은 상황에서는 PID가 한계에 부딪힙니다.

상황PID의 어려움대안
다입출력(MIMO) + 입력 결합변수마다 PID를 따로 두면 상호간섭이 큼LQR, MPC
액추에이터·상태 제약 (안전)하드 제약을 PID 식 안에 못 넣음MPC, CBF
예측이 본질적인 작업 (긴 차량 경로)현재 오차만 봐서는 한참 앞 코너를 못 본다MPC
경제적 최적화 (에너지·시간)비용함수 직접 최소화 불가MPC, LQR

이런 시스템을 다루려면 PID 너머의 도구가 필요합니다. 자연스러운 다음 강의:

  • 로봇공학특론 L3 상태공간 — MIMO를 다루는 언어.
  • L7 피드백 제어 (Pure Pursuit, Stanley) — 차량 경로 추종.
  • L8 유도 & CBF — 안전 보장 제어.
  • L11 MPC — 미래를 예측하는 최적 제어.
  • L12 고급 MPC — Tube MPC, Koopman 등 첨단 주제.

7. 핵심 정리

  • 크루즈 컨트롤(1D) → 모바일 로봇(2D) → 쿼드콥터(3D), 모두 같은 PID 도구로 시작.
  • 2D·3D는 변수를 분해(거리·헤딩, 자세·각속도)해서 PID 여러 개로 풀어준다.
  • 캐스케이드는 시간 척도가 다른 변수를 갈라 안쪽일수록 빠르게 돌리는 게 핵심.
  • PID는 SISO·외란·일부 비선형까지는 잘 처리. MIMO·제약·예측이 본격적으로 들어오면 MPC로 진화.
