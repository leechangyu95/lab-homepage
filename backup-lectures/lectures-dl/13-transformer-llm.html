1. RNN을 버리고 어텐션만 — 2017년의 충격

2017년 6월, 구글 Brain의 Vaswani 등 8명이 발표한 논문 "Attention Is All You Need"는 딥러닝 역사상 가장 큰 파장을 일으킨 논문 중 하나입니다. 제목 그대로 "어텐션만 있으면 충분하다" — RNN도, CNN도, 아무것도 필요 없고 W12 §4 어텐션에서 본 self-attention과 몇 개의 MLP만으로 시퀀스를 처리할 수 있다는 주장을 담고, 이를 실제로 구현한 모델이 바로 Transformer입니다.

왜 RNN을 버리려 했을까요? 세 가지 핵심 문제가 있었습니다:

  • 병렬화 불가 — RNN은 시점 $t$의 계산이 $t-1$의 결과에 의존하므로 순차적으로만 계산할 수 있습니다. 1000 단어 문장이면 1000번을 줄줄이 해야 함. GPU의 수천 개 코어가 대부분 놀고 있습니다.
  • 장거리 의존성의 어려움 — LSTM이 완화는 했지만, 여전히 매우 먼 단어 사이 관계는 약합니다. 각 정보가 여러 LSTM 셀을 거치며 변형됩니다.
  • 경로 길이 — 토큰 $i$와 $j$ 사이 정보 교환의 경로 길이가 $O(|i-j|)$. 먼 토큰은 많은 단계를 거쳐야 서로를 "본다".

Transformer는 이 모두를 한 번에 해결합니다. Self-attention에서는:

  • 완전 병렬 — 모든 토큰이 모든 토큰과 동시에 상호작용하므로 GPU에서 최적.
  • 경로 길이 $O(1)$ — 어떤 두 토큰도 한 번의 어텐션으로 직접 연결. 장거리 의존성이 사라짐.
  • 해석 가능 — 어텐션 가중치가 "어떤 단어가 어떤 단어를 보는지"를 직접 보여줌.

대가가 있긴 합니다 — 메모리 사용량이 시퀀스 길이에 $O(n^2)$로 증가합니다. 하지만 GPU 메모리가 빠르게 커지면서 이 한계는 점점 덜 중요해졌고, 최근엔 Flash Attention, sparse attention 같은 기법으로 더 완화되고 있습니다.

2. Self-Attention — Q·K·V 삼각관계

Self-attention은 "각 토큰이 나머지 모든 토큰을 참조한다"는 생각을 구현합니다. 구현 방식은 영리합니다. 각 토큰 표현 $x_i$를 세 개의 학습 가능한 행렬로 투영해 세 개의 벡터를 만듭니다:

$$ \mathbf q_i = W^Q x_i, \quad \mathbf k_i = W^K x_i, \quad \mathbf v_i = W^V x_i $$
  • Query $\mathbf q$: "내가 지금 찾는 것의 특징" — 이 토큰이 궁금해 하는 질문
  • Key $\mathbf k$: "내가 가진 것의 표지" — 다른 토큰들이 자신을 광고하는 라벨
  • Value $\mathbf v$: "내 실제 내용" — 매칭되면 전달될 정보 본체

비유하자면, 도서관에서 책을 찾는 장면과 같습니다. 내가 "머신러닝에 관한 책을 찾는다"(Query)고 하면, 각 책의 겉표지에 쓰인 제목·키워드(Key)와 비교해 일치 점수를 매기고, 가장 잘 맞는 책의 실제 내용(Value)을 가져다 읽는 것. Self-attention은 이걸 한 번에 모든 쿼리와 모든 키에 대해 수행합니다.

수식으로는 놀랍도록 단순합니다:

$$ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V $$

해체해서 읽어봅시다:

  1. $QK^\top$ — 모든 쿼리와 모든 키의 내적. 결과는 $n \times n$ "유사도 행렬". 셀 $(i, j)$는 "토큰 $i$가 토큰 $j$에 얼마나 관심 있는가".
  2. $\div \sqrt{d_k}$ — 키 차원의 제곱근으로 나눔. 왜 이게 필요한지는 아래 §2.1에서.
  3. $\text{softmax}$ — 각 행을 확률 분포로 정규화. 토큰 $i$가 다른 토큰들에 얼마나 주의를 줄지 합이 1이 되게.
  4. $\cdot V$ — 이 가중치로 Value 벡터들을 가중 평균. 결과는 토큰 $i$의 새 표현.

이 한 줄이 Transformer의 전부이자 최신 LLM의 심장입니다.

2.1 왜 $\sqrt{d_k}$로 나누는가

작은 디테일처럼 보이지만 중요합니다. 차원 $d_k$가 크면 $Q$와 $K$의 내적이 큰 분산을 가집니다 (각 원소의 분산이 $O(d_k)$). softmax의 입력이 너무 크면 한 값이 지배해 그래디언트가 거의 0이 됩니다 (softmax가 "날카로워"져서 one-hot에 가까워짐 → 학습 신호 증발). $\sqrt{d_k}$로 나누면 내적의 분산이 $O(1)$로 유지되어 softmax가 부드러운 분포를 만들고 그래디언트가 건강하게 흐릅니다. 작은 수학적 세부지만 이 한 수 없으면 Transformer는 학습이 안 됩니다.

왜 분산이 $O(d_k)$인지 한 줄 증명: 각 원소가 $\mathcal{N}(0, 1)$인 두 벡터 $\mathbf q, \mathbf k$의 내적은 $\sum_{i=1}^{d_k} q_i k_i$인데, 각 $q_i k_i$의 분산이 1이라 독립 합의 분산은 $d_k$가 됩니다. 따라서 표준편차는 $\sqrt{d_k}$, 그것으로 나누면 단위 분산이 회복됩니다. 단순한 통계학이지만, 이 한 줄을 빠뜨린 초기 어텐션 변형들은 모두 학습이 망가졌습니다.

🎮 인터랙티브: $\sqrt{d_k}$ 정규화의 효과 — softmax가 살아있는가 죽었는가

5개의 키와 1개의 쿼리(랜덤 정규분포)에서 내적을 계산하고, (A) 정규화 없음, (B) $\sqrt{d_k}$로 나누기, 두 경우의 softmax 분포와 그 최대 그래디언트를 비교합니다. 차원 $d_k$를 키워보면, 정규화 없는 쪽은 분포가 "한 칸"에 몰리며 그래디언트가 0에 수렴하는 반면, 정규화한 쪽은 부드러운 분포를 유지합니다.

$d_k$를 16에서 256으로 끌어올려 보세요. (A) 정규화 없는 쪽은 분포가 한 키에 0.99 이상 쏠리고 softmax 야코비안의 최댓값이 1e-3 이하로 떨어집니다 — 역전파에서 어텐션 가중치를 거의 학습할 수 없는 상태. (B) 정규화된 쪽은 분포가 0.1~0.4 정도로 골고루 퍼져 그래디언트가 건강합니다. $\sqrt{d_k}$ 한 글자가 차원이 큰 모델에서 학습 가능 여부를 결정합니다 — 그래서 GPT-3의 $d_k = 128$, GPT-4 추정치 $d_k \ge 128$ 같은 큰 모델에서 이 정규화 없이는 학습이 시작되지도 못합니다.

2.2 간단한 예시로 Q·K·V 감 잡기

"The cat sat on the mat"라는 6단어 문장을 생각해봅시다. 각 단어가 3차원 임베딩이라고 치고, "cat"이라는 단어의 표현이 self-attention을 통해 어떻게 갱신되는지 상상해봅시다. "cat"의 Query는 "나는 동물이고, 내 행동과 위치가 궁금해"라는 의미를 담을 수 있습니다. 각 단어의 Key는 "sat: 동작이야", "mat: 장소야" 등. 내적하면 "sat"와 "mat"이 높은 점수를 받고, softmax로 가중치가 되어 그들의 Value가 "cat"의 새 표현에 섞입니다. 결과적으로 "cat"은 이제 "자기 자신 + 자기가 한 행동(sat) + 자기가 있는 곳(mat)"의 풍부한 의미를 담은 새 벡터가 됩니다. 이 과정을 모든 토큰이 병렬로 수행.

🎮 인터랙티브: Self-Attention 행렬

5단어 문장의 Q·K^T 행렬을 시각화합니다. 행은 query, 열은 key. 각 셀의 색은 그 두 단어의 어울림 점수입니다.

3. Multi-Head Attention — 여러 시점의 앙상블

한 번의 self-attention은 한 종류의 관계만 포착할 수 있습니다. 그런데 언어에는 동시에 여러 층위의 관계가 있습니다:

  • 문법적 관계 (주어-동사 일치)
  • 의미적 관계 (who did what to whom)
  • 공지시(coreference) — "it"이 가리키는 게 무엇인지
  • 담화 관계 — 인과, 대조, 시간 순서

해법은 직관적입니다 — 여러 번의 어텐션을 동시에 돌리자. 이를 멀티헤드 어텐션(Multi-Head Attention)이라 부릅니다. 구체적으로는 Q·K·V를 $h$개의 작은 부분 공간으로 쪼개고 (예: $d = 512, h = 8$이면 각 헤드는 64차원), 각 "헤드"가 독립적으로 attention을 계산합니다:

$$ \text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V) $$ $$ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h) W^O $$

각 헤드는 다른 투영 행렬을 학습하므로 서로 다른 종류의 관계에 집중하도록 저절로 전문화됩니다. BERT 같은 훈련된 모델의 헤드를 분석해보면, 실제로 어떤 헤드는 주어-동사 관계를, 어떤 헤드는 대명사-선행사 관계를 학습한 것을 확인할 수 있습니다. 이를 "attention head specialization"이라 부르고, Transformer의 해석 가능성 연구의 주요 주제입니다.

계산량은 같은데 (헤드가 $h$배 많지만 각 차원은 $1/h$) 표현력은 훨씬 풍부해집니다 — 사실상 공짜 점심입니다.

4. 위치 인코딩 — 순서를 주입하는 트릭

Self-attention의 아름다운 점은 위치에 대한 편향이 없다는 것입니다. 어떤 토큰이 어디에 있든 상호작용 방식은 동일합니다. 그런데 이게 문제도 됩니다 — "철수가 영희를 때렸다"와 "영희가 철수를 때렸다"는 단어 집합이 같지만 의미가 완전히 다릅니다. 순서 정보가 필요합니다!

해법은 단순합니다: 각 위치에 고유한 벡터를 더해주기. 입력 임베딩 $\mathbf e_i$에 위치 벡터 $\mathbf{PE}_i$를 더해 $\mathbf x_i = \mathbf e_i + \mathbf{PE}_i$를 만들어 Transformer에 넣습니다. 이제 모델은 "같은 단어라도 1번째 위치와 10번째 위치의 표현이 다름"을 알게 됩니다.

위치 인코딩을 어떻게 만들까요? 원논문은 사인·코사인 함수를 사용했습니다:

$$ PE_{(pos,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos,2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d}}\right) $$

직관적으로 보면, 각 차원마다 서로 다른 주파수의 사인/코사인 파동을 두어 각 위치에 고유한 "지문"을 만듭니다. 낮은 차원은 빠른 주기(이웃 위치 구별), 높은 차원은 느린 주기(멀리 떨어진 위치 구별). 이 선택의 장점:

  • 훈련 데이터에 없었던 긴 시퀀스에도 일반화 가능 (수식으로 계산되니까)
  • 두 위치의 상대 거리가 선형 변환으로 표현 가능 — 모델이 "상대적 거리"를 학습하기 쉬움

현대 변형에는 학습 가능한 위치 임베딩 (BERT), 상대 위치 인코딩 (T5, Transformer-XL), RoPE (Rotary Position Embedding) (LLaMA) 등이 있습니다. 특히 RoPE는 최근 대부분의 오픈소스 LLM의 표준이 되었습니다.

🎮 인터랙티브: 사인 위치 인코딩 히트맵

각 위치(행)와 각 차원(열)에서 사인/코사인 위치 인코딩 값을 시각화합니다. 낮은 차원은 빠르게 진동하고 높은 차원은 천천히 진동해, 모든 위치에 고유한 "지문"이 만들어져요.

5.1 BERT vs GPT — 같은 블록, 다른 목적

둘 다 Transformer 블록을 쌓은 모델이지만 철학이 정반대입니다.

BERT (Bidirectional Encoder Representations from Transformers, 2018)는 인코더 블록만 씁니다. 훈련 방법: 입력의 약 15% 단어를 마스킹하고 그 단어를 맞추게 합니다 (Masked Language Model). 양방향(앞뒤 모두) 컨텍스트를 사용하므로 "이해"에 뛰어납니다 — 감성 분석, 문장 분류, 질문 응답, 개체명 인식에서 2018년 당시 거의 모든 벤치마크를 갱신했습니다.

GPT (Generative Pre-trained Transformer, 2018~)는 디코더 블록만 씁니다 (인코더-디코더 사이 cross-attention 제거). 훈련 방법: 앞 단어들을 보고 다음 단어를 맞추는 단순한 autoregressive language modeling. 단방향(왼쪽에서 오른쪽)이라 "생성"에 뛰어납니다. 왼쪽 문맥으로부터 텍스트를 이어 쓰는 것이 기본 능력.

처음엔 BERT가 학계에서 우세했습니다. 하지만 2020년 GPT-3가 1750억 파라미터로 "few-shot learning"을 시연하면서 분위기가 역전되었습니다. 프롬프트에 몇 개의 예시만 주면 재훈련 없이 새 작업을 수행할 수 있다는 것이 입증되었고, 이는 BERT가 재훈련(fine-tuning)을 필요로 하는 것과 대조됩니다. 2022년 ChatGPT가 대중에게 공개되면서 GPT 계열이 NLP의 표준이 되었습니다.

🎮 인터랙티브: BERT (양방향) vs GPT (단방향) 마스킹

두 모델이 어텐션을 칠 때 어떤 위치를 볼 수 있는지 보세요. BERT는 [MASK]된 위치를 빼고 모든 곳을 봅니다. GPT는 인과 마스크(causal mask)로 자기 이전 위치만 봅니다 — 그래서 다음 단어 예측이 가능해요.

5.2 스케일링 법칙 (Scaling Laws)

OpenAI의 2020년 논문 "Scaling Laws for Neural Language Models"는 충격적인 관찰을 발표했습니다. 모델 크기 $N$, 데이터셋 크기 $D$, 계산량 $C$가 커질 때 손실이 다음 멱법칙(power law)을 따른다는 것:

$$ L(N) \propto N^{-0.076}, \quad L(D) \propto D^{-0.095}, \quad L(C) \propto C^{-0.050} $$

이는 "모델을 계속 크게 만들면 성능이 예측 가능하게 올라간다"는 의미입니다. 그리고 실제로 이 법칙을 따라 GPT-1 → GPT-2 → GPT-3 → GPT-4로 스케일을 키우자 성능이 계속 올랐습니다. 모델 크기는 GPT-1의 1.1억 개에서 GPT-3의 1750억 개, GPT-4의 추정 1조 개 이상까지 폭발적으로 커졌고, 훈련 데이터도 수십 GB에서 수 TB로.

더 놀라운 것은 창발적 능력(emergent abilities)입니다. 작은 모델에서는 전혀 보이지 않던 능력들 — few-shot 학습, 산술 계산, 추론, 코드 작성 — 이 특정 규모를 넘어가면 갑자기 나타납니다. 이는 단순한 점진적 개선이 아니라 질적 전환으로 보입니다. 왜 이런 현상이 일어나는지는 아직 완전히 설명되지 않았고, 현대 AI 연구의 큰 미스터리입니다.

6. 샘플링과 온도

LLM은 다음 단어의 확률 분포를 출력합니다. 그중 하나를 고르는 방법:

  • Greedy: 최댓값 — 결정적, 단조롭다.
  • Top-k: 상위 k개 중에서 샘플링.
  • Top-p (nucleus): 누적 확률 p까지의 단어들 중에서.
  • 온도: 소프트맥스 전 점수를 $T$로 나눔. $T<1$이면 뾰족(보수적), $T>1$이면 평평(창의적).

🎮 인터랙티브: 온도가 출력 분포에 미치는 영향

5개 후보 단어의 점수를 받아 온도에 따라 확률 분포가 어떻게 변하는지 봅니다. 온도가 0에 가까우면 한 단어만 살아남고, 크면 모두 비슷해집니다.

6.1 그리디·빔서치는 왜 부족한가

가장 단순한 디코딩은 매 스텝 확률 최댓값을 고르는 그리디입니다. 결정적이고 빠르지만, 텍스트 생성에서는 "the the the"처럼 같은 토큰이 반복되거나 단조로운 문장이 자주 나옵니다. 빔서치(W12)는 후보 $K$개를 동시에 추적해 그리디보다 좋지만, 여전히 고확률 경로만 선택하므로 창의성이 떨어지고 반복도 잡지 못합니다.

실제 사람의 글은 "항상 가장 그럴듯한 단어"를 선택하지 않습니다. Holtzman et al. 2020("The Curious Case of Neural Text Degeneration")이 보여준 충격적 결과 — 사람이 쓴 문장의 단어 확률은 매우 들쭉날쭉한데, 그리디·빔이 만든 문장은 확률이 부드럽게 평탄합니다. 즉 너무 "예측 가능"해서 부자연스러운 거예요. 샘플링이 답입니다.

6.2 Top-k 샘플링 — 상위 k개만 후보로

매 스텝 확률 상위 $k$개 토큰만 남기고 나머지를 0으로 만든 뒤 정규화해 샘플링. 보통 $k = 40\sim50$.

  • 장점 — 꼬리(말도 안 되는 단어)를 잘라내 일관성 유지. 단순.
  • 단점 — $k$가 고정이라 분포 형태에 둔감합니다. 분포가 매우 뾰족해서 1등이 90% 차지하는 상황에서도 $k=40$이면 의미 없는 39개 단어를 끌어와 정답을 흐립니다. 반대로 분포가 매우 평평하면 40개로는 너무 좁아요.

6.3 Top-p (Nucleus) — 분포 형태에 적응

Holtzman 2020이 제안한 방식. 누적 확률이 $p$가 되는 최소 집합(nucleus)만 남기고 거기서 샘플링. 보통 $p = 0.9\sim0.95$.

  • 분포가 뾰족하면 1~3개만 남고, 평평하면 수십~수백 개가 남습니다 — 자동 적응이 핵심.
  • 현재 GPT API의 기본값(top_p=1.0이지만 OpenAI 권장은 0.9 부근). Llama·Mistral 등도 top-p가 표준.

6.4 온도와의 결합

온도 $T$는 샘플링 이전에 분포 모양을 조정합니다. 위 데모에서 보듯 $T = 0$ 극한이 그리디, $T = \infty$ 극한이 균등 분포. 실무에선 온도 + top-p를 함께 쓰는 게 표준입니다.

  • 코드 생성·QA·요약 — $T = 0.2\sim0.7$, top-p $= 0.9$. 정확성 우선.
  • 창작·브레인스토밍 — $T = 0.8\sim1.2$, top-p $= 0.95$. 다양성 우선.
  • 결정적 출력 필요 — $T = 0$ (그리디). 디버깅·재현성 확보.

6.5 반복 페널티

샘플링을 써도 같은 구절이 반복되는 일이 종종 있습니다. 두 가지 흔한 처방:

  • Repetition penalty (Keskar 2019) — 이미 등장한 토큰의 점수를 $r$로 나눠 깎음(보통 $r = 1.1\sim1.3$). 점수가 양수면 약화, 음수면 강화 방향이라 부호 처리에 주의.
  • Presence/Frequency penalty (OpenAI API) — 등장 여부(present)와 등장 횟수(frequency)에 비례한 상수를 점수에서 차감. 반복뿐 아니라 어휘 다양성을 끌어올림.
한 줄 정리 — "정답을 맞히는 일"은 낮은 $T$ + 작은 top-p(혹은 그리디), "사람처럼 자연스럽게 쓰는 일"은 적당한 $T$ + top-p. 절대 정답은 없으니 도메인에 맞춰 두세 조합 시험하고 정착하세요.

7. 코드 예제 (Hugging Face)

from transformers import AutoTokenizer, AutoModelForCausalLM

tok = AutoTokenizer.from_pretrained('gpt2')
model = AutoModelForCausalLM.from_pretrained('gpt2')

inputs = tok('Once upon a time', return_tensors='pt')
out = model.generate(**inputs, max_length=50, temperature=0.8,
                     top_p=0.9, do_sample=True)
print(tok.decode(out[0]))

✅ 오늘의 핵심 4가지

  1. RNN을 버리고 어텐션만. Self-attention은 모든 토큰이 모든 토큰을 한 번에 보는 구조 → 병렬화 + 경로 길이 $O(1)$.
  2. Q·K·V 삼각관계. Query는 질문, Key는 라벨, Value는 내용. softmax($QK^\top/\sqrt{d_k}$)V 한 줄이 LLM의 심장.
  3. Multi-Head + Positional Encoding. 여러 종류의 관계를 동시에, 그리고 순서 정보를 따로 주입.
  4. BERT는 인코더, GPT는 디코더. 같은 트랜스포머의 두 얼굴. 오늘날 ChatGPT·Claude 등 모든 LLM이 GPT 계열의 후손.

8. 마무리 — 우리는 어디로 가는가

13주에 걸쳐 W1 KNN에서 시작해 트랜스포머까지 왔습니다. 지난 70년의 AI 역사를 한 학기에 압축한 셈입니다.

마지막으로 한 학기를 돌아보면, 우리가 본 모든 모델이 결국 같은 두 가지 일을 해왔습니다 — "좋은 표현을 만든다"와 "손실을 최소화한다". KNN의 거리, 회귀의 직선, MLP의 은닉층, CNN의 필터, RNN의 은닉상태, Transformer의 어텐션 모두 "같은 클래스끼리는 가까이, 다른 클래스는 멀리" 두는 표현을 만들려 했고, 그러기 위해 손실을 미분해 가중치를 업데이트했습니다. 모델 이름이 어떻게 바뀌어도 이 뼈대는 변하지 않습니다.

다음 단계 — 학기가 끝난 뒤

  • 한 가지를 끝까지 만들어보기. Kaggle 입문 대회(Titanic, Digit Recognizer)를 W1~W8 도구로 풀고, 작은 이미지 분류 프로젝트를 W9~W10으로 풀어보세요. 손이 코드를 외우는 순간 강의 내용이 자기 것이 됩니다.
  • HuggingFace Transformers로 텍스트 생성·분류·번역 미니 프로젝트. 사전학습 모델을 가져와 fine-tune 한번만 해보면 LLM 시대가 손에 잡힙니다.
  • Karpathy의 "Let's build GPT" 영상을 따라 200줄 GPT를 직접 코딩. Transformer의 모든 부품이 머릿속에 정렬됩니다.
  • 다음 주제로 — 강화학습(RL), 생성모델(VAE/GAN/Diffusion), 멀티모달(CLIP/Stable Diffusion), AI 안전성과 정렬(alignment)이 자연스러운 다음 단계입니다.

고생 많으셨습니다. 한 학기 동안 손으로 굴린 모든 데모와 디버깅이 여러분의 진짜 자산입니다. 앞으로 어떤 새 모델이 나와도 — 결국 우리가 본 것들의 변주일 뿐이라는 사실, 그것을 안다는 자신감이 가장 큰 선물이길 바랍니다.

📐 트랜스포머 블록 손계산 — 한 페이지로

입력 시퀀스 길이 T=3, 차원 d=4. 한 트랜스포머 인코더 블록의 동작:

  1. 입력 X = [x_1, x_2, x_3], 각 4차원.
  2. + Positional Encoding — 위치 정보 더함.
  3. Multi-Head Self-Attention — Q, K, V 계산 → softmax(QK^T/√d)V → (3, 4).
  4. + Residual — X에 더함.
  5. LayerNorm — 정규화.
  6. FFN — Linear(4→16) → ReLU → Linear(16→4).
  7. + Residual — 한 번 더.
  8. LayerNorm — 출력 (3, 4).

이 한 블록을 6~96번 쌓으면 트랜스포머. GPT-3는 96 블록.

🎯 In-Context Learning

GPT-3 이후 LLM의 놀라운 능력. 학습 없이 프롬프트의 예시 몇 개만 보고 새 작업을 수행.

프롬프트:
영어→한국어:
hello → 안녕하세요
goodbye → 안녕히 가세요
thank you → ?

GPT 출력: 감사합니다

가중치 업데이트가 전혀 없는데 새 작업을 학습한 것처럼 동작. 이게 LLM이 "범용 도구"로 쓰이는 이유.

🌟 Prompt Engineering — 새로운 기술

  • Zero-shot — 예시 없이 작업 설명만.
  • Few-shot — 예시 2~5개와 작업.
  • Chain-of-Thought — "단계별로 생각해보자". 추론 능력 향상.
  • Self-Consistency — 여러 답을 생성하고 다수결.
  • ReAct — Reasoning + Acting. 도구 사용.

🛠️ RLHF — 인간 피드백 강화학습

ChatGPT의 비밀. 세 단계:

  1. SFT (Supervised Fine-Tuning) — 사람이 쓴 좋은 답으로 fine-tune.
  2. Reward Model — 사람이 답 두 개 비교한 데이터로 보상 모델 학습.
  3. RL — PPO로 보상 모델을 maximize하도록 LLM 추가 학습.

이 세 단계가 GPT-3(1750억) 같은 raw 모델을 ChatGPT처럼 도움이 되고 안전한 모델로 바꿉니다.

📝 연습문제

연습 1. 시퀀스 길이 T=1024, 차원 d=768일 때 self-attention의 메모리 사용량은?
풀이 보기

$T \times T = 1024^2 = $ 약 100만 셀, FP32면 4MB(한 헤드당). 12 헤드면 48MB(한 층당). 12층이면 576MB. 이게 한 입력의 어텐션 메모리. 배치까지 곱하면 GB 단위.

연습 2. Positional Encoding이 필요한 이유?
풀이 보기

Self-Attention은 토큰 순서를 모름(순열 불변). 셔플해도 같은 결과. 그러나 언어는 순서가 매우 중요. → 위치 정보를 임베딩에 더해줘야 함.

연습 3. Multi-Head Attention에서 헤드 수 8, 차원 512면 각 헤드의 차원은?
풀이 보기

512 / 8 = 64. 각 헤드가 64차원에서 어텐션 수행. 결과를 concat해서 다시 512차원. 총 파라미터 수는 single head 512차원과 같지만 표현력은 더 풍부.

연습 4. GPT의 Causal Mask가 BERT의 마스크와 어떻게 다른가?
풀이 보기

BERT — 임의의 위치 15%를 [MASK]로 바꾸고 복원 학습. 양방향 어텐션 가능. GPT — 항상 다음 토큰 예측. 미래 토큰을 못 보는 인과적 마스크. 학습 목표가 다름.

연습 5. Temperature를 0.1로 하면? 2.0으로 하면?
풀이 보기

0.1 — 거의 greedy. 결정적, 단조로움, 사실에 충실. 2.0 — 매우 다양함, 창의적이지만 헛소리 가능성 ↑. 보통 0.7~1.0이 균형.

❓ FAQ

Q1. 왜 트랜스포머가 RNN을 이겼나요?
A. (1) 병렬화 — RNN은 순차적, 트랜스포머는 한 번에. GPU 효율 100배. (2) 거리 무관 의존성. (3) 사전학습 효과. (4) 큰 데이터에서 잘 스케일.
Q2. ChatGPT는 어떻게 만들어졌나?
A. GPT-3.5 base → SFT (사람이 쓴 답) → RLHF (사람 선호 학습 + PPO 미세조정). 핵심은 사람의 피드백을 학습 신호로.
Q3. LLM이 환각(hallucination)하는 이유?
A. LLM은 "다음 단어 확률"을 학습할 뿐 진실을 학습하지 않음. 그럴듯한 패턴이면 사실 여부 무관 생성. RAG, 도구 사용으로 완화.
Q4. GPT-4는 GPT-3보다 얼마나 큰가?
A. OpenAI 비공개. 추정 1조 파라미터(MoE). 그러나 단순 스케일이 아니라 학습 데이터·정렬 품질 차이가 큼.
Q5. 한국어 LLM은?
A. KoGPT(카카오), HyperCLOVA(네이버), Polyglot-Ko(EleutherAI), KULLM(고려대) 등. 한국어 성능은 GPT-4가 여전히 우수.

🌟 LLM 응용 사례

  • 대화 AI — ChatGPT, Claude, Gemini.
  • 코드 생성 — GitHub Copilot, Codex.
  • 번역 — 인간 수준에 가까움.
  • 요약 — 논문, 회의록 자동 요약.
  • 검색 — Perplexity, Bing Chat. 답변 + 출처.
  • 교육 — 개인화 튜터.
  • 자동화 — 이메일 작성, 일정 관리.
  • 창작 — 소설, 시나리오, 광고 카피.

🛡️ LLM 안전성 — 알아둘 5가지 위험

  1. 환각(Hallucination) — 사실이 아닌 그럴듯한 답.
  2. 편향(Bias) — 학습 데이터의 사회적 편향 학습.
  3. 프라이버시 — 학습 데이터 외설.
  4. 오용 — 가짜 뉴스 대량 생성, 사이버 공격.
  5. 일자리 영향 — 일부 직업 자동화.

좋은 엔지니어는 도구를 쓸 수 있을 뿐 아니라 그 한계와 위험을 안다.

🧭 한 학기 정리 — 큰 그림

"AI ⊃ ML ⊃ DL이고, DL의 정점이 트랜스포머와 LLM." KNN으로 시작한 한 학기의 흐름이 모두 한 곳으로 수렴합니다 — 학습 가능한 표현 + 손실 최소화. 이름이 어떻게 바뀌어도 이 뼈대는 그대로입니다.

🎓 졸업 후 추천 진로

  • ML 엔지니어 — 모델 개발 및 배포. 산업의 핵심.
  • 데이터 사이언티스트 — 데이터 분석 + 모델링.
  • AI 연구원 — 새 모델 연구. 대학원 또는 연구소.
  • MLOps — 모델 배포·운영 인프라.
  • 응용 AI — 의료·금융·로보틱스 등 도메인 특화.
  • AI 안전 — 정렬, 해석성, 위험 평가.

🌟 LLM Fine-tuning 방법

  • Full Fine-tuning — 모든 파라미터 학습. 비용 큼.
  • LoRA (Low-Rank Adaptation) — 작은 행렬만 추가 학습. 메모리 1/10.
  • QLoRA — 4-bit 양자화 + LoRA. GPU 1대로 70B 모델 학습.
  • Prefix/Prompt Tuning — 입력 앞에 학습 가능한 토큰만.
  • Adapter — 작은 모듈을 층 사이 추가.

🌟 RAG (Retrieval-Augmented Generation)

LLM의 환각을 줄이는 표준 패턴.

  1. 사용자 질문을 임베딩.
  2. 벡터 DB에서 관련 문서 검색.
  3. 검색된 문서를 프롬프트에 넣어 LLM에 질의.
  4. LLM이 문서 기반으로 답변.

장점: 최신 정보 반영, 출처 제공, 환각 ↓. 도구: LangChain, LlamaIndex, Pinecone.

🌟 멀티모달 모델 — CLIP, GPT-4V, LLaVA

  • CLIP (2021) — 이미지와 텍스트를 같은 임베딩 공간에. zero-shot 분류.
  • Stable Diffusion (2022) — 텍스트 → 이미지 생성. CLIP + Diffusion.
  • GPT-4V/Gemini Vision — 이미지 이해 + 대화.
  • LLaVA — 오픈소스 비전-언어 LLM.

🛡️ AI 윤리 다섯 원칙

  1. 공정성 — 인종·성별·계층에 편향되지 않게.
  2. 투명성 — 모델이 어떻게 결정했는지 설명 가능.
  3. 프라이버시 — 학습 데이터·사용 데이터 보호.
  4. 안전성 — 악용 방지.
  5. 책임 — 잘못된 결정의 책임 주체 명확.

🎓 한 학기 회고 — KNN에서 LLM까지

13주 전 우리는 도미와 빙어를 길이와 무게로 구분하는 KNN으로 시작했습니다. 오늘 우리는 GPT-3, ChatGPT, BERT의 내부 구조를 손으로 그릴 수 있게 됐습니다. 이 거리는 단지 "13주의 학습"이 아니라 "70년 AI 역사의 압축"입니다.

매 주차의 모델이 다음 주차의 부품이 되고, 그 부품이 또 다음 주차의 더 큰 부품이 되는 — 누적의 마법을 경험했습니다.

🌐 한 학기 모델 가족도

  • W1: KNN — 거리 기반.
  • W3: 선형 회귀 → 경사하강.
  • W4: 로지스틱 회귀 → 소프트맥스.
  • W5: SVM, 트리, 랜덤포레스트 — 고전 ML.
  • W6: 비지도 — K-means, PCA.
  • W7: MLP — 비선형 활성.
  • W8: 역전파 — 학습의 엔진.
  • W9-10: CNN — 이미지.
  • W11: RNN/LSTM — 시간.
  • W12: Seq2Seq + Attention.
  • W13: Transformer + LLM — 정점.

🛠️ 졸업 프로젝트 아이디어

  1. 한국어 감성 분석 챗봇 (BERT fine-tune).
  2. 의료 X-ray 분류 (ResNet 전이학습).
  3. 주가 시계열 예측 (LSTM).
  4. 이미지 캡션 생성 (CNN + RNN + Attention).
  5. 나만의 작은 GPT (nanoGPT 따라하기).
  6. RAG 기반 학과 FAQ 봇.
  7. YOLO로 실시간 객체 탐지.
  8. 강화학습으로 게임 AI.

📌 부록 — Transformer Pseudo-Code

def transformer_block(x):
    # Pre-LN 스타일 (현대 표준)
    a = MultiHeadAttention(LayerNorm(x))
    x = x + Dropout(a)
    f = FFN(LayerNorm(x))      # Linear → GELU → Linear
    x = x + Dropout(f)
    return x

def transformer(tokens):
    x = Embedding(tokens) + PositionalEncoding(len(tokens))
    for _ in range(N):
        x = transformer_block(x)
    return LayerNorm(x)

def gpt_generate(prompt, max_tokens):
    tokens = tokenize(prompt)
    for _ in range(max_tokens):
        h = transformer(tokens)            # (T, d)
        logits = h[-1] @ W_out              # (vocab,)
        next_token = sample(softmax(logits / temperature))
        tokens.append(next_token)
        if next_token == EOS: break
    return detokenize(tokens)

📌 부록 — LLM 효율 기술

  • Flash Attention — IO-aware 어텐션. 메모리 효율.
  • KV Cache — 추론 시 K, V 재사용.
  • Speculative Decoding — 작은 모델로 후보 생성, 큰 모델로 검증.
  • Quantization (INT8/INT4) — 메모리 1/4~1/8.
  • Pruning — 불필요한 가중치 제거.
  • Distillation — 큰 모델 → 작은 모델.

📌 부록 — 한국어 LLM 생태계

  • HyperCLOVA X — 네이버. 한국어 특화 거대 모델.
  • KoGPT — 카카오 브레인.
  • Polyglot-Ko — EleutherAI. 12.8B.
  • SOLAR — Upstage. 오픈소스 우수.
  • Gemma-Ko — Google Gemma 한국어 fine-tune.
  • Qwen-Ko, LLaMA-Ko — 오픈 모델 한국어 fine-tune.

🌟 마지막 한 마디

한 학기 동안 만난 모든 모델의 본질은 두 가지였습니다 — "좋은 표현을 만든다"와 "손실을 최소화한다". KNN의 거리, 회귀의 직선, MLP의 은닉층, CNN의 필터, RNN의 게이트, 트랜스포머의 어텐션, 모두 이 두 임무를 다른 형태로 수행해 왔습니다.

앞으로 어떤 새 모델이 나와도 — Diffusion, Mamba, MoE — 모두 같은 두 임무의 새 형태일 뿐입니다. 그 사실을 안다는 것이 한 학기의 가장 큰 선물입니다.

📌 부록 — LLM 평가 지표

  • Perplexity — 학습 시 표준. 낮을수록 좋음.
  • HellaSwag — 상식 추론.
  • MMLU — 다분야 지식 (57개 분야).
  • HumanEval — 코드 생성.
  • GSM8K — 수학 문제.
  • BIG-Bench — 200+ 다양한 작업.
  • MT-Bench, AlpacaEval — 대화 품질 (LLM-as-judge).
  • Chatbot Arena (Elo) — 사람 직접 비교.

📌 부록 — LLM 학습 비용

  • GPT-3 (175B) — 학습 비용 약 460만 달러 (2020년).
  • LLaMA-2 (70B) — 학습 비용 약 200만 달러.
  • GPT-4 — 추정 1억 달러+.
  • 오픈소스 LoRA fine-tune — 100~1000달러로 가능.

그래서 처음부터 학습하지 않고 사전학습된 오픈 모델을 fine-tune하는 게 표준.

📌 부록 — Transformer를 이해하는 가장 좋은 방법

Karpathy의 "Let's build GPT from scratch" 영상(2시간)을 따라하며 직접 구현. 200줄 코드로 작은 GPT를 처음부터 만들면 모든 부품이 머릿속에 정렬됩니다. 본 강의 끝에 강력 추천하는 한 가지.

📌 부록 — Diffusion Model 미리보기 (강의 외)

이미지 생성의 새로운 표준. Stable Diffusion, DALL·E의 기반.

  1. Forward — 이미지에 점차 가우시안 잡음을 더해 완전 잡음으로.
  2. Reverse — 학습된 신경망(보통 U-Net)이 잡음을 단계적으로 제거.
  3. 학습 — 각 단계에서 잡음을 예측하도록.
  4. 생성 — 잡음에서 시작해 학습된 모델로 단계적 복원.

Transformer와 별개의 가족이지만, 최근 DiT (Diffusion Transformer)로 결합되며 멀티모달 생성의 표준이 되어 가는 중.

📌 부록 — Mixture of Experts (MoE)

여러 작은 트랜스포머(전문가)를 두고, 입력마다 일부만 활성화. GPT-4, Mixtral의 추정 구조.

  • 장점 — 같은 계산량으로 더 큰 파라미터(전문성).
  • 단점 — 학습 어려움, 메모리 큼.

📌 부록 — 강화학습과 LLM

  • RLHF — 사람 선호로 정렬.
  • RLAIF — AI 피드백으로 RLHF 자동화.
  • DPO (Direct Preference Optimization) — 보상 모델 없이 직접 선호 학습.
  • PPO — 강화학습 표준 알고리즘.

📌 부록 — 에이전트 (Agent)

LLM이 도구(검색, 계산, 코드 실행)를 사용하는 패러다임.

  • ReAct — Reason + Act 반복.
  • Tool Use — 함수 호출 능력.
  • AutoGPT, BabyAGI — 자율 에이전트 실험.
  • LangChain, LangGraph — 에이전트 구축 프레임워크.

📌 부록 — LLM Inference 옵션

📌 부록 — LLM의 한계와 보완

  • 지식 cutoff — 학습 시점 이후 모름. → RAG로 보완.
  • 수학 약함 — 계산기 도구 사용으로 보완.
  • 긴 컨텍스트 약함 — Long Context 모델, RAG로.

  • 최신 사건 — Web search 도구.

  • Role Prompting — "당신은 전문 의사입니다…"

  • Chain-of-Thought — "단계별로 생각해보세요."

  • Format Specification — "JSON 형식으로 답하세요."

📌 부록 — LLM Hallucination 예방 5가지

  1. RAG로 외부 지식 주입.
  2. "모르면 모른다고 답하라" 명시.
  3. 출처 인용 요구.
  4. Self-consistency 다수결.
  5. 도구 사용 (계산기, 검색).

📌 부록 — 한 학기를 한 줄로

"AI는 데이터로부터 좋은 표현을 학습하고, 그 표현으로 손실을 최소화한다." 이 한 문장이 한 학기 13주의 응축입니다.

📌 부록 — 마지막 자가 학습 추가 문제

  2. BERT의 [MASK] 토큰의 역할?
  3. GPT-3가 GPT-2보다 100배 큰데 학습 시간은 몇 배?
  4. RLHF에서 Reward Model의 입력과 출력?

정답 힌트: 1) Self는 같은 시퀀스, Cross는 다른 시퀀스. 2) 양방향 컨텍스트로부터 [MASK] 단어 예측 학습. 3) 약 10~30배(파라미터 수와 데이터 양 모두 영향). 4) 입력=프롬프트+답, 출력=점수. 5) RAG (fine-tuning은 학습 시점 정보로 고정).

📌 부록 — Cheat Sheet (한 페이지 요약)

  • W1 KNN: 가까운 K명 다수결.
  • W2 데이터: 표준화·분할·누수 방지.

  • W5 분류: KNN·SVM·트리·RF.
  • W6 비지도: K-means·DBSCAN·PCA.
  • W7 MLP: 비선형 활성, 보편 근사.
  • W8 역전파: 연쇄법칙. ReLU·BN·Adam.
  • W9 CNN: 필터 슬라이딩·가중치 공유.
  • W10 전이학습: 사전학습 + fine-tune.
  • W11 RNN/LSTM: 시간 축, 게이트.
  • W12 Seq2Seq + Attention: Q·K·V.
  • W13 Transformer + LLM: Self-Attention·BERT·GPT·RLHF.
