Week 4. CFTOC와 QP 정식화

제약을 더하는 순간 닫힌형 해는 사라지고 우리는 매 스텝 이차계획법(Quadratic Programming, QP)을 풀어야 합니다. 이번 주는 유한 호라이즌 제약 최적 제어 (CFTOC) 문제를 표준 QP로 정확히 어떻게 변환하는지 한 줄씩 따라갑니다.

목차

  1. CFTOC 문제 정식화
  2. 실현 가능 집합 (Feasible Sets)
  3. 치환법(substitution) — 입력만 변수로
  4. 비치환법(non-substitution) — 상태도 변수로
  5. QP 표준형으로 정리
  6. QP 솔버 — OSQP, qpOASES, quadprog

1. CFTOC — 문제 정식화

이산 LTI 시스템 + 이차 비용 + 다면체(polyhedral) 제약:

$$ \min_{U_0,X_0}\ J_0(x(0), U_0) = x_N^\top P x_N + \sum_{k=0}^{N-1}\left(x_k^\top Q x_k + u_k^\top R u_k\right) $$ $$ \text{s.t.}\quad x_{k+1} = A x_k + B u_k,\ k = 0,\ldots,N-1 $$ $$ x_k \in \mathcal{X},\ u_k \in \mathcal{U},\ k = 0,\ldots,N-1 $$ $$ x_N \in \mathcal{X}_f,\ x_0 = x(0) $$

여기서 $\mathcal{X}, \mathcal{U}, \mathcal{X}_f$는 모두 다면체 (선형 부등식 다발). 예: $u_{\min} \le u_k \le u_{\max}$, $H_x x_k \le h_x$.

$U_0 = [u_0, u_1, \ldots, u_{N-1}]$가 결정 변수. 이 문제는 닫힌형 해가 없습니다. 대신 부등식이 활성화되는 패턴이 영역(region)별로 다른 조각별 affine 해를 갖고, 매 스텝 그 영역을 찾아내는 게 QP를 푸는 일.

2. 실현 가능 집합 (Feasible Sets)

주어진 초기 상태 $x_0$에서 위 문제가 실현 가능 (모든 제약 만족 가능)할까요? 그 답이 $N$-단계 실현가능 집합 $\mathcal X_0$입니다:

$$ \mathcal X_0 = \{x_0 \in \mathbb{R}^n \mid \exists\, (u_0,\ldots,u_{N-1})\ \text{s.t. 모든 제약 만족}\} $$

이 집합은 다면체이고, $N$이 클수록 보통 더 큽니다 (시간이 더 있으니 더 멀리서도 회복 가능). $\mathcal X_i$ ($i$ 단계 후 실현가능 집합)을 거꾸로 계산하면 $\mathcal X_0$를 얻을 수 있고, 이는 W6 안정성 분석의 기초가 됩니다.

중요한 점: $\mathcal X_0$ 밖에서 시작하면 MPC가 풀 수 없습니다. 이게 가용성(feasibility) 문제이고, 실무에서는 소프트 제약(slack)으로 우회합니다 — 위반량에 큰 벌점을 주지만 위반은 허용.

3. 치환법 — 입력만 변수로

모델 $x_{k+1} = A x_k + B u_k$를 반복 대입하면 미래 상태가 입력만의 함수로 표현됩니다:

$$ x_k = A^k x_0 + \sum_{j=0}^{k-1} A^{k-1-j} B u_j $$

벡터화하면 (대문자는 쌓은 벡터/행렬):

$$ \mathbf{X} = \mathcal{A}\, x_0 + \mathcal{B}\, \mathbf{U} $$

여기서

$$ \mathbf{X} = \begin{bmatrix}x_0\\x_1\\\vdots\\x_N\end{bmatrix},\ \mathbf{U} = \begin{bmatrix}u_0\\u_1\\\vdots\\u_{N-1}\end{bmatrix},\ \mathcal{A} = \begin{bmatrix}I\\A\\A^2\\\vdots\\A^N\end{bmatrix} $$ $$ \mathcal{B} = \begin{bmatrix}0 & 0 & \cdots & 0\\B & 0 & \cdots & 0\\AB & B & \cdots & 0\\\vdots&&\ddots&\vdots\\A^{N-1}B & A^{N-2}B & \cdots & B\end{bmatrix} $$

이 두 행렬은 모델만으로 결정되어 한 번 계산해두면 끝.

비용도 행렬로 압축 가능. $\bar Q = \text{blkdiag}(Q, Q, \ldots, Q, P)$ ($N+1$개), $\bar R = \text{blkdiag}(R, \ldots, R)$ ($N$개)로 두면:

$$ J = \mathbf{X}^\top \bar Q \mathbf{X} + \mathbf{U}^\top \bar R \mathbf{U} $$

$\mathbf X$를 입력으로 치환하면:

$$ J(\mathbf U; x_0) = \mathbf U^\top \mathbf H \mathbf U + 2 x_0^\top \mathbf F^\top \mathbf U + x_0^\top \mathbf Y x_0 $$

여기서 $\mathbf H = \mathcal B^\top \bar Q \mathcal B + \bar R$, $\mathbf F = \mathcal B^\top \bar Q \mathcal A$, $\mathbf Y = \mathcal A^\top \bar Q \mathcal A$.

마지막 항은 $x_0$만의 함수이므로 최적화에 영향 없음 → 무시. 결과는 다음과 같은 표준 QP:

$$ \min_{\mathbf U}\ \tfrac{1}{2} \mathbf U^\top (2\mathbf H) \mathbf U + (2 \mathbf F x_0)^\top \mathbf U,\quad \text{s.t.}\quad G \mathbf U \le w + S x_0 $$

제약 $\mathbf X = \mathcal A x_0 + \mathcal B \mathbf U \in \mathcal X^{N+1}$도 치환하면 $\mathbf U$만의 부등식이 되어 위 형태로 정리됩니다.

장단점

  • ✓ 변수 개수 적음 ($Nm$개) → 빠름
  • ✗ Hessian $\mathbf H$가 빽빽 (dense) → 큰 $N$에서 비효율
  • ✗ 수치 조건수 나빠질 수 있음

4. 비치환법 — 상태도 변수로

모델을 동등한 등식 제약으로 두고 상태도 결정 변수로 둡니다:

$$ \min_{\mathbf U, \mathbf X}\ \mathbf X^\top \bar Q \mathbf X + \mathbf U^\top \bar R \mathbf U $$ $$ \text{s.t.}\quad x_{k+1} - A x_k - B u_k = 0,\ k = 0,\ldots,N-1 $$ $$ x_0 = x(0),\quad x_k \in \mathcal X,\ u_k \in \mathcal U,\ x_N \in \mathcal X_f $$

변수 개수는 $N(n+m) + n$로 더 많지만, Hessian과 등식 제약 행렬이 희소(sparse) + 띠 구조(banded)를 가져 큰 $N$에서 훨씬 빠릅니다. 현대 MPC 솔버(OSQP, HPIPM, FORCES)는 모두 이 형태를 선호.

5. 표준 QP 형태로

둘 다 결국 다음 표준형으로 떨어집니다:

$$ \min_{z}\ \tfrac{1}{2} z^\top H z + g^\top z,\quad \text{s.t.}\quad A_{eq} z = b_{eq},\ A_{in} z \le b_{in} $$

이 형태이면:

  • $H \succeq 0$ (정정/양반정), $H \succ 0$이면 strict convex → 유일 해.
  • 활성 제약 추적(active set), 내부점법(interior-point), ADMM 등으로 풂.
  • QP는 다항식 시간 풀이가 가능 (NP-hard 아님). 임베디드에서 ms 단위.

🎮 인터랙티브: QP 등고선과 활성 제약

2변수 QP $\min \tfrac{1}{2}(z_1^2 + z_2^2)$를 박스 제약 안에서 푸는 단순한 예. 박스 위치를 움직여 보세요. 박스가 원점을 포함하면 무제약 해(0,0)이 답이고, 그렇지 않으면 박스 모서리·꼭짓점이 답이 됩니다.

7. 코드 예제 — CVXPY로 한 번에

import cvxpy as cp
import numpy as np

A, B = np.array([[1,0.1],[0,1]]), np.array([[0],[0.1]])
N = 10
Q, R, P = np.diag([10,1]), np.array([[1]]), np.diag([10,1])
x0 = np.array([3.0, 0.0])
u_max = 2.0

x = cp.Variable((2, N+1))
u = cp.Variable((1, N))
cost = 0
cons = [x[:,0] == x0]
for k in range(N):
    cost += cp.quad_form(x[:,k], Q) + cp.quad_form(u[:,k], R)
    cons += [x[:,k+1] == A @ x[:,k] + B @ u[:,k]]
    cons += [cp.abs(u[:,k]) <= u_max]
cost += cp.quad_form(x[:,N], P)
prob = cp.Problem(cp.Minimize(cost), cons)
prob.solve(solver=cp.OSQP)
print("u_0 =", u.value[0,0])

8. 핵심 정리

  • 이차 비용 + 선형 모델 + 다면체 제약 → 표준 QP.
  • 치환법은 변수 적지만 dense, 비치환법은 변수 많지만 sparse + 빠름 (큰 $N$).
  • QP는 다항시간 풀이 가능 — 임베디드 ms 단위 실현 가능.
  • 실현 가능 집합 $\mathcal X_0$ 밖에서는 풀이 실패 → 소프트 제약(slack)으로 회피.
  • 실무 표준 콤보: CVXPY 프로토타입 → OSQP 코드 생성.

📚 깊이 1 — CFTOC를 직관으로

CFTOC = Constrained Finite-Time Optimal Control. LQR에 두 가지 제한을 추가한 것:

  1. 유한 호라이즌 $N$ — 무한 대신 N 스텝까지 보고 비용 계산.
  2. 제약 — 입력·상태에 명시적 한계.

이 두 변화로 닫힌 해(Riccati)가 사라지고, 매 시점 QP(이차 계획법)를 풀어야 합니다.

표준 CFTOC 형식

$$\min_{u_0, \dots, u_{N-1}} \sum_{k=0}^{N-1}\big(x_k^TQx_k + u_k^TRu_k\big) + x_N^T P_f x_N$$

$$\text{s.t.}\quad x_{k+1} = Ax_k + Bu_k,\ x_0 = \bar x_0$$

$$\quad x_k \in \mathbb{X},\ u_k \in \mathbb{U},\ x_N \in \mathbb{X}_f$$

📚 깊이 2 — QP 표준 형식 변환

CFTOC는 적절한 변수 묶음으로 표준 QP로 변환됩니다.

$$\min_z \frac{1}{2} z^T H z + q^T z \quad\text{s.t.}\quad A_{\text{ineq}} z \le b_{\text{ineq}},\ A_{\text{eq}} z = b_{\text{eq}}$$

두 가지 표현 방법:

2.1 Sparse 형식 (decision variables = 모든 $x_k$ + $u_k$)

  • $z = [x_0^T, u_0^T, x_1^T, u_1^T, \dots, x_N^T]^T$.
  • 등식 제약: 동역학 $x_{k+1} = Ax_k + Bu_k$ 명시.
  • 차원 큼 ($n+m$ × $N$). 그러나 행렬 sparse → 빠른 풀이 가능.
  • 대규모 NMPC 표준 (CasADi).

2.2 Condensed 형식 (decision variables = $u_0, \dots, u_{N-1}$만)

  • $z = [u_0^T, \dots, u_{N-1}^T]^T$.
  • $x_k$를 $z$의 함수로 명시적 표현 후 비용·제약에 대입.
  • 차원 작음 ($mN$). 행렬 dense.
  • 작은 차원 + 짧은 호라이즌에 효율 (드론, 자동차).

📚 깊이 3 — Condensed 형식 유도 단계별

$x_{k+1} = Ax_k + Bu_k$에서:

$$x_k = A^k x_0 + \sum_{i=0}^{k-1} A^{k-1-i} B u_i$$

벡터 표현:

$$\begin{pmatrix}x_1 \\ x_2 \\ \vdots \\ x_N\end{pmatrix} = \underbrace{\begin{pmatrix}A \\ A^2 \\ \vdots \\ A^N\end{pmatrix}}_{\Phi} x_0 + \underbrace{\begin{pmatrix} B & 0 & \cdots \\ AB & B & \cdots \\ \vdots & & \ddots \\ A^{N-1}B & A^{N-2}B & \cdots\end{pmatrix}}_{\Gamma} \begin{pmatrix}u_0 \\ u_1 \\ \vdots \\ u_{N-1}\end{pmatrix}$$

비용도 $u$만의 이차형으로 환원: $J = \frac{1}{2}u^T H u + q^T(x_0) u + \text{const}$.

📚 깊이 4 — KKT 조건과 QP 해의 구조

QP의 최적해는 Karush–Kuhn–Tucker 조건을 만족합니다.

  1. Stationarity: $\nabla_z L = Hz + q + A_{\text{ineq}}^T \mu + A_{\text{eq}}^T \lambda = 0$.
  2. Primal feasibility: $A_{\text{ineq}}z \le b_{\text{ineq}}$, $A_{\text{eq}}z = b_{\text{eq}}$.
  3. Dual feasibility: $\mu \ge 0$.
  4. Complementary slackness: $\mu_i (A_{\text{ineq},i}z - b_i) = 0$.

활성 제약 집합(active set)이 결정되면 KKT가 선형 시스템 → 빠른 풀이.

📚 깊이 6 — 단계별 예제 — 1차원 CFTOC

$x_{k+1} = x_k + u_k$, $|u_k| \le 1$, $N = 3$, $Q = 1, R = 0.1, P_f = 1$, $x_0 = 5$.

풀이
제약 무시 시 (LQR 닫힌 해): $u_0 = -K x_0 \approx -3.1$, 제약 $|u| \le 1$ 위반.
제약 활성: 첫 단계 $u_0 = -1$, $x_1 = 4$. 그 후 다시 풀이 $u_1 = -1, x_2 = 3$, $u_2$는 $P_f$에 따라 결정 (작은 페널티이면 다시 −1).
관찰: 닫힌 해가 사라지고 active set이 결정한 직선 부분 + 자유 부분의 조합. 이게 explicit MPC의 piecewise-affine 구조의 본질.

📚 깊이 7 — 제약 종류와 표현

  • 입력 박스 제약: $u_{\min} \le u_k \le u_{\max}$. 가장 흔함.
  • 상태 박스 제약: $x_{\min} \le x_k \le x_{\max}$. 안전 영역.
  • 출력 제약: $y_{\min} \le Cx_k \le y_{\max}$. 측정 변수 제한.
  • 입력 변화율 제약: $|u_k - u_{k-1}| \le \Delta u_{\max}$. 액추에이터 슬류율.
  • 다면체 제약: $A_x x \le b_x$. 일반.
  • 이차 제약: $x^TP_x x \le 1$. SOCP/QCQP가 됨 (QP보다 무거움).

📚 깊이 8 — Hard vs Soft 제약

Hard 제약은 절대 위반 안 됨. 그러나 외란이나 모델 오차로 실현불가능(infeasible)이 발생하면 풀이 실패.

Soft 제약 도입 — slack 변수 $\epsilon \ge 0$:

$$x_k \le x_{\max} + \epsilon_k,\quad \min \dots + \rho \|\epsilon\|^2$$

큰 페널티 $\rho$로 가능한 한 제약 만족. 절대 위반 불가능한 입력 제약은 hard, 안전 영역은 soft가 일반.

📚 깊이 9 — Feasibility 분석

  • Recursive feasibility: 한 번 feasible이면 계속 feasible. 종단 제약 + 종단 제어 invariant set으로 보장.
  • Domain of attraction: feasible 초기 상태들의 집합. 호라이즌 $N$ 클수록 커짐.
  • Maximal control invariant set: 어떤 입력으로든 무한히 안전 영역에 머물 수 있는 상태들.

📚 깊이 10 — 수치 예제 — Python으로 CFTOC

import numpy as np
import cvxpy as cp

A = np.array([[1, 0.1], [0, 1]])
B = np.array([[0], [0.1]])
N = 10
Q = np.eye(2)
R = np.array([[0.1]])
Pf = 10*Q  # 종단 비용
x0 = np.array([1.0, 0])

# Decision variables
x = cp.Variable((2, N+1))
u = cp.Variable((1, N))

cost = 0
constraints = [x[:, 0] == x0]
for k in range(N):
    cost += cp.quad_form(x[:, k], Q) + cp.quad_form(u[:, k], R)
    constraints += [x[:, k+1] == A @ x[:, k] + B @ u[:, k]]
    constraints += [cp.abs(u[:, k]) <= 1]   # 입력 제약
cost += cp.quad_form(x[:, N], Pf)

prob = cp.Problem(cp.Minimize(cost), constraints)
prob.solve()
print("Optimal u:", u.value[:, 0])  # 첫 입력만 적용

📚 깊이 11 — 종단 비용 $P_f$ 선택

  • $P_f = 0$: 종단 가치 무시. 안정성 위험.
  • $P_f = Q$: 단순. 종단점도 stage cost와 동일.
  • $P_f = $ DARE 해: 무한 호라이즌 cost-to-go 근사. 안정성 보장 표준.
  • $P_f$ + 종단 제약: 더 강한 안정성 (W6에서).

📚 깊이 12 — 호라이즌 $N$ 선택 가이드

$N$장점단점
짧음 (5~10)빠른 계산근시안적, infeasibility

📚 깊이 13 — Quadratic Programming 풀이 시간 추정

변수 수 $n_v$, 제약 수 $n_c$:

  • Active Set (qpOASES): $O(n_v^2)$, 활성 제약 수 변화에 따라.
  • Interior Point (HPIPM): $O(n_v^3)$ per iteration, ~10 iteration.
  • ADMM (OSQP): $O(n_v^2)$ per iteration, 100+ iteration.

$n_v = 100$ 정도면 ms 단위. $n_v = 1000$이면 100ms~1s.

📚 깊이 14 — Warm Start

매 시점 QP 풀이 시 이전 해를 초기값으로 사용 (warm start). 활성 제약 변화 적으면 1~2 iteration으로 수렴. MPC 실시간성의 핵심.

📚 깊이 16 — 흔한 오답

  1. 유한 호라이즌 끝의 비용을 종단 비용 없이 0으로 끝냄.
  2. Hard 제약만 사용해서 자주 infeasible.
  3. Soft 페널티 $\rho$가 너무 작아 제약 위반 빈번.
  4. Warm start 미사용으로 풀이 시간 낭비.
  5. Condensed 형식을 큰 차원에서 사용 (sparse가 빠름).

📚 깊이 18 — FAQ

Q1. 왜 매 시점 풀이? 한 번이면 안 되나?

모델 오차·외란 보정. Open-loop CFTOC를 적용하면 오차가 누적됨. 매 시점 measure → 풀이 → 첫 입력만 적용으로 closed-loop.

Q2. infeasibility 발생 시 대응?

Soft 제약 도입, 직전 해 재사용, fail-safe 모드.

Q3. QP 해가 LQR 해와 같을 때?

제약 비활성 영역에서. 그러므로 MPC = "제약 활성 시 LQR + 보정".

📚 깊이 19 — 응용 사례

  • 드론 자세 — N = 10~20, 모터 RPM 제약.
  • 차량 LKAS — N = 30, 조향각·가속도 제약.
  • 화학 공정 — N = 30~60, 농도·온도 안전 제약.
  • HVAC — N = 96 (15분 단위 24h), 온도·전력 제약.
