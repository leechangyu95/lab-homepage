1. 경로 계획이란

주어진 환경에서 출발점(start)에서 목표점(end)까지 장애물을 피하면서 갈 수 있는 길을 찾는 문제. 자율주행·드론·로봇청소기·창고 로봇 등 모든 모바일 시스템의 핵심 기능입니다.

"좋은 길"의 기준은 응용에 따라 달라요.

  • 가장 짧은 거리
  • 가장 짧은 시간 (속도 한계 고려)
  • 에너지 최소
  • 가장 안전한(장애물에서 멀리 떨어진) 길
  • 혹은 이들의 가중 평균

2. 모바일 로봇의 3계층 — 전역·지역·추종

실제 로봇 시스템에서 경로 계획은 보통 세 단계로 나뉩니다.

계층주기역할
전역 계획~분 (한 번)지도 위에서 목표까지의 큰 그림 경로 (도로망 위)
지역 계획~Hz현재 시점의 장애물·동적 객체 회피 (로컬 맵 위)
추종 제어10~100Hz경로를 실제로 따라가는 차량 제어 (PID, Pure Pursuit)

이번 강의는 주로 전역·지역 계획을 다룹니다. 추종은 W11 피드백 제어와 로봇공학특론 L7에서 자세히.

3. 맵 표현 — 노드와 간선

알고리즘이 다루기 쉽게 환경을 그래프로 추상화합니다.

  • 노드(node): 위치(셀, 교차로, 방).
  • 간선(edge): 한 노드에서 다른 노드로의 직접 이동 경로. 가중치(거리·시간·비용)를 가질 수 있음.

대표적인 표현 방식 세 가지:

  • Occupancy grid (점유 격자): 환경을 격자로 쪼개고 각 칸이 "장애물 있음/없음"의 확률을 가짐. 라이다·소나로 만든 지도. 로봇청소기·자율주행의 표준.
  • Pathfinding grid: 점유 격자에서 추출한 단순 격자. 각 칸 = 노드, 인접 4·8칸으로 이동. 게임에서도 표준.
  • Topological graph: 의미 있는 위치(방·문·교차로)만 노드로. 도로망·창고·네비게이션에 자주.

3.1 Occupancy Grid — 라이다로 짓는 지도

로봇이 라이다로 거리를 측정하면, 각 광선이 통과한 셀은 "비어 있을 가능성↑", 끝점이 닿은 셀은 "장애물 가능성↑"로 업데이트합니다. 베이즈 갱신을 누적하면 실제 환경의 평면도가 만들어져요. ROS의 nav2·gmapping이 이 방식.

  • 해상도 5cm: 200×200m 환경에서 16M 셀 — 메모리·연산 큰 부담.
  • 해결: Quadtree·Octree(공간을 가변 해상도로 나눔), OctoMap이 3D 표준.

3.2 Cost Map — 비용 격자

경로 계획에서는 단순 "갈 수 있다/없다"가 아니라 "가는 데 드는 비용"을 셀마다 가집니다.

  • 장애물 = ∞ 비용 (가지 못함)
  • 장애물에서 가까울수록 높은 비용 (안전 마진) — inflation layer
  • 경사 큰 지형 = 높은 비용 (자율주행에서 도로 가장자리)
  • 역사적으로 사고가 많은 지점 = 높은 비용 (학습 기반)

이런 비용 격자에서 A*를 돌리면 자연스럽게 장애물에서 멀고, 평탄하고, 안전한 길을 찾습니다. Cost map은 ROS Navigation 스택의 핵심이에요.

4. 그래프 탐색 알고리즘

4.1 BFS (너비 우선 탐색)

출발 노드에서 가까운 순서로 한 겹씩 펼쳐가며 탐색. 모든 간선 비용이 같다고 가정.

  • 장점: 단순, 균등 가중치에서는 최적해 보장.
  • 단점: 가중치가 다를 때 비효율, 탐색 범위가 빨리 커짐.

4.2 Dijkstra 알고리즘

BFS의 일반화. 간선마다 비용이 다를 때 누적 비용이 가장 작은 노드부터 펼침. 우선순위 큐(min-heap)로 구현.

$$ g(n) = \text{출발에서 } n \text{까지의 최소 누적 비용} $$

모든 간선 비용이 0 이상이면 항상 최적해를 보장. 1959년 발표 — 60년 넘은 알고리즘이 지금도 GPS 내비게이션의 기본.

4.3 A* 알고리즘 — 휴리스틱의 도입

Dijkstra는 출발에서 현재까지의 비용 $g(n)$만 사용. 이게 비효율적인 이유는 목표에서 멀어지는 방향까지 탐색하기 때문이에요.

A*는 휴리스틱 $h(n)$을 추가 — "여기서 목표까지의 추정 거리". 두 비용을 합한 $f(n) = g(n) + h(n)$이 가장 작은 노드부터 탐색.

$$ f(n) = g(n) + h(n) $$
  • $g(n)$: 시작에서 $n$까지 실제 비용 (Dijkstra와 같음)
  • $h(n)$: $n$에서 목표까지 추정 비용 (보통 직선거리)

최적성 조건 — $h(n)$이 실제 남은 비용을 절대 초과하지 않으면(admissible) A*가 항상 최적해를 보장합니다. 격자에서는 맨해튼 거리·유클리드 거리가 흔히 admissible.

한 줄 비유:

  • BFS: "출발점에서 한 걸음씩 동심원으로 펼쳐가기" — 모든 방향 균등 탐색.
  • Dijkstra: "가장 싼 노드부터 차례로 펼치기" — 가중치 있을 때 BFS의 일반화.
  • A*: "Dijkstra + 목표 방향 컴퍼스" — 같은 비용이면 목표에 가까운 쪽을 먼저.
알고리즘탐색 폭최적성속도
BFS매우 넓음균등 가중치만느림
Dijkstra넓음 (전 방향)항상 최적중간
A*좁음 (목표 방향)$h$ admissible 시 최적빠름

🎮 인터랙티브: BFS · Dijkstra · A* — 같은 미로, 다른 탐색

같은 격자에서 세 알고리즘을 실행해 비교하세요. 탐색한 칸 수(연한 색)와 최종 경로 길이(진한 선)를 보면 A*가 왜 표준이 됐는지 한눈에 보입니다. 칸을 클릭해 장애물을 추가/제거하고, 출발(S)·목표(G)도 마우스로 끌어 옮길 수 있어요.

🟢 출발 S   🔴 목표 G   ⬛ 장애물   🟦 탐색됨   🟣 최종 경로

5. 샘플링 기반 경로 계획

그래프 탐색은 격자 해상도가 높아지면 차원 폭발 — 6-DOF 로봇팔에서는 격자가 너무 커집니다. 샘플링 기반은 전체 공간을 격자로 나누지 않고 임의의 점들을 샘플링해 그래프를 즉시 만들어가는 접근.

5.1 PRM (Probabilistic Roadmap)

두 단계:

  1. 학습 단계: 자유공간(장애물 없는 영역)에서 랜덤 점 $N$개 샘플링, 가까운 점끼리 연결해 로드맵 그래프 구축.
  2. 질의 단계: 출발-목표 점을 로드맵에 붙이고 그래프 탐색(예: A*).

로드맵은 한 번 만들면 여러 출발-목표에 재활용 가능. 다중 질의에 유리.

5.2 RRT (Rapidly-Exploring Random Tree)

출발점에서 트리를 키워가며 랜덤 점으로 빠르게 확장.

  1. 현재 트리에서 가장 가까운 노드를 찾음.
  2. 그 노드에서 랜덤 점 방향으로 일정 거리만큼 진행한 새 점을 추가.
  3. 충돌이 없으면 트리에 노드·간선으로 등록.
  4. 목표에 충분히 가까워질 때까지 반복.

고차원·복잡한 구속에서도 잘 동작 — 로봇팔, 자율주행 같은 응용에 표준. 단점: 결과가 랜덤이라 매번 다르고, 최적 보장이 없음.

5.3 RRT*

RRT의 개선판. 새 노드를 추가할 때 주변 기존 노드들과의 비용을 비교해 부모를 다시 선택(rewiring). 샘플 수가 무한대로 가면 최적해로 수렴(asymptotically optimal). 자율주행·드론 동적 회피에 자주.

🎮 인터랙티브: RRT가 트리를 어떻게 키우는지

"성장" 버튼을 누르면 한 스텝씩 트리가 자랍니다. 랜덤 점 → 가장 가까운 노드 → 그 방향으로 일정 거리만큼 새 노드 추가. 격자 기반 A*가 모든 칸을 평가하는 것과 달리 RRT는 자유공간을 빠르게 "찔러" 채워가요. 이래서 고차원에서도 동작합니다.

알고리즘특징적합 응용
PRM오프라인 맵 구축, 다중 질의고정된 환경의 반복 작업
RRT빠른 확장, 단일 질의, 비최적고차원 모션 계획
RRT*점근적 최적, 느림품질이 중요한 단발성 계획

5.5 동적 장애물과 시간 — 시공간 계획

지금까지의 알고리즘은 정적 장애물(움직이지 않는 벽·기둥)을 가정했어요. 하지만 자율주행·드론·서비스로봇은 걷는 사람·다른 차·다른 로봇처럼 움직이는 장애물도 피해야 합니다.

5.5.1 시간 차원 추가

장애물의 미래 위치를 예측해 (x, y, t) 3D 공간에서 길을 찾습니다. 즉 "공간뿐 아니라 시간도 한 축"으로 두는 거예요.

  • 장점: "지금 좁은 통로를 빠르게 지나면 통과 가능, 천천히 가면 곧 막힐 예정" 같은 시간 의존 결정이 자연스럽게 됨.
  • 비용: 차원이 하나 늘어 계산량 폭증. 보통 속도 프로파일을 별도로 풀고 공간 경로와 결합하는 방식이 흔함.

5.5.2 동적 윈도우 접근법(DWA)

로봇청소기·자율주행 보조에 자주 쓰는 지역 계획기. 현재 속도·가속 한계 안에서 다음 1초 동안 갈 수 있는 모든 (선속도, 각속도) 조합을 평가하고, 충돌 없이 목표에 가까워지는 조합을 선택. 짧은 호 형태의 경로를 매 사이클마다 다시 계산해 동적 환경에 적응.

5.5.3 사회적 힘 모델·예측 기반 회피

최근에는 보행자의 미래 궤적을 신경망으로 예측하고(Social-LSTM, Trajectron 등) 그 예측 영역을 비용으로 더해 계획. 자율주행의 인지·예측·계획 스택이 이 흐름.

6. 최적화 기반 경로 계획

경로를 직접 변수로 두고 비용 함수를 최소화. 동역학·차량 제약을 그대로 반영할 수 있다는 게 강점.

$$ \min_{\mathbf{x}_0,\dots,\mathbf{x}_N,\mathbf{u}_0,\dots,\mathbf{u}_{N-1}} \sum_k \ell(\mathbf{x}_k, \mathbf{u}_k) \quad \text{s.t.}\;\mathbf{x}_{k+1} = f(\mathbf{x}_k, \mathbf{u}_k),\;\text{충돌 없음} $$

비용에 거리·연료·승차감·안전 마진 등을 자유롭게 넣을 수 있음. 로봇공학특론 L11 MPC가 이 접근법의 정점입니다 — 짧은 시간 단위로 최적화를 반복하며 미래 N스텝을 예측·제어.

7. 동역학 제약과 Dubins 경로

지금까지의 알고리즘은 "임의 방향으로 회전·이동 가능"을 가정합니다. 하지만 자동차는 후진 없이 일정 회전반경 이상으로만 돌 수 있어요. 이런 비홀로노믹(nonholonomic) 제약 하에서의 최단 경로가 Dubins 경로입니다.

Dubins 경로는 직선과 일정 반경 호의 6가지 조합(LRL, RLR, LSL, LSR, RSL, RSR)으로 구성. 자율주행·드론 항법에 표준.

8. 알고리즘 결과를 제어 루프로 보내기

경로가 일련의 점들 또는 곡선으로 나오면, 그것을 차량이 실제로 따라가게 하는 게 다음 단계. 두 가지 표준 방법:

  • Pure Pursuit: 경로 위 일정 거리 앞의 "목표점"을 잡고 그쪽으로 핸들. 단순하고 강건.
  • Stanley Controller: 횡방향 오차 + 헤딩 오차의 결합. Stanford 자율주행팀이 DARPA 우승할 때 사용.

두 컨트롤러 모두 로봇공학특론 L7에 깊은 설명이 있습니다.

8.5 자율주행의 실제 계획 스택 — 한 그림으로

실제 자율주행 차량의 계획 스택은 본 강의에서 다룬 거의 모든 알고리즘이 한 곳에 모입니다.

5. MPC 비용 함수

경로 추종용?

$\|x - x_{ref}\|_Q + \|u\|_R$ + 종단 비용 + 충돌 페널티.

응용 사례

  • 자율주행 — A* (전역) + MPC (지역).
  • 드론 — RRT* + MPC.
  • 매니퓰레이션 — PRM + 트라젝토리 최적화.
  • 모바일 로봇 — A* + DWA.

자주 만나는 함정

  • 휴리스틱이 admissible 아님 → 최적성 깨짐.
  • RRT 무작위성 → 매번 다른 결과.
  • 실시간 제약 미고려.
  • 동적 장애물 미처리.
  • 좁은 통로 어려움.

Python 코드 — A* 의사코드

def a_star(start, goal, heuristic):
    open_set = [(heuristic(start, goal), start)]
    came_from = {}
    g_score = {start: 0}
    while open_set:
        _, current = heapq.heappop(open_set)
        if current == goal:
            return reconstruct_path(came_from, current)
        for neighbor in neighbors(current):
            tentative = g_score[current] + dist(current, neighbor)
            if tentative < g_score.get(neighbor, float('inf')):
                came_from[neighbor] = current
                g_score[neighbor] = tentative
                f = tentative + heuristic(neighbor, goal)
                heapq.heappush(open_set, (f, neighbor))

# 간단한 RRT
import numpy as np
import matplotlib.pyplot as plt

def rrt(start, goal, obstacles, max_iter=1000, step=1.0):
    nodes = [start]
    parents = {0: None}
    for _ in range(max_iter):
        rand = np.random.rand(2) * 100
        nearest_idx = np.argmin([np.linalg.norm(rand - n) for n in nodes])
        nearest = nodes[nearest_idx]
        direction = (rand - nearest) / (np.linalg.norm(rand - nearest) + 1e-6)
        new_node = nearest + step * direction
        if not any(np.linalg.norm(new_node - obs) < 5 for obs in obstacles):
            nodes.append(new_node)
            parents[len(nodes)-1] = nearest_idx
            if np.linalg.norm(new_node - goal) < step:
                return reconstruct(parents, nodes)
    return None
