Week 12. 딥러닝 in 메카트로닉스

모델식을 모르는 시스템·인지가 어려운 환경에서 데이터로부터 직접 학습. 그리고 그 출력을 제어 루프에 연결하기.

🤔 "PID가 산업의 95%를 돈다"는데, 왜 또 딥러닝인가? — PID는 모델식을 적을 수 있는 자리에서만 잘 통합니다. 그러나 메카트로닉스에는 다음 두 자리에서 PID가 막혀요.
  1. 모델을 적기 어려운 자리 — 부드러운 로봇, 4족 보행, 인간 운전자의 행동 등.
  2. 인지가 필요한 자리 — 카메라 픽셀에서 "보행자 검출"을 미분방정식으로 적을 수 있나요? 못 합니다.
이 두 자리에 데이터로 답을 찾아주는 게 딥러닝. 1학년이면 "왜 이걸 쓰는가" + "어디에 쓰이는가"(4절) 두 줄만 잡고 가도 충분합니다. 역전파·경사하강법은 딥러닝 트랙에서 본격 다룹니다.

목차

  1. 왜 메카트로닉스에 딥러닝이 들어왔나

  3. 학습의 3요소: 손실 함수·역전파·옵티마이저
  4. 메카트로닉스 응용 4가지

1. 왜 딥러닝인가

지난 W7~W11에서 우리는 시스템을 미분방정식으로 모델링하고 PID로 제어했어요. 하지만 실제 메카트로닉스에는 두 가지 어려운 상황이 있습니다.

  • 모델을 적기 어려운 경우: 마찰·온도·재료 변형 같은 비선형성, 인간 운전자의 행동, 비정형 환경.
  • 인지가 필요한 경우: 카메라 영상에서 보행자·차선·표지판을 알아내야 함. 미분방정식으로는 답이 안 나오는 문제.

이런 자리에 데이터에서 직접 함수를 학습하는 딥러닝이 들어옵니다. 충분한 데이터가 있다면, 모델을 손으로 적지 않아도 입력→출력 관계를 잡아줍니다.

2. 퍼셉트론과 다층 신경망

2.1 퍼셉트론(Perceptron)

가장 작은 인공 뉴런. 입력에 가중치를 곱해 합한 뒤 활성화 함수를 통과:

$$ y = f\!\left(\sum_{i=1}^{n} w_i x_i + b\right) $$

$w_i$는 가중치(weights), $b$는 바이어스. 학습이란 결국 데이터로 이 $w$, $b$ 값을 찾는 일입니다.

2.2 활성화 함수 — 비선형성의 도입

가중합만으로는 선형 함수만 표현 가능. 활성화 함수가 비선형성을 더해줍니다.

  • Sigmoid $\sigma(x) = 1/(1+e^{-x})$: 0~1 출력, 확률 해석에 좋음
  • ReLU $\max(0, x)$: 가장 흔히 씀. 학습이 빠르고 vanishing gradient 적음
  • Tanh $\tanh(x)$: −1~1 출력, RNN의 게이트에 자주

왜 비선형성이 필요한가? 활성화 함수가 없거나 모두 선형이면, 아무리 층을 깊게 쌓아도 결국 한 번의 행렬 곱 $y = W_{\text{eff}}\,x + b_{\text{eff}}$로 합쳐집니다 — XOR 같은 단순한 문제도 못 풀어요. 비선형 활성화가 끼어들면 층마다 새로운 "꺾임"이 생겨 임의의 곡선·결정경계를 만들 수 있게 됩니다.

🎮 인터랙티브: 활성화 함수와 그 도함수(=그래디언트)

같은 입력에 대해 세 활성화의 모양과 도함수를 비교하세요. 도함수가 학습에 직결됩니다 — 도함수가 0이 되는 영역이 많으면 그래디언트가 사라져(vanishing gradient) 학습이 멈춰요.

  • Sigmoid: 양 끝(|x|>4)에서 도함수 ≈ 0 → 깊은 망에서 학습 멈춤. 1986년 이후 한참 사용했지만 ReLU에 밀림.
  • Tanh: 0 중심, sigmoid보다 좋음. 하지만 여전히 양 끝에서 saturate.
  • ReLU: 양수 영역에서 도함수가 1 → 그래디언트가 살아 있음. 음수 영역에선 도함수 0 ("Dying ReLU" 문제).

2.3 다층 신경망(MLP)과 "Deep"

퍼셉트론을 여러 층으로 쌓으면 다층 신경망(MLP, Multi-Layer Perceptron). 그중 은닉층(hidden layer)이 여러 개 있는 신경망을 심층 신경망(DNN, Deep Neural Network)이라 부르고, 이를 학습하는 분야가 딥러닝입니다.

"Deep"의 핵심은 깊이에서 오는 표현력입니다. 1개 은닉층으로도 임의 함수를 근사할 수 있다는 정리(보편 근사 정리)가 있지만, 실제로는 깊게 쌓을수록 같은 정확도를 더 적은 파라미터로 달성합니다.

🎮 인터랙티브: 퍼셉트론 1개로 풀 수 없는 문제 — XOR

퍼셉트론(은닉층 없음) = 직선 결정 경계만 가능 → XOR(대각선 패턴)을 절대 못 풀어요. 은닉층을 추가하면 휘어진 경계가 가능해지고 XOR이 풀립니다. 데이터 패턴을 바꿔보고, 은닉 뉴런 수를 0과 4로 바꿔 결과를 비교하세요.

3. 학습의 3요소

3.1 손실 함수(Loss Function)

예측 $\hat{y}$가 정답 $y$에 얼마나 가까운가를 수치로. 가장 흔한 두 가지:

  • 회귀 (연속값): 평균제곱오차(MSE) $L = \frac{1}{N}\sum (y_i - \hat{y}_i)^2$
  • 분류 (클래스 라벨): 교차 엔트로피 $L = -\sum y_i \log \hat{y}_i$

3.2 역전파(Backpropagation) (개념만)

각 가중치 $w$가 손실 $L$에 미치는 영향(편미분 $\partial L/\partial w$)을 효율적으로 계산하는 알고리즘. 핵심은 연쇄 법칙(chain rule)으로 출력층에서 입력층으로 거슬러 올라가며 그래디언트를 계산.

$$ \frac{\partial L}{\partial w} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z} \cdot \frac{\partial z}{\partial w} $$

1학년 한 줄 요약 — "출력의 잘못을 거꾸로 한 칸씩 뉴런들에게 책임을 분배하는 알고리즘". 정확한 유도는 미적분(편미분)을 충분히 익힌 후에 봐도 됩니다. 이 알고리즘이 1986년 다시 주목받으면서 신경망의 두 번째 봄이 시작되었어요(W7-W8 딥러닝 트랙에서 자세히).

3.3 옵티마이저(Optimizer)

그래디언트 정보로 가중치를 업데이트하는 규칙. 가장 단순한 형태는 경사 하강법:

$$ w \leftarrow w - \eta \frac{\partial L}{\partial w} $$

$\eta$는 학습률(learning rate). 너무 크면 발산, 너무 작으면 학습이 더딤. 실무에서는 Adam이 표준에 가까움.

직관 — "안개 낀 산에서 가장 낮은 곳 찾기": 손실 함수는 가중치 공간에서의 지형이고, 우리는 가장 낮은 골짜기(=손실 최소점)를 찾고 싶어요. 그래디언트는 "지금 위치에서 가장 가파르게 올라가는 방향"이므로 그 반대로 한 발씩 내려가면 됩니다. 한 발의 크기가 학습률 $\eta$.

🎮 인터랙티브: 경사 하강법으로 손실 최소점 찾기

1차원 손실 $L(w) = (w-2)^2$의 최소점은 $w=2$입니다. 시작점과 학습률을 바꾸며 경로를 관찰하세요. 학습률이 너무 크면(>1) 발산하고, 너무 작으면(<0.05) 한참 걸려요. "적당히 큰 학습률 + 모멘텀"이 실무의 표준입니다.

4. 메카트로닉스에서의 응용 4가지

4.1 인지 (Perception)

카메라·LiDAR로 본 환경을 이해. 자율주행에서 가장 큰 사용처.

  • 객체 검출: YOLO, Faster R-CNN — 보행자·차량·표지판 위치와 종류.
  • 의미 분할: 픽셀마다 클래스 라벨 — 차선·도로·하늘 구분.
  • 3D 인식: PointPillars, PointNet — LiDAR 점군에서 객체 검출.

이 모든 출력이 다시 제어 루프의 입력으로 들어갑니다. 예: "전방 5m에 보행자" → 정지 명령.

4.2 시스템 식별 (System Identification)

물리 모델을 적기 어려운 시스템에서 입출력 데이터로 직접 동역학을 학습. 비선형 마찰·간극·온도 효과까지 잡아낼 수 있음.

  • 응용 예: 4족 로봇의 다리 동역학, 부드러운 로봇(soft robot), 산업 펌프의 마찰 모델.
  • 방법: 신경망 ODE, RNN/LSTM, Transformer 기반 dynamics 모델.

4.3 강화학습 제어 (Reinforcement Learning)

"보상이 큰 행동을 점점 더 자주" 하도록 학습. PID로는 풀기 어려운 다관절·접촉 풍부한 작업에 강력.

  • 응용 예: 로봇 팔의 물체 잡기(grasping), 4족 로봇 보행, 드론 곡예비행.
  • 주의: 시뮬레이션→실제(sim-to-real) 격차, 안전 보장이 약점.

4.4 예지 보전 (Predictive Maintenance)

모터 진동·전류·온도 시계열을 분석해 고장이 일어나기 전에 예측. 산업 IoT의 표준 응용.

  • 방법: LSTM, Transformer, Autoencoder 기반 이상 탐지.
  • 가치: 갑작스런 정지를 사전 경보로 바꿔 가동률·안전 향상.

5. 실시간성과 임베디드 배포

제어 루프는 보통 1~1000Hz로 돕니다. 신경망이 들어가도 같은 속도로 동작해야 한다는 게 임베디드의 큰 도전.

5.1 모델 경량화 기법

  • 양자화(Quantization): 32-bit float → 8-bit int. 모델 크기 4배, 속도 2~4배.
  • 가지치기(Pruning): 영향이 작은 가중치를 0으로 만들고 희소 행렬로 처리.
  • 지식 증류(Knowledge Distillation): 큰 모델의 지식을 작은 모델로 옮김.
  • 전용 NPU/GPU: Jetson Nano/Orin, Coral TPU, Raspberry Pi AI Kit 등.

5.2 안전 보장의 어려움

전통적 제어공학은 안정성 증명이 가능하지만, 신경망 정책은 보장하기 어렵습니다. 그래서 실제 차량·항공기에서는:

  • 인식·계획에는 딥러닝을 쓰되
  • 저수준 제어는 여전히 PID/MPC를 쓰고
  • 안전 조건 위반 시 폴백(fallback)으로 강제 전환하는 구조

신경망과 고전 제어가 함께 가는 하이브리드가 현 단계 산업의 표준입니다.

6.5 추가 워크북 — 메카트로닉스 + 딥러닝 8문제

6.5.1 센서 데이터 학습

IMU 가속도계 데이터로 신경망 학습해 보행자 걸음 수 카운트. 입출력?

입력 — 3축 가속도 시계열(예: 100 Hz × 3초 = 300×3). 출력 — 걸음 수 또는 활동 분류.

6.5.2 RL vs SL

로봇 그립 학습. RL과 SL 중?

데이터 라벨 어려움 + 시도 가능 → RL. 사람 시연 있으면 모방학습. 둘 결합 표준.

6.5.3 실시간 추론

1 kHz 제어 루프, 모델 추론 10 ms. 가능?

불가능. 양자화 + 작은 모델로 단축 또는 GPU/NPU.

6.5.4 데이터 증강

이미지 분류기 표준 증강 5가지?

RandomCrop, Flip, Rotation, ColorJitter, RandomErasing.

6.5.5 전이학습 절차

ImageNet ResNet을 잡초 분류에 적용?

백본 동결 + FC 교체 → fine-tune.

6.5.6 임베디드 추론

Jetson Nano에 ResNet50 가능?

가능하나 무거움. 양자화 + TensorRT 또는 MobileNet으로 교체.

6.5.7 데이터 누수

같은 물체가 학습/테스트에 모두 등장 → 99%. 신뢰?

아님. group split 필요.

6.5.8 안전 critical 결합

자율주행 신경망 + 고전 제어?

신경망 = 인지. 고전 제어 = 명령. 안전 모드 전환.

6.6 메카트로닉스 신경망 응용 5가지

  • 고장 진단 — 진동으로 베어링 마모 탐지.
  • 제어기 모방 — MPC를 신경망으로 → 실시간.
  • System ID — 데이터로 동역학 학습.
  • 비전 인지 — 객체·자세 인식.
  • 강화학습 정책 — 로봇 행동.

6.7 자주 만나는 함정

  • 도메인 차이 (훈련 vs 실제).
  • 실시간 제약 미고려.
  • 안전 보장 부재.
  • 디버깅 어려움.
  • 데이터 누수.

6.8 한 페이지 핵심

  • 딥러닝 = 데이터로 모델 자동 학습.
  • 메카트로닉스 응용 — 인지·진단·System ID·정책.
  • 실시간 + 안전 = 결정적 제약.
  • 고전 제어와 하이브리드 표준.

6.9 PyTorch 한 페이지 코드

import torch
import torch.nn as nn

class IMUClassifier(nn.Module):
    def __init__(self, input_dim=3, hidden=64, num_classes=3):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(hidden*2, num_classes)
    def forward(self, x):
        out, _ = self.lstm(x)
        return self.fc(out[:, -1, :])

model = IMUClassifier()
opt = torch.optim.AdamW(model.parameters(), lr=1e-3)
loss_fn = nn.CrossEntropyLoss()

6.10 신경망의 한계와 보완

  • 일반화 — 훈련 데이터와 다른 환경.
  • 해석 가능성 — 결정 근거 설명 어려움.
  • 안정성 보장 — 수학적 보장 부재.
  • 데이터 효율 — 많은 데이터 필요.

대처 — 도메인 적응, XAI, 강건 제어와 결합, few-shot 학습.

  • 딥러닝 트랙 전체 — 본 강의의 13주 풀버전. CNN(W9-10), RNN(W11-12), Transformer(W13)까지.
  • DL W7 MLP — 퍼셉트론과 XOR 문제, 보편 근사 정리의 깊은 유도.
  • DL W8 역전파 — 2층 신경망 역전파 손계산, 옵티마이저 비교.
한 줄 요약 — 딥러닝은 메카트로닉스의 "모델을 적기 어려운" 자리에 데이터로 답을 찾아주는 도구입니다. 다만 안전 보장이 약해 안전 critical 시스템에서는 고전 제어와 함께 하이브리드로 씁니다.

8. 산업 사례

  • 예측 정비 — 진동·온도 데이터로 고장 임박 예측. 항공·발전소 표준.
  • 외관 검사 — CNN으로 제품 결함 자동 검출.
  • 로봇 시연 학습 — 사람 시연 영상에서 매니퓰레이션 학습.
  • 자율 농업 — 잡초·해충 인식.
  • 스마트 시티 — 교통 흐름·이상 행동 감지.

9. 한국 산업 동향

  • 현대 — 자율주행에 신경망 인지 + 고전 제어.
  • 삼성 — 반도체 검사에 CNN.
  • LG — 가전 사용자 행동 학습.
  • 현대로보틱스 — 매니퓰레이션 강화학습.

11. 다음 주 미리보기 — W13

마지막 주차에서 자율 시스템의 경로 계획. A*, RRT, MPC를 메카트로닉스 관점에서 통합 정리.

12. 한 줄 마무리

"딥러닝은 메카트로닉스의 새로운 도구. 고전 제어와 결합이 미래."

13. 부록 — Imitation Learning

사람의 시연 영상으로 로봇 학습. Behavioral Cloning이 가장 단순하지만 distribution shift 문제. DAgger, GAIL 등 개선 방법.

14. 부록 — Foundation Models for Robotics

RT-1, RT-2, RT-X — Google의 거대 로봇 모델. 다양한 로봇·작업의 시연으로 학습. 새 작업에 zero-shot 또는 few-shot 적응.

15. 부록 — Sim-to-Real

시뮬레이션에서 학습 → 실제 로봇 적용. 도메인 갭 극복 기법:

  • 도메인 무작위화 (lighting, friction).
  • 도메인 적응 (DANN).
  • System ID로 시뮬레이터 보정.

16. 부록 — Reinforcement Learning 기본

  • State, Action, Reward, Policy.
  • Value function — 미래 누적 보상 기대.
  • Q-Learning — 가장 단순한 RL.
  • PPO, SAC — 현대 표준.
  • 딥 RL — 신경망이 정책·가치 함수 근사.

17. 부록 — 한 줄 요약

"메카트로닉스에서 딥러닝은 데이터로 모델·정책 학습. 안전 critical은 하이브리드 표준."

18. 한 페이지 자가 점검

  • ☐ MLP 구조와 학습.
  • ☐ 메카트로닉스 응용 5가지.
  • ☐ 실시간/안전 제약.
  • ☐ 전이학습·Sim-to-Real.
  • ☐ RL 기초.

19. 한 줄 진짜 마지막

"딥러닝 + 고전 제어 = 메카트로닉스의 미래."

20. 부록 — 강화학습 응용 사례

  • 알파고·알파스타 — 게임.
  • 로봇 보행 — Anymal, Boston Dynamics.

21. 부록 — 신경망 라이브러리

  • PyTorch — 연구·교육 표준.
  • TensorFlow — 산업.
  • JAX — 함수형, 구글.
  • Keras — 고수준 API.
  • ONNX — 프레임워크 간 변환.

22. 한 줄 진짜 마지막

"메카트로닉스 + 딥러닝 = 자동화의 새 시대."

— W12 끝.

23. 부록 — Edge AI

임베디드 보드에서 신경망 추론. NVIDIA Jetson, Google Coral, Raspberry Pi 4. 양자화 + TensorRT/OpenVINO 최적화 표준.

24. 부록 — 디지털 트윈

실제 시스템과 시뮬레이터를 데이터로 동기화. 시뮬레이터에서 강화학습 → 실제 시스템 적용 → 결과 데이터로 시뮬레이터 보정. 반복.

25. 한 줄 마지막

"디지털 트윈 = 시뮬-실세계 데이터 폐루프."

26. 부록 — Anomaly Detection

정상 데이터로 학습한 오토인코더의 재구성 오차가 크면 이상. 베어링·모터의 진동 모니터링에 활발 사용.

27. 부록 — Time Series Forecasting

LSTM/Transformer로 센서 시계열 예측. 예: 다음 5초의 진동, 다음 1시간 발전량.

28. 부록 — Model Predictive + Learning

MPC의 모델을 신경망으로 학습. 학습된 동역학으로 미래 시뮬레이션 → 최적 명령. 강건 제어와 결합.

29. 한 줄 끝

"메카트로닉스 W12 = 딥러닝의 메카트로닉스 응용 한 페이지."

30. 부록 — 자가 학습 추가 5문제

  1. 1 kHz 제어 루프에서 신경망 추론 가능 모델 크기?
  2. 도메인 갭 극복 3가지?
  3. RL의 보상 함수 설계 주의점?
  4. Imitation Learning의 distribution shift 문제?
  5. 강건 제어 + 신경망 결합 패턴?

힌트: 1) 작은 모델 + 양자화. 2) 도메인 무작위화, 적응, System ID. 3) Sparse reward · Reward hacking. 4) 정책 변하면 분포 변함 → DAgger. 5) 신경망 인지 + 안전 필터.

31. 부록 — Auto-encoder

인코더(차원 축소) + 디코더(복원). 정상 패턴 학습 → 이상 데이터 시 재구성 오차 큼. 비지도 학습. 기계 모니터링 표준.

32. 부록 — Variational Autoencoder (VAE)

잠재 공간에 확률 분포 부여. 새 샘플 생성 가능. 시나리오 생성, 데이터 증강에 응용.

33. 부록 — GAN

생성자와 판별자의 적대적 학습. 사실적 이미지·데이터 생성. Sim-to-Real에서 시뮬레이션 데이터 사실화.

34. 부록 — Diffusion Model

노이즈 제거 학습. Stable Diffusion 등 이미지 생성 표준. 로봇 매니퓰레이션에서 정책 생성에도 활용.

35. 부록 — Self-supervised Learning

라벨 없이 데이터로 학습. SimCLR, BYOL, DINO. 라벨링 비용 큰 산업 데이터에 매우 유용.

36. 한 줄 끝

"한 학기 메카트로닉스 W12 = 딥러닝 응용 종합."

37. 부록 — 신경망 디버깅 5단계

  1. 입력 데이터 모양/값 확인.
  2. 작은 데이터 과적합 시도.
  3. 학습률·옵티마이저 조정.
  4. 활성·초기화 페어링.
  5. 손실 곡선 시각화.

38. 부록 — 신경망 + 칼만 결합

칼만 필터 + 신경망 융합. 신경망이 측정 잡음·외란 보정, 칼만이 안정성 보장. 자율주행에서 자주.

39. 한 줄 마지막

"신경망 + 칼만 = 데이터 + 안정성. 메카트로닉스의 황금 조합."

40. 부록 — Sensor Fusion + 신경망

여러 센서(IMU, LiDAR, 카메라)의 데이터를 신경망이 융합. 칼만보다 비선형 처리에 강함. 단, 안정성 보장은 어려움.

41. 부록 — Predictive Maintenance 워크플로

  1. 센서 데이터 수집 (진동, 온도, 전류).
  2. 특성 추출 (FFT, wavelet).
  3. 신경망 학습 (정상 vs 이상).
  4. 이상 점수로 알림.
  5. 고장 임박 예측 시 미리 정비.

42. 한 줄 마지막

"예측 정비 = 딥러닝의 메카트로닉스 가장 큰 ROI 응용."

43. 부록 — 신경망과 SystemID

비선형 시스템 식별에 신경망 활용. 입력·출력 데이터 → 동역학 학습. MPC의 모델로 사용. ResNet 형태로 잔차 학습이 안정적.

44. 부록 — Lyapunov-Stable Neural Networks

신경망의 출력이 Lyapunov 안정성 조건을 만족하도록 학습. 안전 critical 시스템에 응용. 최근 연구 활발.

45. 한 줄 진짜 마지막

"신경망에 안전 보장을 추가하는 것이 메카트로닉스 + AI의 최전선."

46. 부록 — XAI (Explainable AI)

신경망의 결정 근거 시각화. SHAP, Grad-CAM, LIME. 의료·자율주행 등 책임 소재가 중요한 영역에 필수.

47. 부록 — 윤리·규제

  • EU AI Act — 위험 기반 분류, 안전 critical 강한 규제.
  • ISO 26262 — 자동차 기능 안전.
  • FDA — 의료기기 ML/AI 가이드.
  • 한국 KS X ISO/IEC — AI 안전성 표준.

48. 한 줄 마무리

"기술 + 윤리 + 규제 = 산업 적용의 3박자."

49. 부록 — 한 학기 W12 결론

딥러닝 = 메카트로닉스의 새로운 도구. 한 학기 마지막에 학습한 이 기법들이 졸업 후 산업에서 매일 사용. 단, 안전·규제·해석성이 항상 고려해야 할 제약.

50. 한 줄 진짜 끝

"메카트로닉스 + 딥러닝 = 산업 자동화의 미래."

51. 부록 — Convolutional Layer 직관

컨벌루션 = 작은 필터를 이미지 위로 슬라이드. 가중치 공유 + 지역 연결로 이미지 효율적 처리. 산업 검사·의료 영상 표준.

52. 부록 — 시계열 RNN/LSTM

은닉 상태로 시간 정보 누적. 메카트로닉스 시계열(진동, 속도) 분석에 자주.

53. 부록 — Transformer 시대

2017년 이후 NLP·비전·로봇까지 트랜스포머가 표준. 대규모 데이터에서 RNN/CNN을 능가.

54. 한 줄 마지막

"트랜스포머가 메카트로닉스에도 진입 — RT-2, GR00T 등."

55. 부록 — Loss Function 카탈로그

  • 회귀 — MSE, MAE, Huber.
  • 이진 분류 — BCE.
  • 다중 분류 — Cross Entropy.
  • 임베딩 — Triplet, Contrastive.
  • RL — Bellman residual.

56. 부록 — 학습률 스케줄

  • Step decay.
  • Cosine annealing.
  • Warmup + decay.
  • OneCycle.
  • ReduceLROnPlateau.

57. 한 줄 끝

"손실 + 옵티마이저 + 스케줄 = 학습 3종 세트."

58. 부록 — 정규화 도구

  • BatchNorm — CNN 표준.
  • LayerNorm — 트랜스포머 표준.
  • Dropout — 무작위 OFF.
  • Weight Decay — 가중치 크기 제한.
  • Data Augmentation — 데이터 다양화.
  • Early Stopping — 검증 손실 모니터링.
  • Label Smoothing — 라벨 부드럽게.

59. 한 줄 마지막

"7종 정규화로 과적합 방지."

60. 부록 — Hyperparameter 튜닝

  • Grid Search — 모든 조합.
  • Random Search — 무작위.
  • Bayesian Optimization — 효율적, Optuna.
  • Hyperband — 자원 분배.
  • Population Based Training — 동시 학습.

61. 한 줄 끝

"하이퍼파라미터 튜닝 = 학습의 마지막 5%."

62. 부록 — Production 배포

  • 모델 서빙 — TorchServe, TF Serving, ONNX Runtime.
  • 모니터링 — 데이터 드리프트, 모델 성능 저하.
  • 버전 관리 — MLflow, DVC.
  • A/B 테스트 — 새 모델 검증.
  • 롤백 — 이상 시 이전 모델로.

63. 한 줄 마지막

"학습은 시작, 배포가 끝. MLOps가 산업의 핵심."

64. 부록 — 자가 학습 추가 5문제

  1. BatchNorm vs LayerNorm 차이?
  2. Dropout이 학습 시와 추론 시 다른 이유?
  3. Imitation Learning의 distribution shift?
  4. System ID에 신경망 vs 고전 방법?
  5. Sim-to-Real 갭 극복법 3가지?

힌트: 1) 배치 vs 특성 차원. 2) 학습 시 무작위 OFF, 추론 시 모두 ON. 3) 정책이 변하면 분포 변함. 4) 고전: 명료한 모델, 신경망: 복잡한 비선형. 5) 도메인 무작위화, 적응, System ID.

65. 한 줄 진짜 끝

"메카트로닉스 + 딥러닝 = 산업의 미래."

66. 부록 — 한 학기 W12 위치

본 강의의 마지막 분기점. W11 피드백 제어 위에 신경망 도구를 얹어, W13 경로 계획에서 통합 응용. 한 학기 끝에서 자율 시스템의 큰 그림.
