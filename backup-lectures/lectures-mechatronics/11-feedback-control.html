Week 11. 피드백 제어 — PID로 시작하는 제어공학

결과를 보고 명령을 다시 조정하는 단순한 발상이 어떻게 자율 시스템의 토대가 되는지.

🚿 PID를 한 문단으로 — 샤워기 비유
  • P (지금): "지금 너무 차갑네 → 따뜻한 쪽으로 많이 돌리자". 오차에 비례.
  • I (지금까지): "계속 차갑네 → 매번 조금씩 더 돌리자". 오차의 누적에 비례.
  • D (앞으로): "온도가 오르고 있는 중이네 → 미리 식는 쪽으로 손을 옮기자". 변화율에 비례.
이 세 사람을 한 컨트롤러에 합친 게 PID입니다. 자율주행 속도 조절, 드론 자세, 에어컨 온도, 모터 속도 제어 — 산업의 95%가 이 한 컨트롤러로 돕니다. 식보다 데모(3절)에서 직접 슬라이더를 조절하면서 직관을 잡으세요.

1. 개루프 vs 폐루프

제어 시스템은 크게 두 가지로 나뉩니다.

1.1 개루프(Open-loop)

입력을 정해 명령만 보내고 결과는 확인하지 않음. 외란이 없을 때만 잘 동작.

  • 예: 토스터(시간 타이머), 신호등, 일반 세탁기 시간 모드.
  • 장점: 단순하고 안정적이다(불안정해질 일이 없음).
  • 단점: 외란·시스템 변동에 그대로 노출. "모터에 부하가 더 걸려도" 명령은 그대로.

1.2 폐루프(Closed-loop) = 피드백

출력을 측정해 목표와의 오차를 계산하고, 그 오차를 줄이는 방향으로 입력을 조정.

  • 예: 자율주행 차량의 속도 제어, 드론 자세 제어, 에어컨 온도 제어.
  • 장점: 외란을 자동으로 보상. 시스템 불확실성에 강함.
  • 단점: 잘못 설계하면 발진·발산 가능 (안정성 분석 필요).

샤워기 온도 조절이 일상의 가장 친근한 폐루프 — 너무 뜨거우면 찬물 쪽으로, 너무 차가우면 따뜻한 쪽으로. 결과(피부 감각)를 보고 입력(밸브)을 조정.

2. 피드백 시스템의 블록 다이어그램

각 신호의 의미:

  • $r(t)$: 참조(reference) — 원하는 값 (예: 목표 속도)
  • $y(t)$: 출력 — 실제 측정된 값
  • $e(t) = r(t) - y(t)$: 오차
  • $u(t)$: 제어 입력 — 제어기가 만들어 대상에 보내는 명령

전체 폐루프 전달 함수 (단위 피드백, $H = 1$):

$$ \frac{Y(s)}{R(s)} = \frac{C(s) P(s)}{1 + C(s) P(s)} $$

분모 $1 + CP$가 0이 되는 $s$ 값이 폐루프 시스템의 극점 — 이것이 좌반평면에 있어야 안정.

3. PID 제어기 — 가장 보편적인 컨트롤러

제어기 $C(s)$의 가장 많이 쓰이는 형태가 PID(Proportional-Integral-Derivative)입니다.

$$ u(t) = K_p\,e(t) + K_i\!\int_0^t e(\tau)\,d\tau + K_d\,\dot{e}(t) $$

라플라스 영역에서:

$$ C(s) = K_p + \frac{K_i}{s} + K_d s $$

세 항의 직관:

  • P (비례): 현재 오차에 비례. 오차가 크면 큰 명령. 빠른 응답을 만들지만 정상상태 오차가 남음.
  • I (적분): 누적된 오차에 비례. 작은 오차도 시간이 지나면 누적되어 컨트롤러가 강하게 반응 → 정상상태 오차 0.
  • D (미분): 오차의 변화율에 비례. "오차가 줄어드는 중이면 명령을 미리 줄여" 오버슈트를 억제.

일상 비유 — 샤워기 온도 조절:

  • P 사람: "지금 너무 차가우니 따뜻한 쪽으로 많이 돌리자" → 너무 많이 돌리면 뜨거워서 다시 차가운 쪽으로… 진동.
  • I 사람: "계속 차갑네, 매번 조금씩 더 따뜻한 쪽으로" → 시간이 지나면 정확히 원하는 온도. 단, 천천히.
  • D 사람: "온도가 오르고 있는 중이니 미리 식는 쪽으로 손을 옮겨 놓자" → 오버슈트 방지. 단, 측정 잡음에 민감.

실제 PID는 이 세 사람을 합친 컨트롤러예요. 각자 약점을 다른 두 사람이 보완합니다.

🎮 인터랙티브: PID로 1차 플랜트 제어

플랜트는 1차 시스템 $1/(\tau s+1)$($\tau=0.7$s, 외란이 출력에 더해짐). 슬라이더로 $K_p, K_i, K_d$를 만져보고 다음 시나리오를 차례로 시도해 보세요:

  1. P만 ($K_p$=2, $K_i$=$K_d$=0): 빠르게 따라오지만 정상상태 오차 남음.
  2. $K_p$를 더 키움 ($K_p$=15): 오버슈트와 진동이 심해짐. 정상상태 오차도 여전히 0이 아님.
  3. I 추가 ($K_p$=2, $K_i$=2): 시간이 지나면 오차가 0으로! 단, 정착이 느려짐.
  4. D 추가 ($K_p$=2, $K_i$=2, $K_d$=0.5): 오버슈트 억제, 응답이 깔끔해짐.
  5. 외란 추가: 외란 슬라이더를 키우면 P-only는 영구 오차, PI는 자동 보상.

🟢 목표 r=1   🔵 출력 y   🟠 제어입력 u (× 0.2 스케일)

4. 각 항의 효과

이득 ↑상승시간오버슈트정착시간정상상태 오차
$K_p$감소증가변화 적음감소
$K_i$감소증가증가0으로
$K_d$변화 적음감소감소변화 없음

이 표가 PID 튜닝의 직관적 가이드가 됩니다. 너무 큰 $K_p$는 오버슈트·발진을, 너무 큰 $K_i$는 늦은 정착을, 너무 큰 $K_d$는 잡음 증폭을 일으킵니다.

5. 튜닝 방법

5.1 Ziegler-Nichols 공식 (가장 고전)

$K_i = K_d = 0$로 두고 $K_p$를 천천히 키우다가 시스템이 등진동(일정 진폭 진동)을 시작하는 임계 이득 $K_u$와 그때의 진동 주기 $T_u$를 측정. 다음 표로 이득을 결정:

제어기$K_p$$T_i$$T_d$
P$0.5 K_u$——
PI$0.45 K_u$$T_u/1.2$—
PID$0.6 K_u$$T_u/2$$T_u/8$

여기서 $K_i = K_p/T_i$, $K_d = K_p T_d$. 시스템을 한계까지 밀어붙이는 방식이라 가전·산업의 안전 시스템에는 직접 못 씁니다.

5.2 실무 권장 — 점진적 튜닝

  1. P만 살리고 시작: $K_p$를 작게 시작해 응답이 빠르되 발진 직전에서 멈춤.
  2. I 추가: 정상상태 오차가 보이면 $K_i$를 천천히 증가 (너무 크면 적분 와인드업).
  3. D 추가: 오버슈트가 거슬리면 $K_d$를 추가. 단, 잡음을 증폭하니 D 앞에 LPF를 두는 게 표준.
  4. 스텝 응답 측정과 반복.

6. 실전 함정과 해결책

6.1 적분 와인드업(Integral Windup)

액추에이터가 포화 상태(예: 모터 PWM이 100%로 다 차 있음)인데 오차가 계속 누적되면 적분 항이 무한정 커집니다. 그 후 오차가 0이 되어도 적분이 너무 커져 시스템이 한참 동안 잘못된 명령을 내보내요.

해결: 액추에이터 포화를 감지하면 적분을 멈추는 anti-windup 기법. 또는 적분값에 상한을 둠.

🎮 인터랙티브: 와인드업 vs Anti-Windup

큰 스텝을 입력하면 P+I 컨트롤러가 매우 큰 명령을 만드는데, 액추에이터 한계가 ±$u_{max}$로 잘립니다. 그동안에도 적분은 계속 누적 — 안정점에 도달한 후에도 적분이 너무 커서 한참을 오버슈트합니다. Anti-windup을 켜면 포화 시 적분을 멈춰 이를 막아요.

6.2 미분 잡음 증폭

D 항은 입력을 미분하므로 고주파 잡음을 키웁니다. 그래서 실제 PID에서 D 앞에는 거의 항상 LPF를 둡니다 — "D + LPF"가 한 모듈로 패키지된 형태.

6.3 디지털 구현 — 샘플링 주기

임베디드에서 PID는 다음과 같이 이산화:

e = ref - y;                        // 현재 오차
integral += e * dt;                 // 적분
derivative = (e - e_prev) / dt;     // 미분
u = Kp*e + Ki*integral + Kd*derivative;
e_prev = e;

샘플링 주기 $dt$는 시스템 시정수의 1/10 이하로 잡는 게 보통. 모터 제어는 보통 1kHz(1ms), 자세제어는 100~500Hz.

6.4 미분 잡음 시각 비교

실제 측정에는 항상 잡음이 섞여 있습니다. 같은 신호를 D 항이 미분하면 잡음이 폭발적으로 키워집니다 — 이것이 D 항을 그대로 쓰지 못하는 이유.

  • 그냥 D: $u_D = K_d \cdot \dfrac{e_n - e_{n-1}}{\Delta t}$ — 한 샘플 사이의 잡음 차이가 $1/\Delta t$로 곱해져 폭발.
  • D + LPF (실무 표준): D 출력에 1차 LPF 통과. 시정수 $T_f \approx K_d/(N K_p)$로 잡고 $N \in [10, 20]$. 안정성과 잡음 억제의 균형.
  • 측정 미분 대신 출력 미분: $D$를 오차 $e$가 아닌 측정 $y$만으로 계산하면 참조 변화 시 D 킥(derivative kick)을 막을 수 있음.

6.5 적분기 와인드업의 또 다른 해법 — Conditional Integration

Anti-windup을 구현하는 방법은 여러 가지가 있습니다.

  1. Clamping: 적분값에 ±한계를 둠. 가장 단순.
  2. Conditional integration: 액추에이터가 포화 상태이면 적분 자체를 멈춤. 위 데모에서 사용한 방식.
  3. Back-calculation: 포화로 잘린 양만큼 적분에서 빼줌. 가장 정확하고 산업 표준에 가까움.

6.6 디지털 PID 구현의 두 형태 — 위치형 vs 증분형

임베디드 코드로 PID를 짤 때 두 가지 표준 형식이 있습니다.

  • 위치형(Positional): $u_n = K_p e_n + K_i \sum e + K_d \frac{e_n - e_{n-1}}{\Delta t}$. 각 시점의 전체 명령을 직접 계산. 위에 보인 예가 이 형식.
  • 증분형(Incremental): $\Delta u_n = K_p(e_n - e_{n-1}) + K_i e_n \Delta t + K_d \frac{e_n - 2e_{n-1} + e_{n-2}}{\Delta t}$. 명령의 변화량만 계산하고 더해 나감. 액추에이터(밸브 등)가 직접 증분을 받는 경우 유리. 와인드업이 자연 방지.

7. 캐스케이드 제어 — 실무의 표준

실제 산업 시스템은 PID 한 개가 아니라 두 개 이상의 PID를 직렬로 묶어 씁니다. 이를 캐스케이드 제어(cascade control)라 해요.

  • 외부 루프(slow): 위치 오차를 보고 속도 명령을 만듦. 보통 ~10–100Hz.
  • 내부 루프(fast): 속도 오차를 보고 토크/전류 명령을 만듦. 보통 ~1kHz.
  • 3중 캐스케이드: 위치 → 속도 → 전류(토크) 도 흔함. 산업용 서보 드라이브의 표준 구성.

왜 캐스케이드? — 내부 루프가 빠르게 외란을 잡아주면 외부 루프 입장에서는 시스템이 "단순한 1차"처럼 보여 튜닝이 쉬워집니다. 또 액추에이터 한계를 내부 루프에서 직접 처리할 수 있어요.

8. 응용 예 — 자율주행 속도 제어

차량의 목표 속도 60km/h에 맞추는 PID:

  • $y$ = 휠 엔코더로 측정한 실제 속도
  • $u$ = 스로틀(또는 브레이크) 명령
  • $P$: 빠르게 속도를 따라가게
  • $I$: 오르막에서 속도가 처지면 보상해 정확히 60에 맞춤
  • $D$: 가속이 너무 급격하면 미리 누름

한 줄 요약 — "오차를 보고 입력을 조정하는" 단순한 발상이 PID라는 이름으로 산업의 95%를 돌리고 있습니다. 더 복잡한 제어(MPC, LQR, 적응제어)는 PID로 풀 수 없는 특수 문제에만 등장해요.

PID 워크북 — 8문제

1. P 단독

$u = K_pe$. 정상상태 오차 존재.

2. PI

$u = K_pe + K_i\int e$. 정상오차 0.

3. PD

$u = K_pe + K_d\dot e$. 응답 빠르게.

4. PID

세 항 결합. 산업 표준.

5. 와인드업

적분기 누적 폭발. 안티-와인드업 필수.

6. 미분 잡음

D 항이 잡음 증폭. LPF로 완화.

7. Ziegler-Nichols

경험적 튜닝. Kp 임계 게인 → PI/PID 게인 결정.

8. 디지털 PID

이산화 구현. 적분 누적, 미분 차분.

PID 튜닝 절차

  1. I, D 끄고 P만 키우기.
  2. 진동 임계점 → 적정 P.
  3. I 추가 → 정상오차 0.
  4. D 추가 → 응답 매끄럽게.
  5. 안티-와인드업 적용.

응용 사례

  • 모터 위치 — 캐스케이드 PID.
  • 온도 — PI.
  • 드론 자세 — PID 3축.
  • 차량 ACC — PID + 적응.
  • 로봇 매니퓰레이터 — PID + 중력 보상.

자주 만나는 함정

  • 적분 와인드업.
  • 미분 잡음 증폭.
  • 샘플링 주파수 부족.
  • 액추에이터 포화.
  • 모델 불확실성.

Python PID 구현

class PID:
    def __init__(self, kp, ki, kd, dt):
        self.kp, self.ki, self.kd, self.dt = kp, ki, kd, dt
        self.integral = 0.0
        self.prev_error = 0.0
    def step(self, sp, pv):
        error = sp - pv
        self.integral += error * self.dt
        derivative = (error - self.prev_error) / self.dt
        self.prev_error = error
        return self.kp*error + self.ki*self.integral + self.kd*derivative

한 페이지 핵심

  • PID = $K_pe + K_i\int e + K_d\dot e$.
  • P → 응답 빠름. I → 정상오차 0. D → 진동 감쇠.
  • 안티-와인드업.
  • Ziegler-Nichols 튜닝.
  • 디지털 구현.

한 줄 마무리

"PID = 산업 95% 제어. 단순하지만 강력."

부록 — 추가 단원

피드백 제어의 본질:

  • 오차 측정 → 입력 조정.
  • 안정성과 정확도의 균형.
  • 외란 거부.
  • 모델 불확실성 강건성.

부록 — Cascade PID

외부 루프(위치) + 내부 루프(속도) + 가장 안쪽(전류). 안쪽 5~10배 빠르게.

부록 — Feedforward + PID

예측 가능한 외란을 미리 보상 + PID로 잔여 오차 처리. 매우 강력한 조합.

부록 — Gain Scheduling

운영 조건(속도, 페이로드)에 따라 PID 게인 변경. 항공기 자동조종, 자동차 ESP에 표준.

부록 — Adaptive Control

시스템 파라미터 실시간 추정 + 게인 조정. 페이로드 변화 큰 시스템에 강함.

부록 — H∞, LQR

현대 제어. PID로 풀기 어려운 다변수 시스템. 학부 외 영역.

부록 — Robust Control

모델 불확실성에 강건한 제어 설계. μ-synthesis. 항공·자동차에서 사용.

부록 — Model Predictive Control

미래 예측 + 최적화. 제약 처리 가능. PID의 진화.

부록 — Digital Implementation

# 안티-와인드업 PID
class APID:
    def step(self, sp, pv):
        error = sp - pv
        u_unsat = self.kp*error + self.integral + self.kd*(error - self.prev_error)/self.dt
        u = max(self.u_min, min(self.u_max, u_unsat))
        # back-calculation
        self.integral += (self.ki * error + self.kt * (u - u_unsat)) * self.dt
        self.prev_error = error
        return u

  • 삼성 반도체 — 챔버 온도.
  • LG 가전 — 냉장고·세탁기.
  • 두산 — 발전 보일러.
  • 로보스타 — 산업 로봇.
