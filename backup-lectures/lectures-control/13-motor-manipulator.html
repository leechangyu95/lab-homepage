1. 왜 단일 PID가 아니라 캐스케이드인가

한 줄 비유부터

고속도로에서 옆 차선으로 들어갈 때 우리는 두 단계로 생각합니다. 먼저 "이쪽으로 차가 향하게 핸들을 살짝 돌리자" (방향 결정), 그다음 "원하는 방향이 됐으니 핸들을 다시 똑바로" (속도 추종). 한 번에 위치만 바라보고 핸들을 돌리면 과민하거나 둔하기 쉽습니다.

캐스케이드 PID도 같은 발상입니다. 안쪽 루프(빠른 변수)가 먼저 자기 일을 빠르게 끝내고, 그 결과를 바깥 루프(느린 변수)가 명령으로 줍니다. 모터에서는 보통 토크(전류)·속도·위치 세 단계.

  • 단일 PID: 위치 오차를 바로 전압으로 변환. 모든 동특성을 한 PID가 떠안아야 해서 튜닝이 어렵다.
  • 캐스케이드: 각 루프가 한 가지만 책임. 안쪽일수록 단순한 시스템이라 튜닝이 쉽다.

2. DC 모터 위치제어 — 3중 루프 구조

전형적인 샘플링 주기:

  • 전류 루프: 10~20 kHz (가장 빠름, 모터의 전기 시정수 ~ms)
  • 속도 루프: 1~5 kHz
  • 위치 루프: 100~500 Hz (가장 느림, 사용자 명령 단위)

보통 안쪽 루프가 바깥 루프보다 5~10배 빨라야 캐스케이드가 정상 동작합니다. 학생용 작은 모터는 전류 루프를 생략하고 위치-속도 두 단으로만 구성하는 일도 흔합니다.

3. 인터랙티브: 단일 vs 캐스케이드 PID 비교

🎮 인터랙티브: 같은 모터, 두 가지 제어 구조

단순화된 모터 (위치 = 속도의 적분, 속도 = 토크의 1차 응답). 위쪽은 위치 PID 단일 루프, 아래는 위치 → 속도 캐스케이드. 같은 위치 명령에 두 구조가 어떻게 다른지 보세요.

관찰:

  • 단일 PID는 위치 Kp를 키우면 빠르지만 진동.
  • 캐스케이드는 안쪽 속도 루프가 빠르게 명령을 따라가므로 외부 위치 루프 게인을 더 크게 줘도 안정.
  • 속도 명령(주황)이 위치 오차에 따라 매끄럽게 만들어지고, 측정 속도(녹)가 그를 따라가는 모습이 보임.

4. 로봇 매니퓰레이터 관절 모델

한 줄 비유부터

창문 손잡이를 떠올려 보세요. 손잡이를 옆으로 돌리는 일과 천장 쪽으로 들어 올리는 일은 힘이 다릅니다. 옆으로는 마찰만 이기면 되지만, 들어 올릴 때는 중력 토크까지 이겨야 합니다. 로봇 팔도 같습니다 — 같은 모터, 같은 PID라도 자세에 따라 필요한 힘이 달라요.

식 한 줄로 가는 길

1자유도 회전 관절(어깨 또는 팔꿈치 하나)을 생각합시다. 끝에 무게 $m$의 페이로드가 있고 길이 $\ell$의 링크가 수직선(아래쪽이 0°) 기준 각도 $\theta$를 이룹니다.

각운동 방정식 만들기 (회전형 뉴턴 제2법칙)

  1. 축 주위의 토크 합 = 관성 × 각가속도: $\sum \tau = J\ddot{\theta}$.
  2. 모터가 주는 토크: $+\tau$.
  3. 점성 마찰 토크: 관절 베어링·기어가 회전 속도에 비례해 저항 → $-b\dot{\theta}$.
  4. 중력에 의한 복원 토크: 무게 $mg$를 팔 방향과 수직 방향으로 분해. 회전을 일으키는 성분은 $mg\sin\theta$, 모멘트팔(축에서 무게중심까지의 거리)은 $\ell$ → $$ \tau_{\text{grav}} = -m g \ell \sin\theta $$ 음의 부호: 팔이 위쪽으로 향하면 ($\theta > 0$) 중력은 아래로 끌어 내려서 $\theta$를 줄이려 함.

모두 합치면:

$$ J\ddot{\theta} = \tau - b\dot{\theta} - mg\ell\sin\theta \;\;\Longleftrightarrow\;\; J\ddot{\theta} + b\dot{\theta} + \underbrace{m g \ell \sin\theta}_{\text{중력 토크}} = \tau $$
왜 $\sin\theta$인가? 중력 벡터는 항상 아래로 향하지만, 회전을 만드는 것은 팔에 수직인 성분뿐입니다(평행 성분은 축이 받아냄). 그 수직 성분이 $mg\sin\theta$라는 사실이 곧 비선형의 출처입니다. $\theta = 0°$(수직 아래)에서는 0, $\theta = 90°$(수평)에서는 최대.

중력 항 $m g \ell \sin\theta$는 비선형이고 위치에 따라 변합니다. 단순 PID로만 제어하면 팔이 무거울수록 처져 있고, 적분기가 그 처짐을 천천히 보정해야 해서 응답이 느려집니다.

📐 다관절 일반 동역학으로 가는 길 — 한 식 미리보기

위 1자유도 식을 $n$자유도로 확장하면 모든 로봇 팔에 통하는 표준형이 나옵니다.

$$ M(\theta)\,\ddot{\theta} + C(\theta, \dot{\theta})\,\dot{\theta} + g(\theta) = \tau $$
  • $M(\theta) \in \mathbb{R}^{n\times n}$ — 관성행렬(자세에 따라 바뀜). 1자유도에서는 그냥 $J$.
  • $C(\theta,\dot\theta) \in \mathbb{R}^{n\times n}$ — 코리올리·원심 항. 한 관절이 움직이면 다른 관절을 끌어당기는 효과.
  • $g(\theta) \in \mathbb{R}^n$ — 중력 벡터. 1자유도에서는 $mg\ell\sin\theta$.
  • $\tau \in \mathbb{R}^n$ — 각 관절 모터 토크.

아래 5절의 중력 보상은 $g(\theta)$만을 피드포워드로 추가하는 것이고, 완전한 computed torque는 위 행렬 $M, C, g$ 모두를 보상합니다. 더 깊이 들어가려면 로봇공학특론 L2 동역학으로 — 라그랑지 식으로 $M, C, g$를 직접 유도합니다.

5. 중력 보상 (Gravity Compensation)

해법은 간단합니다. 알고 있는 중력 토크를 미리 더해주는 피드포워드:

$$ \tau = \underbrace{K_p e + K_i \int e + K_d \dot{e}}_{\text{PID}} + \underbrace{\hat{m}\hat{g}\hat{\ell}\sin\theta}_{\text{중력 보상}} $$

$\hat{m}, \hat{g}, \hat{\ell}$은 추정값이고 실제와 살짝 다를 수 있지만, 그 차이만 PID가 다시 보정합니다. PID가 처음부터 중력을 모두 떠맡지 않아도 되니 작은 게인으로도 부드럽게 동작.

이게 computed torque control의 가장 단순한 형태입니다. 완전한 computed torque는 관성행렬 $M(\theta)$, 코리올리·원심 $C(\theta,\dot\theta)$까지 모두 보상하지만, 보통 입문 단계에서는 중력 보상만으로도 큰 향상이 있습니다.

추정값이 실제와 다르면? (강건성) — 페이로드 무게가 가정한 1 kg가 아니라 실제로는 1.3 kg이라고 합시다.
  • 모자란 만큼($0.3 m g \ell \sin\theta$)은 PID가 떠맡습니다 → 위치 오차가 살짝 발생, 적분 항이 천천히 그 만큼 누적해서 보정.
  • 핵심은 완전한 보상이 아니어도 큰 효과라는 점. 1 kg 추정 → PID는 0.3 kg 분만 처리; 0 kg(보상 없음) → PID가 1.3 kg 전부.
  • 그래서 실무에서는 최소·중간·최대 페이로드의 평균값을 쓰는 일이 흔합니다 — 양 끝의 오차를 절반으로 줄이는 효과.
  • 적응 제어(adaptive control)는 한 걸음 더 나아가 $\hat m$ 자체를 실시간으로 추정·갱신합니다 (대학원 주제).

6. 인터랙티브: 1자유도 팔의 위치제어

🎮 인터랙티브: 중력 보상 ON/OFF 비교

1자유도 회전 팔의 각도 제어. PID는 동일, 중력 보상만 켜고 끄세요. 중력 OFF에서는 PID가 중력을 이기느라 위치 오차가 잔류하거나 큰 적분 와인드업이 일어납니다.

꼭 만져볼 것:

  • 중력 보상 OFF + m = 2 kg: 팔이 목표보다 살짝 처져서 정착 — 정상오차.
  • 중력 보상 ON: 처짐이 사라지고 정확히 목표각에 도달.
  • m을 0으로: 두 모드의 차이가 거의 없어진다 (보상할 중력이 없으니).
  • 목표각 변경 버튼: 큰 외란이 들어왔을 때의 응답.

7. 코드: 임베디드 캐스케이드 PID

class CascadePID:
    """위치 → 속도 → 토크 (3중 캐스케이드, ARM Cortex-M에 그대로 이식 가능)"""
    def __init__(self, kp_pos, kd_pos, kp_vel, dt):
        self.kp_pos, self.kd_pos, self.kp_vel = kp_pos, kd_pos, kp_vel
        self.dt = dt; self.iE_vel = 0.0

    def step(self, theta_ref, theta, omega, omega_max=10.0, tau_max=5.0):
        # 외부 위치 루프  →  속도 명령
        omega_cmd = self.kp_pos * (theta_ref - theta) - self.kd_pos * omega
        omega_cmd = max(-omega_max, min(omega_max, omega_cmd))
        # 내부 속도 루프  →  토크
        tau = self.kp_vel * (omega_cmd - omega)
        return max(-tau_max, min(tau_max, tau))

# 1자유도 팔 + 중력 보상
class ArmController:
    def __init__(self, m, L, g=9.81):
        self.m, self.L, self.g = m, L, g
        self.pid = CascadePID(kp_pos=20, kd_pos=2, kp_vel=8, dt=0.001)

    def step(self, theta_ref, theta, omega):
        tau_pid = self.pid.step(theta_ref, theta, omega)
        tau_grav = self.m * self.g * self.L * np.sin(theta)   # 피드포워드
        return tau_pid + tau_grav

8. DC 모터 전기·기계 모델 깊이 보기

지금까지 모터를 1차 시스템으로 단순화해 다뤘습니다. 이제 진짜 DC 모터의 두 가지 시정수(전기·기계)를 들여다봅시다.

8.1 두 개의 미분방정식

전기자(아마추어) 회로 + 기계 회전:

$$ V = R i + L \frac{di}{dt} + K_e \omega \qquad J \dot\omega = K_t i - b\omega - \tau_{load} $$
  • $V$ — 인가 전압 (PID의 출력)
  • $R, L$ — 권선 저항·인덕턴스
  • $K_e$ — 역기전력 상수 (V·s/rad)
  • $K_t$ — 토크 상수 (Nm/A); SI 단위에서 $K_t = K_e$
  • $J, b$ — 회전 관성·점성 마찰

8.2 두 시정수 — 전기 시정수와 기계 시정수

  • 전기 시정수 $\tau_e = L/R$. 보통 0.1~5 ms. 코일에 전류가 자리잡는 시간.
  • 기계 시정수 $\tau_m = JR/(K_t K_e)$. 보통 10~500 ms. 회전 속도가 자리잡는 시간.
  • $\tau_e \ll \tau_m$이 정상. 전류 루프가 전기 시정수 안에서 끝나면, 외부 속도 루프 입장에서 모터는 "토크 = 명령" 처럼 보임 — 단순한 1차 회전 시스템.

8.3 라플라스 변환과 전달함수

$\tau_{load} = 0$, 입력 $V$ → 출력 $\omega$의 전달함수:

$$ \frac{\omega(s)}{V(s)} = \frac{K_t}{(Ls + R)(Js + b) + K_t K_e} $$

$L \to 0$ (전기 응답 무시) 근사:

$$ \frac{\omega(s)}{V(s)} \approx \frac{K_t/R}{Js + b + K_t K_e/R} = \frac{K_m}{\tau_m s + 1} $$

이게 1차 시스템 근사의 정체입니다. 전류 루프가 빠르게 닫히면 모터는 깨끗한 1차 시스템으로 보임.

9. 손계산 예제

예제 1. Maxon RE25 — 시정수 계산

Maxon RE25 24 V 모터: $R = 2.32$ Ω, $L = 0.238$ mH, $K_t = K_e = 0.0238$ Nm/A, $J = 10.5$ g·cm² $= 1.05 \times 10^{-6}$ kg·m².

  1. 전기 시정수 $\tau_e = L/R = 0.238 \times 10^{-3} / 2.32 \approx 0.103$ ms.
  2. 기계 시정수 $\tau_m = JR/(K_t K_e) = 1.05 \times 10^{-6} \cdot 2.32 / (0.0238)^2 \approx 4.3$ ms.
  3. 비율 $\tau_m/\tau_e \approx 42$ → 캐스케이드 분리 설계 안전.
  4. 전류 루프 대역폭 권장 $\sim 1/\tau_e \approx 10$ kHz, 속도 루프 $\sim 1/\tau_m \approx 200$ Hz.

예제 2. 전류 PI 게인 결정

예제 1의 모터에 대해 전류 루프 PI를 IMC(영점-극점 상쇄)로 설계.

  1. 플랜트 $G_i(s) = 1/(Ls+R) = 1/(2.32(0.103 \times 10^{-3} s + 1))$.
  2. $T_i = \tau_e = 0.103$ ms로 잡으면 영점이 극점을 상쇄.
  3. 닫힌계 시정수를 $\tau_{cl} = 0.05$ ms (기존의 1/2)로 설정 ⇒ $K_p = R \cdot \tau_e / \tau_{cl} = 2.32 \cdot 0.103/0.05 \approx 4.78$.
  4. $K_i = K_p / T_i \approx 4.78 / (0.103 \times 10^{-3}) \approx 46400$ — 시간 단위가 ms이면 큰 수치, 초로 환산하면 자연스러움.

예제 3. 외부 속도 PI 게인

전류 루프가 빠르게 닫혔다고 가정, 외부에서 본 모터는 $G_\omega(s) = K_m/(\tau_m s + 1)$. $K_m = K_t/(b + K_t K_e/R)$.

  1. $b = 1.5 \times 10^{-6}$ Nm·s/rad로 가정. $K_t K_e/R = 0.0238^2/2.32 \approx 2.44 \times 10^{-4}$.
  2. $K_m \approx 0.0238/2.44 \times 10^{-4} \approx 97.5$ rad/(s·A).
  3. 닫힌계 시정수 1 ms 목표. IMC: $K_p = \tau_m/(K_m \tau_{cl}) \approx 4.3 \times 10^{-3}/(97.5 \cdot 10^{-3}) \approx 0.044$.
  4. $K_i = K_p/\tau_m \approx 10.2$.

예제 4. 위치 P 게인 (외부 외부 루프)

속도 루프가 1 ms 시정수의 1차 시스템처럼 보임. 위치 = 속도의 적분 ⇒ 위치 입장에서 모터는 적분기 $1/s$ + 1차.

  1. 외부 위치 루프는 단순 P. 닫힌계 대역폭을 속도의 1/5 (200 Hz)로 잡으면 $K_p^{(\theta)} \approx 2\pi \cdot 200 \approx 1260$ — 사실상 매우 큰 값.
  2. 현실에선 인코더 양자화·기어 백래시 때문에 50~100 Hz 수준에 정착.
  3. 위치 P + 속도 D(피드포워드) 조합이 흔한 절충. 자세한 건 W14의 게인 스케줄 참고.

예제 5. 1자유도 팔 — 자세별 게인 차이

$J = 0.5$ kg·m², $b = 0.3$ Nm·s/rad, $m = 1$ kg, $\ell = 0.5$ m. 두 자세에서 ZN 임계 게인을 추정.

  1. 수직 아래 자세 ($\theta = 0$): $\sin\theta = 0$이라 중력 항이 미분형 안에서 0. 작은 신호 선형화는 $J\ddot\theta + b\dot\theta + (mg\ell)\theta = \tau$. 자연 진동수 $\omega_n = \sqrt{mg\ell/J} = \sqrt{4.905/0.5} \approx 3.13$ rad/s.
  2. 수평 자세 ($\theta = 90°$): $\sin\theta$의 도함수 $\cos\theta = 0$. 중력의 선형화 기여 0이라 시스템은 적분기 + 1차에 가까움.
  3. 같은 PID 게인이어도 두 자세 응답이 완전히 다름 → 게인 스케줄링 또는 중력 보상 필요.

11. 산업 응용 사례

11.1 ABB IRB 6700 — 산업용 6축 로봇

각 관절에 저항-기반 전류 측정 + 16-bit 인코더, 4 kHz 위치 루프. 캐스케이드 PID + 동역학 보상(M, C, g 모두). 페이로드 1500 kg, 반복정밀도 ±0.05 mm.

11.2 Universal Robots UR5

BLDC 관절 모듈 6개, 각 모듈에 자체 STM32가 PID + Modbus 통신. 협동로봇이라 토크 제어 모드를 강조 — 사람과 부딪히면 즉시 정지. PID 외에도 디스터번스 옵저버(DOB)로 외력 추정.

11.3 SimpleFOC / VESC — 오픈소스 모터 컨트롤러

아두이노 + 드라이버로 만드는 누구나 따라할 수 있는 BLDC FOC 제어. d-q 변환 + 전류·속도·위치 3단 PID. 학생 프로젝트와 전기자전거 시장에서 광범위 사용.

11.4 Tesla 모터 — IPM 동기 모터

차량 모터는 BLDC가 아닌 IPM(매입형 영구자석). 자속 약화(field weakening)로 고속 영역 확장. 전류 루프 PID는 d-q 좌표계에서 두 개 — 토크 분 $i_q$, 자속 약화 분 $i_d$.

11.5 보스턴 다이내믹스 SPOT — 4족 로봇

각 다리 3개 관절 × 4다리 = 12 관절. 각 관절에 큰 D 게인의 PD + 토크 피드포워드. 1 kHz 토크 명령. PID 한 줄로는 못 가지만 PD가 마지막 안전망.

12. 흔한 함정 — 모터·로봇 팔 편

함정 1. 인코더 양자화로 미분 잡음

1024 PPR 인코더에서 미세한 회전을 측정하면 한 카운트가 0.35° 점프로 보임. 이 점프를 미분하면 거대한 펄스. 해결: 미분 필터 또는 위치 차분 평균화(이동평균).

함정 2. 기어 백래시

모터가 기어를 통해 부하를 구동할 때 기어 사이 빈틈이 짧은 시간 동안 모터를 무부하 상태로 만듭니다. PID가 그 순간 명령을 키워 부하 측에 충격이 가해짐. 해결: 모터-부하 양쪽에 인코더(이중 엔코더), 또는 deadband 보상.

함정 3. PWM 노이즈가 전류 측정 오염

PWM 스위칭(20 kHz)이 전류 측정 신호에 그대로 실립니다. 해결: PWM 주기와 동기된 샘플링 시점(중앙 트리거), 또는 LPF 컷오프를 PWM 주파수의 1/10 수준으로.

함정 4. 캐스케이드 시정수 분리 부족

안쪽 루프가 바깥보다 5배 이상 빠르지 않으면 두 루프가 상호작용하며 진동. 해결: 안쪽 대역폭을 의식적으로 키우거나, 바깥 대역폭을 줄이거나.

함정 5. 페이로드 변화에 게인 고정

로봇 팔이 빈손과 5 kg을 들었을 때 동특성이 완전히 다름. 해결: 페이로드 추정(토크 측정 → mass 추정) + 게인 스케줄, 또는 적응 제어.

함정 6. 중력 보상 부호 오류

$+m g \ell \sin\theta$로 적은 사람과 $-m g \ell \sin\theta$로 적은 사람의 부호 합의가 안 되어 있으면 보상이 외란을 더 키움. 해결: 시뮬레이션에서 한 자세의 정적 평형으로 부호 검증.

13. 인터랙티브 ② — 페이로드 변화에 따른 게인 스케줄링

🎮 인터랙티브: 적응 vs 고정 게인

페이로드 mass가 시간에 따라 변합니다(예: 물건을 잡았다가 놨다가). 고정 게인은 한 mass에 맞춰져 있고, 적응 게인은 mass에 비례해 자동 조정.

관찰: 적응 OFF에서 페이로드를 0.5→2.5 kg으로 바꾸면 응답이 무거워져 둔하거나, 반대로 가벼워져 진동. 적응 ON에서는 mass에 비례해 게인이 조정돼 일정한 응답을 유지.

14. 연습문제

  1. 전기 시정수: 어떤 모터의 $L = 0.5$ mH, $R = 1.5$ Ω. $\tau_e$를 구하고, 권장 전류 루프 대역폭을 정하라.
  2. 기계 시정수: 위 모터에 $J = 5 \times 10^{-5}$ kg·m², $K_t = K_e = 0.05$ Nm/A. $\tau_m$을 구하라.
  3. 캐스케이드 분리: $\tau_m/\tau_e$ 비를 계산하고, 캐스케이드 분리 설계가 안전한지 판단하라.
  4. 전류 PI 게인: 위 모터의 전류 루프를 IMC로 설계, 닫힌계 시정수 $\tau_{cl} = \tau_e/3$로 설정. $K_p, K_i$를 구하라.
  5. 속도 PI 게인: 외부 속도 루프 시정수 목표 5 ms. 게인을 설계하라.
  6. 위치 P 게인: 외부 위치 루프 P 게인을 정하라(속도 루프의 1/5 대역폭).
  7. 1자유도 팔 — 자세 ω_n 변동: $J = 0.4$ kg·m², $m = 0.8$ kg, $\ell = 0.4$ m. $\theta = 0°$와 $90°$에서의 선형화 자연 진동수 $\omega_n$을 비교하라.
  8. 중력 보상 효과: 위 팔이 30°에서 정적 평형을 유지하려면 보상 토크 $\hat m \hat g \hat\ell \sin 30°$를 얼마로 둬야 하나? PID Ki = 0인 상태에서.
  9. 적분 와인드업과 모터 한계: 모터 토크 한계가 ±5 Nm. 큰 setpoint 변경 시 와인드업 효과를 정성적으로 설명하고, 그 결과를 위 인터랙티브 ②에서 재현해보라.
  10. 인코더 분해능 영향: 1024 PPR vs 4096 PPR 인코더에서 미분 잡음이 어떻게 달라지나? RMS 잡음 비율을 추정하라.

15. 다관절 매니퓰레이터로 일반화 — 표준 동역학식 풀이

15.1 N-자유도 식

$$ M(q)\ddot q + C(q,\dot q)\dot q + g(q) = \tau $$
  • $q \in \mathbb{R}^n$ — 관절각 벡터
  • $M(q)$ — 관성 행렬 (대칭, 양정치)
  • $C(q,\dot q)$ — 코리올리/원심 행렬
  • $g(q)$ — 중력 벡터
  • $\tau$ — 관절 토크 명령

15.2 Computed Torque Control (역동역학 보상)

관절별 PD를 외곽으로 두고, 그 출력을 동역학에 대입해 토크 명령을 만듦:

$$ \tau = M(q)\,a_{ref} + C(q,\dot q)\dot q + g(q) $$ $$ a_{ref} = \ddot q_d + K_d (\dot q_d - \dot q) + K_p (q_d - q) $$

완벽한 모델이면 닫힌계는 $\ddot e + K_d \dot e + K_p e = 0$인 선형 2차 시스템 — 자연 진동수와 댐핑을 게인으로 직접 결정. 실제로는 모델 오차가 있어 외곽에 추가 안전망 PID.

15.3 PD + 중력 보상의 안정성 (Takegaki-Arimoto, 1981)

중력 보상만으로도 정역학 평형(setpoint regulation)이 점근 안정함을 Lyapunov 함수로 증명할 수 있습니다:

$$ V = \tfrac{1}{2}\dot q^T M(q) \dot q + \tfrac{1}{2} e^T K_p e $$

$\dot V = -\dot q^T K_d \dot q \leq 0$ (관성 행렬의 시간 도함수 - 2C 항이 반대칭이라는 성질 사용). 이 결과가 산업 로봇 PD + 중력 보상의 이론적 기반.

17. 케이스 스터디 — 6축 산업 로봇의 게인 결정 절차

  1. 각 관절의 동역학 식별: 천천히 움직이며 토크-가속도 관계로 $J_i, b_i$ 추정. 정역학 자세에서 $g(q)$ 검증.
  2. 가장 안쪽 전류 루프: 모터 데이터시트의 $L, R$로 IMC. 전류 루프 대역폭 5~10 kHz.
  3. 속도 루프: 한 관절씩 격리(다른 관절 잠금). 인코더 차분으로 속도 측정 + IMC 또는 ZN. 1 kHz 대역폭.
  4. 위치 루프: 단순 P 또는 PD. 100~200 Hz 대역폭.
  5. 중력 보상 + 동역학 보상 추가: 페이로드별로 보상 계수 측정.
  6. 경로 계획 → 토크 피드포워드: 미리 알 수 있는 가속·코리올리 명령을 피드포워드로 추가. PID는 차이만 보정.
  7. 실측 응답 검증: 대표 동작(픽앤플레이스, 용접 궤적)으로 추종 오차·진동 확인. 필요시 게인 미세 조정.

18. 흔한 디버깅 체크리스트

  • 모터가 진동하면? → Kp 절반, Kd가 잡음 증폭 가능 (필터 N 늘리기).
  • 응답이 둔하면? → Kp 키우기, 액추에이터 한계 확인.
  • 정상오차 잔류 → Ki 추가 또는 키움, 적분 한계(클램프) 확인.
  • 큰 setpoint 변경 시 큰 오버슈트 → 와인드업, setpoint 램프 적용.
  • 외부 충격 후 회복 느림 → Kd 키우기, 외란 옵저버(DOB) 추가 검토.
  • 온도 따라 응답 변함 → 모터 권선 저항 온도 보상, 또는 게인 스케줄.
  • 기어 진동 → 백래시 보상, 출력측 인코더(이중 엔코더).
  • 전원 공급 약하면 ESC가 PWM 누락 → 전류 한계 ↓, 적분 클램프.

19. 인터랙티브 ③ — 캐스케이드 시정수 분리

🎮 인터랙티브: 안쪽 루프가 충분히 빠른가?

안쪽 속도 루프 대역폭을 바꿔가며 외부 위치 루프가 어떻게 영향받는지 확인합니다. 시정수 분리가 5배 미만이면 진동.

관찰: 비율을 1로 줄이면 외부 위치 루프가 안쪽 시정수와 같아 응답이 진동·느림. 5 이상이면 외부 P가 1차 시스템 응답처럼 깨끗.

20. 모터 컨트롤러 - 임베디드 구현 디테일

20.1 PWM 인버터와 데드타임

3상 인버터에서 상측·하측 IGBT가 동시에 ON되면 단락. 이를 막기 위해 두 신호 사이 50~500 ns의 데드타임(dead-time)을 둠. 그러나 이 데드타임이 출력 전압에 비선형 왜곡을 일으킴 → 저속에서 토크 리플. 보상: 부호 기반 데드타임 보상 알고리즘.

20.2 션트 저항 vs 홀 효과 센서

방식장점단점
션트 저항저렴, 정확발열, 절연 필요
홀 효과절연 자체로 됨, 빠른 응답드리프트, 자기장 간섭
커런트 트랜스절연, 큰 전류 OKDC 측정 불가, 부피 큼

20.3 ADC 샘플링 동기화

전류는 PWM 캐리어와 동기되어 샘플링되어야 정확. STM32의 경우 TIM1과 ADC를 트리거 연결, PWM 중앙 정렬에서 캐리어 정점에서 샘플 → 평균 전류 측정.

20.4 d-q 좌표 변환 (FOC)

3상 → 2상 (Clarke) → 회전 좌표 (Park):

$$ \begin{bmatrix} i_d \\ i_q \end{bmatrix} = \begin{bmatrix} \cos\theta & \sin\theta \\ -\sin\theta & \cos\theta \end{bmatrix} \begin{bmatrix} i_\alpha \\ i_\beta \end{bmatrix} $$

$\theta$는 회전자 위치. 회전 좌표에서 PI는 DC 신호를 보는 셈이라 정상오차 0이 가능. SimpleFOC 라이브러리에서도 이 변환이 핵심.

21. 매니퓰레이터 — 작업 공간 vs 관절 공간

21.1 두 공간의 차이

  • 관절 공간(Joint Space) 제어: 각 관절각 $q_i$를 PID. 단순, 안정. 그러나 끝단(end-effector) 위치는 자코비안에 따라 비선형으로 결정됨.
  • 작업 공간(Task Space) 제어: 끝단 위치 $x$를 직접 PID. 직관적, 그러나 자코비안 역(혹은 의사역)이 필요. 특이점에서 발산.

21.2 작업 공간 제어식

$$ \tau = J^T(q) \big( K_p (x_d - x) + K_d (\dot x_d - \dot x) \big) + g(q) $$

자코비안 전치 $J^T$를 통해 작업 공간 힘이 관절 토크로 매핑.

21.3 임피던스 제어 (협동 로봇의 핵심)

외력에 대해 가상 스프링·댐퍼처럼 동작:

$$ \tau = J^T (M_d (\ddot x_d - \ddot x) + B_d (\dot x_d - \dot x) + K_d (x_d - x)) + g(q) $$

$M_d, B_d, K_d$를 작게 하면 부드럽게 양보, 크게 하면 강하게 위치 유지. UR, Franka Panda, Kuka iiwa의 핵심 모드.

22. 백래시·컴플라이언스 처리

22.1 백래시(Backlash)

하모닉 드라이브, 평기어 모두 ±0.1° ~ ±2° 사이의 백래시. 모터-부하 사이가 한순간 무부하 상태. 대처:

  • 이중 인코더(모터 측 + 부하 측). 모터 측은 캐스케이드 안쪽, 부하 측은 위치 추정.
  • 전동기 제동 토크가 늘 한 방향이 되도록 사전 부하(preload) 적용.
  • 방향 전환 시 dead-band 보상 펄스.

22.2 컴플라이언스(유연성)

긴 링크는 굽힘이 있어 무거운 페이로드 들 때 끝단이 처짐. 모드 진동수가 PID 대역폭 안에 들어오면 진동. 대처:

  • 저주파 PID + 노치 필터(특정 진동 모드 제거).
  • 입력 셰이핑(Input Shaping) — 명령을 두 펄스로 분리해 진동 자체를 안 만듦.
  • endpoint 가속도계 + active vibration damping.

23. 페이로드 추정 알고리즘

23.1 정역학 기반 추정

로봇이 정지한 자세에서 측정된 토크는 거의 모두 중력 토크. 자세별 토크 측정 → 페이로드 mass 추정:

$$ \hat m = \frac{\tau_{measured} - \tau_{noload}}{g \ell \sin\theta} $$

23.2 동적 추정 (RLS, EKF)

운동 중 토크-가속도 데이터를 실시간으로 회귀해 mass 추정. UR 로봇의 "auto-payload" 기능이 이 방식.

23.3 적응 제어 (MRAC)

모델 참조 적응 제어. 게인을 직접 추정해 갱신:

$$ \dot{\hat\theta} = -\Gamma \phi e $$

$\hat\theta$는 추정 파라미터, $\phi$는 회귀 벡터, $e$는 추적 오차. 학부 수준 너머의 주제.

24. 시뮬레이터로 PID 검증

24.1 무엇을 시뮬레이션할 것인가

  • 모터 동역학 (전기 + 기계).
  • 액추에이터 한계 (전류·전압·토크).
  • 인코더 양자화 + 잡음.
  • 샘플링 지연.
  • 외란(중력, 마찰, 페이로드 변화).

25. 학습용 미니 프로젝트 제안

  1. STM32 + DC 모터: 인코더로 위치 PID. 1 ms 주기, 안티 와인드업 + 미분 필터. 팔 회전·파지 동작.
  2. SimpleFOC + BLDC: d-q FOC + 캐스케이드 PID. 0.1° 위치 정밀도 달성.
  3. 2자유도 팔 (Pan-Tilt): 카메라 짐벌 형태. 사람 얼굴 추적.
  4. 3D 프린터 보드 분해: Marlin 펌웨어의 PID 코드 읽기, 온도 제어 분석.
  5. 1자유도 도립진자(Inverted Pendulum): 가장 어려운 PID 응용. 대부분 LQR 사용.

27. 부록 B — 인코더 종류와 분해능

27.1 절대형 vs 증분형

  • 절대형(Absolute): 위치값을 직접 출력. 전원 ON 즉시 위치 알 수 있음. 가격 비쌈.
  • 증분형(Incremental): A/B 채널 펄스로 상대 변화만 측정. 원점 복귀(homing) 필요. 저렴.

27.2 분해능 등급

  • 저해상도 (200~1000 PPR): DC 모터 일반.
  • 중해상도 (2048~10000 PPR): BLDC 정밀 위치 제어.
  • 고해상도 (4-quadrant decoding 후 100k+ PPR): 머신툴, 반도체.

27.3 PPR vs CPR

PPR (Pulses Per Revolution) — 한 채널의 펄스 수. CPR (Counts Per Revolution) — 4 × PPR (4-quad decoding 시). 헷갈리기 쉬움.

28. 부록 C — 매니퓰레이터 좌표계 약속

28.1 Denavit-Hartenberg 표기

관절 i에서 관절 i+1로의 변환을 4개 파라미터 ($a_i, \alpha_i, d_i, \theta_i$)로 표현:

$$ T_i^{i-1} = R_z(\theta_i) T_z(d_i) T_x(a_i) R_x(\alpha_i) $$

모든 관절에 대해 곱하면 끝단까지의 변환. 자코비안 유도의 출발점.

28.2 ZYX vs ZYZ Euler

드론은 ZYX(요-피치-롤), 매니퓰레이터는 ZYZ(요-피치-요)이 흔함. 짐벌 락(gimbal lock) 방지를 위해 quaternion 사용 추세.

29. 케이스 연구 — 자동차 EPS(Electric Power Steering)

운전자가 핸들에 가하는 토크를 토크 센서로 측정 → BLDC 모터가 보조 토크 제공.

  • 전류 루프 (10 kHz) → 토크 PI.
  • 모델: 핸들 + 콜럼 + 랙 = 다관성 시스템 (3-mass model).
  • 도로 외란(노면 충격, 풍속, 비대칭 마찰)을 적극적으로 보상.
  • 핵심: 운전자가 도로감을 잃지 않도록 보조 토크 곡선 설계 — 단순 PID 외에 룩업 테이블 + 동적 게인.

30. 시뮬레이션 실습 가이드 — 1자유도 팔

Python 한 페이지로 1자유도 팔을 시뮬레이션:

import numpy as np
import matplotlib.pyplot as plt

# 시스템 파라미터
J, b, m, L, g = 0.5, 0.3, 1.0, 0.5, 9.81
dt, T = 1e-3, 5.0
N = int(T/dt)

# PID 게인
Kp, Kd, Ki = 20, 5, 5
gravity_comp = True

# 시뮬레이션
theta_ref = np.pi/3
theta, omega = 0.0, 0.0
iE = 0.0
hist_theta, hist_tau = [], []
for k in range(N):
    e = theta_ref - theta
    de = -omega
    iE += e*dt
    tau = Kp*e + Kd*de + Ki*iE
    if gravity_comp:
        tau += m*g*L*np.sin(theta)
    tau = np.clip(tau, -30, 30)
    thdd = (tau - b*omega - m*g*L*np.sin(theta))/J
    omega += thdd*dt
    theta += omega*dt
    hist_theta.append(theta)
    hist_tau.append(tau)

t = np.arange(N)*dt
plt.subplot(2,1,1)
plt.plot(t, np.array(hist_theta)*180/np.pi, label='θ')
plt.axhline(theta_ref*180/np.pi, color='r', linestyle='--', label='θ_ref')
plt.ylabel('각도 [°]'); plt.legend(); plt.grid()
plt.subplot(2,1,2)
plt.plot(t, hist_tau)
plt.ylabel('토크 [Nm]'); plt.xlabel('시간 [s]'); plt.grid()
plt.tight_layout(); plt.show()

이 코드 한 페이지로 학생은 (1) 게인 변화 효과, (2) 중력 보상 ON/OFF, (3) 페이로드 변화의 영향을 모두 실험 가능.

30.5 캐스케이드 게인 튜닝 절차 — 한 페이지

  1. 가장 안쪽 루프(전류·토크) 먼저. 단독으로 안정 + 충분히 빠르게(주파수 대역폭 ~kHz).
  2. 다음 루프(속도) — 안쪽 루프가 5~10배 빠르도록 P/PI 게인 조정.
  3. 가장 바깥 루프(위치) — P 게인으로 시작, 오버슈트 크면 D 추가.
  4. 외란 응답 확인 — 페이로드 추가, 마찰 변화에 회복 속도 측정.
  5. 액추에이터 포화 체크 — 큰 명령에 토크 한계 도달 시 안티-와인드업.

30.6 자주 만나는 문제와 해결

증상원인해결
위치 정상상태 오차중력 보상 부재중력 항을 피드포워드로 추가
고주파 진동D 게인 너무 큼 + 측정 잡음D 게인 ÷2 또는 LPF 추가
큰 페이로드에서 느림I 게인 부족, 적응 부재게인 스케줄링
가속 시 떨림역기전력 보상 부재속도 피드포워드

30.7 한 줄 요약

"캐스케이드 + 피드포워드 + 적응" — 모터/로봇 제어의 3종 세트. 단독 PID로는 한계가 빠르게 옴.

31. 핵심 정리

  • 모터는 토크 ↔ 속도 ↔ 위치 세 시간 척도가 다르다 → 캐스케이드가 자연스럽다.
  • 안쪽이 5~10배 빠르게 돌면서 단순화 → 바깥 게인을 더 크게 줄 수 있다.
  • 전기 시정수($L/R$)와 기계 시정수($JR/(K_t K_e)$)의 분리가 캐스케이드 설계의 출발점.
  • 로봇 팔처럼 비선형 외력(중력)이 있는 곳은 PID + 피드포워드 보상이 표준.
  • 페이로드가 변하는 환경은 게인 스케줄링·적응 제어로 — PID 단독은 한계.
  • 완전한 computed torque로 가는 길은 로봇공학특론 L2 동역학으로.
