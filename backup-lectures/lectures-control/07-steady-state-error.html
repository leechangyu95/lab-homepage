Week 7. 정상상태 오차

시간이 충분히 흐른 뒤에도 남는 오차를 어떻게 예측하고 없앨까? Type 번호 한 숫자로 시스템의 성능을 한 줄에 정리합니다.

목차

  1. 최종값 정리로 정상상태 오차 계산하기
  2. Type 번호와 시스템 분류
  3. 오차 상수 (Kp, Kv, Ka)
  4. 입력 종류별 오차 표
  5. 인터랙티브: Type별 응답 비교

1. 최종값 정리로 정상상태 오차 계산하기

단위 피드백 폐루프에서 오차 $e(t) = r(t) - y(t)$의 라플라스는

$$ E(s) = \frac{1}{1 + G(s)}\, R(s) $$

W2에서 본 최종값 정리(Final Value Theorem)를 쓰면 시간이 무한대로 갈 때의 오차를 역변환 없이 바로 계산할 수 있습니다.

$$ \boxed{\, e_{ss} = \lim_{t\to\infty} e(t) = \lim_{s\to 0} sE(s) = \lim_{s\to 0} \frac{s\, R(s)}{1 + G(s)} \,} $$
전제 조건 — 폐루프가 안정해야 합니다. 불안정 시스템에 최종값 정리를 적용하면 엉뚱한 값이 나옵니다. 그래서 정상상태 오차 분석은 항상 안정성 확인(W8) 뒤에 합니다.

2. Type 번호와 시스템 분류

$G(s)$를 다음과 같이 적었을 때, $s = 0$의 극점 개수 $N$을 Type 번호라고 합니다.

$$ G(s) = \frac{K\,(s+z_1)(s+z_2)\cdots}{s^N\,(s+p_1)(s+p_2)\cdots} $$

즉 분모에 $s$가 몇 개 곱해져 있느냐. 이게 시스템의 "오차 성능 등급"을 한 숫자로 압축합니다.

Type예시 G(s)실제 예
0$\dfrac{K}{(s+1)(s+2)}$온도제어 (적분기 없음)
1$\dfrac{K}{s(s+1)}$위치제어 (속도→위치 적분 1회)
2$\dfrac{K}{s^2(s+1)}$가속도→위치 (적분 2회)

3. 오차 상수 (Kp, Kv, Ka)

입력의 종류(스텝/램프/포물선)에 따라 정의되는 세 가지 오차 상수입니다.

이름정의해당 입력
위치 오차상수 $K_p$$\displaystyle\lim_{s\to 0} G(s)$스텝 입력
속도 오차상수 $K_v$$\displaystyle\lim_{s\to 0} sG(s)$램프 입력
가속도 오차상수 $K_a$$\displaystyle\lim_{s\to 0} s^2 G(s)$포물선 입력

이 정의에서 어떤 시스템이 어떤 입력에 0 오차를 낼 수 있는지가 자동으로 결정됩니다.

4. 입력 종류별 오차 표 — 한 장으로 정리

$R(s)$가 단위 스텝, 램프, 포물선일 때 정상상태 오차:

입력$R(s)$$e_{ss}$ 공식Type 0Type 1Type 2
스텝 (위치)$1/s$$\dfrac{1}{1 + K_p}$유한00
램프 (속도)$1/s^2$$\dfrac{1}{K_v}$$\infty$유한0
포물선 (가속도)$1/s^3$$\dfrac{1}{K_a}$$\infty$$\infty$유한
한 줄 요약 — Type N 시스템은 $N$차 다항식 입력까지는 어떻게든 따라가지만, 그보다 한 차수 높은 입력에는 따라가지 못한다 (오차 = $\infty$). 같은 차수에서는 유한한 오차, 더 낮은 차수에서는 0 오차.

예제: 모터 위치제어가 왜 자연스럽게 Type 1인가?

DC 모터의 전달함수 $G_m(s) = K_m / [s(\tau_m s + 1)]$은 분모에 $s$ 하나가 이미 있습니다. 이건 "각도는 각속도의 적분"이라는 물리적 사실에서 나옵니다. 그래서 P 제어만 걸어도 폐루프가 안정한 한 스텝 위치 명령에 대해 정상상태 오차가 0이 됩니다 — 적분기를 따로 넣지 않아도.

5. 인터랙티브: Type별 응답 비교

🎮 인터랙티브: Type 0/1/2 시스템에 스텝과 램프 넣어보기

같은 게인 $K$로 세 시스템 ($\dfrac{K}{s+1}$, $\dfrac{K}{s(s+1)}$, $\dfrac{K}{s^2(s+1)}$)에 스텝/램프 입력을 가했을 때 어느 쪽이 따라가는지 보세요.

꼭 만져볼 것:

  • 스텝 입력: Type 0은 잔류 오차, Type 1·2는 0으로 수렴.
  • 램프 입력: Type 0은 시간이 갈수록 발산하는 오차, Type 1은 일정한 오차(= 1/Kv), Type 2는 0.
  • K를 키우면 Type 0/1의 오차가 작아지지만, 너무 키우면 진동이 시작 → W11 안정여유와의 트레이드오프.

6. 왜 적분기를 추가하면 오차가 0이 되는가?

I 제어가 정상상태 오차를 0으로 만든다는 사실은 마법이 아니라 위 표의 직접적 결과입니다. PID에서 적분 항 $K_i/s$를 추가한다는 건 시스템 분모에 $s$가 하나 더 곱해진다는 뜻 → Type 번호가 1 증가. Type 0이었던 시스템이 Type 1이 되면서 스텝 입력 오차가 0이 됩니다.

이게 실무에서 "외란이 있는 항온조에는 PI를 써라"의 이론적 근거입니다. 외란도 결국 입력 신호이고, Type 1이 되면 상수 외란에 대해서도 정상상태 오차가 0이 됩니다.

그럼 적분기를 마구 추가하면? — Type 번호를 키울수록 정상오차는 좋아지지만 안정성이 위태로워집니다. 각 적분기는 Bode 위상에 −90°를 추가해서 위상여유를 잡아먹습니다(W11). 그래서 실무 PID에는 거의 항상 적분기 1개만 씁니다.

7. 손계산 예제 1 — Type 0 + 스텝 입력

문제

$G(s) = \dfrac{4}{(s+1)(s+2)}$ 인 단위 피드백 시스템에 단위 스텝 $r(t) = 1$ 을 넣었을 때 정상상태 오차를 구하시오.

풀이

1단계 — Type 확인: 분모에 $s$ 가 없음 → Type 0.

2단계 — 위치 오차상수 계산:

$$ K_p = \lim_{s\to 0}G(s) = \lim_{s\to 0}\frac{4}{(s+1)(s+2)} = \frac{4}{1\cdot 2} = 2 $$

3단계 — 정상오차: $e_{ss} = \dfrac{1}{1+K_p} = \dfrac{1}{1+2} = \dfrac{1}{3} \approx 0.333$.

해석: 시간이 충분히 지나도 출력은 $1$ 이 아니라 $\tfrac{2}{3} \approx 0.667$ 에서 안착. 입력의 33%가 영구 오차로 남음.

오차를 줄이려면?

  • 방법 1: $K$ 를 키운다. $K=20$ 이면 $K_p = 10$, $e_{ss} = 1/11 \approx 9\%$. 단, 너무 크면 진동·불안정.
  • 방법 2: 적분기 추가 → Type 1 → $e_{ss} = 0$. 트레이드오프는 안정여유 감소(W11).

8. 손계산 예제 2 — Type 1 + 램프 입력

문제

$G(s) = \dfrac{10}{s(s+5)}$ (모터 위치제어 모델) 시스템에 램프 입력 $r(t) = 2t$ 를 넣었을 때 정상상태 오차는?

풀이

Type 확인: 분모에 $s^1$ → Type 1.

속도 오차상수:

$$ K_v = \lim_{s\to 0}sG(s) = \lim_{s\to 0}\frac{10}{s+5} = \frac{10}{5} = 2 $$

정상오차: 램프 입력 $r(t) = At$ (여기서 $A=2$) 에 대해 $e_{ss} = \dfrac{A}{K_v} = \dfrac{2}{2} = 1$.

해석: 입력이 $r(t) = 2t$ (속도 2 로 증가) 일 때, 출력은 항상 $1$ 만큼 뒤처진 채 따라감. "속도는 따라가지만 위치는 1 만큼 뒤짐".

실제 의미 — 자율주행 자동차의 차로 추종

자동차가 곡선 도로(연속 변화하는 헤딩) 를 따라가야 할 때, Type 1 컨트롤러는 차의 헤딩 속도는 따라잡지만 위치(차로 중심선) 는 일정한 횡오차를 가진 채 추종. 이걸 0 으로 만들려면 적분기 추가 → Type 2 또는 추가 보상기 설계.

9. 손계산 예제 3 — PI 제어로 Type 승격하기

문제

플랜트 $G_p(s) = \dfrac{1}{(s+1)(s+2)}$ (Type 0) 에 PI 제어기 $C(s) = K_p + \dfrac{K_i}{s}$ 를 적용한다. 단위 스텝과 단위 램프 입력에 대한 정상오차를 비교하시오.

풀이 — 개루프 전달함수 만들기

$$ L(s) = C(s)\,G_p(s) = \frac{K_p s + K_i}{s(s+1)(s+2)} $$

분모의 $s$ 하나가 적분기에서 옴 → Type 1 로 승격.

스텝 입력

$K_p = \lim_{s\to 0}L(s) = \infty$ → $e_{ss} = \dfrac{1}{1+\infty} = 0$ ✓

램프 입력

$K_v = \lim_{s\to 0}sL(s) = \dfrac{K_i}{2}$. $e_{ss} = \dfrac{1}{K_v} = \dfrac{2}{K_i}$.

$K_i$ 를 키우면 램프 추종 오차도 줄지만, $K_i$ 를 너무 키우면 안정여유가 사라짐. 보통 $K_i = 1\sim 10$ 사이에서 튜닝.

10. 외란(disturbance) 정상오차 — Type의 진짜 가치

문제 상황

실제 시스템에는 일정한 상수 외란 $d$ 가 항상 있습니다 (예: 모터에 작용하는 마찰토크, 항온조에 들어오는 외풍). 이 외란이 출력에 영구 오차를 만들지 않게 하려면?

외란 응답 계산

플랜트 $G_p(s)$ 와 컨트롤러 $C(s)$ 의 단위 피드백 시스템에서, 입력 $R = 0$ 으로 두고 외란만 본다면:

$$ Y(s) = \frac{G_p(s)}{1 + C(s)G_p(s)}\,D(s) $$

외란이 상수 $D(s) = d/s$ 일 때:

$$ y_{ss} = \lim_{s\to 0}\frac{G_p(s)}{1 + C(s)G_p(s)}\cdot d $$

P 제어 vs PI 제어

  • P 제어 $C(s) = K_p$: $y_{ss} = \dfrac{G_p(0)}{1 + K_p G_p(0)}\cdot d \neq 0$. 외란이 출력에 영구 영향.
  • PI 제어 $C(s) = K_p + K_i/s$: $s\to 0$ 에서 $C(s) \to \infty$, 분모도 $\infty$, 분자는 유한 → $y_{ss} = 0$. 외란을 완전히 제거 ✓
핵심 통찰: PI 제어가 산업 현장에서 압도적으로 많이 쓰이는 이유 = "상수 외란을 자동으로 제거" 한다는 보장. 적분기 한 개의 위력.

11. 정상오차 vs 응답속도 — 통합 트레이드오프

제어 방식정상오차 (스텝)정상오차 (램프)응답속도안정여유
P (작은 K)큼$\infty$느림좋음
P (큰 K)작음$\infty$빠름나쁨 (진동)
PI0유한적당중간
PID0유한빠름중간 (D가 도와줌)

각 행을 외우기보다 "왜 이런 트레이드오프가 생기는가" 의 직관을 잡아두세요:

  • K ↑ → 오차 ↓ but 안정 ↓ — 빠른 반응 = 진동 위험.
  • I 추가 → 정상오차 = 0 but 위상 −90° → 안정여유 감소.
  • D 추가 → 응답 빠르게 + 안정여유 회복 but 잡음 증폭.

12. 흔한 함정 — 시험에 자주 나오는 실수

  1. 안정성 확인 안 하고 최종값 정리 적용: 폐루프가 불안정한데 $\lim sE(s)$ 를 계산하면 0 또는 유한값이 나오지만 실제로는 발산. 항상 W8 안정성 분석 후 정상오차 계산.
  2. Type 계산 시 분모만 보기: $G(s) = \dfrac{s}{s+1}$ 처럼 분자에 $s$ 가 있어도 분모의 $s^N$ 만 세는 것이 Type 정의. 분자에는 영향 없음.
  3. 피드백이 단위가 아닐 때: $H(s) \neq 1$ 인 비단위 피드백에서는 위 공식 그대로 안 됨. 폐루프 오차 식을 다시 유도해야 함.
  4. $K_p$ 와 게인 $K$ 혼동: $K_p$ 는 위치 오차상수 (정의: $\lim G$), 게인 $K$ 는 전달함수의 비례계수. 둘은 같은 글자 사용해도 다른 양.

13. 응용 — 실제 산업 시스템의 Type 분석

(1) 가전 — 에어컨 온도제어

  • 플랜트: 방의 열용량 → $G_p \approx \dfrac{K}{\tau s + 1}$ (Type 0).
  • P 제어 → 설정온도와 실제온도 사이 ~1°C 잔류 오차.
  • PI 제어 → 외란(외부 기온, 사람 입실) 있어도 정확히 설정온도 유지.
  • 실제 인버터 에어컨은 PI + 적응제어 조합.

(2) 산업로봇 — 위치제어

  • 관절 모터: $G \approx \dfrac{K_m}{s(\tau s + 1)}$ → 자연스럽게 Type 1.
  • 스텝 명령("이 각도로 가라") 추종 오차 = 0 (단순 P 제어로도).
  • 등속 회전 명령(궤적 추종) 추종 오차 = $\dot\theta_{\text{cmd}}/K_v$ → 속도 곱 만큼 뒤처짐.
  • 고정밀 추종이 필요하면 PI 또는 외란 관측기(Disturbance Observer) 추가.

(3) 자율주행 — 차량 종방향 제어

  • 입력: 가속 명령, 출력: 차량 속도. 플랜트는 차량 동역학 (Type 0 ~ Type 1 사이).
  • P 제어 — 등속 명령에 대해 약간의 잔류 속도 오차.
  • PI 제어 — 평지에서는 정확히 추종. 경사로에서 외란(중력)도 자동 보상.

(4) 항공우주 — 위성 자세제어

  • 관성 모먼트 → $G \approx K/s^2$ (Type 2).
  • 적분기 2개가 이미 있어 위치+속도 명령 모두 0 오차.
  • 그러나 위상 −180° → 안정 위해 D 항(미분 보상기) 필수.

14. 연습문제

문제 1 (Type 판정)

다음 시스템의 Type 번호를 답하시오.

  1. $G(s) = \dfrac{2}{s+3}$
  2. $G(s) = \dfrac{5(s+1)}{s^2(s+2)}$
  3. $G(s) = \dfrac{s+1}{s(s+2)(s+3)}$
  4. $G(s) = \dfrac{10}{s^3(s^2+2s+5)}$

문제 2 (스텝 정상오차)

$G(s) = \dfrac{8}{(s+2)(s+4)}$ 단위 피드백 시스템에 $r(t) = 5$ 단위 스텝 입력. $K_p$ 와 $e_{ss}$ 를 구하시오.

문제 3 (램프 정상오차)

$G(s) = \dfrac{20}{s(s+1)(s+5)}$ 시스템에 $r(t) = 3t$ 램프 입력. $K_v$ 와 $e_{ss}$ 를 구하시오.

문제 4 (PI 설계)

문제 5 (외란 분석)

15. 핵심 정리

  • $e_{ss} = \lim_{s\to 0} sE(s)$ — 시간 영역 적분 없이 정상오차를 바로 계산.
  • Type 번호 = $G(s)$의 $s=0$ 극점 개수.
  • Type N은 $t^{N-1}$까지의 입력에서 0 오차, $t^N$ 입력에서 유한 오차, 그 이상은 발산.
  • I 제어를 추가하면 Type +1, 정상오차 ↓ 안정여유 ↓ — 트레이드오프.

16. 정상오차와 안정성의 트레이드오프

적분기 추가 → 정상오차 ↓. 그러나 위상이 -90° 추가 → 안정여유 ↓. 보통 PI 제어로 절충.

17. PI vs PID 정상오차 차이

  • P 제어 — 정상오차 존재 (Type 0).
  • PI 제어 — 정상오차 0 (Type +1).
  • PID — PI와 동일하지만 D가 응답 속도 향상.

18. 외란 종류별 정상오차

  • 입력 외란 — 적분기 제어로 0.
  • 출력 외란 — 적분기 제어로 0.
  • 측정 외란 — 0이 안 됨 (제어기가 잘못된 정보).
  • 모델 외란 — 강건 설계 필요.

19. Python 코드 — 정상오차 검증

import control as ct
import numpy as np

# Type 0 시스템
G0 = ct.tf([1], [1, 1])
T0 = ct.feedback(G0, 1)
t, y = ct.step_response(T0, T=np.linspace(0, 10, 200))
print('Type 0 단위 계단 정상상태:', y[-1])  # 1/2

# Type 1 시스템
G1 = ct.tf([1], [1, 1, 0])
T1 = ct.feedback(G1, 1)
t, y = ct.step_response(T1, T=np.linspace(0, 30, 200))
print('Type 1 단위 계단 정상상태:', y[-1])  # 1.0

"적분기는 자동제어의 마법 — 정상오차 0의 비결."

72. 부록 — 추가 실무 팁

  • Bumpless transfer — 자동/수동 전환 시 적분기 초기화.
  • Tracking — 동시 제어기 운영 시 적분기 동기화.
  • Anti-saturation — 미리 정의된 한계 + 적분기 보정.
  • Reset windup — 큰 외란 후 적분기 재초기화.

  1. $G = K/(s(s+5))$, P=2 적용. 단위 계단 정상오차?
  2. 같은 시스템에 PI(Kp=2, Ki=1)면 정상오차?
