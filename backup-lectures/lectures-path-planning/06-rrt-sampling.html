Week 6. RRT — 뿌리처럼 뻗어 길을 찾는 알고리즘

격자 위를 한 칸씩 살피는 A*와 달리, RRT(Rapidly-exploring Random Tree)는 "공간 한복판에 무작위로 점을 던지고 그쪽으로 한 걸음씩 가지를 뻗는" 단순한 규칙으로 경로를 찾습니다. 좁은 통로·고차원 공간에서도 통하는 강력한 기법이라, 자율주행 주차·매니퓰레이터 경로계획의 표준 도구입니다.

1. 경로계획의 두 세계 — 격자 vs 샘플링

한 줄 비유부터

모르는 동네에서 친구 집을 찾아야 합니다. 두 가지 전략이 있어요.

  • 격자 전략: 동네를 모눈종이처럼 격자로 자르고 한 칸씩 빠짐없이 살핍니다 — A*(W7)의 방식. 정확하지만 칸이 많아지면 느려집니다.
  • 샘플링 전략: 손전등을 든 채 어둠 속에서 사방으로 무작위로 빛을 쏘다가, 그쪽으로 한 걸음씩 다가가며 길을 잇습니다. 이게 RRT입니다.

차원이 높아지거나 좁은 통로가 많아질수록 격자는 폭발적으로 느려지지만, 샘플링은 균등하게 공간을 메꿔 가며 빠르게 길을 찾아냅니다.

방식대표장점단점
격자 탐색BFS, Dijkstra, A* (W7)최단 경로 보장고차원·연속 공간에서 폭발
샘플링PRM, RRT, RRT*고차원·복잡 환경에 강함최단이 아닐 수 있음 (RRT)

2. RRT 5단계 알고리즘

RRT는 단 5줄로 요약됩니다. 아래 5줄을 외우면 끝.

  1. Sample — 공간 안에 점 $q_\text{rand}$를 무작위로 찍는다.
  2. Nearest — 트리 안에서 그 점에 가장 가까운 노드 $q_\text{near}$를 찾는다.
  3. Steer — $q_\text{near}$에서 $q_\text{rand}$ 방향으로 작은 한 걸음만 가서 $q_\text{new}$를 만든다.
  4. Collision Check — $q_\text{near} \to q_\text{new}$ 선분이 장애물을 통과하지 않는지 확인.
  5. Add — 통과면 트리에 추가, $q_\text{new}$가 목표 근처면 종료.
외우는 비결 — "Sample → Nearest → Steer → Check → Add". 한국어로는 "샘플 → 이웃 → 걸음 → 체크 → 추가". 
# Ref: PythonRobotics/PathPlanning/RRT/rrt.py (MIT, A. Sakai) — 교육용 단순화
import numpy as np

class RRT:
    def __init__(self, start, goal, obs, bounds, step=0.5, max_iter=2000):
        self.start = np.array(start); self.goal = np.array(goal)
        self.obs, self.bounds = obs, bounds
        self.step, self.max_iter = step, max_iter
        self.nodes = [self.start]; self.parents = [-1]

    def sample(self):
        lo, hi = self.bounds
        return np.random.uniform(lo, hi, size=2)

    def nearest(self, q):
        return int(np.argmin(np.linalg.norm(np.array(self.nodes) - q, axis=1)))

    def steer(self, q_from, q_to):
        v = q_to - q_from; d = np.linalg.norm(v)
        return q_to if d <= self.step else q_from + v/d * self.step

    def plan(self):
        for _ in range(self.max_iter):
            q_rand = self.sample()
            i_near = self.nearest(q_rand)
            q_new  = self.steer(self.nodes[i_near], q_rand)
            if not self.collision_free(self.nodes[i_near], q_new): continue
            self.nodes.append(q_new); self.parents.append(i_near)
            if np.linalg.norm(q_new - self.goal) < self.step:
                return self.extract_path(len(self.nodes) - 1)
        return None

핵심은 10줄짜리 메인 루프입니다. 이걸 머릿속에 넣고 시작하세요.

3. 인터랙티브: 트리가 자라는 모습

🎮 인터랙티브: RRT 트리 성장

한 걸음씩 버튼을 눌러 알고리즘이 한 번 반복하는 모습을 직접 보세요. 노란 점은 이번 회 샘플 $q_\text{rand}$, 회색 선이 트리 가지, 파란 점이 목표 도달 시 추출된 경로입니다.

4. Goal Bias — "목표 쪽 자주 찍기"

완전한 무작위 샘플은 아무 방향이나 균등하게 탐색하느라 목표에 도달이 늦습니다. 5~10% 확률로 $q_\text{rand}$ 자리에 목표점을 그대로 넣어주는 트릭이 거의 모든 RRT 구현의 표준입니다.

def sample(self):
    if np.random.rand() < 0.05:        # 5% goal bias
        return self.goal
    return np.random.uniform(*self.bounds, size=2)
50% 이상은 금물 — 너무 높이면 목표 직선 방향으로만 시도하다가 장애물에 막혀 영영 못 갑니다. 경험적 스위트 스팟은 5–10%입니다.

5. 인터랙티브: Goal Bias 효과 비교

🎮 인터랙티브: Bias 0% / 5% / 30% 동시 실행

같은 환경·같은 step에서 세 가지 Bias로 동시에 RRT를 돌린 결과입니다. 자동 재생으로 한 번에 보세요.

관찰: Bias 0%는 트리가 사방으로 균등하게 퍼지지만 목표 도달이 느립니다. Bias 5%면 같은 노드 수로도 더 빨리 도달합니다. Bias 30%는 운이 좋으면 가장 빠르지만 좁은 통로에서는 막히기 쉽습니다.

6. RRT*: 점점 좋아지는 경로

기본 RRT는 최초 발견된 경로에서 멈추는데, 이 경로는 보통 지그재그입니다. RRT*는 두 가지를 추가합니다:

  1. Choose Parent — $q_\text{new}$ 주변 반경 안의 후보들 중 가장 비용이 적은 부모를 고름.
  2. Rewire — $q_\text{new}$를 트리에 넣은 뒤, 주변 노드들 중 "$q_\text{new}$를 부모로 두면 더 싸지는" 것들을 재연결.

이 두 단계 덕분에 반복할수록 경로가 점근적으로 최적(asymptotically optimal)에 가까워집니다.

def choose_parent(self, q_new, near_ids):
    best_i, best_cost = None, np.inf
    for i in near_ids:
        if not self.collision_free(self.nodes[i], q_new): continue
        c = self.cost[i] + np.linalg.norm(q_new - self.nodes[i])
        if c < best_cost: best_i, best_cost = i, c
    return best_i, best_cost

def rewire(self, new_id, near_ids):
    for i in near_ids:
        c = self.cost[new_id] + np.linalg.norm(self.nodes[new_id] - self.nodes[i])
        if c < self.cost[i] and self.collision_free(self.nodes[new_id], self.nodes[i]):
            self.parents[i] = new_id
            self.cost[i]    = c

8. Shortcut · Hybrid A* · Reeds-Shepp

Shortcut — 거친 경로 다듬기

경로 위 임의의 두 점이 직선으로 연결 가능하면 그 사이를 잘라냅니다. 빠르고 효과적이에요.

def shortcut(path, obstacles, iters=200):
    for _ in range(iters):
        if len(path) <= 2: break
        i = np.random.randint(0, len(path)-2)
        j = np.random.randint(i+2, len(path))
        if collision_free(path[i], path[j], obstacles):
            path = path[:i+1] + path[j:]
    return path

비홀로노믹 차량 — Hybrid A* / Reeds-Shepp

자동차는 옆으로 미끄러질 수 없는(비홀로노믹) 시스템이라 RRT가 만든 지그재그를 그대로 못 탑니다. 두 가지 길이 있어요:

  • 후처리: RRT 결과를 Shortcut + Cubic Spline으로 매끈하게.
  • Hybrid A*: 격자 탐색의 이웃 확장을 실제 차량 모션(조향각별 한 걸음)으로 두는 기법. 주차 시나리오 표준.
  • Reeds-Shepp 곡선: 전진+후진을 모두 허용하는 최단 경로 — 좁은 주차에서 필수.
변종특징
RRT기본형. 첫 발견에 멈춤.
RRT-Connect시작·목표 양쪽에서 트리를 동시에 키움. 빠름.
RRT*점근 최적. 시간이 곧 품질.
Informed RRT*이미 찾은 경로보다 짧을 수 있는 영역(타원 안)만 샘플링. 2~5배 빠름.
Hybrid A*차량 모션 반영. 주차·자율주행 표준.

9. 핵심 정리

  • RRT 5단계 = Sample → Nearest → Steer → Check → Add. 메인 루프 10줄.
  • Goal Bias 5–10%가 표준. 50% 이상은 지역해 빠짐.
  • RRT는 빠르지만 최적이 아님 → RRT*로 점근 최적, Informed RRT*로 가속.
  • 차량용으로는 Shortcut + Spline 후처리가 필수. 주차는 Hybrid A* / Reeds-Shepp.
  • 한 줄 비유: "어둠 속에서 손전등 비추며, 씨앗에서 가지 뻗기".

9c. 손풀이 — RRT 직관 점검

Q1. step 크기 영향

step = 0.5 m, 시작-목표 직선 거리 10 m, 장애물 없음. 평균 노드 수는?

풀이

이론적으로 20 노드면 직선으로 도달 가능. 하지만 랜덤 샘플은 균등 분포 → 매 스텝 효율적 진행 보장 X.

Goal bias 5%면 평균 ~50 노드, 0%면 ~200 노드 가량.

Q2. RRT* 의 Choose Parent

새 노드 $q_\text{new}$ 주변 반경 $R=2$ 내에 노드 3개. 비용 $g_1=5, g_2=4, g_3=6$, 거리 $d_1=1, d_2=1.5, d_3=0.8$. 어느 노드가 부모?

풀이

총 비용 비교:

  • $q_1$ 통해: $5 + 1 = 6$
  • $q_2$ 통해: $4 + 1.5 = 5.5$ ← 최소
  • $q_3$ 통해: $6 + 0.8 = 6.8$

답: $q_2$ 가 부모. 가장 가까운 게 아니라 총 비용이 최소인 노드.

Q3. Goal bias 영향

Bias = 0% vs 100% 의 극단을 비교하라.

풀이
  • 0%: 완전 균일 샘플 → 매우 느린 수렴, 트리가 모든 방향 균등하게 자람
  • 100%: 항상 목표를 샘플 → 직선 방향만 시도. 장애물 있으면 무한 반복 (실패)
  • 5~10% (권장): 충분한 탐색 + 약간의 목표 편향. 균형

9d. PRM (Probabilistic Roadmap) — RRT의 사촌

RRT가 "단일 쿼리(start→goal 한 번)" 라면, PRM은 "다중 쿼리" 알고리즘입니다.

2단계 구성

  1. Roadmap 구축 (오프라인): 공간에 N개 점 무작위로 찍고, 충돌 없는 점끼리 가까운 이웃 K개와 연결 → 그래프 완성
  2. Query (온라인): start, goal 추가 후 Dijkstra/A* 로 그래프 탐색
