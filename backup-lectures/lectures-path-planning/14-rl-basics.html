Week 14. 강화학습(RL) 기초 — 경험으로 배우는 제어

PID·MPC·CBF가 모두 모델을 손에 쥐고 설계하는 도구라면, 강화학습은 정반대 — 모델 없이 시행착오로 정책을 학습합니다. 오늘은 강아지 훈련 비유로 큰 그림을 잡고, 그리드월드에서 Q-Learning이 어떻게 최적 경로를 스스로 찾아가는지를 직접 돌려봅니다.

1. RL이란? — 강아지 훈련 비유

한 줄 비유부터

강아지에게 "앉아!"를 가르치는 장면을 떠올려 보세요. 처음엔 강아지가 엉뚱한 행동을 합니다. 앉으면 간식을 주고, 안 앉으면 안 줍니다. 시간이 지나면 강아지는 "앉으면 간식이 나오는구나"를 시행착오로 배웁니다. 강아지에게 어떤 근육을 어떻게 움직이라고 가르치지 않았는데도요.

이게 강화학습(Reinforcement Learning)입니다. "잘하면 보상, 못하면 보상 없음/벌점"이라는 신호 하나로 정책을 학습합니다.

2. 4가지 구성 요소

요소기호강아지 예시자율주행 예시
State (상태)$s$강아지의 자세·상황차량 위치·속도·차선
Action (행동)$a$앉다 / 일어서다 / 짖다조향각 / 가속 / 제동
Reward (보상)$r$간식 +1 / 무반응 0경로 추종 +1, 충돌 -100
Policy (정책)$\pi(s)$상황별 행동 규칙상태→조향/속도 함수

목표는 단 하나: 누적 할인 보상 $\sum_t \gamma^t r_t$를 최대화하는 정책 $\pi$를 찾기.

3. 탐험(Exploration) vs 활용(Exploitation)

RL의 가장 유명한 딜레마입니다. 지금 알고 있는 최선을 계속 쓸까(활용)? 아니면 새 행동을 시도해 더 좋은 길을 찾아볼까(탐험)?

  • 탐험만 하면 → 보상이 낮은 길도 무작정 시도, 학습이 느림.
  • 활용만 하면 → 처음 발견한 그저 그런 길만 반복, 더 좋은 길을 못 찾음.

가장 단순한 절충안이 ε-greedy입니다. 확률 $\varepsilon$로 랜덤 행동, $1-\varepsilon$로 현재 최선의 행동. 학습 초반에 ε이 크고, 시간이 지날수록 줄여 갑니다.

4. Q-Learning 업데이트 규칙

Q-Learning은 "상태 $s$에서 행동 $a$를 했을 때 앞으로 받을 총 보상의 추정값" $Q(s, a)$을 학습합니다.

$$ Q(s, a) \leftarrow Q(s, a) + \alpha\Big[\,r + \gamma\,\max_{a'} Q(s', a') - Q(s, a)\,\Big] $$

$\alpha$: 학습률 (0.01–0.3), $\gamma$: 할인율 (0.9–0.99). 정책은 $\pi(s) = \arg\max_a Q(s, a)$.

5. 인터랙티브: ε-greedy 탐험 시각화

🎮 인터랙티브: ε에 따른 행동 분포

두 슬롯머신 A, B 중에 진짜 보상은 B가 더 큽니다. ε 값에 따라 에이전트의 행동 분포와 누적 보상이 어떻게 바뀌는지 보세요.

ε이 0이면 처음 운 좋은 슬롯에 갇혀 진짜 좋은 슬롯을 못 찾고, 1이면 늘 무작위라 학습 의미가 없어요. 0.05–0.2 정도가 표준 출발점.

6. 인터랙티브: 그리드월드 Q-Learning

🎮 인터랙티브: 5×5 그리드월드 학습

좌측 하단(녹색)에서 우측 상단(빨강)으로 가야 합니다. 가운데 회색은 함정(보상 -5). 매 스텝 -0.1, 도달 +10. 학습 시작을 누르면 Q-table이 갱신되고, 셀 화살표가 점차 최적 경로 쪽으로 정렬됩니다.

처음에는 화살표가 무작위 방향이지만, 수십~수백 에피소드 후에 함정을 피해 목표로 향하는 일관된 패턴이 만들어집니다. γ를 너무 낮추면 멀리 있는 +10 보상이 감지되지 않아 학습이 더뎌요.

7. 인터랙티브: 보상 설계의 함정

🎮 인터랙티브: 잘못 설계된 보상은 이상한 행동을 학습한다

같은 그리드월드에 보상 설계만 다르게 줍니다. 어느 보상 설계가 "원하는 행동"을 만들어내는지 비교해 보세요.

"속도만 +"는 도달 보상을 추구하긴 하지만 함정 패널티가 없어 종종 함정 옆을 무심히 지나갑니다. "희박" 보상은 신호가 너무 적어 정책이 잘 안 잡히고, 도착까지 무작위에 가까운 행동이 남기 쉽습니다. 보상 설계는 RL 성패의 절반이에요.

8. Deep RL과 자율주행

Q-table은 상태가 적은 그리드에서만 동작합니다. 자율주행처럼 상태가 $(x, y, \psi, v, \ldots)$ 연속이고 행동도 연속이면 신경망으로 $Q(s,a)$ 또는 $\pi(s)$를 근사하는 Deep RL을 씁니다.

알고리즘특징자율주행 적합도
Q-Learning테이블, 이산 행동저차원 그리드만
DQNCNN으로 Q, 이산 행동비디오 게임에 강함
DDPG / TD3연속 행동, off-policy★ 자율주행 연구
PPO안정적 학습, 산업 표준★★ 자율주행/로봇
SAC샘플 효율 높음★★ 시뮬→실차 전이
실차에 그대로 못 쓰는 이유 — Deep RL은 보통 수백만 에피소드의 시뮬이 필요하고, 안전이 보장되지 않습니다. 실무에서는 Sim-to-Real 전이 + 안전 필터(W12 CBF) + 사람 감독의 조합이 표준이고, RL은 행동 선택(Behavior layer)이나 보조 정책에 주로 쓰입니다.

둘은 경쟁이 아니라 역할 분담입니다. 안쪽 추종 루프는 PID·MPC가, 차선 변경/양보 같은 의사결정은 RL이, 안전 보장은 CBF가 — 이게 2026년 현대 자율주행의 표준 구조입니다.

9b. 자율주행 RL 의 진짜 장벽

1. 시뮬→실차 갭 (Sim-to-Real Gap)

시뮬에서 완벽한 정책이 실차에서 무력해지는 이유:

  • 시뮬의 타이어·공기저항 모델 단순 → 실차는 비선형 강함
  • 센서 노이즈, 지연(latency), 통신 지터 → 시뮬에선 부재
  • 도로 마찰, 풍향, 비/눈 → 도메인 시프트

대응: Domain Randomization. 시뮬 학습 시 파라미터를 무작위로 흔들어 정책이 다양한 환경에 robust하게.

2. 보상 해킹 (Reward Hacking)

RL 에이전트는 "보상을 최대화"만 합니다. "안전하게 잘 운전"이 아닙니다. 예시:

  • "빨리 도착하면 +" → 신호 무시, 과속 학습
  • "차선 유지하면 +" → 출발 자체를 안 함 (안전한 정지)
  • "가속도 작게 -" → 후진하며 천천히 도착

대응: 다중 목표 보상 설계 + 인간 피드백 (RLHF).

3. 검증 불가능성

"이 정책이 어떤 상황에서도 안전한가?" 를 수학적으로 증명할 수 없음. 100만 km 시뮬해도 "그 다음 1 km" 가 안전하다는 보장 X.

대응: Shielded RL — 학습된 정책 출력에 CBF 같은 안전 필터를 강제로 통과시킴.

9c. Imitation Learning — 전문가를 모방하라

순수 RL의 학습 시간(수백만 에피소드)을 단축하는 방법: 전문가 행동을 직접 학습.

Behavioral Cloning (BC)

전문가 (사람 운전자) 의 (state, action) 쌍을 모아 지도학습:

$$ \min_\theta \sum_i \|\pi_\theta(s_i) - a_i^\text{expert}\|^2 $$
  • 장점: 빠른 학습, 안전 (전문가 데이터)
  • 단점: 분포 시프트 — 전문가가 가본 적 없는 상태에서 무력 (compounding error)

DAgger (Dataset Aggregation)

BC로 학습 → 시뮬 돌림 → 학습 정책이 실수한 곳에서 전문가 데이터 추가 수집 → 재학습. 분포 시프트 점진 해결.

RL + IL 조합

현실적 패턴: IL로 초기 정책 학습 + RL로 미세조정. Wayve, Waymo 모두 이 형태로 자율주행 정책을 학습.

9e. 손풀이 — RL 개념 점검

Q1. 보상 설계

"교차로에서 왼쪽 회전" 행동을 학습시키려 한다. 다음 중 가장 좋은 보상은?

  1. 왼쪽으로 갔으면 +1, 그 외 0
  2. 매 스텝 -0.1, 회전 완료 시 +10, 사고 시 -100
  3. 회전 중 부드럽게 가면 +1
  4. 모든 행동 +0.01
풀이

(b). (a)는 너무 sparse, (c)는 모호, (d)는 신호 부족.
(b)는 ① 빠르게 (매스텝 페널티) ② 목표 (도달 보너스) ③ 안전 (사고 페널티) 모두 포함 → 균형 잡힌 설계.

Q2. 탐험 vs 활용

$\varepsilon$-greedy에서 $\varepsilon$=0.05로 너무 작게 시작하면?

풀이

탐험 부족 → 처음 발견한 평범한 정책에 갇힘 (지역 최적). 더 좋은 경로가 있어도 못 찾음.
권장: $\varepsilon$=0.3~0.5에서 시작 → 점차 0.01까지 감소 (annealing).

Q3. Q-Learning 한 스텝

$Q(s,a) = 5$, 실제 받은 $r=2$, 다음 상태 최대 Q $= 4$, $\alpha=0.1$, $\gamma=0.9$. 업데이트 후 $Q(s,a)$는?

풀이

$Q \leftarrow 5 + 0.1 \times (2 + 0.9 \times 4 - 5) = 5 + 0.1 \times 0.6 = 5.06$.

10. 핵심 정리

  • RL = 경험으로 정책을 학습. 모델 없이도 잘 작동하는 강력한 도구.
  • 4가지 요소 (S, A, R, π)와 ε-greedy 탐험은 RL 책의 1장에 모두 들어 있음.
  • Q-Learning의 한 줄: $Q(s,a) \leftarrow Q(s,a) + \alpha[r + \gamma\max_{a'}Q(s',a') - Q(s,a)]$.
  • 보상 설계가 절반 — 잘못 짜면 엉뚱한 행동을 학습한다.
  • 자율주행에서는 Deep RL을 단독으로 쓰기보다 고전 제어 + 안전 필터 + RL 의사결정 형태로 결합.

10c. 실무 자율주행 RL 사례

Wayve (영국)

"End-to-End Learning" — 카메라 → 조향·속도 직접 예측. RL + IL 조합. 런던 도심 시범 운행 중.

Waymo

주로 모방 학습 (전문가 운전자 데이터) 기반. RL은 합류·차선변경 같은 의사결정 보조에 사용.

Tesla FSD

RL 기반 행동 결정 + MPC 추종. 양자 결합으로 안전성 확보.

10e. RL 의 미래 — 2025+ 트렌드

  • Foundation Model + RL: GPT 같은 대형 모델이 상황 이해, RL이 행동 결정
  • World Model: 환경 모델을 학습 후 내부 시뮬로 빠른 학습 (Dreamer 시리즈)
  • Multi-agent RL: 여러 차량이 협력 — 합류·교차로 협상
  • Diffusion Policy: 확산 모델 기반 정책 (행동 생성)
  • Offline RL: 실차 로그 만 가지고 학습 (시뮬 불필요)

이 모든 것이 "안전 제약 (CBF) + 강건성 + 인간 피드백" 위에서 발전 중. 2030년쯤엔 자율주행 표준 스택이 될 가능성.
