Week 5. 수신지평선 MPC

CFTOC를 한 번 푸는 것은 오프라인 계획일 뿐. 매 샘플마다 다시 푸는 것이 진짜 MPC입니다. 이번 주는 수신지평선 전략의 의미, 종단 비용·종단 제약의 역할, 그리고 폐루프 동작을 직접 시뮬레이션합니다.

목차

  1. 왜 수신지평선인가 — 폐루프의 회복
  2. MPC 알고리즘 (수도코드)
  3. 종단 비용 $P$ — LQR과 만나는 다리
  4. 종단 제약 $\mathcal X_f$ — 안정성 보장의 핵심
  5. 워밍 스타트 — 빠른 수렴의 비결
  6. 참조 추적 — 원점 안정화에서 trajectory 추적으로

1. 왜 수신지평선인가

한 번 풀고 끝내면 안 되나?

$N$ 스텝 입력 시퀀스 $U^* = [u_0^*, \ldots, u_{N-1}^*]$을 한 번 풀고 그대로 적용하면 어떨까? 답: 외란이나 모델 오차에 무방비. 한 번이라도 예측이 빗나가면 그 후 입력 시퀀스는 잘못된 상태에 대한 답입니다. 이는 사실상 개루프 제어예요.

해법 — 첫 입력만 적용, 다시 풀기

매 샘플 $t$마다:

  1. 현재 상태 $x(t)$ 측정 (또는 추정)
  2. CFTOC를 풀어 $U^*_t = [u_t^*, u_{t+1}^*, \ldots, u_{t+N-1}^*]$ 얻음
  3. 첫 입력 $u_t^*$만 시스템에 적용
  4. 한 샘플 후 새 측정값으로 1번부터 다시

이렇게 풀고 버리고 다시 풀고 버리는 반복이 폐루프 효과를 만듭니다. 측정값이 매 스텝 다시 들어오니까요. "낭비처럼 보이지만 강건성의 원천"이라는 점이 핵심.

비유 — 자동차로 1시간 거리를 운전할 때, 떠나기 전에 1시간 분량의 핸들·엑셀 시퀀스를 미리 다 정해놓고 그대로 따라가지 않습니다. 매 순간 "지금 어디에 있고, 앞에 무엇이 있나"를 보고 다시 결정하죠. 그게 수신지평선.

3. 종단 비용 $P$ — LQR과 만나는 다리

유한 호라이즌 비용은 $\sum_{k=0}^{N-1}(\cdot) + x_N^\top P x_N$. 마지막 항 $x_N^\top P x_N$이 종단 비용입니다. "$N$ 스텝 후의 상태에 얼마나 벌점을 줄까."

$P$를 어떻게 고를까?

  • $P = Q$ — 가장 단순. "마지막도 그냥 한 스텝처럼 취급". 호라이즌이 짧으면 근시안 위험.
  • $P = $ DARE 해 — 무한 호라이즌 LQR의 최적 비용. "$N$ 이후도 LQR로 마저 풀었을 때의 비용"을 정확히 반영. 표준 권장.
  • $P \succ Q$ 큰 값 — 종단 상태를 강하게 0 근처로 강제. 안정성 보장에 보수적이지만 가용성 영역이 좁아짐.

$P = $ DARE인 경우의 의미: 만약 $N$이 충분히 길고 종단에서 제약이 활성화되지 않는다면, 유한 호라이즌 MPC와 무한 호라이즌 LQR이 같은 입력을 냅니다. 즉 MPC는 "제약이 있으면 LQR을 약간 보정"한 결과.

4. 종단 제약 $\mathcal X_f$ — 안정성 보장의 핵심

실현가능 집합 안에서 시작해도, MPC가 폐루프 동안 항상 풀이 가능하다는 보장은 자동으로 안 됩니다. 표준 해법: 종단 집합 $\mathcal X_f$를 LQR이 $\mathcal X, \mathcal U$를 만족하는 제어 불변 집합 (control invariant set)으로 두기.

"제어 불변": $x \in \mathcal X_f$이면 LQR 입력 $u = -K x$로 $A x + B u \in \mathcal X_f$이고 $-Kx \in \mathcal U$. 즉 한 번 들어가면 안에서 LQR로 무한히 머물 수 있는 영역.

이 종단 집합을 강제하면 재귀적 가용성(recursive feasibility)과 폐루프 안정성이 보장됩니다 (W6 주제).

5. 워밍 스타트 (Warm Start)

한 스텝 풀 때 이전 스텝 답을 시작점으로 쓰면 같은 QP가 훨씬 빨리 풉니다. "한 칸 미끄러뜨리기":

$$ U_t^{\text{init}} = [u_{t}^*|_{t-1},\ u_{t+1}^*|_{t-1},\ \ldots,\ u_{t+N-2}^*|_{t-1},\ u_{N-1}^{\text{aux}}] $$

마지막 $u_{N-1}^{\text{aux}}$는 보통 LQR 게인으로 채움. OSQP 등 ADMM 솔버는 워밍 스타트로 5–10배 가속됩니다 — 임베디드 실시간 MPC의 표준.

6. 참조 추적 — 원점에서 trajectory로

지금까지는 "$x \to 0$ 안정화"였지만 실제 응용은 보통 "$x \to r$ 추적" 또는 시변 참조 trajectory. 변환은 단순합니다 — 오차 $\tilde x = x - x_r$, 오차 입력 $\tilde u = u - u_r$로 재정의하면 정확히 같은 형태.

여기서 $(x_r, u_r)$은 "정상상태 평형점"으로 $x_r = A x_r + B u_r$, $r = C x_r$를 만족. 이는 참조에 대응하는 정상 상태/입력 페어를 한 번 풀어두면 됨.

🎮 인터랙티브: 폐루프 MPC vs 개루프 — 외란 회복

두 컨트롤러가 같은 시스템을 같은 목표로 보냅니다. 중간에 외란(빨간 화살표)이 들어오면, 폐루프 MPC는 즉시 새 계획으로 복구하지만 개루프는 미리 짜둔 시퀀스를 그대로 적용해 회복 못 해요.

7. 핵심 정리

  • 매 샘플마다 측정 → CFTOC 풀이 → 첫 입력 적용 → 반복.
  • 이 반복이 폐루프를 만들고 외란·모델 오차에 강건성을 부여.
  • 종단 비용 $P$는 LQR DARE 해, 종단 제약 $\mathcal X_f$는 LQR 불변 집합이 표준.
  • 워밍 스타트로 같은 QP를 5–10배 빨리 풂 — 임베디드 핵심.
  • 참조 추적은 오차 좌표 변환으로 같은 형식.
