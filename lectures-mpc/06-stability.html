Week 6. 안정성과 가용성

"MPC를 돌리면 시스템이 정말 0으로 수렴하는가?", "어떤 시점에서 풀이가 갑자기 실패하지 않는가?" — 두 질문에 답하는 핵심 도구가 Lyapunov 함수와 제어 불변 집합입니다. 종단 비용·종단 제약을 어떻게 설계해야 폐루프가 안정·실현 가능한지 이론적으로 보장하는 방법.

목차

  1. 두 질문 — 안정성과 재귀적 가용성
  2. Lyapunov 함수 — "에너지가 줄어든다"
  3. 제어 불변 집합 (Control Invariant Set)
  4. 표준 안정성 정리 — 3대 가정
  5. 종단 집합 $\mathcal X_f$ 계산법
  6. 가용성을 잃을 때의 처방 — 소프트 제약

1. 두 질문

안정성 (Closed-Loop Stability)

MPC 제어 하의 폐루프 시스템 $x_{k+1} = A x_k + B u_k^{\text{MPC}}(x_k)$가 원점으로 점근 수렴(asymptotic stability)할까? PID는 게인 마진·위상 마진으로 답하지만 MPC는 매 스텝 다른 입력을 내므로 직접 분석이 어렵습니다.

재귀적 가용성 (Recursive Feasibility)

$x_t$에서 CFTOC가 풀렸다면, 다음 시점 $x_{t+1} = A x_t + B u_t^*$에서도 풀이 가능할까? 만약 어느 순간 풀이가 실패하면 컨트롤러가 멈춰 버립니다 — 안전이 걸린 시스템에선 치명적.

두 질문은 깊이 연결되어 있습니다. 표준 결과는 적절한 종단 비용·종단 제약 설계로 둘 다 동시에 보장됨.

2. Lyapunov 함수 — "에너지가 줄어든다"

아이디어는 단순합니다. 시스템에 "에너지" 같은 양 $V(x)$가 있고, 그것이 매 스텝 줄어들기만 한다면 결국 0(또는 가장 낮은 점)으로 수렴할 수밖에 없습니다.

이산시간 Lyapunov 정리: 다음 조건을 만족하는 함수 $V: \mathbb R^n \to \mathbb R_{\ge 0}$이 존재하면 폐루프가 점근 안정.

  1. $V(0) = 0$, $V(x) > 0$ for $x \ne 0$ (양정치)
  2. $V(A x + B \kappa(x)) - V(x) \le -\alpha \|x\|^2$ ($\alpha > 0$, 매 스텝 감소)

$\kappa(x)$는 "x에서 적용되는 입력 정책". MPC라면 $\kappa = u_0^*$ (CFTOC 첫 입력).

Lyapunov 함수 후보 — 최적 비용 자체

핵심 통찰: MPC의 최적 비용 함수 $V^*(x) = J^*(x)$를 그대로 Lyapunov 후보로 쓴다. 이 비용이 매 스텝 줄어든다면 안정성이 자동으로 따라옴.

3. 제어 불변 집합 (Control Invariant Set)

집합 $\mathcal X_f \subseteq \mathcal X$가 제어 불변이라는 건:

$$ \forall x \in \mathcal X_f,\ \exists u \in \mathcal U:\ A x + B u \in \mathcal X_f $$

풀어 말하면 "이 집합 안에서 시작하면, 입력 제약 안에서 입력을 골라 다음 스텝도 이 집합 안에 머물 수 있다". MPC의 종단 제약 $x_N \in \mathcal X_f$는 정확히 이 성질을 활용합니다.

불변 집합 위계

  • 최대 제어 불변 집합 $\mathcal C_\infty$ — 가능한 가장 큰 제어 불변 집합. 이론적 한계.
  • 최대 양의 불변 집합 $\mathcal O_\infty$ — 고정된 입력 정책(예: LQR) 하에서 머무는 가장 큰 집합. 계산이 더 쉬움.
  • $\lambda$-수축 집합 — 매 스텝 $\lambda < 1$ 비율로 줄어드는 집합. 강한 안정성 보장.

실무에서 가장 흔한 선택: LQR 양의 불변 집합 — LQR 입력 $u = -Kx$로 폐루프를 만들었을 때 $(\mathcal X, \mathcal U)$ 안에 머무는 가장 큰 집합. MPT3 (MATLAB), polytope (Python) 같은 다면체 라이브러리로 계산.

4. 표준 안정성 정리 — 3대 가정

Mayne·Rawlings·Rao·Scokaert의 2000년 논문이 정리한 표준 결과 ("Constrained model predictive control: stability and optimality"):

가정:

  1. $\mathcal X_f \subseteq \mathcal X$는 제어 불변, $0 \in \mathcal X_f$.
  2. $\mathcal X_f$ 안에서 작동하는 종단 제어 정책 $\kappa_f$가 존재 ($u = \kappa_f(x) \in \mathcal U$, $A x + B \kappa_f(x) \in \mathcal X_f$).
  3. 종단 비용 $P$가 다음 부등식을 만족: $$ V_f(A x + B \kappa_f(x)) - V_f(x) \le -\ell(x, \kappa_f(x)),\quad \forall x \in \mathcal X_f $$ 여기서 $V_f(x) = x^\top P x$, $\ell(x,u) = x^\top Q x + u^\top R u$. 즉 "종단 안에서는 종단 비용이 단계 비용 이상 줄어듬".

이 세 가정이 만족되면 다음이 모두 성립:

  • 재귀적 가용성: $x_0 \in \mathcal X_0$이면 모든 $t \ge 0$에서 CFTOC가 풀이 가능.
  • 점근 안정성: $x_t \to 0$ as $t \to \infty$.
  • 최적성 부등식: $V^*(x_{t+1}) \le V^*(x_t) - \ell(x_t, u_t^*)$.

증명 스케치

핵심 트릭은 "수신지평선의 한 칸 미끄러뜨린 입력 시퀀스"가 가용성을 유지함을 보이는 것:

$t$에서의 최적 시퀀스 $U_t^* = (u_0^*, u_1^*, \ldots, u_{N-1}^*)$를 한 칸 밀고 마지막에 $\kappa_f(x_N^*)$를 붙입니다:

$$ \tilde U_{t+1} = (u_1^*, u_2^*, \ldots, u_{N-1}^*, \kappa_f(x_N^*)) $$

가정 1, 2에 의해 $\tilde U_{t+1}$은 가용 (모든 제약 만족). 따라서 $V^*(x_{t+1}) \le J(\tilde U_{t+1})$. 가정 3을 활용해 정리하면 $V^*$가 $\ell$ 만큼 감소 — Lyapunov.

5. 종단 집합 $\mathcal X_f$ 계산법

가장 일반적 방법: LQR 양의 불변 집합 + 다면체 표현.

  1. 무한 호라이즌 LQR 해 $K = $ DARE 계산.
  2. 폐루프 $A_K = A - BK$의 Lyapunov 함수 $V_f(x) = x^\top P x$ ($P$는 DARE 해).
  3. 다면체 $\Omega_\alpha = \{x \mid x^\top P x \le \alpha\}$ 중에서 $A_K^k x \in \mathcal X$, $-K A_K^k x \in \mathcal U$ ($\forall k \ge 0$)을 만족하는 가장 큰 $\alpha$ 찾기.
  4. $\mathcal X_f = \Omega_\alpha$.

실용 버전 — 더 단순한 박스 또는 LMI 기반 접근:

  • 박스 종단 집합: $\mathcal X_f = \{x \mid \|x\|_\infty \le \epsilon\}$, 작게 잡으면 손쉽지만 가용성 영역이 좁아짐.
  • LMI 접근: $\Omega = \{x \mid x^\top P x \le 1\}$ 형태로 두고 $P$를 LMI로 결정. cvxpy로 한 번 풀면 끝.
  • MPT3: 입력 시 자동으로 polyhedral 불변 집합 계산.

6. 가용성을 잃을 때 — 소프트 제약

실제 시스템에서 외란이 너무 크거나 모델 오차가 누적되면 $x_t \notin \mathcal X_0$가 될 수 있습니다. 이때 CFTOC는 풀이 불가능. 두 처방:

소프트 제약 (Slack Variables)

제약을 절대 위반 못 하는 "하드"에서 큰 벌점만 받는 "소프트"로 완화:

$$ x_k \in \mathcal X \quad\Rightarrow\quad H_x x_k \le h_x + s_k,\ s_k \ge 0 $$

비용에 $\rho \|s_k\|_1$ 추가 ($\rho$ 매우 큼). 가능하면 $s = 0$으로 두지만, 불가능하면 최소한으로 위반. 이렇게 하면 풀이는 항상 가능합니다.

다만 안전 제약(충돌, 최대 압력)은 절대 소프트하게 만들면 안 됨. 입력 제약과 성능 제약만 소프트하게.

회복 모드

풀이 실패 시 안전한 기본 컨트롤러(예: LQR 또는 fixed safe action)로 fallback. 산업용 MPC에선 표준.

🎮 인터랙티브: Lyapunov 함수 감소

2D 시스템의 폐루프 궤적과 Lyapunov 함수 $V(x) = x^\top P x$를 봅니다. 궤적이 어떤 등고선을 가로지를 때마다 $V$가 줄어드는 게 보여요. $Q/R$ 비율을 바꾸면 등고선과 수렴 속도가 동시에 변합니다.

7. 핵심 정리

  • MPC의 안정성은 자동 보장되지 않음 → 종단 비용 + 종단 제약으로 설계.
  • 3대 가정: $\mathcal X_f$ 제어 불변, 종단 정책 존재, 종단 비용이 단계 비용 이상 감소.
  • 표준 선택: $\mathcal X_f$ = LQR 양의 불변 집합, $V_f = x^\top P x$ ($P$ = DARE 해).
  • Lyapunov 후보로 최적 비용 $V^*$ 자체를 사용 — "한 칸 미끄러뜨리기" 트릭으로 감소 입증.
  • 외란·오차로 가용성 잃을 위험에는 소프트 제약(slack)으로 대응 — 단, 안전 제약은 하드 유지.

📚 깊이 1 — 안정성의 의미

MPC가 "안정"이라 함은 닫힌 루프 시스템 $x_{k+1} = Ax_k + Bu_0^*(x_k)$의 점근 안정성을 의미합니다. 즉 $x_k \to 0$ (또는 reference로).

  • Lyapunov 안정: $\|x_k\|$가 영구히 작은 영역 안.
  • 점근 안정: $x_k \to x^*$.
  • 지수 안정: $\|x_k\| \le c\rho^k\|x_0\|$, $0 \le \rho < 1$.

📚 깊이 2 — Recursive Feasibility

안정성의 전제. 한 시점 feasible이면 다음 시점도 feasible해야 함.

증명 트릭 (shifted solution): 시점 $k$의 최적해 $u_0^*, u_1^*, \dots, u_{N-1}^*$, $x_N^*$. 시점 $k+1$의 후보 입력:

$$\tilde u = (u_1^*, u_2^*, \dots, u_{N-1}^*, K_f x_N^*)$$

마지막을 종단 제어 $K_f$로 채움. $\mathcal X_f$가 $K_f$에 대해 control invariant이면 끝점도 $\mathcal X_f$ 안 → 다음 시점 feasible.

📚 깊이 3 — Lyapunov 안정성 증명

최적 비용 함수 $V^*(x)$를 Lyapunov 함수 후보로 사용.

$$V^*(x_{k+1}) - V^*(x_k) \le \tilde J - V^*(x_k)$$

여기서 $\tilde J$는 shifted solution의 비용. 계산해보면:

$$\tilde J = V^*(x_k) - l(x_k, u_0^*) - V_f(x_N^*) + l(x_N^*, K_f x_N^*) + V_f(A_K x_N^*)$$

여기서 $A_K = A - BK_f$. 안정성 조건:

$$V_f(A_K x) - V_f(x) + l(x, K_f x) \le 0,\ \forall x \in \mathcal X_f$$

즉 종단 비용 + 종단 제어가 $\mathcal X_f$ 안에서 자체로 Lyapunov 함수가 되어야 함. 보통 $V_f = $ LQR DARE 해, $K_f = $ LQR 게인이면 자동 만족.

📚 깊이 4 — 안정성 보장 4단계 표준 트릭

  1. 종단 비용 $V_f(x) = x^TP_fx$, $P_f$ = LQR DARE 해.
  2. 종단 제어 $K_f$ = LQR 게인 (옵션).
  3. 종단 제약 $x_N \in \mathcal X_f$, $\mathcal X_f$ = invariant set 또는 origin.
  4. 가관측 가정 — Lyapunov 함수 양정값 보장.

📚 깊이 5 — 종단 제약이 없는 경우 (Quasi-Infinite Horizon)

종단 제약 $x_N \in \mathcal X_f$를 강제하면 feasibility 영역(domain of attraction)이 줄어듦. 종단 제약 없이도 안정 보장 방법:

  • 호라이즌 $N$을 충분히 길게.
  • $V_f$를 충분히 크게.
  • Chen-Allgöwer 1998 — Quasi-infinite horizon NMPC.

📚 깊이 6 — Domain of Attraction

MPC가 안정적으로 동작하는 초기 상태들의 집합. 종단 제약 → 시작점에서 N 스텝 안에 $\mathcal X_f$ 도달 가능한 상태. 호라이즌이 클수록 영역 큼.

📚 깊이 7 — Stability vs Performance Trade-off

  • 큰 $V_f$: 안정성 강화, 보수적 제어.
  • 작은 $V_f$: 적극적 제어, 안정성 약함.
  • 긴 $N$: 안정성 + 성능 모두 좋음, 계산량↑.
  • 짧은 $N$: 빠른 계산, 안정성 위험.

📚 깊이 8 — Suboptimal MPC의 안정성

실시간 시간 부족으로 정확한 최적해 대신 근사해 사용해도 안정 가능?

핵심 조건: 다음 부등식 만족하는 근사해 $\tilde u$:

$$V(x_{k+1}, \tilde u) - V(x_k, u^*_{k-1}) \le -l(x_k, \tilde u_0)$$

즉 비용이 stage cost만큼 감소. Diehl, Findeisen 등의 연구로 RTI(Real-Time Iteration)도 안정 가능.

📚 깊이 9 — 종단 영역 $\mathcal X_f$ 계산

일반적인 방법:

  1. LQR 닫힌 루프 $A_K = A - BK_f$ 안정 확인.
  2. Lyapunov 부등식 $A_K^T P A_K - P \prec 0$ 만족 $P$ 계산 (또는 DARE 해).
  3. $\mathcal X_f = \{x : x^TPx \le \alpha\}$ ellipsoidal invariant set.
  4. $\alpha$를 입력·상태 제약 만족하도록 결정.

📚 깊이 10 — Maximal Positively Invariant Set

$K_f$ 사용 시 무한히 안전 영역에 머물 수 있는 최대 상태 집합. MPT(Multi-Parametric Toolbox)로 자동 계산.

📚 깊이 11 — 안정성과 모델 오차

Nominal MPC의 안정성 증명은 정확한 모델 가정. 모델 오차가 있으면:

  • 점근 안정 → 한정 영역 안에서 진동 (Input-to-State Stability, ISS).
  • 외란 한계가 클수록 한정 영역 큼.
  • 견실 MPC (W7)에서 명시적 처리.

📚 깊이 12 — Tracking 안정성

정상상태 추종 시 안정성:

  • 참조점 $(x_{ref}, u_{ref})$가 평형점 ($Ax_{ref} + Bu_{ref} = x_{ref}$).
  • $\tilde x = x - x_{ref}$로 변환 후 표준 분석.
  • 참조 변경 시 transient 안정 분석은 더 복잡.

📚 깊이 13 — Soft 제약과 안정성

Soft 제약 도입 시 infeasibility는 사라지지만 안정성 분석은 추가 주의 필요. 큰 페널티 $\rho$로 hard 제약 근사 → 안정성 보존.

📚 깊이 14 — 안정성 인증의 표준 정리 (Mayne 등 2000)

다음 4조건 만족 시 RHC 점근 안정 + recursive feasible:

  1. $\mathcal X_f \subseteq \mathbb X$ (안전 영역 포함).
  2. $K_f x \in \mathbb U$ for $x \in \mathcal X_f$ (입력 제약 만족).
  3. $A_K \mathcal X_f \subseteq \mathcal X_f$ ($\mathcal X_f$ 정positive invariant).
  4. $V_f(A_K x) - V_f(x) + l(x, K_f x) \le 0\ \forall x \in \mathcal X_f$ (Lyapunov 감소).

이게 MPC 안정성 이론의 핵심 정리.

📚 깊이 15 — 흔한 오답

  1. 종단 비용 무시 → 짧은 호라이즌에서 불안정.
  2. 종단 제약 미설정 → recursive feasibility 깨짐.
  3. $P_f$를 임의로 잡음 (LQR DARE 해 권장).
  4. 가관측 조건 미확인.
  5. Soft 제약의 페널티 너무 작아 제약 위반.

📚 깊이 17 — FAQ

Q1. 종단 제약 없이도 안정 가능?

Quasi-infinite horizon (Chen-Allgöwer)으로 가능. 호라이즌 길게 + 큰 $V_f$.

Q2. Soft 제약은 안정성 깨나?

아닙니다. 큰 페널티이면 hard 제약 근사. 단 분석은 더 정교.

Q3. 비선형 MPC는?

같은 원리이지만 종단 영역 계산이 어려움. NMPC 안정성 이론은 더 복잡 (W8).
