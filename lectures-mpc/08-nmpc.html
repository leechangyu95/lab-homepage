Week 8. 비선형 MPC (NMPC)

로봇 매니퓰레이터·드론·자율차의 동역학은 본질적으로 비선형입니다. 선형화로는 큰 동작 범위를 다루기 어려워, 비선형 모델을 그대로 쓰는 NMPC가 표준이 됐어요. QP가 NLP가 되고, SQP·Real-Time Iteration 같은 영리한 트릭들로 ms 단위에 풀어냅니다.

목차

  1. 왜 NMPC인가 — 선형화의 한계
  2. NLP 정식화
  3. Direct vs Indirect — 두 학파
  4. SQP — 비선형을 QP로 잘게 풀기
  5. Real-Time Iteration (RTI)
  6. iLQR — 큰 문제를 위한 그래디언트 기반

1. 왜 NMPC인가

선형 MPC는 동작점 근처의 작은 영역에선 잘 작동하지만:

  • 로봇 팔이 0°에서 90°까지 큰 각도로 움직이면 자코비안이 크게 변함 → 선형화 오차 ↑
  • 드론의 자세 제어는 회전 행렬 자체가 비선형 manifold
  • 차량의 횡방향 동역학은 슬립 각도, 타이어 비선형성으로 dominant
  • 화학 공정의 Arrhenius 항 ($e^{-E/RT}$) 같은 강한 비선형

여러 동작점에서 선형화한 모델을 갈아끼우는 gain-scheduling이나 LPV-MPC도 있지만, 비선형 모델을 그대로 들고 가는 NMPC가 가장 일반적이고 정확.

2. NLP 정식화

비선형 시스템 $x_{k+1} = f(x_k, u_k)$, 비용·제약도 비선형 가능:

$$ \min_{X, U}\ \sum_{k=0}^{N-1} \ell(x_k, u_k) + V_f(x_N) $$ $$ \text{s.t.}\quad x_{k+1} = f(x_k, u_k) $$ $$ g(x_k, u_k) \le 0,\quad x_0 = x(t),\quad x_N \in \mathcal X_f $$

이건 일반 비선형 계획법 (Nonlinear Programming, NLP). QP보다 훨씬 어렵습니다 (비볼록 가능, 지역 최솟값).

3. Direct vs Indirect

Indirect (변분법 기반)

최적성 조건 (Pontryagin Maximum Principle, KKT)을 미분방정식으로 유도해 풀음. 수학적으로 우아하나 수치적으로 까다로움.

Direct (이산화 후 유한차원 NLP)

시간을 이산화하고 직접 NLP 솔버에 넘김. 현재 표준. 두 가지 sub-방법:

  • Single Shooting — 입력 $U$만 결정 변수, 상태는 forward simulation으로 계산. 변수 적지만 수치적으로 민감.
  • Multiple Shooting — 상태와 입력 모두 결정 변수, 동역학을 등식 제약으로. 변수 많지만 sparse + 수치적으로 안정 + 병렬화 가능. 현재 표준.
  • Collocation — 다항식 근사로 동역학을 더 정확히. 화학 공정에서 인기.

4. SQP — Sequential Quadratic Programming

NLP를 푸는 가장 흔한 방법. 아이디어: 현재 추정 $z_k$ 주변에서 NLP를 QP로 근사 → 풀이 → 업데이트 → 반복.

한 SQP 반복

  1. 현재 $z_k$에서 비용·제약을 1·2차 Taylor 전개: $$ \min_{\Delta z}\ \nabla f(z_k)^\top \Delta z + \tfrac{1}{2} \Delta z^\top H \Delta z $$ $$ \text{s.t.}\ \nabla c(z_k)^\top \Delta z + c(z_k) = 0,\ \nabla g(z_k)^\top \Delta z + g(z_k) \le 0 $$ ($H$는 Lagrangian의 Hessian 또는 quasi-Newton 근사 BFGS).
  2. 이 QP를 풀어 step $\Delta z$.
  3. $z_{k+1} = z_k + \alpha \Delta z$ ($\alpha$는 line search).
  4. 수렴 검사 후 반복.

각 반복이 QP 한 번 풀이. 따라서 NLP 한 번 = QP 여러 번. 보통 5~20 반복.

대표 솔버

  • IPOPT — interior-point. 대규모 문제에 강함. CasADi의 기본.
  • SNOPT — active-set SQP. 상용. 큰 변화 추적 빠름.
  • fmincon (MATLAB) — 범용.
  • FORCES Pro / acados — 임베디드 NMPC 특화.

5. Real-Time Iteration (RTI)

실시간 NMPC의 핵심 트릭. NLP를 매 샘플마다 완전히 풀지 않고 SQP를 한 번만 반복. Diehl 등(2002)이 개발.

아이디어

  • 이전 샘플에서 풀어둔 해를 워밍 스타트로 사용.
  • 현재 샘플에서는 SQP를 한 번만 돌려 step $\Delta z$를 얻고 그 게 새 입력.
  • 다음 샘플에서 또 한 번 SQP.
  • 샘플 간격이 짧으면 시스템이 천천히 변하니, 한 번씩 SQP만 해도 점근적으로 수렴.

장점

  • 예측 가능한 계산 시간 (한 번의 QP 풀이만).
  • 임베디드에 적합 — ms 단위로 NMPC 실현.
  • ACADO Toolkit, acados의 표준 모드.

단점

큰 외란이 들어와 상태가 갑자기 멀어지면 한 번의 SQP로 따라잡기 부족. 그럴 때만 다회 반복으로 fallback.

6. iLQR — 큰 문제를 위한 그래디언트 기반

iLQR (iterative LQR)은 NMPC의 또 다른 인기 접근. 로봇 운동 계획에서 표준.

아이디어

현재 trajectory 주변에서 동역학을 선형화 + 비용을 이차근사 → 시변 LQR 풀이 → 새 trajectory → 반복. SQP의 단순화 버전으로 볼 수도 있음.

한 반복

  1. Forward pass: 현재 입력 시퀀스로 trajectory 시뮬레이션.
  2. Backward pass: 시간을 거꾸로 돌면서 LQR Riccati 재귀로 게인 $K_k$, $k_k$ 계산.
  3. Forward rollout: 새 게인으로 다시 시뮬레이션.
  4. 비용 감소 확인하고 line search.

장점: 제약 처리는 부족하지만 큰 N (수백 스텝)에서도 매우 빠름. 로봇 매니퓰레이션, 사족 보행에서 "DDP / iLQR"이 표준.

제약 추가 버전: Constrained iLQR, AL-iLQR (Augmented Lagrangian).

🎮 인터랙티브: 비선형 진자 NMPC

비선형 진자 $\ddot\theta = -\sin\theta + u$를 $\theta = 0$ (위쪽)으로 swing-up하는 NMPC. 호라이즌 N을 키우면 swing-up 동작이 부드러워지고, $u_{\max}$을 줄이면 여러 번 흔들어 올려야 합니다.

7. 핵심 정리

  • 실제 동역학이 큰 동작 범위에서 비선형이면 NMPC.
  • NLP는 직접 이산화 + multiple shooting + collocation이 표준.
  • SQP는 NLP를 QP의 연속으로 풂. IPOPT, SNOPT가 대표 솔버.
  • RTI: 매 샘플 SQP 한 번만. 임베디드 실시간 NMPC의 핵심.
  • iLQR/DDP: 제약 없거나 약한 큰 문제에 빠른 그래디언트 기반.
  • 도구: CasADi (모델링) + acados (실시간 풀이).
