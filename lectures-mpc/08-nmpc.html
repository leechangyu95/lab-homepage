Week 8. 비선형 MPC (NMPC)

로봇 매니퓰레이터·드론·자율차의 동역학은 본질적으로 비선형입니다. 선형화로는 큰 동작 범위를 다루기 어려워, 비선형 모델을 그대로 쓰는 NMPC가 표준이 됐어요. QP가 NLP가 되고, SQP·Real-Time Iteration 같은 영리한 트릭들로 ms 단위에 풀어냅니다.

목차

  1. 왜 NMPC인가 — 선형화의 한계

  3. Direct vs Indirect — 두 학파
  4. SQP — 비선형을 QP로 잘게 풀기

  6. iLQR — 큰 문제를 위한 그래디언트 기반

1. 왜 NMPC인가

선형 MPC는 동작점 근처의 작은 영역에선 잘 작동하지만:

  • 로봇 팔이 0°에서 90°까지 큰 각도로 움직이면 자코비안이 크게 변함 → 선형화 오차 ↑
  • 드론의 자세 제어는 회전 행렬 자체가 비선형 manifold
  • 차량의 횡방향 동역학은 슬립 각도, 타이어 비선형성으로 dominant
  • 화학 공정의 Arrhenius 항 ($e^{-E/RT}$) 같은 강한 비선형

여러 동작점에서 선형화한 모델을 갈아끼우는 gain-scheduling이나 LPV-MPC도 있지만, 비선형 모델을 그대로 들고 가는 NMPC가 가장 일반적이고 정확.

2. NLP 정식화

비선형 시스템 $x_{k+1} = f(x_k, u_k)$, 비용·제약도 비선형 가능:

$$ \min_{X, U}\ \sum_{k=0}^{N-1} \ell(x_k, u_k) + V_f(x_N) $$ $$ \text{s.t.}\quad x_{k+1} = f(x_k, u_k) $$ $$ g(x_k, u_k) \le 0,\quad x_0 = x(t),\quad x_N \in \mathcal X_f $$

이건 일반 비선형 계획법 (Nonlinear Programming, NLP). QP보다 훨씬 어렵습니다 (비볼록 가능, 지역 최솟값).

3. Direct vs Indirect

Indirect (변분법 기반)

최적성 조건 (Pontryagin Maximum Principle, KKT)을 미분방정식으로 유도해 풀음. 수학적으로 우아하나 수치적으로 까다로움.

Direct (이산화 후 유한차원 NLP)

시간을 이산화하고 직접 NLP 솔버에 넘김. 현재 표준. 두 가지 sub-방법:

  • Single Shooting — 입력 $U$만 결정 변수, 상태는 forward simulation으로 계산. 변수 적지만 수치적으로 민감.
  • Multiple Shooting — 상태와 입력 모두 결정 변수, 동역학을 등식 제약으로. 변수 많지만 sparse + 수치적으로 안정 + 병렬화 가능. 현재 표준.
  • Collocation — 다항식 근사로 동역학을 더 정확히. 화학 공정에서 인기.

4. SQP — Sequential Quadratic Programming

NLP를 푸는 가장 흔한 방법. 아이디어: 현재 추정 $z_k$ 주변에서 NLP를 QP로 근사 → 풀이 → 업데이트 → 반복.

한 SQP 반복

  1. 현재 $z_k$에서 비용·제약을 1·2차 Taylor 전개: $$ \min_{\Delta z}\ \nabla f(z_k)^\top \Delta z + \tfrac{1}{2} \Delta z^\top H \Delta z $$ $$ \text{s.t.}\ \nabla c(z_k)^\top \Delta z + c(z_k) = 0,\ \nabla g(z_k)^\top \Delta z + g(z_k) \le 0 $$ ($H$는 Lagrangian의 Hessian 또는 quasi-Newton 근사 BFGS).
  2. 이 QP를 풀어 step $\Delta z$.
  3. $z_{k+1} = z_k + \alpha \Delta z$ ($\alpha$는 line search).
  4. 수렴 검사 후 반복.

각 반복이 QP 한 번 풀이. 따라서 NLP 한 번 = QP 여러 번. 보통 5~20 반복.

대표 솔버

  • IPOPT — interior-point. 대규모 문제에 강함. CasADi의 기본.
  • SNOPT — active-set SQP. 상용. 큰 변화 추적 빠름.
  • fmincon (MATLAB) — 범용.
  • FORCES Pro / acados — 임베디드 NMPC 특화.

5. Real-Time Iteration (RTI)

실시간 NMPC의 핵심 트릭. NLP를 매 샘플마다 완전히 풀지 않고 SQP를 한 번만 반복. Diehl 등(2002)이 개발.

아이디어

  • 이전 샘플에서 풀어둔 해를 워밍 스타트로 사용.
  • 현재 샘플에서는 SQP를 한 번만 돌려 step $\Delta z$를 얻고 그 게 새 입력.
  • 다음 샘플에서 또 한 번 SQP.
  • 샘플 간격이 짧으면 시스템이 천천히 변하니, 한 번씩 SQP만 해도 점근적으로 수렴.

장점

  • 예측 가능한 계산 시간 (한 번의 QP 풀이만).
  • 임베디드에 적합 — ms 단위로 NMPC 실현.
  • ACADO Toolkit, acados의 표준 모드.

단점

큰 외란이 들어와 상태가 갑자기 멀어지면 한 번의 SQP로 따라잡기 부족. 그럴 때만 다회 반복으로 fallback.

6. iLQR — 큰 문제를 위한 그래디언트 기반

iLQR (iterative LQR)은 NMPC의 또 다른 인기 접근. 로봇 운동 계획에서 표준.

아이디어

현재 trajectory 주변에서 동역학을 선형화 + 비용을 이차근사 → 시변 LQR 풀이 → 새 trajectory → 반복. SQP의 단순화 버전으로 볼 수도 있음.

한 반복

  1. Forward pass: 현재 입력 시퀀스로 trajectory 시뮬레이션.
  2. Backward pass: 시간을 거꾸로 돌면서 LQR Riccati 재귀로 게인 $K_k$, $k_k$ 계산.
  3. Forward rollout: 새 게인으로 다시 시뮬레이션.
  4. 비용 감소 확인하고 line search.

장점: 제약 처리는 부족하지만 큰 N (수백 스텝)에서도 매우 빠름. 로봇 매니퓰레이션, 사족 보행에서 "DDP / iLQR"이 표준.

제약 추가 버전: Constrained iLQR, AL-iLQR (Augmented Lagrangian).

🎮 인터랙티브: 비선형 진자 NMPC

비선형 진자 $\ddot\theta = -\sin\theta + u$를 $\theta = 0$ (위쪽)으로 swing-up하는 NMPC. 호라이즌 N을 키우면 swing-up 동작이 부드러워지고, $u_{\max}$을 줄이면 여러 번 흔들어 올려야 합니다.

7. 핵심 정리

  • 실제 동역학이 큰 동작 범위에서 비선형이면 NMPC.
  • NLP는 직접 이산화 + multiple shooting + collocation이 표준.
  • SQP는 NLP를 QP의 연속으로 풂. IPOPT, SNOPT가 대표 솔버.
  • RTI: 매 샘플 SQP 한 번만. 임베디드 실시간 NMPC의 핵심.
  • iLQR/DDP: 제약 없거나 약한 큰 문제에 빠른 그래디언트 기반.
  • 도구: CasADi (모델링) + acados (실시간 풀이).

📚 깊이 1 — NMPC가 필요한 이유

비선형 시스템에서 선형화 + LMPC가 부족한 경우:

  • 큰 작동 영역: 평형점 주변 선형화의 정확도 한계.
  • 강한 비선형성: 로봇 매니퓰레이터의 관성 결합, 차량의 타이어 슬립.
  • 상태 의존 동특성: 제어 게인이 상태에 강하게 의존.
  • 고차원 운동: 3D 자세, 멀티 에이전트.

📚 깊이 2 — NMPC 표준 형식

$$\min_{u_0,\dots,u_{N-1}} \sum_{k=0}^{N-1} l(x_k, u_k) + V_f(x_N)$$

$$\text{s.t.}\quad x_{k+1} = f(x_k, u_k),\ x_0 = \bar x_0$$

$$\quad x_k \in \mathbb X,\ u_k \in \mathbb U,\ x_N \in \mathbb X_f$$

$f$가 비선형, $l, V_f, \mathbb X, \mathbb U, \mathbb X_f$도 일반(비볼록 가능). 이 NLP(Nonlinear Program)를 매 시점 풀이.

📚 깊이 4 — Multiple Shooting 자세히

Single shooting은 시간 적분 오차 누적이 문제. Multiple shooting은 호라이즌을 여러 구간으로 나누고 각 구간 끝점도 변수로:

  • 변수: $u_0, \dots, u_{N-1}, x_1, \dots, x_N$.
  • 등식 제약: $x_{k+1} = f(x_k, u_k)$ (각 구간).
  • 비볼록이지만 sparse 구조 → IPOPT 등으로 효율적 풀이.

📚 깊이 5 — Real-Time Iteration (RTI) Scheme

NMPC가 실시간에서 무거우니 SQP 1 iteration만 수행 (Diehl 2002):

  1. Preparation phase: 직전 trajectory에서 모든 도함수·QP 행렬 미리 계산.
  2. Feedback phase: 새 measure 도착 후 1 iteration QP 풀이 + 첫 입력 적용.

Feedback phase가 매우 빠름 (μs~ms). 자동차·드론 NMPC 표준.

📚 깊이 6 — iLQR (Iterative LQR)

비선형 시스템을 nominal trajectory 주변에서 선형화 + LQR 풀이 → 갱신 → 반복.

  1. Nominal $\bar x, \bar u$ 시뮬.
  2. 각 시점 선형화: $A_k = \partial f/\partial x|_{\bar x_k, \bar u_k}$, $B_k = \partial f/\partial u|$.
  3. 이차 비용 근사 + Backward Riccati로 LQR 게인 계산.
  4. 업데이트: $u_k = \bar u_k + \alpha\, K_k (x_k - \bar x_k) + \alpha\, k_k$.
  5. 수렴까지 반복.

로봇·자율주행에서 인기 (Drake, Crocoddyl).

📚 깊이 7 — DDP (Differential Dynamic Programming)

iLQR의 일반화. 동역학과 비용의 2차 미분까지 사용. 더 빠른 수렴, 더 큰 계산.

📚 깊이 8 — CasADi 입문

NMPC 구현의 표준 도구. 자동 미분 + 다양한 NLP 솔버 인터페이스.

import casadi as ca

x = ca.MX.sym('x', 2)
u = ca.MX.sym('u', 1)

# 비선형 동역학 (예: 진자)
g, L = 9.8, 0.5
xdot = ca.vertcat(x[1], -g/L*ca.sin(x[0]) + u/L)

# 적분기 (RK4)
T_s = 0.1
F = ca.integrator('F', 'rk', {'x': x, 'p': u, 'ode': xdot}, {'tf': T_s})

# NMPC NLP 정의
N = 20
opti = ca.Opti()
X = opti.variable(2, N+1)
U = opti.variable(1, N)
X0 = opti.parameter(2)

cost = 0
for k in range(N):
    cost += ca.sumsqr(X[:, k]) + 0.1*ca.sumsqr(U[:, k])
    Xn = F(x0=X[:, k], p=U[:, k])['xf']
    opti.subject_to(X[:, k+1] == Xn)
opti.subject_to(X[:, 0] == X0)
opti.subject_to(opti.bounded(-2, U, 2))
opti.minimize(cost)
opti.solver('ipopt')

# 매 시점 풀이
opti.set_value(X0, [0.5, 0])
sol = opti.solve()
u_apply = sol.value(U)[0, 0]

📚 깊이 9 — 비선형 모델 예시

9.1 진자 (Pendulum)

$\ddot \theta = -(g/L)\sin\theta + u/(mL^2)$. $\sin\theta$ 비선형.

9.2 차량 Bicycle (Kinematic)

$\dot x = v\cos\psi$, $\dot y = v\sin\psi$, $\dot\psi = v\tan\delta/L$, $\dot v = a$. 삼각함수 + 곱.

9.3 쿼드콥터

12상태 (위치 3, 속도 3, 자세 3, 각속도 3). 회전 행렬 $R(\eta)$ 곱셈으로 비선형.

9.4 화학 반응기 (Arrhenius)

$\dot C = -k_0 e^{-E/RT} C + \dots$. 지수함수 비선형.

📚 깊이 10 — NMPC 안정성

LMPC와 비슷한 종단 비용 + 종단 영역 트릭. 그러나:

  • $V_f$를 LQR DARE 해 (선형화 모델)로 두면 지역 안정.
  • $\mathcal X_f$를 nonlinear invariant set으로 (계산 어려움).
  • Quasi-infinite horizon NMPC (Chen-Allgöwer 1998) — 표준.

📚 깊이 11 — NMPC 풀이 시간

  • SQP: 10~100 ms (변수 수 100~500).
  • RTI: 1~10 ms (1 iteration만).
  • iLQR: 1~10 ms (작은 차원).
  • FORCES Pro/HPIPM: μs~ms (코드 생성).

📚 깊이 12 — 흔한 오답

  1. 비볼록이라 지역 최적 가능 — warm start로 완화.
  2. Single shooting으로 큰 호라이즌 → 수치 불안정.
  3. 풀이 시간 예산 무시.
  4. 경계 조건 누락 → 발산.
  5. 자동 미분 도구 미사용으로 손계산 도함수 오류.

📚 깊이 14 — FAQ

Q1. NMPC vs 선형화 + LMPC?

큰 작동 영역이거나 강한 비선형이면 NMPC. 작은 평형점 주변이면 LMPC로 충분.

Q2. NMPC의 실시간성?

RTI + 코드 생성으로 ms 단위 가능. 드론·자동차에서 실시간 동작.

Q3. 학습 모델 + NMPC?

Gaussian Process, Neural Network로 모델 학습 후 NMPC 적용. 활발한 연구 분야.
