Week 2. 이산 상태공간 모델

MPC는 매 샘플 시간마다 "내부 시뮬레이터"로 미래를 예측합니다. 그 시뮬레이터의 표준 형태가 이산시간 상태공간 모델이에요. 모델을 어떻게 얻고, 어떻게 검증하고, 무엇을 보장하는지 한 페이지에 정리합니다.

목차

  1. 왜 상태공간인가 — 전달함수와 비교
  2. 이산시간 모델의 형태 (A, B, C, D)
  3. 연속 → 이산 변환 (ZOH, Tustin)
  4. 가제어성·가관측성 — MPC가 동작하기 위한 조건
  5. 모델 식별 — 데이터에서 (A, B) 뽑기
  6. 외란·미스매치 — 적분 외란 모델

1. 왜 상태공간인가

한 줄 비유부터

전달함수 $G(s)$는 "입력→출력의 주파수 관계"를 한 식에 담는 SISO용 도구입니다. 스마트폰의 음향 EQ 같은 신호 처리에는 충분합니다. 그런데 자율차의 종방향·횡방향 동역학처럼 "입력 여러 개 → 출력 여러 개 + 내부 상태가 서로 연결"인 시스템에는 부족해요. 상태공간은 시스템의 내부 변수들이 서로 어떻게 영향을 주고받는지를 행렬로 펼쳐서 보여줍니다.

이산시간 표준형

MPC가 가장 자주 쓰는 형태:

$$ x_{k+1} = A x_k + B u_k + E w_k $$ $$ y_k = C x_k + D u_k + v_k $$
  • $x_k \in \mathbb{R}^n$ — 상태 (시스템의 "기억"). 위치, 속도, 온도, 농도 등.
  • $u_k \in \mathbb{R}^m$ — 제어 입력. 모터 전압, 밸브 개도, 토크 등.
  • $y_k \in \mathbb{R}^p$ — 측정 출력. 센서로 직접 잴 수 있는 양.
  • $w_k, v_k$ — 프로세스/측정 외란. 평균 0, 공분산 알려진 가정.
  • $A, B, C, D, E$ — 상수 행렬 (선형 시불변, LTI). 모델링 또는 식별로 결정.

$D = 0$인 경우가 많습니다 (출력에 입력이 직접 포함되지 않음). MPC에서는 일반적으로 출력 추적 $y \to r$이 목표지만, 수식 단순화를 위해 자주 $y = x$ (상태 직접 측정)로 두는 경우도 있습니다.

2. 연속시간 → 이산시간 변환

물리 모델은 보통 연속시간 형태로 나옵니다:

$$ \dot x(t) = A_c x(t) + B_c u(t) $$

이걸 샘플링 시간 $T_s$로 이산화합니다. 가장 정확하면서 흔한 방법은 Zero-Order Hold (ZOH) — "한 샘플 간격 동안 입력 $u$가 일정하다"는 가정:

$$ A = e^{A_c T_s},\quad B = \int_0^{T_s} e^{A_c \tau} d\tau \cdot B_c $$

다른 방법으로 Forward Euler($A = I + A_c T_s$, 단순하지만 큰 $T_s$에서 불안정), Tustin (Bilinear)($A = (I + \tfrac{T_s}{2}A_c)(I - \tfrac{T_s}{2}A_c)^{-1}$, 안정성·주파수 보존이 좋음)이 있습니다.

실전 팁 — Python에서는 scipy.signal.cont2discrete((Ac, Bc, Cc, Dc), Ts, method='zoh')로 한 줄. MATLAB은 c2d(sys, Ts, 'zoh'). ZOH는 거의 항상 옳고, Tustin은 주파수 응답을 정확히 보존하고 싶을 때 좋습니다.

예시 — 1차 RC 회로

연속시간: $\dot x = -\tfrac{1}{RC} x + \tfrac{1}{RC} u$. $RC = 1$, $T_s = 0.1$일 때 ZOH 결과는 $A \approx 0.905$, $B \approx 0.095$. 즉 이산 모델 $x_{k+1} = 0.905\, x_k + 0.095\, u_k$.

3. 가제어성 (Controllability)과 가관측성 (Observability)

MPC가 의미 있게 동작하려면 두 가지 구조적 성질이 필요합니다.

가제어성 — "어디든 갈 수 있는가"

임의의 초기 상태 $x_0$에서 임의의 목표 $x_f$로 유한 스텝 안에 도달 가능한가? 판정 행렬:

$$ \mathcal{C} = [B \quad AB \quad A^2 B \quad \cdots \quad A^{n-1} B] \in \mathbb{R}^{n \times nm} $$

$\text{rank}(\mathcal C) = n$이면 시스템은 가제어. 그렇지 않으면 어떤 모드는 입력으로 절대 제어할 수 없습니다 — MPC도 LQR도 그런 모드는 손댈 수 없어요.

가관측성 — "내부를 다 볼 수 있는가"

출력만 보고 모든 상태를 복원할 수 있는가? 판정 행렬:

$$ \mathcal{O} = \begin{bmatrix} C \\ CA \\ \vdots \\ CA^{n-1} \end{bmatrix} \in \mathbb{R}^{np \times n} $$

$\text{rank}(\mathcal O) = n$이면 가관측. MPC는 보통 상태 $x$를 직접 안다고 가정하지만, 실제로는 출력 $y$만 측정 가능하므로 관측기(observer)로 $\hat x$를 추정해 사용합니다 (보통 칼만 필터, L4 1D 칼만 참조).

약화된 조건 — 가제어성 대신 안정화 가능성(stabilizability)이면 충분합니다 ("불안정 모드는 모두 제어 가능"). 마찬가지로 검출 가능성(detectability) ("불안정 모드는 모두 관측 가능")이면 관측기 설계 가능.

4. 모델 식별 — 데이터에서 (A, B) 추정

물리 1원리로 $A, B$를 유도하기 어려운 시스템(생물·경제·복잡 기계)은 입출력 데이터로 직접 식별합니다.

최소제곱 ARX

$N$ 스텝의 입출력 $\{u_k, y_k\}$에서 다음을 최소화:

$$ \min_{A, B}\ \sum_{k=0}^{N-1} \|y_{k+1} - (A y_k + B u_k)\|^2 $$

이는 결국 단일 큰 최소제곱 문제 — 머신러닝 W3 회귀의 정규방정식과 동일한 구조입니다. SciPy의 numpy.linalg.lstsq 한 줄.

고급 기법

  • Subspace identification (N4SID, MOESP) — 출력 데이터의 SVD로 상태 차원과 행렬을 한 번에. 산업 표준.
  • Prediction Error Method (PEM) — 잡음 모델까지 함께 추정. MATLAB idtfest, n4sid.
  • Sparse identification (SINDy) — 비선형 시스템에서 sparse 표현 추정.

🎮 인터랙티브: 시스템 응답 시각화

2차 시스템 $x_{k+1} = A x_k + B u_k$의 행렬 원소를 슬라이더로 바꾸며 임펄스 응답이 어떻게 변하는지 봅니다. $A$의 고윳값 절댓값이 1보다 작으면 안정, 1을 넘으면 발산해요.

5. 외란과 모델 미스매치

실제 시스템은 식별된 모델과 정확히 일치하지 않습니다. MPC가 정상상태에서도 오차를 남기는 가장 흔한 원인은 일정 외란(constant disturbance)입니다. 해법은 적분 외란 모델(integrating disturbance model):

$$ x_{k+1} = A x_k + B u_k + B_d d_k,\quad d_{k+1} = d_k $$

외란 $d$를 상태에 추가해 함께 추정하면 MPC가 자동으로 보상하게 됩니다. 이는 PID의 적분기 ($I$ 항)와 같은 역할.

6. 핵심 정리

  • 이산시간 상태공간 $x_{k+1} = Ax_k + Bu_k$가 MPC의 표준 모델 형식.
  • 연속 → 이산은 ZOH가 거의 항상 정답 (scipy/MATLAB 한 줄).
  • 가제어성·가관측성 (또는 약화 버전)이 MPC가 동작하기 위한 구조적 조건.
  • $x$를 직접 못 보면 칼만 필터로 $\hat x$ 추정해 사용.
  • 정상상태 오차를 없애려면 적분 외란 모델로 $d$를 상태에 추가.
