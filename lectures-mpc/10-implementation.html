Week 10. 실시간 구현 & 응용

이론을 다 배웠으니 실전으로. 실제 하드웨어에 MPC를 올리려면 모델링 → 수치 최적화 → 코드 생성 → 실시간 검증의 파이프라인이 필요합니다. 현장에서 가장 많이 쓰이는 도구 체인과 종합 응용 사례 4가지를 정리합니다.

1. 실시간 MPC의 4가지 요건

  1. 샘플 시간 안에 계산 완료 — $T_s = 10$ ms이면 풀이 + 통신 + 적용을 모두 10 ms 안에. 워스트 케이스 보장 필수.
  2. 예측 가능한 실행 시간 — 평균이 5 ms여도 가끔 50 ms 걸리면 실패. RTI 같은 결정적 알고리즘 선호.
  3. 메모리 한정 (스택, 힙) — 임베디드 마이크로컨트롤러는 KB 단위. 동적 할당 금지.
  4. 수치 강건성 — 부동소수점 정밀도, 조건수, NaN 회피. fixed-point CPU에선 고정소수점 변환까지.

2. 도구 체인 — 표준 4종

역할도구특징
모델링 / 자동미분CasADi심볼릭 표현, autodiff, NLP/QP/SX. Python·MATLAB·C++.
실시간 NMPCacadosRTI·SQP·HPIPM. C 코드 생성, hardware-in-the-loop.
QP 솔버 (선형 MPC)OSQPADMM, 임베디드용 작은 코드, 워밍 스타트.
고수준 인터페이스do-mpc / CVXPY프로토타입에 적합. 느림. 산업 응용은 acados로 이전.

전형적 워크플로:

  1. CVXPY/do-mpc로 알고리즘 검증 (시뮬레이션)
  2. CasADi로 정확한 모델 + 비용·제약 정의
  3. acados로 RTI 솔버 구성, C 코드 생성
  4. 마이크로컨트롤러/RTOS에 통합
  5. HIL (Hardware-in-the-Loop) 시험 → 실차/실 플랜트

3. 코드 생성 워크플로 — acados 예시

# Python: acados로 NMPC 정의 + C 코드 생성
from acados_template import AcadosModel, AcadosOcp, AcadosOcpSolver
import casadi as ca

# 1. CasADi로 모델 정의
x = ca.MX.sym('x', 4)   # [px, py, vx, vy]
u = ca.MX.sym('u', 2)   # [ax, ay]
xdot = ca.vertcat(x[2], x[3], u[0], u[1])
model = AcadosModel()
model.f_expl_expr = xdot
model.x = x
model.u = u
model.name = 'point_mass'

# 2. OCP 정의
ocp = AcadosOcp()
ocp.model = model
ocp.dims.N = 20
ocp.dims.nu = 2
ocp.solver_options.tf = 2.0   # 호라이즌 길이 (초)
ocp.cost.cost_type = 'LINEAR_LS'
# ... Q, R, ref 설정

# 3. 제약
ocp.constraints.lbu = [-2, -2]
ocp.constraints.ubu = [+2, +2]

# 4. 실시간 옵션
ocp.solver_options.qp_solver = 'PARTIAL_CONDENSING_HPIPM'
ocp.solver_options.nlp_solver_type = 'SQP_RTI'

# 5. C 코드 생성
solver = AcadosOcpSolver(ocp, json_file='mpc.json')
# → c_generated_code/ 폴더에 모든 C 코드 생성됨

생성된 C 코드는 STM32, NVIDIA Jetson, dSPACE 같은 임베디드 플랫폼에 그대로 컴파일 가능.

4. 응용 ① 자율주행 Racing

문제

1:43 스케일 RC 자동차 (ETH ORCA 프로젝트). 트랙을 가장 빠르게 도는 것이 목표. 비선형 자전거 모델 + 슬립 + 트랙 경계 제약.

모델

$$ \dot p_x = v\cos(\psi+\beta),\ \dot p_y = v\sin(\psi+\beta),\ \dot \psi = (v/L) \tan\delta\cos\beta $$

$\beta$는 측슬립 각, $\delta$는 조향. 4D state ($p_x, p_y, \psi, v$), 2D input ($\delta, a$).

비용 + 제약

  • 비용: $-v$ (속도 최대화) + 트랙 중심선 추종 + 제어 부드러움
  • 제약: $|\delta| \le \delta_{\max}$, $a \in [a_{\min}, a_{\max}]$, 트랙 경계 (다면체)

샘플 시간 50 ms, 호라이즌 1.5 s (30 스텝). NMPC + RTI + acados로 실시간 구현. 사람보다 빠른 lap time 달성.

5. 응용 ② 쿼드콥터 곡예 비행

문제

UPenn GRASP 랩의 곡예 비행 시연. 좁은 창문 통과, 공중제비, 협력 운반 — 모두 NMPC로.

모델

13D state (위치 3, 속도 3, 회전 행렬 9 또는 쿼터니언 4) + 4D 입력 (모터 4개의 추력). 회전 동역학은 매우 비선형.

핵심 트릭

  • Differential flatness — 위치와 yaw만 결정하면 나머지 상태가 대수적으로 결정됨. 호라이즌을 위치 공간에서 풀고 입력은 inverse로 회복.
  • Cascade 구조 — 외부 NMPC (10 Hz) + 내부 attitude PID (1 kHz). NMPC가 무리할 정도로 빠를 필요 없음.
  • Soft 제약 — 속도, 추력 한계는 hard지만 충돌 회피는 soft (slack 사용).

6. 응용 ③ 빌딩 HVAC

문제

ETH OptiControl 프로젝트. 사무 빌딩의 난방·냉방·환기·블라인드를 통합 제어. 예보된 날씨와 전기 가격을 활용해 쾌적성 유지하며 에너지 비용 최소화.

모델

벽·바닥·천장의 RC 등가 회로. 6~12개 상태 (방·벽 온도). 외란: 외기 온도, 일사량, 사람 활동.

특징

  • 샘플 시간 15분, 호라이즌 24~48시간 (96~192 스텝)
  • 예보 데이터를 모델에 통합 — 날씨 + 전기 가격 + 점유율 예측
  • Stochastic MPC (chance constraint)로 예보 불확실성 처리
  • 실험에서 RBC (rule-based) 대비 17% 에너지 절감

7. 응용 ④ 화학 공정 — Heat Exchanger

문제

두 유체가 열을 교환하는 헤어핀 열교환기. 출구 온도를 일정하게 유지하면서 압력 제약 만족.

모델

분포 매개변수 PDE → 공간 이산화 (10~50 노드). 결과적 ODE는 선형 ($N = 10$ 노드면 10D).

왜 MPC인가?

  • 긴 시간 지연 (관에서 유체 이동) — PID로는 어려움
  • 동시 추적 + 제약 (압력 한계, 펌프 용량)
  • 상호작용 (hot side 유량 ↑ → cold side 출구 온도 ↑)

샘플 시간 1초, 호라이즌 60초. 선형 MPC + OSQP. PID 대비 정착 시간 50% 단축, 에너지 8% 절감.

8. 흔한 실수와 디버깅

  • 모델 식별 검증 부족 — 시뮬레이션 모델과 실제 시스템 응답이 가장 다른 곳에서 MPC가 실패. 항상 stress test.
  • 제약을 너무 엄격하게 — Hard 제약이 가용성을 잃으면 풀이 실패. 소프트 제약 + 큰 벌점이 안전.
  • 호라이즌 너무 짧게 — 비용은 줄이지만 미래 제약을 못 봄. 정착 시간의 1~2배가 권장.
  • 워밍 스타트 안 함 — 같은 QP를 매번 처음부터 풀어 5~10× 느림. OSQP·acados는 자동 지원.
  • 외란 모델 빠뜨림 — 정상상태 오차 발생. 적분 외란 추가 필수.
  • 관측기 없이 상태 직접 사용 — 잡음이 입력으로 직접 들어가 진동. 칼만 필터 필수.
  • 샘플 지터(jitter) 무시 — RTOS에서 샘플 간격이 들쭉날쭉하면 안정성 깨짐. 우선순위 고정.

🎮 인터랙티브: 자율주행 차선 추종 NMPC

차량(파랑)이 곡선 차선을 따라가는 NMPC. 호라이즌 N을 키우면 곡선 미리 보기로 안쪽으로 깔끔하게 컷, 줄이면 늦은 회전. 속도도 조절 가능.

9. 핵심 정리 — MPC 한 학기 마무리

  • MPC는 단일 알고리즘이 아니라 최적화 + 모델 + 제약 + 수신지평선의 조합 패러다임.
  • 이론(W1–W6) → 강건성·비선형성(W7–W8) → 임베디드(W9–W10)의 흐름.
  • 현장 도구: CasADi(모델링) + acados(실시간 솔버) + OSQP(QP) — 이 셋이 사실상 표준.
  • 응용은 ms 단위 드론부터 시간 단위 빌딩까지 같은 프레임워크.
  • 흔한 실수의 90%는 모델 식별·외란 모델·관측기 부재 — 이론이 아닌 시스템 통합 문제.
  • MPC는 끝이 아니라 출발점 — Learning-based MPC, Distributed MPC, Economic MPC가 현재 활발한 연구 영역.

📖 더 깊이 — 다음에 배울 주제

  • Learning-based MPC — Gaussian Process, deep learning으로 모델 학습. 머신러닝과의 융합.
  • Distributed MPC — 큰 시스템을 작은 MPC들의 협력으로. 스마트 그리드, 군집 로봇.
  • Economic MPC — 추적이 아닌 직접 경제 비용 최적화. 화학 공정, 에너지.
  • Hybrid MPC — 연속 + 이산 (스위치, 모드) 결합. MILP/MIQP 사용.
  • Reinforcement Learning vs MPC — RL은 모델 없이도 학습, MPC는 모델 활용. 현재 활발한 비교 연구.
