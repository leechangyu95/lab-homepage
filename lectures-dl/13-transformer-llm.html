Week 13. Transformer와 LLM

RNN을 대체하고 ChatGPT·Claude를 가능케 한 핵심 아키텍처입니다.

목차

  1. RNN의 한계
  2. 셀프 어텐션(Self-Attention)
  3. Q · K · V의 의미
  4. 멀티 헤드와 트랜스포머 블록
  5. 대규모 언어모델(LLM)
  6. 인터랙티브 어텐션 데모
  7. HuggingFace로 즉시 사용

1. RNN의 한계

RNN/LSTM은 단어를 순서대로만 처리할 수 있어 병렬화가 안 됩니다. 또한 매우 긴 문장에서는 여전히 정보가 흐릿해집니다.

2. 셀프 어텐션의 핵심 아이디어

한 단어가 문장 안의 모든 다른 단어를 동시에 본 뒤, "지금 나에게 가장 중요한 단어가 누구인지" 가중치를 계산합니다.

$$ \text{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V $$

3. Q · K · V란?

  • Query (Q) — "내가 찾고 싶은 것"
  • Key (K) — "내가 가지고 있는 것의 색인"
  • Value (V) — "실제 내용"

단어 도서관에 비유하면 Q는 검색어, K는 책 제목, V는 책 내용입니다. Q와 K의 내적이 클수록 그 단어의 V가 많이 섞여 들어갑니다.

4. 멀티 헤드 & 트랜스포머 블록

여러 개의 어텐션을 병렬로 두면 각 헤드가 다른 종류의 관계(문법, 의미, 위치 등)를 따로 학습합니다.

$$ \text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h) W^O $$

한 트랜스포머 블록 = Self-Attention → 잔차+LayerNorm → FFN → 잔차+LayerNorm. 이걸 12층, 24층, 96층씩 쌓으면 BERT, GPT-3, GPT-4가 됩니다.

5. 대규모 언어모델 (LLM)

인터넷 텍스트 수조 개 단어로 트랜스포머를 학습시키면, 다음 단어 예측이라는 단순한 과제만으로도 놀라운 일반 지능이 나타납니다.

  • BERT — 양방향, 빈칸 채우기 학습. 이해 과제에 강함.
  • GPT — 단방향, 다음 단어 예측. 생성 과제에 강함.
  • T5 / BART — 입력 → 출력 변환 (요약, 번역).

🎮 인터랙티브: 어텐션 가중치 히트맵

단어 5~7개의 짧은 문장을 입력하면 단어 간 어텐션 가중치를 시각화합니다. (간이 데모 — 실제 모델 가중치는 아니지만 패턴은 유사)

6. HuggingFace로 즉시 사용

거대한 모델을 직접 학습할 필요는 없습니다. 사전학습된 모델을 단 몇 줄로 불러올 수 있습니다.

from transformers import pipeline

# 1) 감성 분석
clf = pipeline('sentiment-analysis')
print(clf("This movie was wonderful!"))
# → [{'label': 'POSITIVE', 'score': 0.9999}]

# 2) 텍스트 생성 (GPT-2)
gen = pipeline('text-generation', model='gpt2')
print(gen("AI will change the future", max_length=40)[0]['generated_text'])

# 3) 요약 (DistilBART)
summarize = pipeline('summarization', model='sshleifer/distilbart-cnn-12-6')
print(summarize(long_text, max_length=60, min_length=20))

# 4) 질문 응답
qa = pipeline('question-answering')
print(qa(question="When was PyTorch released?", context=ctx))

# 5) 제로샷 분류 (학습 없이 새 카테고리 분류)
zs = pipeline('zero-shot-classification')
print(zs("이 영화는 정말 재미있었다",
         candidate_labels=['긍정', '부정', '중립']))

7. 프롬프트 엔지니어링

LLM의 성능은 프롬프트 작성 방식에 크게 좌우됩니다.

  • 역할 부여: "당신은 친절한 수학 선생님입니다…"
  • 예시 제시(few-shot): 원하는 출력 형식의 예 2~3개
  • 단계 분해: "단계별로 차근차근 생각해보세요"
  • 명확한 형식 지정: "JSON 형식으로 답하세요"
여기까지 마치면: 머신러닝의 기본부터 최신 LLM의 핵심 원리까지 모두 둘러본 셈입니다. 이제 실전 프로젝트(이미지 분류기 만들기, 챗봇 만들기, 시계열 예측)로 손을 움직여 보세요. 또는 로봇공학특론의 강화학습·MPC와 결합해 자율 시스템에 응용할 수 있습니다.
