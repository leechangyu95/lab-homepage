Week 2. 데이터 핸들링 & Python 기초

데이터를 깨끗이 정리하는 것이 모델보다 더 중요할 때가 많습니다.

목차

  1. NumPy 한 줄 정리
  2. Pandas 한 줄 정리
  3. Train / Test 분리
  4. 표준화 (StandardScaler)
  5. 전처리의 효과 데모

1. NumPy

파이썬 기본 리스트는 느립니다. NumPy의 ndarray는 C로 구현되어 있어 100배 이상 빠릅니다.

import numpy as np

a = np.array([1, 2, 3, 4])
b = np.array([10, 20, 30, 40])
print(a + b)        # [11 22 33 44]   (요소별 덧셈)
print(a * 2)        # [2 4 6 8]       (브로드캐스팅)
print(a.mean())     # 2.5
print(a[1:3])       # [2 3]           (슬라이싱)

2. Pandas

DataFrame은 엑셀 시트 같은 표 자료구조입니다.

import pandas as pd

df = pd.read_csv('fish.csv')
print(df.shape)        # (49, 3)
print(df.head())       # 첫 5행
print(df.describe())   # 통계 요약 (mean, std, min, max ...)
df['weight'].hist()    # 히스토그램

4. 표준화 (Standardization)

특성마다 값의 단위가 다르면 KNN처럼 거리 기반 알고리즘이 망가집니다. 예를 들어 길이(cm)는 10~40, 무게(g)는 5~1000이라면 무게가 거리를 지배합니다.

$$ z = \frac{x - \mu}{\sigma} $$

각 특성을 평균 0, 표준편차 1로 만드는 변환입니다.

from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)   # ← 같은 평균/표준편차로 변환
주의 — 테스트 데이터에는 fit_transform이 아니라 transform만 써야 합니다. 안 그러면 테스트 정보가 학습으로 새어 나갑니다(데이터 누수).
