3. 속도 프로파일 — 곡률 기반 감속

곡선이 급할수록 횡가속 한도($a_{y,\max}$, 보통 2.5 m/s²) 안에서만 돌 수 있는 속도가 정해집니다.

$$ v_\text{limit}(s) = \sqrt{\frac{a_{y,\max}}{|\kappa(s)|}}, \qquad v_\text{ref}(s) = \min\!\big(v_\text{limit}(s),\;v_\text{cruise}\big) $$ 
dx = csx(s_fine, 1); dy = csy(s_fine, 1)
ddx = csx(s_fine, 2); ddy = csy(s_fine, 2)
kappa = (dx*ddy - dy*ddx) / (dx**2 + dy**2)**1.5
v_limit = np.sqrt(a_y_max / np.clip(np.abs(kappa), 1e-3, None))
v_ref   = np.minimum(v_limit, 15.0)            # 최대 15 m/s 순항

이렇게 미리 계산된 $v_\text{ref}(s)$를 W9의 PID가 추종합니다. 곡선에서 자동으로 감속, 직선에서 최대속.

5. 인터랙티브: 동적 장애물 → 재계획

🎮 인터랙티브: 트리거 버튼으로 새 장애물 등장

주행 중 새 장애물이 경로 위로 떨어지면 시스템은 즉시 A*로 재계획하고 새 경로를 따라갑니다. 이게 자율주행의 표준 패턴입니다.

7. 흔한 통합 버그와 해결

증상의심해결
차가 경로와 반대 방향으로 출발각도 wrap 누락$\psi \in [-\pi,\pi]$로 매 스텝 정규화
차가 경로 끝에서 가만히 못 멈춤look-ahead가 경로 길이보다 큼경로 끝에 도달하면 $L_d$를 줄이거나 PID 종료 모드
곡선에서 과속 → 슬립속도 프로파일 미적용$v_\text{limit} = \sqrt{a_y/\kappa}$ 기반 감속
장애물 관통Costmap 미사용 / 인플레이션 부족안전 마진 ≥ 차폭 절반
제어 입력이 톡톡 튐지터 + Kd 너무 큼저역 필터 또는 Rate limit
좌표계 혼동(World vs Body)변환 누락모듈 입력에 좌표계 명시 + 단위 테스트

8. 메인 루프 — 코드로 정리

# 1) 한 번만: 지도 → A* → smoothing → 속도 프로파일
grid       = load_map()
path_grid  = astar(grid, start_idx, goal_idx)
path_xy    = grid_to_xy(path_grid, res=0.1)
ref_x, ref_y = smooth(path_xy)         # Shortcut + Cubic spline
kappa        = curvature(ref_x, ref_y)
v_ref        = np.minimum(np.sqrt(2.5 / np.clip(np.abs(kappa), 1e-3, None)), 15.0)

# 2) 50 Hz 루프
car       = CarSim(x=2, y=2, psi=0, v=0)
speed_pid = PID(1.2, 0.3, 0.05, u_min=-5, u_max=3)
ref_path  = np.c_[ref_x, ref_y]
log = []
for t in np.arange(0, 60, 0.02):
    state    = car.state
    delta, i = pure_pursuit(state, ref_path, Lfc=2.0, k=0.1)
    a        = speed_pid.step(v_ref[i] - state[3], dt=0.02)
    car.step(a, delta, dt=0.02)
    log.append((t, *state, v_ref[i], delta, a))
    if np.linalg.norm(state[:2] - ref_path[-1]) < 1.0:
        break

# 3) 평가
log = np.array(log)
err_y  = compute_cross_track_error(log[:, 1:3], ref_path)
print(f"RMSE_ey = {np.sqrt((err_y**2).mean()):.3f} m")

9. 핵심 정리

  • 통합은 새 알고리즘이 아니라 인터페이스 설계의 문제. 데이터를 누가 누구에게 어떤 형식으로 주는지가 핵심.
  • 경로계획은 1회, 추종 루프는 50 Hz. 주기 분리가 시스템 안정성의 출발.
  • 곡률 기반 속도 프로파일이 안전·승차감의 핵심.
  • 버그 1번 원인: 좌표계·각도 wrap·단위. 단위 테스트로 막을 수 있다.
  • 실차 가기 전 시뮬에서 RMSE·최소 거리·입력 변화율 세 지표는 반드시 통과시키자.
