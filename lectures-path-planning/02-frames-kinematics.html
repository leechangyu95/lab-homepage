1. 왜 좌표계가 먼저인가?

한 줄 비유부터

친구에게 길을 알려준다고 해봅시다. "오른쪽으로 가"는 누구의 오른쪽일까요? 나의 오른쪽? 친구의 오른쪽? 지도상의 동쪽? — 똑같은 점도 누구의 시점에서 말하느냐에 따라 숫자가 달라집니다.

자율주행에서 같은 일이 매 순간 일어납니다. 센서가 본 좌표, 차체에서 본 좌표, 지도에서의 절대 좌표 — 이 셋이 다 다른 숫자입니다. 잘못 환산하면 알고리즘은 엉뚱한 곳에 차를 보냅니다.

실무 격언 — "어떤 좌표계에서 이야기하는 거야?"가 자율주행 디버깅의 첫 질문입니다. 버그의 절반 이상은 좌표계 오해에서 옵니다.

2. 자율주행의 4가지 주요 좌표계

좌표계기호원점 / 축용도
세계 (Global)$\{W\}$지도 상 고정점, $x$ 동, $y$ 북전역 경로, HD Map
차체 (Body)$\{B\}$차 후륜축 중앙, $x$ 전방추종제어 수식 작성
센서 (Sensor)$\{S\}$센서 장착 지점인식 출력
도로 (Frenet)$\{F\}$도로 중심선 기준 $(s, d)$지역 경로계획 (4주차)

이번 주는 $\{W\}, \{B\}, \{S\}$에 집중합니다. $\{F\}$는 4주차에서 본격 등장합니다.

3. 2D 회전행렬 복습

한 줄 비유부터

회전행렬은 종이 위 점을 원점 둘레로 핀처럼 돌리는 도구입니다. 각도만 정해주면, 어느 방향으로 얼마나 돌릴지 결정됩니다.

점을 $\theta$ 만큼 반시계로 회전시키는 행렬:

$$ R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix} $$

꼭 외워야 할 성질 4개:

  • 직교행렬: $R^\top R = I$
  • 역회전 = 전치: $R^{-1} = R^\top$ (덕분에 역행렬 계산이 공짜)
  • $\det R = 1$ (길이·각 보존)
  • 합성: $R(\alpha) R(\beta) = R(\alpha + \beta)$
import numpy as np

def R(theta):
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, -s],
                     [s,  c]])
Active vs Passive — 같은 행렬이 두 가지 의미로 쓰입니다. Active: "점 자체를 회전" ($\mathbf{p}' = R\mathbf{p}$). Passive: "좌표축을 회전" ($\mathbf{p}^B = R(-\theta)\mathbf{p}^W$). 자율주행에서 "World 좌표를 차체 좌표로" 변환할 땐 Passive 관점입니다 — 점은 안 움직이고 우리가 보는 축만 돌아가니까요.

4. 동차 변환행렬 (Homogeneous Transform)

왜 필요?

실제 차는 회전만 하지 않고 위치도 움직입니다. 회전과 평행이동을 한 번의 행렬 곱으로 처리하기 위한 꼼수가 동차 좌표(homogeneous coordinates)입니다. 점에 1을 하나 더 붙여 3차원 벡터로 만들면, 변환이 모두 행렬 곱이 됩니다.

$$ T^W_B = \begin{bmatrix} R(\theta) & \mathbf{t} \\ \mathbf{0}^\top & 1 \end{bmatrix} = \begin{bmatrix} \cos\theta & -\sin\theta & t_x \\ \sin\theta & \cos\theta & t_y \\ 0 & 0 & 1 \end{bmatrix} $$

점 변환 (동차 좌표):

$$ \tilde{\mathbf{p}}^W = T^W_B \, \tilde{\mathbf{p}}^B, \quad \tilde{\mathbf{p}} = [x, y, 1]^\top $$

역변환은 다음과 같이 깔끔하게 떨어집니다:

$$ T^B_W = (T^W_B)^{-1} = \begin{bmatrix} R^\top & -R^\top \mathbf{t} \\ \mathbf{0}^\top & 1 \end{bmatrix} $$ 
def T_wb(x, y, theta):
    """Body -> World 동차 변환 행렬"""
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, -s, x],
                     [s,  c, y],
                     [0,  0, 1]])

def transform(T, p):
    """2D 점 p=[x,y]를 T로 변환"""
    ph = np.array([p[0], p[1], 1.0])
    return (T @ ph)[:2]

# 차량 (10,5), 헤딩 30°. 차체 기준 전방 2m 점은?
T   = T_wb(10.0, 5.0, np.deg2rad(30))
p_w = transform(T, np.array([2.0, 0.0]))
print(p_w)   # [11.732  6.   ]

체인 법칙 — 좌표계를 줄줄이 잇기

센서 점을 곧장 월드로 옮길 때 자주 등장하는 패턴입니다.

$$ T^W_S = T^W_B \, T^B_S $$
  • $T^B_S$ — 센서 장착 캘리브레이션 (한 번 측정, 정적)
  • $T^W_B$ — 시시각각 차량 자세 (Localization 출력, 동적)

ROS의 tf2 라이브러리가 이런 트리를 자동으로 관리해 줍니다. 원리는 똑같아요.

5. 인터랙티브: World ↔ Body 변환

🎮 인터랙티브: 차를 끌어보세요 — 월드의 빨간 점이 차체에서 어떻게 보이나?

회색 격자가 월드 프레임, 파란 차량의 자체 축이 차체 프레임입니다. 차량 위치 $(x,y)$와 헤딩 $\psi$를 슬라이더로 바꾸면 오른쪽 패널의 차체 좌표값이 실시간으로 갱신됩니다. 빨간 점 자체는 가만히 있어요 — 우리가 보는 축이 돌고 옮겨지는 것이죠.

6. Kinematic Bicycle Model — 가정 & 유도

왜 "자전거"라 부르나?

실제 차는 4륜이지만 좌우 한 쌍을 한 점으로 합치면 자전거처럼 단순한 모델이 됩니다. 앞바퀴 하나, 뒷바퀴 하나, 휠베이스 L. 이 단순화가 Planning·저속 Control에서 충분히 정확합니다.

가정

  1. 좌우 바퀴를 하나로 합친 자전거
  2. 타이어 슬립 없음 — 바퀴는 바라보는 방향으로만 굴러감
  3. 조향은 앞바퀴만 (Ackermann 근사)
  4. 평지 2D, Pitch·Roll 무시

기호

  • $(x, y, \psi)$ — 후륜축 중앙 위치 + 차량 헤딩
  • $v$ — 후륜축 속도 ($\approx$ 차량 속도)
  • $\delta$ — 전륜 조향각
  • $L$ — 휠베이스 (앞뒤 차축 거리)

상태방정식 유도

후륜은 헤딩 방향으로, 전륜은 헤딩+조향 방향으로 굴러갑니다. 후륜축 중앙을 기준으로 잡으면:

$$ \dot x = v\cos\psi, \quad \dot y = v\sin\psi, \quad \dot\psi = \frac{v}{L}\tan\delta $$
한 줄 암기 — "조향각의 탄젠트 ÷ 휠베이스 × 속도 = 요레이트"
$\dot\psi = (v/L)\tan\delta$. 이 한 줄이 본 강의 끝까지 천 번 등장합니다.

7. 회전반경 R = L/tan(δ)

$\dot\psi = v/R$라는 원운동 관계와 위 식을 결합하면 곧바로:

$$ \tan\delta = \frac{L}{R} \;\Longrightarrow\; R = \frac{L}{\tan\delta} $$
예시LδR
완만 선회2.7 m5°≈ 30.9 m
일반 선회2.7 m15°≈ 10.1 m
급선회2.7 m30°≈ 4.7 m

도심 U턴 최소 반경(약 5 m)이 조향각 30° 근처라는 사실, 직관과 잘 맞죠?

🎮 인터랙티브: 조향각 → 회전반경 곡선

슬라이더로 조향각 $\delta$를 움직이면 (a) 차가 그리는 원호와 회전중심, (b) $R$ vs $\delta$ 곡선 위의 현재 위치가 동시에 갱신됩니다. δ가 0에 가까워질수록 R이 폭발적으로 커지는(=거의 직진하는) 모습을 눈으로 확인하세요.

8. 인터랙티브: Bicycle 시뮬 & 8자 궤적

🎮 인터랙티브: Kinematic Bicycle 시뮬레이터

속도 $v$와 조향 $\delta$를 슬라이더로 직접 잡아 차를 운전해 보세요. 회색 곡선이 차가 지나간 자취, 파란 점선이 이론 회전반경 $R = L/\tan\delta$입니다. δ를 일정하게 두면 차가 정확히 그 원 위로 굴러가는 걸 보게 됩니다.

🎮 인터랙티브: 정현파 조향 → 8자 궤적

조향을 시간에 따라 $\delta(t) = \delta_{\max}\sin(\omega t)$로 흔들면 차는 8자를 그립니다. $\delta_{\max}$가 폭, $\omega$가 빠르기를 결정합니다. 자율주행 대회 시험 주행의 단골 패턴이에요.

9. Ackermann 조향 · 좌표 기준점

실제 4륜은 좌·우 조향각이 다르다

회전 중심이 후륜축 연장선 위 한 점 $O$를 공유해야 한다는 기하 조건에서 자연스럽게 따라옵니다. 내측 바퀴가 더 많이 꺾여야 하죠. 이를 Ackermann Geometry라고 합니다.

$$ \cot\delta_o - \cot\delta_i = \frac{W}{L} $$

여기서 $W$는 트레드(좌우 바퀴 간격), $L$은 휠베이스. Bicycle 모델의 $\delta$는 실무적으로 내·외 조향의 평균이라고 보면 됩니다.

기준점을 어디로 잡느냐 — 후륜 vs 중심 vs 전륜

기준점$\dot x$$\dot y$특징
후륜축 중앙$v\cos\psi$$v\sin\psi$수식 단순. Planning에서 표준
차체 중심 G$v\cos(\psi+\beta)$$v\sin(\psi+\beta)$실측 IMU 위치에 가깝다
전륜축 중앙$v_f\cos(\psi+\delta)$$v_f\sin(\psi+\delta)$Stanley 제어(W11)의 기준

차체 슬립각 $\beta$ (속도 방향과 차체 방향의 차)는 Kinematic에서 $\tan\beta = (l_r \tan\delta)/L$. Stanley가 "전륜 기준"으로 설계되는 이유 — 11주차에서 다시 등장합니다.

10. PythonRobotics 기반 — 한 스텝 시뮬레이터

# 참고: PythonRobotics/PathTracking/pure_pursuit 의 상태 갱신 로직을
# 교육용으로 단순화 (MIT License, A. Sakai).
import numpy as np

L = 2.9  # 휠베이스 [m]

def update(state, v, delta, dt=0.1):
    x, y, psi = state
    x   += v * np.cos(psi) * dt
    y   += v * np.sin(psi) * dt
    psi += v / L * np.tan(delta) * dt
    return np.array([x, y, psi])

# 예: 5초간 직진 → 오른쪽 조향
state = np.array([0.0, 0.0, 0.0])
traj  = [state.copy()]
for t in np.arange(0, 5, 0.1):
    delta = np.deg2rad(10) if t > 2 else 0.0
    state = update(state, v=5.0, delta=delta)
    traj.append(state.copy())

traj = np.array(traj)

11. 수치 안정성 팁

적분 — Euler vs Runge-Kutta 4

강의에서는 Euler 적분($\Delta t = 0.01\sim0.1$)을 사용합니다. 고정밀이 필요하면 RK4:

def rk4(f, s, u, dt):
    k1 = f(s, u)
    k2 = f(s + dt/2*k1, u)
    k3 = f(s + dt/2*k2, u)
    k4 = f(s + dt*k3,   u)
    return s + dt/6*(k1 + 2*k2 + 2*k3 + k4)

각도 정규화 — wrap to [-π, π]

버그 1위 원인: $\psi$가 +180°/-180° 경계에서 불연속하게 튀어 비교가 깨집니다.

def wrap(a):
    return (a + np.pi) % (2*np.pi) - np.pi
흔한 함정 5 — (1) rad↔deg 혼용, (2) 각도 wrap 누락, (3) World↔Body 부호 오류, (4) 휠베이스 단위(미터로 통일!), (5) 후륜 vs 중심 기준 혼동. 이 다섯 개만 조심하면 시뮬 디버깅이 90% 줄어듭니다.

13. Kinematic Model의 한계

상황왜 문제가 되나

그럼에도 왜 쓰나? 간단하고 선형화 쉽고, Planning 용도에는 충분하기 때문입니다. 추종제어가 어차피 보정해 주니까요. 다음 주(W3)에서 이 한계를 부수고 Dynamic Model로 확장합니다.

14. 핵심 정리

  • 좌표계 4종: World, Body, Sensor, Frenet. 모든 수식은 어느 프레임에서 쓴 것인지 먼저 명시.
  • 동차 변환행렬 $T = \begin{bmatrix}R & t \\ 0 & 1\end{bmatrix}$ 하나로 회전·평행이동을 한 번에 처리.
  • 체인 법칙: $T^W_S = T^W_B \, T^B_S$. 캘리브레이션(정적) × 차량 자세(동적).
  • Kinematic Bicycle: $\dot x = v\cos\psi,\; \dot y = v\sin\psi,\; \dot\psi = (v/L)\tan\delta$.
  • 회전반경 $R = L/\tan\delta$. 이 한 줄이 후속 알고리즘 곳곳에서 다시 등장.
  • 한계: 고속·급조향·미끄러움. 그래서 다음 주는 Dynamic Bicycle.

15. 손풀이 연습

Q1. $L = 2.8$ m, $\delta = 10°$, $v = 5$ m/s. 회전반경과 요레이트는?
→ $R = 2.8/\tan 10° \approx 15.88$ m, $\dot\psi = (5/2.8)\tan 10° \approx 0.315$ rad/s ($\approx 18°$/s).

Q2. 월드 점 $(3, 4)$를 차량 위치 $(1, 1)$, 헤딩 $90°$ 기준으로 차체 좌표는?
→ 평행이동 $(2, 3)$ → $R(-90°)$ 곱 → $(3, -2)$. 차량 앞 3 m, 오른쪽 2 m.

Q3. Bicycle 식만으로 차가 옆으로 평행이동할 수 있나?
→ 불가능. $\dot x, \dot y$가 항상 헤딩 방향으로만 갖는 비홀로노믹(non-holonomic) 시스템.
