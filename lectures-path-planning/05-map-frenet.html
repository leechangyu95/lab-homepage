1. 차의 지도는 사람의 지도와 어떻게 다른가

한 줄 비유

사람이 카페 위치를 "스타벅스 옆 건물"이라고 말하면, 자동차는 "위도 37.5°, 경도 127.0°, 주차장 입구 (x, y)=(10, 15) m" 또는 "중심선 진행거리 $s=120$ m, 옆방향 $d=-1.5$ m"로 이해해야 합니다.

  • 사람의 지도 — 이미지로 읽음. "여기 좌회전" 같은 감각적 지시.
  • 차의 지도 — 컴퓨터가 읽을 수 있는 숫자·구조. 좌회전을 "헤딩 90°만큼 회전" 같은 수학 연산으로 표현.

3. Occupancy Grid — 마인크래프트 픽셀

핵심 아이디어

  • 세상을 정사각형 격자로 자르기
  • 각 칸 = 장애물 있음(1) / 없음(0) / 모름(-1)

해상도(resolution) — "한 칸이 실제로 몇 m인가". 도심은 0.1 m/칸, 고속도로는 0.5 m/칸 정도. 차량 폭 1.8 m라면 0.2 m/칸이면 충분합니다.

실세계 좌표 ↔ 격자 인덱스

차량은 m 단위로 위치를 인식하지만 격자는 (행, 열)이니까 변환이 필요합니다.

def xy_to_ij(x, y, res=0.1, x0=0, y0=0):
    j = int((x - x0) / res)   # 열(col) ← x
    i = int((y - y0) / res)   # 행(row) ← y
    return i, j

def ij_to_xy(i, j, res=0.1, x0=0, y0=0):
    x = j * res + x0
    y = i * res + y0
    return x, y
가장 흔한 실수 — 행과 열을 바꿔 쓰는 것! 이미지 좌표는 위→아래로 행이 증가합니다. i = y/res, j = x/res 주석을 항상 달아두세요. imshow로 그릴 때 origin='lower'를 지정하면 차량 좌표계와 시각적으로 맞아집니다.

Costmap — 난로 비유

0/1 이분법은 "장애물 바로 옆"도 안전이라고 하는 문제가 있습니다. 실제로는 1 cm 차이로 충돌이니까요. 그래서 장애물 주변까지 위험도를 부드럽게 퍼뜨린 costmap을 씁니다.

"난로를 떠올려보세요. 바로 앞은 매우 뜨겁고, 1 m 떨어지면 따뜻하고, 3 m 멀어지면 거의 안 뜨겁죠." — 장애물에서 멀어질수록 비용이 줄어드는 구조입니다.

from scipy.ndimage import distance_transform_edt

def make_costmap(occ, max_dist=1.5, res=0.1):
    free = (occ == 0).astype(np.uint8)
    dist_m = distance_transform_edt(free) * res     # 각 칸의 가장 가까운 장애물까지 거리 [m]
    cost = 1.0 - np.clip(dist_m / max_dist, 0, 1)   # 0(안전) ~ 1(충돌)
    return cost

5. HD Map & Centerline CSV

일반 네비 지도는 도로를 선 하나로 표현합니다. HD Map(Lanelet)은 도로를 차선 단위로 쪼개고, 차선 경계·신호등·정지선·속도제한 등 의미 정보까지 같이 들고 다닙니다.

  • Lanelet: 한 차선 조각. 왼쪽·오른쪽 경계선 점열로 정의.
  • 규제 요소: 신호등, 정지선, 속도 제한, 우선권.
  • 연결 관계: 합류·분기·교차로.

본 강의에서는 가장 단순화된 형태인 중심선(centerline) CSV만 사용합니다. 실제 대학생 자율주행 경진대회들도 보통 이 형식으로 트랙을 줍니다.

6. Cubic Spline — 점들을 부드럽게 잇기

10개 정도의 이산 점을 직선으로 이으면 각진 경로가 되어 차가 못 탑니다. Cubic Spline은 구간마다 3차 다항식으로 부드럽게 연결하는 보간법입니다.

비유 — 종이에 점 10개 찍고 자로 그으면 울퉁불퉁하지만, 유연한 자(혹은 길고 가는 막대)로 이으면 자연스러운 곡선이 됩니다. CubicSpline이 그 "유연한 자"입니다. 
from scipy.interpolate import CubicSpline

# 누적 거리(arc-length)를 매개변수로
s = np.cumsum(np.hypot(np.diff(wps.x), np.diff(wps.y)))
s = np.insert(s, 0, 0.0)

csx = CubicSpline(s, wps.x)
csy = CubicSpline(s, wps.y)

s_fine = np.linspace(0, s[-1], 500)
plt.plot(csx(s_fine), csy(s_fine), '-')

7. Frenet 좌표 — 지하철 노선도

아이디어

도로를 쫙 펴서 1차원 자처럼 다룹시다.

  • $s$ — 중심선을 따라 진행한 누적 거리(arc-length)
  • $d$ — 중심선에서 옆방향으로 벗어난 거리 (왼쪽 +, 오른쪽 −)

지하철 노선도 비유: 강남 → 교대 → 서초 (순서 = $s$), 1번 출구 ↔ 2번 출구 (옆방향 = $d$). 실제 지하철은 구불구불하지만 노선도는 직선으로 그리죠? 그래서 "강남에서 3번째 역"처럼 단순한 표현이 됩니다.

왜 강력한가

  • 차선 유지 = $d \to 0$
  • 차선 변경 = $d: 0 \to 3.5$ m
  • 앞차와 거리 = $\Delta s$ 한 숫자
  • 복잡한 2D가 1D + 1D로 분해됨

수식

참조경로(중심선) $\mathbf{r}(s) = (x(s), y(s))$에 대해

  • 접선 벡터 (진행방향): $\mathbf{t}(s) = (x'(s),\, y'(s))$
  • 법선 벡터 (옆방향): $\mathbf{n}(s) = (-y'(s),\, x'(s))$

Frenet → World 변환:

$$\boxed{\;\mathbf{p}(s, d) = \mathbf{r}(s) + d \cdot \mathbf{n}(s)\;}$$

"중심선 위 점 + 옆으로 $d$만큼" — 이 한 줄이 Frenet의 본질입니다.

8. 인터랙티브: World ↔ Frenet 변환

인터랙티브: 점을 끌어 옮겨보면 (s, d)가 실시간 갱신

S자 모양 중심선 위에 빨간 차량 점이 있습니다. 점을 드래그해서 옮기면, 가장 가까운 중심선 점($s$)과 옆으로 벗어난 거리($d$)가 실시간으로 계산됩니다. 좌측이 양수, 우측이 음수임을 확인하세요.

World → Frenet 코드

def to_frenet(ref, x, y):
    # 1. 참조경로를 촘촘히 샘플링
    s_grid = np.linspace(0, ref.s_max, 2000)
    xs, ys = ref.xy(s_grid)
    # 2. 가장 가까운 점 찾기
    i = int(np.argmin((xs - x)**2 + (ys - y)**2))
    s0 = s_grid[i]
    yaw = ref.yaw(s0)
    # 3. 접선과 수직 거리 = d (왼쪽 +)
    d = -np.sin(yaw) * (x - xs[i]) + np.cos(yaw) * (y - ys[i])
    return s0, d

RefPath 클래스 (재사용)

# Ref: PythonRobotics/PathPlanning/CubicSpline (MIT, A. Sakai)
from scipy.interpolate import CubicSpline

class RefPath:
    def __init__(self, xs, ys):
        s = np.cumsum(np.hypot(np.diff(xs), np.diff(ys)))
        s = np.insert(s, 0, 0.0)
        self.s_max = s[-1]
        self.cx = CubicSpline(s, xs)
        self.cy = CubicSpline(s, ys)

    def xy(self, s):  return self.cx(s),    self.cy(s)
    def yaw(self, s): return np.arctan2(self.cy(s, 1), self.cx(s, 1))

    def to_xy(self, s, d):
        x0, y0 = self.xy(s); yaw = self.yaw(s)
        return x0 - d * np.sin(yaw), y0 + d * np.cos(yaw)

9. Frenet 의 진짜 위력 — 장애물 질의

인터랙티브: 같은 장면을 World vs Frenet 두 관점으로

왼쪽은 World (실제 모양) — 장애물(빨강)이 곡선 도로 여기저기 흩어져 있어 "내 앞에 있는지" 질의가 까다롭습니다. 오른쪽은 같은 장면을 Frenet $(s, d)$로 펼친 것 — "$s_\text{obs} - s_\text{ego} \in [0, 10]$" 한 줄로 끝납니다. 장애물을 드래그해서 비교해 보세요.

"전방 10 m 안에 차폭 1.75 m 이내 장애물이 있는가?" — World에서는 곡선 거리·수직선 같은 기하 계산이지만, Frenet에서는 한 줄 비교문으로 끝납니다.

for (x_o, y_o) in obstacles_xy:
    s_o, d_o = to_frenet(ref, x_o, y_o)
    if 0 < s_o - s_ego < 10 and abs(d_o) < 1.75:
        print("앞에 장애물!")
이 단순한 질의가 13주차 MPC의 제약식으로 그대로 들어갑니다. 차선 변경 궤적도 "$d$를 0 → 3.5로 5차 다항식으로 부드럽게"라는 1차원 문제로 단순화됩니다.

10. 곡률과 안전 속도

중심선이 얼마나 꺾이는지를 곡률 $\kappa$로 표현합니다. 회전반경 $R = 1/|\kappa|$. 횡가속 한계 $a_{y,\max}$가 주어지면, 그 곡률에서 차가 미끄러지지 않을 최대 속도가 결정됩니다.

$$\kappa(s) = \frac{x'(s)\,y''(s) - y'(s)\,x''(s)}{(x'(s)^2 + y'(s)^2)^{3/2}},\qquad v_{\max}(s) = \sqrt{\frac{a_{y,\max}}{|\kappa(s)|}}$$

승차감 기준 $a_{y,\max} = 2 \sim 3$ m/s². 곡률이 커지면 안전 속도가 급격히 떨어집니다.

12. 핵심 정리 & 치트시트

  • Occupancy Grid = 마인크래프트 픽셀, 0/1 (또는 -1 모름).
  • HD Map = 차선·신호·규제 포함 정밀 도면. 본 강의는 centerline CSV로 단순화.
  • Frenet $(s, d)$ = 도로를 편 1+1차원. 장애물 질의·차선 변경이 단순해짐.
  • 핵심 식: $\mathbf{p}(s, d) = \mathbf{r}(s) + d\,\mathbf{n}(s)$.
  • 좌표 ↔ 인덱스: i = int(y/res); j = int(x/res).
  • 안전 속도: $v_{\max} = \sqrt{a_{y,\max} / |\kappa|}$.

13. 손풀이 연습

  1. 해상도 0.2 m, 격자 (i, j) = (30, 50)의 실좌표는? $x = j \cdot \text{res} = 10$ m, $y = i \cdot \text{res} = 6$ m. 답: $(10, 6)$ m.
  2. 차량이 중심선 왼쪽 1.5 m에 있을 때 $d$의 부호와 값? 관례상 왼쪽 = 양수. 답: $d = +1.5$ m.
  3. $a_{y,\max} = 2.5$ m/s², $|\kappa| = 0.1$일 때 최대 속도? $v_{\max} = \sqrt{2.5/0.1} = 5$ m/s.
