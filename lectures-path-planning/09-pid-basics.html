Week 9. PID 제어 기초

전반부에서 "어디로 갈까(Planning)"를 다뤘다면, 후반부는 "어떻게 갈까(Control)"입니다. 그 출발점이 되는 가장 단순하고 가장 강력한 도구가 PID(Proportional–Integral–Derivative)예요. 오늘은 샤워기·에어컨 비유로 직관을 먼저 잡고, 슬라이더로 게인을 직접 흔들면서 P·I·D 각각의 역할을 손에 익힙니다.

1. 피드백 제어 — 샤워기 비유

한 줄 비유부터

샤워기 손잡이를 돌리며 딱 좋은 온도를 맞추는 일을 머릿속에 떠올려 보세요. 우리는 매 순간 "지금 너무 차갑다 → 빨강 쪽으로 살짝, 어이쿠 너무 뜨겁다 → 다시 파랑 쪽으로"를 반복합니다. 이 단순한 행동 안에 자동제어의 모든 본질이 담겨 있어요.

  • P(비례): 지금 너무 차갑다 → 뜨거운 쪽으로 큰 폭으로 돌린다.
  • I(적분): 아까부터 계속 약간 차가웠다 → 조금 더 돌린다.
  • D(미분): 지금 빠르게 뜨거워지고 있다 → 이제는 오히려 덜 돌린다.

현재 오차 + 누적 오차 + 변화 속도, 이 셋을 합쳐 결정하는 것이 바로 PID입니다.

2. PID 수식과 각 항의 역할

$$ u(t) = K_p\,e(t) + K_i \int_0^t e(\tau)\,d\tau + K_d\,\dot e(t),\qquad e(t) = r(t) - y(t) $$
항역할키우면너무 키우면
P $K_p e$현재 오차에 비례 대응응답 빨라짐진동 시작
I $K_i \int e\,dt$누적 오차 제거 (정상상태 0)정상상태 오차 줄어듦오버슈트, 와인드업
D $K_d \dot e$오차 변화에 브레이크오버슈트 억제잡음 증폭
P만 쓰면 정상상태 오차가 남는 이유 — 오르막에서 차가 목표 20 m/s에 못 미치는 18 m/s에서 멈췄다고 합시다. 이때 $u = K_p\cdot 2$가 정확히 중력으로 인한 감속과 균형을 이룹니다. 오차를 0으로 만들려면 더 큰 입력이 필요한데 $e$는 그대로니 $u$도 그대로 — 영원히 2 m/s 모자랍니다. 적분 항이 들어와야 시간이 갈수록 $\int e\,dt$가 커져 차이를 메꿔줍니다.

3. 인터랙티브: PID 게인 튜닝

🎮 인터랙티브: 1차 플랜트 + PID 스텝 응답

플랜트 $\tau\dot y + y = K u + d$ ($\tau=2$, $K=1$, 외란 $d$)에 PID를 걸었습니다. 슬라이더로 $K_p, K_i, K_d$를 흔들며 응답이 어떻게 변하는지 직접 보세요. 오버슈트(%), 정착시간, 정상상태 오차가 라이브로 표시됩니다.

꼭 만져볼 시나리오:

  • P만(Ki=0, Kd=0): Kp를 키우면 빨라지지만 진동 시작. 작으면 느리고 정상상태 오차 잔존.
  • PI(Kd=0): Ki를 켜면 정상상태 오차가 0으로 떨어짐. 오버슈트가 함께 늘어남에 주의.
  • PID: Kd를 추가하면 오버슈트가 줄어 깔끔. 너무 키우면 노이즈에 민감.
  • 외란 d: 외란을 켜면 P만으로는 영원히 못 메꿈. I가 도움.

4. 인터랙티브: P / PI / PID 항 비교

🎮 인터랙티브: 같은 플랜트, 세 가지 제어기

빨강 P, 주황 PI, 파랑 PID를 동시에 돌려 차이를 봅니다. 외란을 켜면 차이가 더 명확해져요.

P만으로는 외란이 들어왔을 때 회복되지 않고 정상상태 오차가 남는 게 보입니다. PI를 켜면 외란 후 다시 1.0으로 돌아오죠. PID는 거기에 오버슈트까지 잡습니다.

5. 적분기 와인드업과 Anti-windup

실제 액추에이터에는 한계가 있습니다(예: 가속 ±3 m/s²). 이 한계에 도달했는데도 적분기가 계속 누적되면, 한계를 벗어난 후에 큰 반동이 와서 시스템이 흔들립니다. 이를 적분기 와인드업(integrator windup)이라 합니다.

가장 단순한 대처는 포화가 발생한 스텝에서 적분을 되돌리는 클램핑(back-calculation):

u_raw = Kp*e + Ki*I + Kd*dedt
u = np.clip(u_raw, u_min, u_max)
if u_raw != u:           # 포화 시
    I -= e * dt          # 이번 스텝의 적분 누적을 되돌림

6. 인터랙티브: Anti-windup 효과

🎮 인터랙티브: 액추에이터 포화에서의 동작

큰 스텝 입력(0 → 2.0)을 줘서 액추에이터가 포화되는 상황입니다. Anti-windup ON/OFF를 토글해 차이를 보세요.

Anti-windup을 OFF로 두면 큰 스텝에서 적분이 폭발적으로 쌓여 오버슈트가 매우 커지고 회복도 늦어집니다. ON으로 두면 깔끔하게 정착하죠.

7. Ziegler-Nichols 튜닝

실차에서 손쉽게 출발점을 잡는 방법입니다.

  1. $K_i = K_d = 0$.
  2. $K_p$를 점점 올려가며 출력이 지속 진동을 시작하는 임계 게인 $K_u$, 진동 주기 $T_u$를 측정.
  3. 다음 권장값으로 시작:
$$ K_p = 0.6\,K_u,\qquad K_i = \frac{2 K_p}{T_u},\qquad K_d = \frac{K_p\,T_u}{8} $$

이는 어디까지나 초기값입니다. 실차에서는 미세 조정이 필요해요. 시뮬에서 먼저 잡고 실차에서 다듬는 게 표준 워크플로입니다.

증상별 조치 가이드

증상조치
정상상태 오차 잔존$K_i$ ↑
지속 진동$K_p$ ↓ 또는 $K_d$ ↑
응답이 느림$K_p$ ↑
큰 오버슈트$K_d$ ↑ 또는 $K_i$ ↓
입력이 톡톡 튐(노이즈)$K_d$ ↓ 또는 미분 항에 저역필터

8. 자율주행 Cruise Control 적용

import numpy as np

class PID:
    def __init__(self, Kp, Ki, Kd, u_min=-np.inf, u_max=np.inf):
        self.Kp, self.Ki, self.Kd = Kp, Ki, Kd
        self.u_min, self.u_max = u_min, u_max
        self.I = 0.0; self.e_prev = None

    def step(self, e, dt):
        self.I += e * dt
        dedt = 0.0 if self.e_prev is None else (e - self.e_prev) / dt
        self.e_prev = e
        u_raw = self.Kp*e + self.Ki*self.I + self.Kd*dedt
        u = np.clip(u_raw, self.u_min, self.u_max)
        if u != u_raw:                  # anti-windup
            self.I -= e * dt
        return u

# 평지 → 오르막(중력 -1) → 평지 시나리오
pid = PID(1.5, 0.5, 0.0, u_min=-5, u_max=3)
v, v_ref, dt = 0.0, 20.0, 0.02
for t in np.arange(0, 9, dt):
    gravity = -1.0 if 3 < t < 6 else 0.0
    a = pid.step(v_ref - v, dt)
    v += (a - 0.05*v + gravity) * dt

오르막에서 P만 쓰면 영원히 못 따라잡지만, $K_i$가 시간이 지날수록 부족분을 메꿔줍니다. 자율주행에서는 속도 제어는 PI(또는 PID), 조향 제어는 Pure Pursuit/Stanley(다음 주)가 표준 조합입니다.

8b. ACC — Adaptive Cruise Control

실제 양산 차량의 ACC 기능은 두 개의 PID가 상황에 따라 전환하는 구조입니다.

두 가지 모드

  • Cruise 모드: 앞차 없거나 멀 때. 목표 속도 PID.
  • Following 모드: 앞차 가까울 때. 안전거리 PID.

안전거리 모델 — Time Headway

$$ d_\text{safe}(v) = d_0 + \tau\,v $$
  • $d_0$: 정지 시 최소 간격 (5 m 정도)
  • $\tau$: 시간 간격 (1.5 ~ 2 초, 운전자 반응 시간 + 마진)

$v=20$ m/s 에서 $d_\text{safe} = 5 + 2\times 20 = 45$ m 정도. 충분히 멀어 보이지만 고속에선 1초 만에 좁혀질 수 있어요.

def acc_step(v, v_ref, d_front, dt):
    if d_front - d_safe(v) > margin:        # 앞차 멀면 Cruise
        a = pid_speed.step(v_ref - v, dt)
    else:                                     # 가까우면 거리 유지
        a = pid_dist.step(d_front - d_safe(v), dt)
    return np.clip(a, -5, 3)                 # 가속도 한계

8c. PID 의 친척들 — 한 장 비교

변종특징언제 쓰나
PI (D 없음)노이즈에 강함속도 제어, 일반 산업 (가장 흔함)
PD (I 없음)정상상태 오차 허용 시로봇 관절 제어
PID3축 모두 사용온도·압력 등 정밀 추종
PID + Feedforward모델로 미리 대응 + 잔차 제거현대 자율주행 실무 표준
Cascade PID외부 루프(위치) + 내부 루프(속도)드론 자세 제어

Feedforward + PID 패턴

def feedforward_pid(v_ref, slope, e, dt):
    # 모델 기반 미리 대응
    u_ff = a_required(v_ref, slope)        # 중력 보상 등
    # PID로 잔차 제거
    u_fb = pid.step(e, dt)
    return u_ff + u_fb

오르막을 알면 PID가 0부터 시작하지 않아도 됨 → 응답 더 빠르고 부드러움.

8d. 손풀이 연습 — PID 직관 익히기

Q1. P 만의 정상 오차 계산

$K_p = 2$, 외란(중력) $-3$, 시스템 $\dot v = a + d$ ($d$=외란). P 제어 단독으로 도달하는 정상 속도 오차는?

풀이

정상상태에서 $\dot v = 0$이므로 $a + d = 0 \Rightarrow a = 3$. P 제어: $a = K_p \cdot e = 2 e \Rightarrow e = 1.5$.

즉, 목표보다 1.5 m/s 부족한 상태로 균형. I 추가 시 0으로 수렴.

Q2. Anti-windup이 없을 때

가속 한계 +3 m/s², 큰 오차로 PID 출력이 +5를 요청. 시간 지나면서 적분기는 어떻게 되나?

풀이

출력은 +3에서 포화(saturate)되지만 적분기 $I = \int e\,dt$는 계속 누적 → I가 폭주. 오차가 작아진 후에도 I가 너무 커서 한참 동안 +3 명령 지속 → 큰 오버슈트.

해결: 포화 시 적분 갱신 중단 (clamp) 또는 역방향 보정 (back-calc).

Q3. dt가 너무 클 때

제어 주기 $\Delta t = 0.5$ s, $K_d = 1$ 으로 미분 제어. 잡음 0.1 m/s 가 한 스텝에 들어오면 D 출력은?

풀이

$\dot e \approx 0.1/0.5 = 0.2$ → $u_D = 1 \times 0.2 = 0.2$ m/s². 작은 잡음이 큰 미분 명령으로 증폭.

해결: ① $\Delta t$ 줄이기, ② D에 LPF 추가, ③ D 자체를 빼기 (PI만).

9. 핵심 정리

  • $u = K_p e + K_i\int e + K_d \dot e$. 현재 + 누적 + 변화 세 가지를 합쳐 결정.
  • P만으로는 외란·중력 같은 정상상태 오차를 못 잡음. I가 그걸 메꿈.
  • I가 너무 크거나 액추에이터가 포화하면 와인드업 → Anti-windup 필수.
  • Ziegler-Nichols로 초기값을 잡고 실차에서 미세 조정.
  • 자율주행에서 PID는 속도 제어의 기본기. 조향 추종은 다음 주 PP/Stanley.

9b. PID 의 한계 — 다음 단계로 가는 이유

PID 가 못 하는 5가지

한계왜 문제인가대안
제약 조건 직접 반영 X가속 한계, 차선 폭 등을 비용함수로만 간접 다룸W13 MPC — 부등식 제약 자연스럽게
다입출력(MIMO) 어려움여러 PID 묶으면 게인 결합 폭증LQR, MPC
예측 불가곡선 진입 등 미래 정보 활용 XMPC (예측 + 최적화)
비선형 강하면 튜닝 난감속도별 게인 다르게 (Gain Scheduling) 필요NMPC, RL
안전 보장 X충돌 가능성 직접 막지 못함W12 CBF

그래서 PID는 "제어의 기본기" 이지 "최선" 이 아닙니다. 후반부 강의는 PID가 못하는 영역을 점차 다룹니다.

실무 패턴 — PID 와 친구들

속도 (종방향)   ├ PID (대부분 충분)
                ├ MPC + PID (고급)

조향 (횡방향)   ├ Pure Pursuit / Stanley   ← 다음 주
                ├ LQR (선형 동역학 가정)
                ├ MPC (제약 + 예측)         ← W13
                └ + CBF (안전)              ← W12

9c. PID 의 인터뷰 단골 질문

Q. PID 의 각 항을 한 줄로 설명해 보세요.
P는 현재 오차에 비례 — 즉시 대응. I는 누적 오차 제거 — 정상상태 오차 0. D는 오차 변화율에 반응 — 오버슈트 억제.

Q. 적분 와인드업이 뭔가요? 어떻게 막죠?
액추에이터가 포화 상태인데도 적분기가 계속 누적되는 현상. 포화 후 큰 오버슈트 야기. 해결: clamp(포화 시 갱신 중단), back-calculation(역으로 보정), conditional integration(오차 큰 영역만 적분).

Q. PID 만으로 자율주행 조향을 하면 안 되나요?
가능은 하지만 권장 X. 차량 기구학(휠베이스, 비홀로노믹 제약)이 PID 입력에 반영되지 않아 곡선에서 큰 오차. Pure Pursuit/Stanley가 기구학 내장이라 더 안정.
