Week 12. CBF — 안전을 수학적으로 보장하기

Pure Pursuit는 경로를 잘 따라가지만 장애물 자체는 모릅니다. A*+Costmap은 정적 장애물엔 강하지만 갑자기 튀어나오는 보행자엔 약합니다. 오늘은 제어 입력에 안전 제약을 수학적으로 부과하는 도구 — Control Barrier Function(CBF) — 을 만나봅니다. 핵심 비유는 "개 목줄"입니다.

1. 왜 안전 보장이 어려운가?

한 줄 비유부터

강아지를 산책 시킨다고 상상해 보세요. 강아지(기본 제어기)는 자유롭게 뛰어다니고 싶어합니다. 주인은 차도 쪽으로는 못 가게 하고 싶어요. 둘을 화해시키는 도구가 목줄입니다.

  • 안전 영역 안에서는 목줄이 느슨 — 강아지는 마음대로 뜁니다.
  • 경계에 가까워지면 목줄이 팽팽해지면서 최소한만 잡아당겨 안전 영역 안으로 돌립니다.

CBF는 정확히 이 "목줄"을 수학적 부등식으로 쓴 것입니다. 평소엔 컨트롤러를 그대로 두고, 위험할 때만 살짝 수정해요.

왜 "최소한만 수정"이 중요할까?

안전한 행동을 강제하는 가장 단순한 방법은 "장애물 가까우면 무조건 정지"입니다. 하지만 이러면 차가 너무 자주 멈춰서 도착을 못 합니다. CBF의 묘미는 "안전을 깨지 않는 한도 내에서" 원하는 명령을 최대한 살려준다는 점입니다. 수학적으로는 "원본 명령 $u_\text{des}$ 와의 거리를 최소화하는 안전 명령 $u^*$" 를 찾는 최적화 문제로 표현됩니다(7장 참조).

오늘 배우는 CBF는 이 위에 "실시간으로, 명령 단계에서, 수학적으로" 안전을 강제합니다. 즉:

  • 경로 생성 후가 아니라 매 제어 사이클에서 동작.
  • 장애물이 갑자기 등장해도 같은 사이클에서 반응.
  • "증명 가능한 안전(provably safe)" — 부등식 만족만 확인하면 충돌이 수학적으로 불가능.

3. Safe Set과 안전 함수 $h(x)$

먼저 "안전한 상태들의 집합"을 함수 한 줄로 표현합니다.

  • $h(x) \ge 0$ → 안전 (Safe Set 안)
  • $h(x) < 0$ → 충돌 (Safe Set 바깥)

예를 들어 장애물 중심 $(x_o, y_o)$, 반지름 $R$의 원형 장애물이라면:

$$ h(x, y) = (x - x_o)^2 + (y - y_o)^2 - R^2 $$

차량 위치가 장애물 밖에 있으면 $h > 0$, 충돌 직전엔 $h = 0$, 안에 있으면 $h < 0$입니다. "$h$를 0 이상으로 영원히 유지"하는 게 우리의 목표예요.

$h(x)$ 설계 예시 모음

상황안전 함수 $h(x)$의미
원형 장애물$(x-x_o)^2 + (y-y_o)^2 - R^2$장애물 밖이면 양수
벽 앞 정지$x_\text{wall} - x$벽까지 남은 거리
차선 이탈$(W/2)^2 - d^2$차선 안이면 양수 ($d$=Frenet 횡오차)

설계 자유도가 높습니다. 한 시스템에 여러 $h_i$ 동시 부과도 가능 (다중 제약).

4. CBF 핵심 부등식

$h(x)$를 "0 이상"으로 유지하는 가장 간단한 충분조건은 다음과 같습니다.

$$ \dot h(x, u) \ge -\alpha\,h(x), \qquad \alpha > 0 $$

의미를 풀어 쓰면:

  • $h$가 큰 값(안전한 상태)일 땐 $\dot h$가 음수여도 OK — 약간 줄어도 괜찮음.
  • $h$가 0 가까이 갈수록 $\dot h$의 허용 하한 $-\alpha h$가 0에 가까워져 — 더는 줄어들 수 없음.
  • $h = 0$ 경계에서는 $\dot h \ge 0$이 강제 — 안전 집합에서 절대 빠져나갈 수 없음.

$\alpha$가 작을수록 멀리서부터 조심(보수적), 클수록 경계 근처까지 풀가속(공격적).

제어 affine 시스템 $\dot x = f(x) + g(x)u$에서는 부등식이 입력 $u$에 대해 선형이 됩니다.

$$ L_f h(x) + L_g h(x)\,u \ge -\alpha\,h(x) $$

여기서 $L_f h = \nabla h \cdot f$, $L_g h = \nabla h \cdot g$입니다. 이 부등식을 만족하는 $u$만 허용하면 안전이 보장됩니다.

5. 수식 한 줄을 풀어 읽는 법 (1학년 친화)

겁먹지 말기 — $L_f h, L_g h$ 같은 기호는 "Lie 미분"이라 부르지만, 하는 일은 그냥 편미분 곱하기입니다. 이 강의는 의미만 잡고 가요. 자세한 표기는 비선형 제어 과목에서.

한 줄씩 번역

기호실제 계산"무슨 일을 하는가"
$\nabla h$$h$의 편미분 벡터 $[\partial h/\partial x_1, \partial h/\partial x_2, \ldots]$"$h$가 가장 빨리 커지는 방향"
$L_f h = \nabla h \cdot f$위 벡터와 $f$의 점곱"제어 안 줘도 자연스럽게 $h$가 변하는 속도"
$L_g h \cdot u$벡터 점곱 후 $u$ 곱"제어 $u$가 $h$에 미치는 영향"
$\dot h$$L_f h + L_g h \cdot u$"실제로 $h$가 변하는 속도"

부등식 $\dot h + \alpha h \ge 0$ 의 직관

좌변을 한 단어로 부르면 "여유 줄어드는 속도 + 여유의 α배". 이게 0 이상이어야 한다는 건:

  • 여유가 많을 땐(=$h$ 큼) 좀 줄어도 OK ($\alpha h$가 큰 양수라서)
  • 여유가 거의 없을 땐(=$h \to 0$) 더 이상 못 줄게 됨 ($\dot h \ge 0$)
  • 이미 위험 영역에 있을 땐(=$h<0$) 빠져나오는 방향만 허용 ($\dot h \ge -\alpha h > 0$)

그래서 한 번 안전한 상태로 들어오면 절대 빠져나갈 수 없는 게 보장됩니다 (Nagumo 정리).

6. 인터랙티브: 1D — 벽 앞에 자동 정지

🎮 인터랙티브: $h(x) = x_\text{wall} - x$

1차원 차량이 벽($x = 10$)을 향해 갑니다. PID는 항상 "5 m/s로 가자"고 명령하지만, CBF가 "벽에서 멀 때만 그렇게 가도 된다"고 자동 제한합니다. α를 흔들어 보세요.

α가 작으면 멀리서부터 부드럽게 감속, 크면 벽 근처까지 풀가속하다 급제동. 어느 쪽도 절대 벽을 통과하지 않습니다 — 그게 CBF의 보장입니다.

8. 인터랙티브: 2D — 장애물 우회 (CBF on/off 비교)

9. 인터랙티브: α 게인 효과

🎮 인터랙티브: α를 작게/크게 설정해 보세요

같은 환경에서 α만 다르게 — 작으면 멀리서부터 우회(보수적), 크면 장애물에 더 다가간 뒤 회피(공격적).

α=0.5는 멀리서부터 부드럽게 곡선을 그리며 우회, α=5.0은 장애물 바로 앞까지 직진하다 급격히 회피. 후자는 짧지만 승차감이 나쁘고 동역학 제약을 위반할 위험이 있습니다.

10. QP-CBF 안전 필터 — 기존 제어기에 끼우기

실제 시스템에서는 기본 컨트롤러(Pure Pursuit, MPC, …)가 만든 명령 $u_\text{des}$를 그대로 쓰지 않고, 가장 가까운 안전 명령으로 살짝 수정합니다. 이게 QP-CBF 안전 필터입니다.

$$ u^* = \arg\min_{u}\;\|u - u_\text{des}\|^2 \quad \text{s.t.}\quad L_f h + L_g h\,u \ge -\alpha h $$

제약이 1개면 닫힌 해(projection)가 있고, 여러 장애물이면 여러 부등식을 동시에 만족하는 QP를 풉니다.

import numpy as np

def cbf_filter(u_des, ego, obstacles, alpha=1.5):
    """단일 적분기 ego, 다수 원형 장애물에 대한 QP-CBF (간단 projection)."""
    u = u_des.copy()
    for (cx, cy, R) in obstacles:
        d  = ego - np.array([cx, cy])
        h  = d @ d - R**2
        gh = 2*d
        lhs = gh @ u + alpha * h
        if lhs < 0:                     # 안전 제약 위반
            u = u - (lhs / (gh @ gh)) * gh   # projection
    return u

# 메인 루프
ego = np.array([0.0, 0.0])
goal = np.array([20.0, 0.0])
obs = [(10.0, 0.0, 2.5), (14.0, 1.0, 1.5)]
for t in np.arange(0, 12, 0.05):
    u_des = (goal - ego) / np.linalg.norm(goal - ego) * 2.0
    u = cbf_filter(u_des, ego, obs, alpha=1.5)
    ego += u * 0.05

Pure Pursuit과의 결합

실차에서는 입력이 (조향, 가속)이고 시스템이 비홀로노믹 차량입니다. 위 1차 적분기 예제의 아이디어는 그대로지만, $\nabla h$의 그래디언트를 차량 동역학으로 한 번 더 유도해야 해요. cvxpy로 손쉽게 풀 수 있습니다.

11. 다중 장애물·Soft 제약·HO-CBF

다중 장애물 — 제약 N개

장애물이 K개 있으면 단순히 부등식을 K개 추가합니다.

$$ u^* = \arg\min_{u}\;\|u - u_\text{des}\|^2 \quad \text{s.t.}\quad L_f h_i + L_g h_i\,u \ge -\alpha h_i,\ \ i = 1,\ldots,K $$

QP solver(예: cvxpy, OSQP)가 이를 1ms 내외로 풉니다. 단, 제약이 너무 많거나 서로 모순될 땐 infeasible(해 없음)이 발생할 수 있어요.

Soft 제약 — 해를 항상 보장

실차에서는 "해 없음"이 가장 위험합니다. 해법은 슬랙 변수 $s$ 도입.

$$ \min_{u, s\ge 0}\;\|u - u_\text{des}\|^2 + M\,s^2 \quad \text{s.t.}\quad L_f h_i + L_g h_i\,u \ge -\alpha h_i - s $$
  • $M$ 큼 → 슬랙 사용을 매우 싫어함 → 가능한 한 hard에 가까움
  • $M$ 작음 → 위반 더 허용

이렇게 두면 어떤 상황에서도 해는 나오고, 위반량 $s$는 모니터링 지표로 활용합니다.

HO-CBF — 차량처럼 가속도까지 있는 시스템

1차 적분기 $\dot x = u$ 는 너무 단순합니다. 실제 차량은 2차 시스템 ($\ddot x = u$, 즉 가속도 입력). 이런 경우 단순 CBF로는 부족합니다 — 한 번 미분된 $\dot h$ 가 입력 $u$ 에 직접 의존하지 않거든요.

해결책: Higher-Order CBF (HO-CBF). $h$를 두 번 미분해 $\ddot h$ 가 $u$에 의존하도록 만듭니다.

$$ \ddot h + \alpha_1 \dot h + \alpha_0 h \ge 0 $$

이 식은 마치 2차 LTI 시스템처럼 생겼고, $\alpha_0, \alpha_1$ 을 잘 고르면 안전 영역으로 부드럽게 수렴합니다.

12. 다른 안전 기법과의 비교

기법안전 보장장점단점

13. 실무 사례 — AEB · 차선유지 · 로보틱스

AEB (Autonomous Emergency Braking)

대부분 신차에 탑재된 ADAS 기능. 앞차 거리 $\Delta s$ 에 대한 CBF-유사 조건:

$$ h(\Delta s, v) = \Delta s - (d_0 + \tau v) \ge 0 $$

"안전거리 미만으로 줄어들지 않도록" 가속 제한. 일반 운전 중에는 운전자 명령 그대로, 위험할 때만 자동 제동.

LKAS (Lane Keeping Assist System)

차선 이탈을 막는 안전 함수: $h(d) = (W/2)^2 - d^2$. 일반 주행 중엔 운전자가 자유, 차선 가까이 가면 자동 조향 토크 추가. 정확히 CBF의 "최소 수정" 철학과 같습니다.

로보틱스 — 협동 로봇

공장의 협동 로봇(Cobot)은 사람과 가까이 작업합니다. 각 관절·끝점에 대해 사람과의 거리를 $h$로 두고, "최소한만 수정"되는 CBF 안전 필터가 표준 솔루션이 되어 가는 추세.

13b. CBF vs Potential Field — 닮은 듯 다른 두 방법

CBF는 종종 Potential Field(PF)와 비교됩니다. 둘 다 "장애물 회피"가 목적이고, 장애물 근처에서 반응이 강해진다는 공통점이 있어요. 하지만 본질이 다릅니다.

Potential Field — "보이지 않는 자석"

장애물에서 멀어지는 방향으로 가짜 힘을 만들고, 차량은 그 힘에 따라 움직입니다.

$$ \mathbf{F}_\text{rep} = k_\text{rep} \frac{\mathbf{r} - \mathbf{r}_o}{\|\mathbf{r} - \mathbf{r}_o\|^3}, \qquad u = -\nabla U(x) $$
  • 장점: 직관적, 경로 부드러움
  • 단점: 지역 최소(local min) — U자 장애물 안에서 갇힘. 안전 보장 X.

CBF — "넘지 말아야 할 선"

제어 입력에 부등식 제약을 부과. 부드러운 힘이 아니라 경계 강제.

  • 장점: 수학적 안전 보장, 기존 컨트롤러와 자유 결합
  • 단점: 모델 필요, 자체적으로 목표 도달 안 함 (별도 컨트롤러 필요)

한 줄 정리

  • "부드럽게 회피" → Potential Field
  • "절대 안 부딪히게" → CBF
  • 둘 다 → "PF 형태의 비용 + CBF 안전 제약" 결합 (실무 표준)

14. 동적 장애물 — 시간 변하는 $h(x, t)$

지금까지의 $h$는 "정적 장애물"이었습니다. 보행자나 다른 차량이 움직이는 경우엔 어떻게 할까요?

방법 1 — 장애물 속도 측정 후 안전 함수에 포함

장애물 위치 $(x_o(t), y_o(t))$가 시간에 따라 변하면 안전 함수도 시간 의존:

$$ h(x, y, t) = (x - x_o(t))^2 + (y - y_o(t))^2 - R^2 $$ $$ \dot h = 2(x - x_o)(\dot x - \dot x_o) + 2(y - y_o)(\dot y - \dot y_o) $$

이때 $\dot x_o$ (장애물 속도) 가 추가 항으로 들어갑니다. 인지(perception)에서 추정한 속도를 그대로 사용하면 됩니다.

방법 2 — 보수적 마진 추가

장애물 속도가 불확실하면 안전 마진 $r_\text{margin}$ 을 추가:

$$ h(x, y) = (x - x_o)^2 + (y - y_o)^2 - (R + r_\text{margin})^2 $$

$r_\text{margin}$ 은 "장애물이 가장 빠르게 움직일 때 한 제어 사이클 사이에 이동할 수 있는 거리". 일반적으로 0.5 ~ 2 m.

방법 3 — TTC 기반 (Time-to-Collision)

"몇 초 후 충돌하나"를 직접 안전 척도로:

$$ h_\text{TTC}(x, v_\text{rel}) = \tau_\text{min} - \frac{|x_o - x|}{|v_\text{rel}|} $$

$\tau_\text{min}$ = 운전자 반응 시간 + 마진 (보통 2~3초). 자동차 산업 표준 ADAS 안전 지표.

FAQ

Q1. CBF가 있으면 충돌이 0%인가요?
이론적으론 그렇습니다 — 단, 다음 가정이 모두 만족할 때: ① 시스템 모델이 정확, ② 외란 없음, ③ 시간 이산화 충분히 작음. 실차는 이 가정들이 깨질 수 있어 안전 마진을 더 둡니다.

Q2. CBF로 차선 변경 같은 복잡한 행동도 할 수 있나요?
CBF는 "안전 유지" 도구입니다. "차선 변경"은 경로 생성 단계의 결정 — 별개 모듈 (예: Frenet planner) 이 담당합니다. CBF는 그 위에서 "안전하게만 따라가도록" 강제.

Q3. PID와 CBF, 어느 쪽이 더 좋나요?
비교 대상이 아닙니다. PID는 "추종 도구", CBF는 "안전 필터". 대부분 시스템에서 PID + CBF 형태로 함께 씁니다.

Q4. 학부생이 직접 시도해 볼 수 있나요?
네! 1D 예제(이 페이지의 인터랙티브)는 Python 10줄이면 충분합니다. 2D 예제도 cvxpy 라이브러리만 깔면 30줄 정도. 시간 두고 하나씩 해보면 큰 자산이 됩니다.

Q5. 학부 1학년인데 어디까지 이해해야 하나요?
① "$h \ge 0 =$ 안전" 정의, ② "$\dot h + \alpha h \ge 0$" 조건, ③ "QP-CBF는 안전 필터" 컨셉 — 이 셋만 잡으면 시험·인터뷰에서 충분합니다. 나머지는 대학원이나 실무에서!

16. 핵심 정리 & 다음 주 연결

  • $h(x) \ge 0$ = 안전, $\dot h \ge -\alpha h$ = 안전 집합 이탈 금지.
  • CBF는 안전 필터: 평소엔 컨트롤러를 그대로, 위험할 때만 최소한 수정.
  • α가 작으면 보수적, 크면 공격적. 차량 동역학 한계와 연동해 선택.
  • QP-CBF는 단일/다중 장애물에 모두 일반화 가능.
  • 지역 최소 회피·동적 모델 적용은 더 발전된 형태(HO-CBF, robust CBF)가 다룸.
  • 한 줄: "개 목줄 = CBF".

다음 주(W13 MPC)와의 연결

13주차 MPC는 "예측 + 최적화" 제어입니다. CBF의 부등식은 MPC의 제약식으로 자연스럽게 들어갑니다:

  • MPC 단독: 비용함수에 "장애물 거리 페널티" 추가 → 부드러운 회피, 그러나 안전 보장 X
  • MPC-CBF: 매 예측 스텝마다 $\dot h_k + \alpha h_k \ge 0$ 부등식 → 예측 + 안전

이 조합이 2024-2026년 자율주행 연구의 가장 뜨거운 주제 중 하나입니다.

def cbf_steering_filter(delta_des, state, obstacles, alpha=1.5):
    """ Pure Pursuit가 만든 조향 명령을 CBF로 안전 필터 통과. """
    x, y, yaw, v = state
    delta = delta_des
    for (cx, cy, R) in obstacles:
        dx, dy = x - cx, y - cy
        h = dx*dx + dy*dy - R*R
        # 단순화: ∇h · v 형태로 (자세한 차량 모델은 강의 너머)
        h_dot = 2 * (dx * v * np.cos(yaw) + dy * v * np.sin(yaw))
        if h_dot + alpha * h < 0:
            # 조향 보정 (간단 휴리스틱)
            delta = delta + 0.05 * np.sign(dx*np.sin(yaw) - dy*np.cos(yaw))
    return np.clip(delta, -np.deg2rad(30), np.deg2rad(30))

18. 손풀이 연습 — 시험 대비

Q1. 안전 함수 설계

차선 중앙에서 좌우 각 1.75 m 가 안전 영역, 차량 좌우 횡오차 $d$ ∈ [-1.75, 1.75] 일 때:

  • (a) 안전 함수 $h(d)$ 를 한 줄로 써라.
  • (b) $\alpha = 2.0$ 일 때 $d = 1.5$ m 에서 허용 최대 횡속도 $\dot d$ 는?
풀이 보기

(a) $h(d) = (1.75)^2 - d^2 = 3.0625 - d^2$. ($d=0$이면 $h$ 최대, 경계에서 $h=0$, 바깥이면 음수)

(b) $\dot h = -2 d \cdot \dot d$. CBF 조건 $\dot h + \alpha h \ge 0$:

$-2(1.5) \dot d + 2.0 (3.0625 - 2.25) \ge 0$

$-3 \dot d + 1.625 \ge 0$ → $\dot d \le 0.542$ m/s.

오른쪽 경계로 가까이 갈수록 허용 횡속도 자동 감소.

Q2. α 선택 — 차량 동역학 한계 고려

차량 최대 가속 $a_\max = 4$ m/s²이고, 벽까지 거리 $h(x) = x_w - x = 8$ m, 현재 속도 $v = 10$ m/s. 안전한 정지를 위한 최소 $\alpha$ 는?

풀이 보기

물리적 정지 거리: $d_\text{stop} = v^2 / (2 a_\max) = 100/8 = 12.5$ m.

현재 8 m 남았는데 12.5 m 필요 → $a_\max=4$로는 부족. CBF α 무관, 시스템 자체가 안전 보장 불가.

교훈: CBF는 시스템 능력 안에서만 안전 보장. $\alpha$만 키운다고 해결되지 않음.

Q3. 다중 장애물에서 infeasible 판단

장애물 2개가 너무 가까이 있어 사이로 못 지나가는 경우, QP-CBF의 결과는?

풀이 보기

두 부등식이 동시에 만족 불가 → infeasible. Hard 제약이면 solver가 에러 반환.

해결: ① Soft CBF (slack 변수), ② "더 급한 것만 hard로" 휴리스틱, ③ 상위 Planner가 다른 경로 선택하도록 신호.

19. 한 페이지 요약 카드

  • CBF 한 줄: 평소엔 그대로, 위험할 때만 최소 수정 → 수학적 안전 보장
  • 핵심 부등식: $\dot h + \alpha h \ge 0$, 단 $h(x) \ge 0$ = 안전 집합
  • 구현: QP 한 번 풀기 (cvxpy/OSQP, 수 ms)
  • α 튜닝: 작 = 보수적, 큼 = 공격적, 시작값 1.0
  • 실무 패턴: A* (전역) + PP/MPC (추종) + CBF (안전 필터)
  • 한계: 모델 필요, 지역적, infeasible 가능 → Soft 제약
  • 가까운 친척: AEB, LKAS, 협동 로봇 안전 — 모두 같은 철학
