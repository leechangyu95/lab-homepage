Week 4. 오일러 방법 & 시뮬레이션

"수식을 컴퓨터로 굴리는 법". 미분방정식이 종이 위에서 그치지 않고 실제로 자동차가 움직이는 시뮬레이터가 되려면, 시간을 잘게 끊어서 변화율을 계속 더하는 수치 적분(numerical integration)이 필요합니다. 오늘은 가장 단순한 오일러 방법부터 실무 표준 RK4까지, 그리고 우리 자전거 모델을 직접 굴리는 시뮬레이터를 만듭니다.

1. 왜 시뮬레이션이 필요한가?

속도계만 있는 차

2·3주차에서 우리는 자전거 모델 수식을 얻었습니다.

$$\dot{x} = v\cos\psi,\quad \dot{y} = v\sin\psi,\quad \dot{\psi} = \frac{v}{L}\tan\delta$$

이 식이 알려주는 건 "지금 이 순간 얼마나 변하는지"(변화율)뿐입니다. 그런데 우리가 진짜 알고 싶은 건 10초 후 차가 어디 있을까?죠. 변화율을 시간에 따라 계속 누적해야 위치가 나옵니다.

비유 — 속도계만 보이는 차 — 속도는 매 순간 알 수 있지만, "지금까지 얼마나 갔는지"는 직접 계산해야 합니다. 1초마다 속도를 곱해서 더하면 위치 — 이게 바로 수치 적분이고, 자율주행 시뮬레이터의 심장입니다.

왜 시뮬이 필수인가?

  • 실차는 위험·비싸다 — 알고리즘 버그를 시뮬에서 잡으면 99% 효율.
  • 제어기 튜닝 — PID·LQR·MPC 게인을 시뮬에서 미리 맞춰 실차엔 거의 완성된 상태로 올림.
  • 알고리즘 비교 — 같은 환경에 여러 Planner를 돌려 성능을 정량 비교.

2. 오일러 방법 — 한 줄 공식

핵심 아이디어

"지금 방향(속도)으로 아주 짧은 시간 $\Delta t$ 동안 그냥 직진시키자."

$$\boxed{\;x_{k+1} = x_k + \dot{x}_k \cdot \Delta t\;}$$
기호의미예시 (자전거 모델)
$x_k$지금(k 스텝)의 상태$(x, y, \psi)$ 현재 위치·헤딩
$\dot{x}_k$지금의 변화율$v\cos\psi,\;v\sin\psi,\;v\tan\delta/L$
$\Delta t$한 걸음의 시간 폭0.01 ~ 0.05 s 권장
$x_{k+1}$$\Delta t$ 후의 상태한 스텝 전진한 위치·헤딩

걸음마 비유

오일러 방법은 "보폭이 $\Delta t$인 산책"입니다. 매 순간 (1) 지금 어디인지 보고 (2) 어디로 갈지(변화율) 정한 뒤 (3) 한 걸음만큼 직진합니다. 이걸 수천 번 반복하면 곡선 궤적이 만들어집니다.

인터랙티브: 오일러 한 걸음씩 — 커피 식는 곡선

뉴턴 냉각 법칙 $\dot{T} = -k(T - T_\text{room})$을 오일러로 풀어봅니다. "한 걸음" 버튼을 누를 때마다 빨간 점이 한 스텝씩 전진합니다. $\Delta t$를 키워보면 같은 시간 동안 정답(검정 곡선)에서 점점 멀어지는 걸 볼 수 있어요.

3. $\Delta t$가 정확도에 미치는 영향

오일러는 한 걸음 안에서는 변화율이 일정하다고 가정합니다. 실제로는 매 순간 바뀌니까, 걸음이 클수록 곡선을 직선으로 잘라먹는 셈이 되어 오차가 누적됩니다.

$\Delta t$10초 시뮬 스텝 수정확도속도
1.0 s10매우 거침매우 빠름
0.1 s100거침빠름
0.01 s1,000좋음 (자율주행 표준)적당
0.001 s10,000거의 완벽느림
비유 — 카메라 셔터 속도 — 빠르면 선명하지만 사진 용량이 폭증하고, 느리면 뭉개집니다. 실무 표준은 $\Delta t = 0.01 \sim 0.05$ s (100~20 Hz)입니다.

4. 인터랙티브: 자전거 모델 시뮬레이터

인터랙티브: 직진 후 회전 — $\Delta t$를 흔들어보세요

$v=5$ m/s, 3초까지 직진 후 조향 각 $\delta$로 회전. 같은 명령에 $\Delta t$만 바꿔 그려보면, 큰 $\Delta t$일수록 이론상 원호(빨간 점선)에서 벗어나는 게 보입니다.

한 함수 10줄 — 시뮬레이터의 심장

import numpy as np

def step_euler(state, v, delta, dt=0.01, L=2.9):
    x, y, psi = state
    # 1) 변화율 계산 (자전거 모델)
    dx   = v * np.cos(psi)
    dy   = v * np.sin(psi)
    dpsi = v / L * np.tan(delta)
    # 2) 한 걸음 전진
    return np.array([
        x   + dx   * dt,
        y   + dy   * dt,
        psi + dpsi * dt
    ])

이 한 함수가 5~14주차의 모든 실습이 사용하는 시뮬레이터의 핵심입니다. 클래스로 감싸두면 더 편합니다.

class CarSim:
    def __init__(self, x=0, y=0, psi=0, v=0, L=2.9):
        self.state = np.array([x, y, psi, v])
        self.L = L

    def step(self, accel, delta, dt=0.02):
        x, y, psi, v = self.state
        x   += v * np.cos(psi) * dt
        y   += v * np.sin(psi) * dt
        psi += v / self.L * np.tan(delta) * dt
        v   += accel * dt
        self.state = np.array([x, y, psi, v])
        return self.state

5. 오일러의 함정 — 누적 오차와 에너지 드리프트

오일러가 "현재 변화율"만 쓰기 때문에, 한 걸음 안에서 변화율이 바뀌는 시스템에서는 매 스텝 조금씩 틀리고 그게 계속 누적됩니다. 단진동(흔들리는 진자)에서 가장 잘 보입니다.

인터랙티브: 진자(단진자)에서 에너지 드리프트

이상적인 단진자 $\ddot\theta = -\omega^2 \sin\theta$는 마찰이 없으면 영원히 같은 진폭으로 흔들려야 합니다. 오일러로 풀면 매 사이클마다 진폭이 커지면서 발산합니다(에너지 증가). RK4로 바꾸면 수백 사이클을 돌려도 거의 일정합니다. 직접 비교해 보세요.

관찰 포인트 — Explicit Euler는 큰 $\Delta t$에서 진폭이 점점 커지면서 시스템이 발산합니다. "에너지가 늘어나는 진자"는 물리적으로 불가능한데, 이게 바로 수치 불안정성입니다. Semi-implicit Euler는 같은 1차 방법이지만 에너지를 거의 보존하고, RK4는 거의 완벽하게 일정합니다.

6. RK4 — "걸음 전에 4번 정찰"

아이디어

오일러는 출발점에서 방향 1번 보고 한 걸음 갑니다. RK4는 한 걸음 전에 4지점에서 방향을 봐서 가중평균합니다.

  1. $k_1$: 시작점에서 변화율
  2. $k_2$: 절반만 가본 지점($k_1$ 사용)에서 변화율
  3. $k_3$: 다시 절반 지점($k_2$ 사용)에서 변화율
  4. $k_4$: 끝 지점($k_3$ 사용)에서 변화율
  5. 4개 가중평균으로 진짜 한 걸음
$$ \begin{aligned} k_1 &= f(x_k) \\ k_2 &= f\left(x_k + \tfrac{\Delta t}{2} k_1\right) \\ k_3 &= f\left(x_k + \tfrac{\Delta t}{2} k_2\right) \\ k_4 &= f(x_k + \Delta t\, k_3) \\ x_{k+1} &= x_k + \tfrac{\Delta t}{6}(k_1 + 2k_2 + 2k_3 + k_4) \end{aligned} $$
방법한 스텝 계산오차 차수같은 정확도에서 권장 $\Delta t$
Explicit Euler1번$O(\Delta t)$0.005 ~ 0.01 s
Midpoint (RK2)2번$O(\Delta t^2)$0.02 s
RK44번$O(\Delta t^4)$0.05 ~ 0.1 s (실무 표준)
Implicit Euler반복(뉴턴)$O(\Delta t)$큰 $\Delta t$에도 안정 (강체 시스템용)

RK4는 한 스텝에 4번 계산하지만, 같은 정확도를 얻는 데 $\Delta t$를 훨씬 크게 잡을 수 있어서 오히려 빠를 때가 많습니다.

def rk4_step(f, x, u, dt):
    k1 = f(x,                u)
    k2 = f(x + dt/2 * k1,    u)
    k3 = f(x + dt/2 * k2,    u)
    k4 = f(x + dt   * k3,    u)
    return x + dt/6 * (k1 + 2*k2 + 2*k3 + k4)

def bicycle(state, u, L=2.9):
    x, y, psi = state
    v, delta  = u
    return np.array([
        v*np.cos(psi),
        v*np.sin(psi),
        v/L*np.tan(delta)
    ])

# 사용
state = rk4_step(bicycle, state, (v, delta), dt=0.05)

7. 인터랙티브: 오일러 vs RK4 — 원운동 누적 오차

인터랙티브: 같은 자전거가 원을 그릴 때 누가 정확한가

일정 속도, 일정 조향각에서 자전거 모델은 이론상 완벽한 원을 그려야 합니다(반경 $R = L/\tan\delta$). 오일러는 큰 $\Delta t$에서 점점 바깥으로 나가는 나선이 되고, RK4는 거의 정확하게 원을 따릅니다.

실험해 볼 것 — $\Delta t = 0.3$ s로 키워보세요. Euler는 명백한 나선이 되고 RK4는 여전히 거의 원을 따라갑니다. 같은 정확도를 얻으려면 Euler는 $\Delta t$를 1/10로 줄여야 합니다.

8. 실전 트릭 — 각도 wrap, clip, 제어 주기

각도 wrapping

$\psi$를 그대로 더하다 보면 $\pi$를 넘어 계속 증가합니다. 그러면 추후 $\psi_r - \psi$ 계산에서 엉뚱하게 큰 숫자가 나와요. 매 스텝 wrap을 적용하세요.

def wrap(a):
    return (a + np.pi) % (2 * np.pi) - np.pi

psi = wrap(psi + dpsi * dt)   # 항상 [-π, π]

입력 clip

가속·조향에 물리적 한계가 있는데 제어기가 이를 모르고 큰 값을 보내면 시뮬이 비현실적이 됩니다.

accel = np.clip(accel, -3.0, 1.5)            # m/s²
delta = np.clip(delta, -np.deg2rad(35), np.deg2rad(35))

제어 주기 vs 시뮬 주기

실차에서 센서·제어는 보통 20~50 ms 주기로 도는데, 시뮬은 더 작은 $\Delta t$가 필요합니다. "내부 시뮬은 작게, 제어는 그보다 느린 외부 루프" 패턴을 씁니다.

for outer in range(N_ctrl):
    accel, delta = controller(car.state)   # 50 ms마다 한 번
    for _ in range(5):                     # 50 ms = 10 ms × 5
        car.step(accel, delta, dt=0.01)

노이즈·지연 주입

"시뮬에서는 잘 됐는데 실차에선 왜?"의 1순위 원인은 모델 오차·지연·노이즈입니다. 시뮬에 미리 넣어 두면 제어기가 강건해집니다.

cmd_buf = [(0.0, 0.0)] * 3                # 60 ms 지연
for t in np.arange(0, 10, 0.02):
    cmd = controller(state)
    cmd_buf.append(cmd)
    accel, delta = cmd_buf.pop(0)         # 지연된 명령 사용
    state = step_euler(state, accel, delta, dt=0.02)
    state += np.random.randn(3) * [0.005, 0.005, 0.001]   # 측정 노이즈

10. 핵심 정리 & 치트시트

  • 시뮬레이션 = 미분방정식을 컴퓨터로 푸는 것 = 변화율을 시간 따라 누적.
  • 오일러 한 줄: $x_{k+1} = x_k + \dot x_k \Delta t$. 직관적이지만 누적 오차에 약함.
  • $\Delta t$ 표준: 자율주행 0.01 ~ 0.05 s. 작을수록 정확하지만 느림.
  • RK4 = "걸음 전에 4번 정찰". 같은 정확도에 큰 $\Delta t$ 사용 가능 → 실무 표준.
  • 실전 3트릭: 각도 wrap, 입력 clip, 시뮬/제어 주기 분리.

11. 손풀이 연습

  1. 등속 운동 $v=10$ m/s, $\Delta t=0.5$, 4 스텝 후 위치는? 스텝별로 0 → 5 → 10 → 15 → 20. 답: 20 m.
  2. 커피 냉각 $T_0=80°$C, $T_\text{room}=20°$C, $k=0.1$, $\Delta t=1$ s, 3 스텝 후 $T_3$?
    • 스텝 1: $\dot T = -0.1\times(80-20) = -6$ → $T_1 = 74$°C
    • 스텝 2: $\dot T = -0.1\times(74-20) = -5.4$ → $T_2 = 68.6$°C
    • 스텝 3: $\dot T = -0.1\times(68.6-20) = -4.86$ → $T_3 \approx 63.7$°C
  3. $\Delta t$를 2배로 키우면 Euler 오차는? — 약 2배 ($O(\Delta t)$). RK4는 16배 ($O(\Delta t^4)$).
